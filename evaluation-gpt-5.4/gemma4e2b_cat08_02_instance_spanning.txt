### 2. The differentiation between within-instance and between-instance delay is conceptually flawed
This is one of the biggest weaknesses.

The answer says:

- ōWithin-instance waiting time = activity durationö
- ōBetween-instance waiting time = time before/after an activity because resource is occupiedö

That is not correct.

**Activity duration is not waiting time.** It is service/processing time.  
A proper answer should distinguish:
- **Processing time**: start to complete of an activity.
- **Waiting time due to within-case factors**: case not enabled yet because prior steps/rework/data issues/approval dependency within the same order.
- **Waiting time due to between-case factors**: case enabled, but cannot start because of shared resource contention, batching dependency, priority interruption, or global WIP cap.

This mistake is significant because the prompt explicitly asked for this differentiation.
