- **Core SQL syntax is wrong** for DuckDB:
  - `STRING_AGG(activity, ',') ORDER BY timestamp AS activity_sequence`
  - The `ORDER BY` must be inside the aggregate call, not outside it.
- **`@K` is not standard DuckDB SQL syntax** for a parameter placeholder in a plain query.

- It does not provide a fully robust or clearly DuckDB-correct final query.

- build ordered activity sequences per case,
- group and count variants,
- take top K,
- join back to return original events for matching cases.
