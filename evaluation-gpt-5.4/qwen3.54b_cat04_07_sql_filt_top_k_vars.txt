   `ORDER BY case_count DESC LIMIT :k` is not deterministic when multiple variants have the same frequency. A stricter answer would define tie-breaking explicitly.

   `:k` is not the most standard DuckDB placeholder form. The note partially mitigates this, but the query as written may not run unchanged in many DuckDB contexts.

   Using `ARRAY_AGG(...)` and then grouping/joining on the resulting array/list is plausible in DuckDB, but a more robust answer might use a canonical string representation of the variant instead of relying on list equality semantics.
