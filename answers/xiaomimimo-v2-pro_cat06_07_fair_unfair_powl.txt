## Model 1: POWL Model with Potential Unfairness

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define all activities
receive_application = Transition(label="ReceiveApplication")
data_completeness_check = Transition(label="DataCompletenessCheck")
request_more_info = Transition(label="RequestMoreInfo")
skill_assessment = Transition(label="SkillAssessment")
cultural_fit_check = Transition(label="CulturalFitCheck")
community_affiliation_check = Transition(label="CommunityAffiliationCheck")
managerial_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# Create the loop for data completeness (Step 1)
# The loop executes DataCompletenessCheck, then either exits or
# executes RequestMoreInfo and then DataCompletenessCheck again
data_loop = OperatorPOWL(
    operator=Operator.LOOP, 
    children=[data_completeness_check, request_more_info]
)

# Create XOR choice for cultural fit evaluation (Step 3 - bias point)
# This represents the choice between standard cultural fit check
# and community affiliation check that gives preferential treatment
cultural_evaluation_xor = OperatorPOWL(
    operator=Operator.XOR, 
    children=[cultural_fit_check, community_affiliation_check]
)

# Build the overall workflow as a partial order
unfair_model = StrictPartialOrder(nodes=[
    receive_application,
    data_loop,
    skill_assessment,
    cultural_evaluation_xor,
    managerial_review,
    final_decision
])

# Define the execution order
unfair_model.order.add_edge(receive_application, data_loop)
unfair_model.order.add_edge(data_loop, skill_assessment)
unfair_model.order.add_edge(skill_assessment, cultural_evaluation_xor)
unfair_model.order.add_edge(cultural_evaluation_xor, managerial_review)
unfair_model.order.add_edge(managerial_review, final_decision)

print("POWL Model with Potential Unfairness created successfully!")
```

**Explanation of the unfair model:**
- The `* (DataCompletenessCheck, RequestMoreInfo)` loop ensures all applications have complete data before proceeding
- The `X (CulturalFitCheck, CommunityAffiliationCheck)` is where unfairness can occur - applicants can take either path, but the community affiliation path provides implicit advantages
- This XOR structure creates a branching point where bias can enter the process

fair_model = StrictPartialOrder(nodes=[
