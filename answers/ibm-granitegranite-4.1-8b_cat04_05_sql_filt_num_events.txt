To achieve the task described, we need to first count the number of events per `case_id` and then filter out those `case_id`s that have more than six events. After identifying the `case_id`s with six or fewer events, we can select all events belonging to these `case_id`s. Here is how you can write the DuckDB SQL query to accomplish this:

```sql
WITH case_event_counts AS (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT e.*
FROM event_log e
JOIN case_event_counts cec ON e.case_id = cec.case_id;
```

### Explanation:

1. **`WITH case_event_counts AS (...)`**: This common table expression (CTE) calculates the number of events for each `case_id` in the `event_log` table. The `GROUP BY case_id` clause groups the rows by `case_id`, and the `HAVING COUNT(*) <= 6` clause filters these groups to include only those where the count of events is six or fewer.

2. **Main Query**: The main query selects all columns (`*`) from the `event_log` table for rows (`e`) where the `case_id` matches any `case_id` in the `case_event_counts` CTE (`ce`). This effectively returns all events belonging to cases that have six or fewer events.

This approach ensures that we first identify the qualifying cases and then retrieve all events from those cases, aligning with the requirements of the task.
