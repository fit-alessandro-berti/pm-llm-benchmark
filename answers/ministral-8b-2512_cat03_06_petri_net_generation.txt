Below is the Python code to construct an accepting Petri net for the complex insurance claim management process described. The Petri net includes places for each state, transitions for activities, arcs to model the flow (including loops for re-submission and in-depth investigation), and initial/final markings.

### Key Features:
1. **Places** represent states (e.g., `claim_registration`, `document_verification`, `fraud_check`, etc.).
2. **Transitions** represent activities (e.g., `CR`, `DV`, `FC`, etc.).
3. **Arcs** connect places and transitions to model the process flow, including loops for re-submission (`RDS`) and in-depth investigation (`II`).
4. **Initial marking** (`im`) starts with a token in `claim_arrival`.
5. **Final marking** (`fm`) has a token in `claim_closed`.

### Python Code:
```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# Create the Petri net
net = PetriNet(name="Insurance_Claim_Management_Process")

# Define places (states)
claim_arrival = PetriNet.Place(name="claim_arrival")
claim_registration = PetriNet.Place(name="claim_registration")
document_verification = PetriNet.Place(name="document_verification")
re_document_submission = PetriNet.Place(name="re_document_submission")
fraud_check = PetriNet.Place(name="fraud_check")
in_depth_investigation = PetriNet.Place(name="in_depth_investigation")
claim_assessment = PetriNet.Place(name="claim_assessment")
approval = PetriNet.Place(name="approval")
payment = PetriNet.Place(name="payment")
notification = PetriNet.Place(name="notification")
claim_closed = PetriNet.Place(name="claim_closed")

# Add places to the net
net.places.add(claim_arrival)
net.places.add(claim_registration)
net.places.add(document_verification)
net.places.add(re_document_submission)
net.places.add(fraud_check)
net.places.add(in_depth_investigation)
net.places.add(claim_assessment)
net.places.add(approval)
net.places.add(payment)
net.places.add(notification)
net.places.add(claim_closed)

# Define transitions (activities)
cr = PetriNet.Transition(name="CR", label="Claim Registration")
dv = PetriNet.Transition(name="DV", label="Document Verification")
rds = PetriNet.Transition(name="RDS", label="Re-Document Submission")
fc = PetriNet.Transition(name="FC", label="Fraud Check")
ii = PetriNet.Transition(name="II", label="In-Depth Investigation")
ca = PetriNet.Transition(name="CA", label="Claim Assessment")
ap = PetriNet.Transition(name="AP", label="Approval")
p = PetriNet.Transition(name="P", label="Payment")
n = PetriNet.Transition(name="N", label="Notification")

# Add transitions to the net
net.transitions.add(cr)
net.transitions.add(dv)
net.transitions.add(rds)
net.transitions.add(fc)
net.transitions.add(ii)
net.transitions.add(ca)
net.transitions.add(ap)
net.transitions.add(p)
net.transitions.add(n)

# Add arcs to model the process flow
# Initial flow: claim_arrival -> CR -> claim_registration -> DV -> document_verification
petri_utils.add_arc_from_to(claim_arrival, cr, net)
petri_utils.add_arc_from_to(cr, claim_registration, net)
petri_utils.add_arc_from_to(claim_registration, dv, net)
petri_utils.add_arc_from_to(dv, document_verification, net)

# Loop for re-submission (DV -> RDS -> DV)
petri_utils.add_arc_from_to(document_verification, rds, net)
petri_utils.add_arc_from_to(rds, document_verification, net)

# Flow after document verification: DV -> FC -> fraud_check
petri_utils.add_arc_from_to(document_verification, fc, net)
petri_utils.add_arc_from_to(fc, fraud_check, net)

# Loop for in-depth investigation (FC -> II -> FC)
petri_utils.add_arc_from_to(fraud_check, ii, net)
petri_utils.add_arc_from_to(ii, fraud_check, net)

# Flow after fraud check: FC -> CA -> claim_assessment
petri_utils.add_arc_from_to(fraud_check, ca, net)
petri_utils.add_arc_from_to(ca, claim_assessment, net)

# Flow for approval (CA -> AP -> approval)
petri_utils.add_arc_from_to(claim_assessment, ap, net)
petri_utils.add_arc_from_to(ap, approval, net)

# Flow for payment (approval -> P -> payment)
petri_utils.add_arc_from_to(approval, p, net)
petri_utils.add_arc_from_to(p, payment, net)

# Flow for notification (payment -> N -> notification)
petri_utils.add_arc_from_to(payment, n, net)
petri_utils.add_arc_from_to(n, notification, net)

# Final flow: notification -> claim_closed
petri_utils.add_arc_from_to(notification, claim_closed, net)

# Define initial marking (im): token in claim_arrival
im = Marking()
im[claim_arrival] = 1

# Define final marking (fm): token in claim_closed
fm = Marking()
fm[claim_closed] = 1

# Optional: Remove unused places (e.g., claim_arrival, notification) if not needed in final marking

# del im[claim_arrival]
# petri_utils.remove_place(net, notification)
# del fm[notification]
```

### Explanation:
1. **Places** (`claim_arrival`, `claim_registration`, etc.) represent states in the process.
2. **Transitions** (`CR`, `DV`, `RDS`, etc.) represent activities.
3. **Arcs** connect places and transitions to model:
   - Linear flow (e.g., `claim_arrival -> CR -> claim_registration`).
   - Loops (e.g., `document_verification -> RDS -> document_verification` for re-submission).
   - Conditional paths (e.g., `fraud_check -> II -> fraud_check` for in-depth investigation).
4. **Initial marking** (`im`) starts with a token in `claim_arrival`.
5. **Final marking** (`fm`) ends with a token in `claim_closed`.
