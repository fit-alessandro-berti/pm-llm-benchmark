Assuming the intended process allows only these two valid paths:

1. **Fulfillment path:**  
   **Receive order  Check inventory  Authorize payment  Package items  Ship order  Close order**

2. **Cancellation path:**  
   **Receive order  Check inventory  Cancel order  Close order**

then the violating cases are:

| Case ID | Conformance | Explanation |
|---|---:|---|
| **1** |  Conforming | Follows the normal fulfillment path exactly: receive, check inventory, authorize payment, package, ship, close. |
| **2** |  Violating | **Package items occurs before Authorize payment.** The description says payment should be authorized before packaging, and packaging should occur only after authorization. The observed sequence is: receive  check  **package**  **authorize**  ship  close. |
| **3** |  Violating | Several issues occur. After **Cancel order**, the case continues with **Ship order**, which is explicitly disallowed: shipping should never occur after cancellation. Also, if the order is canceled due to insufficient inventory, the expected path is check inventory  cancel  close, not authorize payment  cancel  ship  close. In addition, shipping occurs without a prior **Package items** step. |
| **4** |  Conforming | Follows the cancellation path correctly: receive, check inventory, cancel, close. |
| **5** |  Violating | **Ship order occurs without Package items.** The normative process states that after payment authorization, items should be packaged before shipment. The observed sequence skips packaging: receive  check  authorize  **ship**  close. |

Therefore, the cases that violate the normative process are:

**Cases 2, 3, and 5.**

Summary of violated expectations:

- **Case 2:** Packaging before payment authorization.
- **Case 3:** Cancellation followed by shipping; shipment without packaging; mixed cancellation and fulfillment behavior.
- **Case 5:** Shipment without prior packaging.
