## POWL Model 1: Process with Potential Unfairness (XOR Branching)

This model includes an XOR choice after the Skill Assessment. One path is the standard Cultural Fit Check, and the other is the potentially biased Community Affiliation Check, which provides an implicit advantage to certain candidates.

**Activities/Nodes:**

| Label | Type | Description |
| :--- | :--- | :--- |
| `ReceiveApplication` | Transition | Initial application submission. |
| `DataCompletenessCheck` | Transition | Automated check for missing information (start of loop). |
| `RequestMoreInfo` | Transition | Applicant is prompted to provide missing details (loop body). |
| `SkillAssessment` | Transition | Online, domain-specific skill testing. |
| `CulturalFitCheck` | Transition | Standard assessment of values alignment. |
| `CommunityAffiliationCheck` | Transition | Check for local/club affiliations (potential bias point). |
| `ManagerialReview` | Transition | Human review of borderline candidates. |
| `FinalDecision` | Transition | Approve or reject candidate. |

**Model Structure:**

1. **Loop for Data Completeness:** A loop is defined between `DataCompletenessCheck` (A) and `RequestMoreInfo` (B).
   * $LoopNode = * ( DataCompletenessCheck, RequestMoreInfo )$

2. **Choice for Cultural/Affiliation Check:** An exclusive choice (XOR) is introduced between the two evaluation paths.
   * $XORNode = X ( CulturalFitCheck, CommunityAffiliationCheck )$

3. **Overall Partial Order:** Sequential execution of stages.

$$
PO_1 = \text{PO}(
    \text{nodes}=\{
        \text{ReceiveApplication},
        LoopNode,
        \text{SkillAssessment},
        XORNode,
        \text{ManagerialReview},
        \text{FinalDecision}
    \},
    \text{order}=\{
        \text{ReceiveApplication} \to LoopNode,
        LoopNode \to \text{SkillAssessment},
        \text{SkillAssessment} \to XORNode,
        XORNode \to \text{ManagerialReview},
        \text{ManagerialReview} \to \text{FinalDecision}
    \}
)
$$

## POWL Model 2: Process Without Unfairness (Eliminating Biased XOR)

This model ensures all candidates undergo the same cultural evaluation path by removing the `CommunityAffiliationCheck` as an alternative path. The process is now purely sequential and includes the required data completeness loop and skill checks.

**Activities/Nodes:**

| Label | Type | Description |
| :--- | :--- | :--- |
| `ReceiveApplication` | Transition | Initial application submission. |
| `DataCompletenessCheck` | Transition | Automated check for missing information (start of loop). |
| `RequestMoreInfo` | Transition | Applicant is prompted to provide missing details (loop body). |
| `SkillAssessment` | Transition | Online, domain-specific skill testing. |
| `StandardCulturalFit` | Transition | Universal, merit-based assessment of values alignment. |
| `ManagerialReview` | Transition | Human review of borderline candidates. |
| `FinalDecision` | Transition | Approve or reject candidate. |

**Model Structure:**

1. **Loop for Data Completeness:** The loop structure remains the same to ensure data quality.
   * $LoopNode = * ( DataCompletenessCheck, RequestMoreInfo )$

2. **Overall Partial Order:** Sequential execution of stages, where all applicants must pass through `StandardCulturalFit`.

$$
PO_2 = \text{PO}(
    \text{nodes}=\{
        \text{ReceiveApplication},
        LoopNode,
        \text{SkillAssessment},
        \text{StandardCulturalFit},
        \text{ManagerialReview},
        \text{FinalDecision}
    \},
    \text{order}=\{
        \text{ReceiveApplication} \to LoopNode,
        LoopNode \to \text{SkillAssessment},
        \text{SkillAssessment} \to \text{StandardCulturalFit},
        \text{StandardCulturalFit} \to \text{ManagerialReview},
        \text{ManagerialReview} \to \text{FinalDecision}
    \}
)
$$
