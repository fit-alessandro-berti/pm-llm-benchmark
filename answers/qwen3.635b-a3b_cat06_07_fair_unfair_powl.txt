### 1. POWL Model with Potential Unfairness (Biased)
This model includes the XOR branching at the cultural fit stage. One path represents the standard evaluation, while the other represents the `CommunityAffiliationCheck` that introduces implicit preferential treatment.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# --- Activity Definitions ---
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# --- Control Flow Operators ---
# Loop: Check completeness -> if missing, request more -> loop back to check
loop_data = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck, RequestMoreInfo])

# XOR: Standard cultural fit OR biased community affiliation check
xor_culture = OperatorPOWL(operator=Operator.XOR, children=[CulturalFitCheck, CommunityAffiliationCheck])

# --- Root Partial Order ---
root_biased = StrictPartialOrder(
    nodes=[ReceiveApplication, loop_data, SkillAssessment, xor_culture, ManagerialReview, FinalDecision]
)

# Define sequential execution order
root_biased.order.add_edge(ReceiveApplication, loop_data)
root_biased.order.add_edge(loop_data, SkillAssessment)
root_biased.order.add_edge(SkillAssessment, xor_culture)
root_biased.order.add_edge(xor_culture, ManagerialReview)
root_biased.order.add_edge(ManagerialReview, FinalDecision)
```

**Bias Explanation:** The `xor_culture` node creates a divergent path. In practice, applicants routed to `CommunityAffiliationCheck` receive an implicit score uplift based on local/community ties, while others go through the merit-based `CulturalFitCheck`. This violates the principle of uniform evaluation.

# --- Control Flow Operators ---
# Loop: Identical to biased model (ensures data completeness for all)
loop_data = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck, RequestMoreInfo])

# No XOR branch. All applicants proceed sequentially through the same cultural fit evaluation.
# (xor_culture is removed entirely)

root_fair = StrictPartialOrder(

**Bias Removal Explanation:** By eliminating the `XOR` operator and the `CommunityAffiliationCheck` activity, the workflow enforces a single, deterministic path for the cultural evaluation stage. This guarantees that no applicant receives preferential routing or implicit scoring based on community affiliations, aligning the process with fair and transparent hiring principles.
