# DuckDB Query: Filter Cases by Event Count

Here's a query that returns all events belonging only to cases with **six or fewer events**:

## Solution using a window function (recommended)

```sql
SELECT case_id, activity, timestamp
FROM (
    SELECT
        *,
        COUNT(*) OVER (PARTITION BY case_id) AS events_per_case
    FROM event_log
)
WHERE events_per_case <= 6
ORDER BY case_id, timestamp;
```

**How it works:**
1. The window function `COUNT(*) OVER (PARTITION BY case_id)` computes the total number of events per case **without collapsing rows**, so every event keeps its full detail.
2. The outer query filters out rows belonging to cases where the count exceeds six.
3. Ordering by `case_id, timestamp` keeps each case's events together in chronological order (optional, but typical for event logs).

## Alternative: Subquery with `GROUP BY` / `HAVING`

```sql
SELECT case_id, activity, timestamp
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
ORDER BY case_id, timestamp;
```

**How it works:**
1. The subquery groups events by `case_id` and keeps only those cases whose event count is at most six (`HAVING COUNT(*) <= 6`).
2. The outer query returns all original event rows for those qualifying cases.

## Notes

- **Why not just `GROUP BY` alone?** A plain `GROUP BY case_id` would collapse each case into one row — you'd lose the individual events. That's why we need either a window function or a semi-join back to the full table.
- **Performance:** Both approaches work well in DuckDB; the window-function version requires only a single scan of the table, which is typically the most efficient on large event logs.
- If `timestamp` clashes with a reserved word in your setup, quote it as `"timestamp"`.
