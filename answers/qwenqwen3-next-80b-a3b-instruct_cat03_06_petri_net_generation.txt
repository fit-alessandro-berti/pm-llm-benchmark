Here's the complete Python code to construct an **accepting Petri net** for the **insurance claim management process** as described. The net includes:

- **Places** for states between activities (e.g., “after CR”, “after DV”, etc.)
- **Transitions** labeled with activity names
- **Loops** for:
  - Re-Document Submission (RDS) after failed DV
  - In-depth Investigation (II) after suspicious FC (can repeat)
- **Initial marking**: one token in the start place (`start`)
- **Final marking**: one token in the closure place (`closed`)

We use `pm4py.objects.petri_net.obj.PetriNet`, `Marking`, and `petri_utils` as required.

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# Initialize the Petri net
net = PetriNet(name="Insurance_Claim_Management")

# Create places (states between activities)
start = PetriNet.Place(name="start")
after_cr = PetriNet.Place(name="after_CR")
after_dv = PetriNet.Place(name="after_DV")
after_rds = PetriNet.Place(name="after_RDS")  # after re-document submission
after_fc = PetriNet.Place(name="after_FC")
after_ii = PetriNet.Place(name="after_II")    # after in-depth investigation
after_ca = PetriNet.Place(name="after_CA")
after_ap = PetriNet.Place(name="after_AP")
after_p = PetriNet.Place(name="after_P")
after_n = PetriNet.Place(name="after_N")
closed = PetriNet.Place(name="closed")

# Add places to the net
net.places.add(start)
net.places.add(after_cr)
net.places.add(after_dv)
net.places.add(after_rds)
net.places.add(after_fc)
net.places.add(after_ii)
net.places.add(after_ca)
net.places.add(after_ap)
net.places.add(after_p)
net.places.add(after_n)
net.places.add(closed)

# Create transitions (activities) with labels
cr = PetriNet.Transition(name="transition_CR", label="Claim Registration")
dv = PetriNet.Transition(name="transition_DV", label="Document Verification")
rds = PetriNet.Transition(name="transition_RDS", label="Re-Document Submission")
fc = PetriNet.Transition(name="transition_FC", label="Fraud Check")
ii = PetriNet.Transition(name="transition_II", label="In-depth Investigation")
ca = PetriNet.Transition(name="transition_CA", label="Claim Assessment")
ap = PetriNet.Transition(name="transition_AP", label="Approval")
p = PetriNet.Transition(name="transition_P", label="Payment")
n = PetriNet.Transition(name="transition_N", label="Notification")
cl = PetriNet.Transition(name="transition_CL", label="Closure")

# Add transitions to the net
net.transitions.add(cr)
net.transitions.add(dv)
net.transitions.add(rds)
net.transitions.add(fc)
net.transitions.add(ii)
net.transitions.add(ca)
net.transitions.add(ap)
net.transitions.add(p)
net.transitions.add(n)
net.transitions.add(cl)

# Define arcs (using petri_utils.add_arc_from_to)
# Start  CR
petri_utils.add_arc_from_to(start, cr, net)

# CR  after_CR
petri_utils.add_arc_from_to(cr, after_cr, net)

# after_CR  DV
petri_utils.add_arc_from_to(after_cr, dv, net)

# DV  after_DV
petri_utils.add_arc_from_to(dv, after_dv, net)

# after_DV  RDS (if documents incomplete) — loop back
petri_utils.add_arc_from_to(after_dv, rds, net)
# RDS  after_RDS
petri_utils.add_arc_from_to(rds, after_rds, net)
# after_RDS  DV (loop back to re-verify)
petri_utils.add_arc_from_to(after_rds, dv, net)

# after_DV  FC (if documents are valid)
petri_utils.add_arc_from_to(after_dv, fc, net)

# FC  after_FC
petri_utils.add_arc_from_to(fc, after_fc, net)

# after_FC  II (if fraud suspected) — loop back
petri_utils.add_arc_from_to(after_fc, ii, net)
# II  after_II
petri_utils.add_arc_from_to(ii, after_ii, net)
# after_II  FC (loop back for multiple investigations)
petri_utils.add_arc_from_to(after_ii, fc, net)

# after_FC  CA (if no fraud detected)
petri_utils.add_arc_from_to(after_fc, ca, net)

# CA  after_CA
petri_utils.add_arc_from_to(ca, after_ca, net)

# after_CA  AP (if amount > threshold)
petri_utils.add_arc_from_to(after_ca, ap, net)

# AP  after_AP
petri_utils.add_arc_from_to(ap, after_ap, net)

# after_CA  after_AP (if no approval needed — bypass AP)
petri_utils.add_arc_from_to(after_ca, after_ap, net)

# after_AP  P
petri_utils.add_arc_from_to(after_ap, p, net)

# P  after_P
petri_utils.add_arc_from_to(p, after_p, net)

# after_P  N
petri_utils.add_arc_from_to(after_p, n, net)

# N  after_N
petri_utils.add_arc_from_to(n, after_n, net)

# after_N  CL
petri_utils.add_arc_from_to(after_n, cl, net)

# CL  closed
petri_utils.add_arc_from_to(cl, closed, net)

# Define initial marking: one token at start
im = Marking()
im[start] = 1

# Define final marking: one token at closed
fm = Marking()
fm[closed] = 1

# Optional: Print net structure for verification
print("Petri Net constructed successfully.")
print(f"Places: {[p.name for p in net.places]}")
print(f"Transitions: {[t.label for t in net.transitions]}")
print(f"Initial Marking: {dict(im)}")
print(f"Final Marking: {dict(fm)}")
```

###  **Key Design Choices Explained**

| Feature | Implementation |
|--------|----------------|
| **Re-Document Submission (RDS)** | Loop: `after_DV  RDS  after_RDS  DV` — allows multiple re-submissions |
| **In-depth Investigation (II)** | Loop: `after_FC  II  after_II  FC` — allows multiple investigations until cleared |
| **Approval (AP) Bypass** | Direct arc from `after_CA` to `after_AP` to allow skip if amount  threshold |
| **Final State** | Only `closed` place holds the final token — ensures process completed fully |
| **Initial State** | Token only in `start` — represents new claim arrival |
| **No Deadlocks** | Every transition has a clear path to `closed` via defined logic |
