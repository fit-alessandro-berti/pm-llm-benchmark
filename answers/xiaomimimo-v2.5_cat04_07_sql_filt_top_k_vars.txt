Here's a DuckDB SQL query that addresses all the requirements:

```sql
WITH case_sequences AS (
    -- Step 1: Extract ordered activity sequence for each case
    SELECT 
        case_id,
        LIST(activity ORDER BY timestamp) AS activity_sequence
    FROM event_log
    GROUP BY case_id
),
variant_counts AS (
    -- Step 2 & 3: Count cases per variant and rank them
    SELECT 
        activity_sequence,
        COUNT(*) AS case_count,
        RANK() OVER (ORDER BY COUNT(*) DESC) AS variant_rank
    FROM case_sequences
    GROUP BY activity_sequence
)
-- Step 4: Return events only from cases in top K variants
SELECT el.*
FROM event_log el
JOIN case_sequences cs ON el.case_id = cs.case_id
JOIN variant_counts vc ON cs.activity_sequence = vc.activity_sequence
WHERE vc.variant_rank <= {{K}};  -- Replace {{K}} with desired integer (e.g., 10)
```

## Explanation:

1. **`case_sequences` CTE**: 
   - Groups by `case_id` and creates an ordered list of activities using `LIST(activity ORDER BY timestamp)`
   - This gives us the complete activity sequence (variant) for each case

2. **`variant_counts` CTE**:
   - Groups by the `activity_sequence` to count how many cases follow each variant
   - Uses `RANK()` to assign a rank based on frequency (highest count = rank 1)
   - This handles ties correctly (variants with same count get same rank)

3. **Final query**:
   - Joins back to the original event log through the case sequences and variant counts
   - Filters to only include cases belonging to variants with rank  K
   - Returns all original events from those qualifying cases

**Example usage:**
If you want the top 5 variants:
```sql
-- Set K value in your query
WHERE vc.variant_rank <= 5;
```

**Key features:**
- Preserves the original timestamp ordering within each case
- Handles ties in variant counts (using RANK instead of ROW_NUMBER)
- Returns complete event records, not just the sequences
- Clean and efficient with proper CTE organization
