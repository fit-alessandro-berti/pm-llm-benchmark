### 1. Sales Lane (Top)
The process begins in the Sales lane on a scheduled basis.

*   **Trigger:** Starts with a Timer Event labeled **"Monday morning"**.
*   **Subprocess ("Check customer's order"):**

---

### 2. Production Lane (Second from Top)
This lane handles the planning and manufacturing of the product.

*   **Trigger:** Starts with a Message Start Event labeled **"Order details arrive"** (triggered by the "Send order details" task in Sales).
*   **Parallel Gateway (AND - splitting):**
    *   **Top branch:** Directs to the task **"Plan production"**.
    *   **Bottom branch:** Waits at an Intermediate Message Catch Event labeled **"Material not faulty"** (waiting for confirmation from Procurement).
*   **Parallel Gateway (AND - joining):** Both the production plan and the material confirmation must be complete to proceed.
*   **Task:** **"Manufacture product"**.
*   **End:** Ends with a Message End Event labeled **"Product complete"**, which sends a message down to the Warehouse lane.

---

### 3. Procurement Lane (Third from Top)
This lane manages material sourcing, checking, and potential compensation logic.

*   **Trigger:** Starts with a Message Start Event labeled **"Order details arrive"** (triggered by the "Send order details" task in Sales).
*   **Subprocess ("Material Compensation Subprocess"):**
    *   Starts with a None Start Event.
    *   **Task:** **"Order materials"**. This task has a Compensation Boundary Event attached.
    *   If compensation is triggered, it executes the **"Return material"** task.
*   **Task:** **"Check materials"**.
    *   **Normal Flow:** Leads to a Message End Event that sends a **"Material not faulty"** message to the Production lane.
    *   **Error Flow (Boundary Event):** An Error Boundary Event is attached to "Check materials". If an error/defect is detected, it triggers a Compensation Throw Event labeled **"Order cancelation"** (which initiates the "Return material" compensation task in the previous subprocess) and then ends the process.

---

### 4. Warehouse Lane (Bottom)
This lane handles the physical delivery of the completed product.

*   **Trigger:** Starts with a Message Start Event labeled **"Product complete"** (triggered by the "Product complete" Message End Event in the Production lane).
*   **Task:** **"Send product"**.
*   **End:** Ends with a Message End Event labeled **"Product sent"**, which sends a message flow back up to the "Product sent" Intermediate Catch Event in the Sales lane.
