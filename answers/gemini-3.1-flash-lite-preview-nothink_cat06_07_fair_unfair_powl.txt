Below are the POWL model definitions corresponding to the GlobalTeq Solutions hiring process.

### 1. POWL Model with Potential Unfairness
This model implements an XOR choice after the Skill Assessment. One branch forces a standard cultural evaluation, while the other branch allows for a "CommunityAffiliationCheck" that provides the documented preferential treatment.

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
receive = Transition(label="ReceiveApplication")
data_check = Transition(label="DataCompletenessCheck")
req_info = Transition(label="RequestMoreInfo")
skill_test = Transition(label="SkillAssessment")
std_cultural = Transition(label="StandardCulturalFitCheck")
comm_check = Transition(label="CommunityAffiliationCheck")
mgr_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# Loop for data completeness: Loop(DataCompletenessCheck, RequestMoreInfo)
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, req_info])

# XOR branch for cultural fit (The bias point)
cultural_choice = OperatorPOWL(operator=Operator.XOR, children=[std_cultural, comm_check])

# Sequential structure
# Receive -> DataLoop -> SkillTest -> CulturalChoice -> ManagerialReview -> FinalDecision
root_unfair = StrictPartialOrder(nodes=[receive, data_loop, skill_test, cultural_choice, mgr_review, final_decision])
root_unfair.order.add_edge(receive, data_loop)
root_unfair.order.add_edge(data_loop, skill_test)
root_unfair.order.add_edge(skill_test, cultural_choice)
root_unfair.order.add_edge(cultural_choice, mgr_review)
root_unfair.order.add_edge(mgr_review, final_decision)
```

---

### 2. POWL Model Without Unfairness
In this model, the XOR branch is removed. All candidates must pass through the `StandardCulturalFitCheck`, ensuring that the preferential treatment provided by the `CommunityAffiliationCheck` is entirely eliminated from the process.

```python
# Reuse activities from above
# Loop remains the same
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, req_info])

# Sequential structure without the biased XOR branch
# Receive -> DataLoop -> SkillTest -> StandardCulturalFitCheck -> ManagerialReview -> FinalDecision
root_fair = StrictPartialOrder(nodes=[receive, data_loop, skill_test, std_cultural, mgr_review, final_decision])
root_fair.order.add_edge(receive, data_loop)
root_fair.order.add_edge(data_loop, skill_test)
root_fair.order.add_edge(skill_test, std_cultural)
root_fair.order.add_edge(std_cultural, mgr_review)
root_fair.order.add_edge(mgr_review, final_decision)
```

### Key Differences Explained:
*   **Structural Change:** In the "Unfair" model, we use an `OperatorPOWL` with `Operator.XOR` to introduce a branching path. By including `CommunityAffiliationCheck` as a child node in that XOR branch, the process design explicitly supports a mechanism for differentiated (and biased) treatment.
*   **Standardization:** In the "Fair" model, the `OperatorPOWL` for the choice is removed. By replacing the branch with a direct sequential execution of `StandardCulturalFitCheck`, we ensure that every applicant is subject to the same evaluation criteria, thereby removing the architectural source of the bias described.
