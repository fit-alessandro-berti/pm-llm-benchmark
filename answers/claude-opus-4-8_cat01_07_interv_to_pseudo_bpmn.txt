# BPMN-Like Process Model: Invoice Approval Process

## 1. Participants (Pools / Lanes)

| Role / System | Description |
|---|---|
| **Accounts Payable Clerk (Mary)** | Receives, logs, matches invoices; initiates manager approval; schedules payment |
| **Purchasing Officer (Dan)** | Confirms receipt of goods/services; validates invoice against PO |
| **Department Manager (Carlos)** | Provides final approval/rejection |
| **AP Supervisor (Rita)** | Handles escalated exceptions; coordinates retrospective POs |
| **Supplier Relationship Manager (Karen)** | Resolves supplier-side issues (missing PO, pricing) |
| **Supplier** (external) | Provides invoice, corrected invoice, missing PO details |
| **Accounting/ERP System** | Logs invoices, performs matching, sends automated approval notifications |

---

## 2. Artifacts / Data Objects

- **Invoice** (incoming, usually via email)
- **Purchase Order (PO)** — reference for matching
- **Goods/Services Receipt Confirmation** (from Purchasing)
- **Confirmation Email** (Purchasing  AP)
- **Approval/Rejection Notification** (System  Manager)
- **Corrected/Revised Invoice** (from Supplier)
- **Retrospective PO** (created during exception handling)
- **Payment Schedule Entry** (output)

---

## 3. Main Process Flow

```
[START EVENT] Invoice received (via email) — Lane: AP Clerk
        
        
(Task A) Check for valid PO number — Lane: AP Clerk
        
        
<Gateway G1: PO present?> (Exclusive XOR)
         NO  (Task B) Request PO/details from supplier  [See Exception Path E1]
        
         YES 
        
(Task C) Log invoice into accounting system — Lane: AP Clerk / System
        
        
(Task D) Forward to Purchasing to confirm receipt of goods/services — Lane: AP Clerk
        
        
(Task E) Confirm goods/services received & compare invoice vs PO
         (quantities, prices, delivery charges) — Lane: Purchasing (Dan)
        
        
<Gateway G2: Invoice matches PO & receipt confirmed?> (Exclusive XOR)
         NO (discrepancy / mismatch)  [See Exception Path E2]
        
         YES 
        
(Task F) Send confirmation email to Accounts Payable — Lane: Purchasing
        
        
(Task G) Match invoice line items against PO in system — Lane: AP Clerk
         [Note: see Ambiguity #1 regarding overlap with Task E]
        
        
<Gateway G3: All matched?> (Exclusive XOR)
         NO (discrepancy)  [See Exception Path E2]
        
         YES 
        
(Task H) System sends automatic approval request to Manager — Lane: System
        
        
(Task I) Manager reviews invoice, PO, and confirmation — Lane: Manager (Carlos)
        
        
<Gateway G4: Manager decision?> (Exclusive XOR)
         REJECT / SUSPICIOUS (e.g., large budget variance,
             unexpected items)  [See Exception Path E3]
        
         APPROVE 
        
(Task J) Schedule invoice for next payment run — Lane: AP Clerk
        
        
[END EVENT] Invoice approved & scheduled for payment
```

## 4. Exception / Alternate Paths

### Exception Path E1 — Missing PO Number (from Gateway G1 = NO)
```
(Task B) Request PO/details from supplier — Lane: AP Clerk
        
        
<Gateway E1-G: Source of PO resolution?>
         Standard case  Supplier provides PO/revised invoice
                              RETURN to (Task A) Check for valid PO  [rework loop]
        
         No PO exists (e.g., informal/contracted services)
                             Escalate to AP Supervisor (Rita)
                             (Task K) Decide: create Retrospective PO
                                          OR reject invoice — Lane: AP Supervisor
                                   Retrospective PO created  RETURN to (Task C) Log invoice
                                   Reject  [END EVENT] Invoice rejected
```

### Exception Path E2 — Invoice/PO Discrepancy (from Gateway G2 or G3 = NO)
*(e.g., incorrect pricing, missing/extra items, delivery charge mismatch)*
```
<Gateway E2-G: Resolution route?> (parallel/alternative options described by interviewees)
         (Task L) Contact supplier directly — Lane: Purchasing
        
         (Task M) Ask AP to HOLD invoice until corrected version arrives — Lane: Purchasing/AP
        
         (Task N) Loop in Supplier Relationship Manager (Karen) for:
                       - missing PO retrieval, or
                       - pricing verification against contract
                       — Lane: Supplier Relationship Mgr
        
        
<Gateway E2-G2: Discrepancy resolved easily?>
         YES  Supplier sends corrected invoice / correct price confirmed
                     RE-ENTER main flow at (Task E) Purchasing confirmation
        
         NO (cannot be resolved easily)
                   (Task O) Escalate to AP Supervisor (Rita) — Lane: AP Clerk
                   (Task P) Supervisor coordinates between departments/suppliers
                        Resolved  RE-ENTER main flow at (Task E)
                        Unresolvable  [END EVENT] Invoice rejected
```

### Exception Path E3 — Manager Rejection / Hold (from Gateway G4)
```
(Task Q) Manager rejects OR requests hold & investigation — Lane: Manager
        
        
(Task R) AP Clerk holds & investigates — Lane: AP Clerk
          (may involve Supervisor / Purchasing / Supplier Mgr)
        
<Gateway E3-G: Issue resolved?>
         YES  RE-ENTER main flow at (Task H) request manager approval
         NO   [END EVENT] Invoice rejected
```

## 5. Consolidated Decision Points (Gateways) Summary

| ID | Location | Type | Condition | Outcomes |
|---|---|---|---|---|
| G1 | After PO check | XOR | Is a valid PO number present? | Yes  Log; No  Request from supplier (E1) |
| G2 | After Purchasing review | XOR | Receipt confirmed & invoice matches PO? | Yes  Confirm; No  Discrepancy (E2) |
| G3 | After system matching | XOR | Line items match? | Yes  Manager approval; No  Discrepancy (E2) |
| G4 | After Manager review | XOR | Approve / Reject / Hold? | Approve  Schedule payment; Reject/Hold  E3 |
| E1-G | PO missing path | XOR | Does a PO exist or can supplier supply one? | Provide  loop; None  Supervisor (retrospective PO or reject) |
| E2-G2 | Discrepancy path | XOR | Easily resolvable? | Yes  re-enter flow; No  escalate to Supervisor |
| E3-G | Manager hold path | XOR | Resolved after investigation? | Yes  re-approve; No  reject |

## 6. Noted Ambiguities & Conflicting Accounts

1. **Overlap between Purchasing review (Task E) and AP matching (Task G):**
   Mary describes **two separate matching activities** — Purchasing confirms receipt + matches, *then* she matches line items against the PO in the system. Dan describes himself as the primary matcher. **Uncertainty:** these may be one consolidated matching step or two redundant ones. Modeled here as two sequential tasks, but they may merge in practice.

2. **Order of confirmation vs. matching:**
   Mary: forward to Purchasing  confirmation  *then* match line items  manager.
   Dan: he matches *and* confirms, then it goes to the manager.
   The exact sequencing of AP's own line-item match relative to Purchasing's confirmation is **not fully consistent** across accounts.

3. **Who initiates "hold":**
   Both Purchasing (Dan) and AP can request a hold during discrepancies. The triggering owner varies by case.

4. **Manager rejection handling:**
   Carlos says he "might reject it or ask Mary to hold it." It is unclear whether rejection is terminal or always routes back through investigation (E3). Modeled as routing to investigation with possible terminal rejection.

5. **Karen's (Supplier Mgr) entry point:**
   She is "looped in" by either AP or Purchasing during discrepancies/missing PO. She is modeled as a sub-participant within Exception Path E2, but could also be invoked during E1. **Non-deterministic entry.**

6. **Retrospective PO scenario:**
   Only Rita mentions invoices with *no PO at all* (informal services). This is a distinct variant feeding into E1's escalation branch rather than the standard "supplier forgot to include PO" case.

## 7. Suggested Diagram Construction Notes

- Use a **collaboration diagram with 5–7 pools/lanes** (AP Clerk, Purchasing, Manager, AP Supervisor, Supplier Relationship Manager, plus external Supplier pool and a System lane).
- Place **happy path** (Tasks A  C  D  E  F  G  H  I  J) as the central horizontal flow.
- Render **Exception Paths E1–E3** as branches that **loop back** into the main flow via re-entry points (Task A, C, E, or H), with possible terminal **"Invoice Rejected" end events**.
- Use **message flows** (dashed lines) for all supplier communications and system notifications.
- Mark ambiguous tasks (G overlap, sequencing) with **annotation/text-annotation artifacts** referencing the notes in Section 6.
