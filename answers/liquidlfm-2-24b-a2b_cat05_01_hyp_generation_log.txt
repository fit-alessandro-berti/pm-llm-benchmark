### SQL Queries to Investigate Anomalies

#### **1. Validate Event Sequence for Case 1002**
```sql
SELECT 
  case_id, 
  event_id, 
  activity, 
  timestamp
FROM order_event_log
WHERE case_id = 1002
ORDER BY event_id;
```
*Purpose:* Check if events are logged in the correct order.  

#### **2. Check for Missing "Validate Stock" Event**
```sql
SELECT 
  case_id, 
  event_id, 
  activity
FROM order_event_log
WHERE case_id = 1002 
  AND activity != 'Validate Stock'
  AND event_id NOT IN (
    SELECT event_id FROM order_event_log 
    WHERE case_id = 1002 AND activity = 'Perform Credit Check'
  );
```
*Purpose:* Identify missing `Validate Stock` events.  

#### **3. Detect Late Shipment Confirmations**
```sql
SELECT 
  case_id, 
  activity, 
  timestamp, 
  resource
FROM order_event_log
WHERE activity = 'Confirm Shipment' 
  AND late_confirmation = 'Y';
```
*Purpose:* Highlight delayed shipments.  

#### **4. Verify Payment Receipt for Orders**
```sql
SELECT 
  o.case_id, 
  o.order_value, 
  oe.amount AS received_amount
FROM orders o
LEFT JOIN order_event_log oe ON o.case_id = oe.case_id 
  AND oe.activity = 'Receive Payment'
WHERE oe.amount IS NULL;
```
*Purpose:* Find orders where payment was not received.  

#### **5. Identify Resource Duplication**
```sql
SELECT 
  resource, 
  COUNT(*) AS event_count
FROM order_event_log
GROUP BY resource
HAVING COUNT(*) > 1;
```
*Purpose:* Detect overused resources.
