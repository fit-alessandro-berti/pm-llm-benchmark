Each key is a pair of activities (the first activity always occurs before the second in a trace) and each value is a tuple **(average_time_seconds, standard_deviation_seconds)**.  
The numbers are rough, datadriven estimates that capture typical leadtimes, processing times, and variability in a hightech electronics supply chain.

### How to use the dictionary
```python
pair = ('OP', 'PT')
avg, std = temporal_profile[pair]
print(f"Average time from OP to PT: {avg/86400:.1f} days "
      f"Ý {std/86400:.1f} days")
```

The dictionary captures a **variety of temporal relationships**:

* **Direct neighbours** (e.g., `('SS','OP')`, `('PT','PK')`)  short, relatively lowvariance intervals.  
* **Multistep gaps** (e.g., `('SS','DT')`, `('RC','AS')`)  longer intervals with larger variability because they accumulate the uncertainties of all intermediate activities.  

Feel free to extend the profile with additional pairs or refine the numbers based on actual eventlog statistics.
