```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# ============================================================
# POWL Model 1: With Potential Unfairness (Biased XOR Branch)
# ============================================================

# --- Activity Definitions ---
receive_app_1 = Transition(label="ReceiveApplication")
data_check_1 = Transition(label="DataCompletenessCheck")
request_info_1 = Transition(label="RequestMoreInfo")
skill_assess_1 = Transition(label="SkillAssessment")
cultural_fit_1 = Transition(label="CulturalFitCheck")
community_aff_1 = Transition(label="CommunityAffiliationCheck")
manager_review_1 = Transition(label="ManagerialReview")
final_decision_1 = Transition(label="FinalDecision")

# --- Operators ---
# Loop: DataCompletenessCheck, and if incomplete, RequestMoreInfo then repeat
data_loop_1 = OperatorPOWL(operator=Operator.LOOP, children=[data_check_1, request_info_1])

# XOR: Either standard CulturalFitCheck OR biased CommunityAffiliationCheck
# This is the unfairness point — applicants with community affiliations
# are routed to a path that gives them a subtle advantage.
biased_xor_1 = OperatorPOWL(operator=Operator.XOR, children=[cultural_fit_1, community_aff_1])

# --- Partial Order (Sequential Workflow) ---
unfair_model = StrictPartialOrder(
    nodes=[receive_app_1, data_loop_1, skill_assess_1, biased_xor_1, manager_review_1, final_decision_1]
)
unfair_model.order.add_edge(receive_app_1, data_loop_1)
unfair_model.order.add_edge(data_loop_1, skill_assess_1)
unfair_model.order.add_edge(skill_assess_1, biased_xor_1)
unfair_model.order.add_edge(biased_xor_1, manager_review_1)
unfair_model.order.add_edge(manager_review_1, final_decision_1)

print("=== POWL Model 1: With Potential Unfairness ===")
print("Structure: ReceiveApplication -> *(DataCompletenessCheck, RequestMoreInfo) -> "
      "SkillAssessment -> X(CulturalFitCheck, CommunityAffiliationCheck) -> "
      "ManagerialReview -> FinalDecision")
print("Bias point: The XOR branch allows applicants with community affiliations "
      "to bypass the standard cultural fit check and receive preferential scoring.\n")


# ============================================================
# POWL Model 2: Without Unfairness (Fair Process)
# ============================================================

# --- Activity Definitions ---
receive_app_2 = Transition(label="ReceiveApplication")
data_check_2 = Transition(label="DataCompletenessCheck")
request_info_2 = Transition(label="RequestMoreInfo")
skill_assess_2 = Transition(label="SkillAssessment")
cultural_fit_2 = Transition(label="CulturalFitCheck")
manager_review_2 = Transition(label="ManagerialReview")
final_decision_2 = Transition(label="FinalDecision")

# --- Operators ---
# Loop: same data completeness loop as before
data_loop_2 = OperatorPOWL(operator=Operator.LOOP, children=[data_check_2, request_info_2])

# No XOR branch — ALL applicants go through the same CulturalFitCheck.
# The CommunityAffiliationCheck activity is entirely removed.

# --- Partial Order (Sequential Workflow) ---
fair_model = StrictPartialOrder(
    nodes=[receive_app_2, data_loop_2, skill_assess_2, cultural_fit_2, manager_review_2, final_decision_2]
)
fair_model.order.add_edge(receive_app_2, data_loop_2)
fair_model.order.add_edge(data_loop_2, skill_assess_2)
fair_model.order.add_edge(skill_assess_2, cultural_fit_2)
fair_model.order.add_edge(cultural_fit_2, manager_review_2)
fair_model.order.add_edge(manager_review_2, final_decision_2)

print("=== POWL Model 2: Without Unfairness ===")
print("Structure: ReceiveApplication -> *(DataCompletenessCheck, RequestMoreInfo) -> "
      "SkillAssessment -> CulturalFitCheck -> ManagerialReview -> FinalDecision")
print("All applicants follow the same path with no community-affiliation-based branching.")
```

### Explanation of the Two Models

**Model 1 — With Potential Unfairness:**

| Step | Element | Description |
|------|---------|-------------|
| 1 | `ReceiveApplication` | Application intake |
| 2 | `*(DataCompletenessCheck, RequestMoreInfo)` | **Loop**: Check data; if incomplete, request more info and re-check |
| 3 | `SkillAssessment` | Automated skill test |
| 4 | **`X(CulturalFitCheck, CommunityAffiliationCheck)`** | ** BIAS POINT — XOR choice**: Applicants are routed to *either* a standard cultural fit evaluation *or* a community-affiliation-based path that implicitly boosts their score |
| 5 | `ManagerialReview` | Human review of borderline candidates |
| 6 | `FinalDecision` | Hire or reject |

The **XOR branch** is the source of unfairness: applicants flagged with community affiliations (e.g., "Global Business Leaders Club" members, local residents) are diverted to `CommunityAffiliationCheck`, which applies a subjective uplift. Other applicants go through the standard `CulturalFitCheck` without this advantage.

**Model 2 — Without Unfairness:**

| Step | Element | Description |
|------|---------|-------------|
| 1 | `ReceiveApplication` | Application intake |
| 2 | `*(DataCompletenessCheck, RequestMoreInfo)` | **Loop**: Same data completeness loop |
| 3 | `SkillAssessment` | Same automated skill test |
| 4 | `CulturalFitCheck` | **Single, uniform** cultural fit evaluation for **all** applicants |
| 5 | `ManagerialReview` | Human review |
| 6 | `FinalDecision` | Hire or reject |

The **XOR branch is removed entirely**. Every applicant follows the same sequential path through a single `CulturalFitCheck`, eliminating the structural mechanism through which community-affiliation bias could be introduced.
