Here is the DuckDB SQL query that fulfills all your requirements. It uses Common Table Expressions (CTEs) to break down the logic into clear, manageable steps. 

*(Note: Replace `5` in the `LIMIT` clause with your desired value for **K**).*

```sql
WITH CaseVariants AS (
    -- 1. Determine the sequence of activities for each case_id, ordered by timestamp
    SELECT 
        case_id,
        string_agg(activity, ' -> ' ORDER BY timestamp) AS variant
    FROM event_log
    GROUP BY case_id
),
TopVariants AS (
    -- 2 & 3. Group cases by their variant, count the frequency, and select the top K
    SELECT 
        variant
    FROM CaseVariants
    GROUP BY variant
    ORDER BY COUNT(case_id) DESC
    LIMIT 5 -- Replace 5 with your desired K
)
-- 4. Return all original events that belong to cases matching the top K variants
SELECT 
    e.case_id,
    e.activity,
    e.timestamp
FROM event_log e
INNER JOIN CaseVariants cv 
    ON e.case_id = cv.case_id
INNER JOIN TopVariants tv 
    ON cv.variant = tv.variant
ORDER BY 
    e.case_id, 
    e.timestamp;
```

### How it works:
1. **`CaseVariants` CTE**: Uses DuckDB's `string_agg` function to concatenate the activities for each `case_id` into a single string (e.g., `'Start -> Process -> End'`), strictly ordered by the `timestamp`. This defines the process variant for each case.
2. **`TopVariants` CTE**: Groups the results of the first CTE by the generated `variant` string. It counts the number of `case_id`s per variant, orders them in descending order of frequency, and applies a `LIMIT` to isolate only the top **K** variants.
3. **Final `SELECT`**: Joins the original `event_log` back to the `CaseVariants` to tag each event with its variant, and then inner-joins with `TopVariants` to filter out any cases that do not belong to the top **K** sequences. Finally, it orders the output by `case_id` and `timestamp` for readability.

### Alternative for Complex Activity Names:
If your `activity` names might contain the `' -> '` delimiter, you can safely use DuckDB's native array handling instead of string aggregation. Simply replace the `CaseVariants` CTE with:
```sql
    SELECT 
        case_id,
        array_agg(activity ORDER BY timestamp) AS variant
```
DuckDB natively supports equality checks on arrays, so the rest of the query will work identically and flawlessly.
