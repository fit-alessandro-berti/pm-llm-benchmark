To solve this problem, we need to identify and exclude cases from an event log that contain a specific consecutive sequence of activities with a time gap greater than 5 days between two of the activities. Here's the step-by-step DuckDB SQL solution:

### Approach
1. **Identify Consecutive Sequences**: Use window functions (`LEAD`) to find cases where the three target activities appear consecutively (no other events in between) in the exact order: *Approve Purchase Requisition*  *Create Order*  *Confirm Order*.
2. **Calculate Time Difference**: For valid sequences, compute the time elapsed between the *Create Order* and *Confirm Order* events.
3. **Identify Excluded Cases**: Collect distinct `case_id`s where the time difference between the two activities is more than 5 days.
4. **Filter Final Results**: Return all events from the original log that do not belong to the excluded cases.

### Solution Code
```sql
WITH sequence_candidates AS (
    -- For each event, get the next two activities and their timestamps per case, ordered by time
    SELECT 
        case_id,
        activity AS current_activity,
        timestamp AS current_ts,
        LEAD(activity, 1) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_activity_1,
        LEAD(timestamp, 1) OVER (PARTITION BY case_id ORDER BY timestamp) AS create_order_ts,
        LEAD(activity, 2) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_activity_2,
        LEAD(timestamp, 2) OVER (PARTITION BY case_id ORDER BY timestamp) AS confirm_order_ts
    FROM event_log
),
valid_sequence_cases AS (
    -- Filter to only rows that start the exact consecutive target sequence
    SELECT 
        case_id,
        create_order_ts,
        confirm_order_ts,
        confirm_order_ts - create_order_ts AS time_between_create_confirm
    FROM sequence_candidates
    WHERE 
        current_activity = 'Approve Purchase Requisition'
        AND next_activity_1 = 'Create Order'
        AND next_activity_2 = 'Confirm Order'
),
excluded_cases AS (
    -- Get distinct case IDs where the time between Create and Confirm Order is >5 days
    SELECT DISTINCT case_id
    FROM valid_sequence_cases
    WHERE time_between_create_confirm > INTERVAL '5 days'
)
-- Final result: all events NOT from excluded cases
SELECT 
    *
FROM event_log
WHERE 
    NOT EXISTS (
        SELECT 1
        FROM excluded_cases ec
        WHERE ec.case_id = event_log.case_id
    )
-- Order results by case and timestamp for readability
ORDER BY case_id, timestamp;
```

### Explanation
1. **`sequence_candidates` CTE**: Uses `LEAD` window functions to look ahead 1 and 2 rows (per case, ordered by timestamp) to capture the next two activities and their timestamps. This ensures we check for consecutive events with no gaps.
2. **`valid_sequence_cases` CTE**: Filters rows to only those that match the exact start of the target sequence (`Approve Purchase Requisition` followed immediately by `Create Order` then `Confirm Order`). It also calculates the time difference between the *Create Order* and *Confirm Order* events of the sequence.
3. **`excluded_cases` CTE**: Collects unique `case_id`s where the time difference between the two activities exceeds 5 days. These are the cases we need to exclude entirely.
4. **Final Query**: Uses `NOT EXISTS` to efficiently filter out all events belonging to excluded cases, returning only events from valid cases. The results are ordered by `case_id` and `timestamp` for clarity.

This approach efficiently handles consecutive sequence detection, timing validation, and filtering using DuckDB's window functions and set operations, ensuring correctness and performance even for large event logs.
