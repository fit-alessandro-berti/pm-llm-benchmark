Here are the two POWL models implemented in Python using the `pm4py` library.

### 1. POWL Model with Potential Unfairness

This model contains an exclusive choice (`XOR`) operator after the skill assessment. One branch leads to the standard cultural fit evaluation, while the other branch allows for a "CommunityAffiliationCheck" that introduces biased advantages for certain candidates.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# --- Define Activities ---
receive_app = Transition(label="ReceiveApplication")
data_check = Transition(label="DataCompletenessCheck")
request_info = Transition(label="RequestMoreInfo")
skill_assess = Transition(label="SkillAssessment")

# Cultural fit activities (with potential unfairness branch)
cultural_fit = Transition(label="CulturalFitCheck")
community_check = Transition(label="CommunityAffiliationCheck")

managerial_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# --- Define Control Flow Structures ---

# Loop: Data Completeness Check
# Executes data_check, then either exits or requests more info and checks again
loop_completeness = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# XOR Choice: Cultural Fit Check vs. Community Affiliation Check (Potential Bias)
xor_cultural_bias = OperatorPOWL(operator=Operator.XOR, children=[cultural_fit, community_check])

# --- Construct the Workflow Graph (Partial Order) ---
root_unfair = StrictPartialOrder(nodes=[
    receive_app, 
    loop_completeness, 
    skill_assess, 
    xor_cultural_bias, 
    managerial_review, 
    final_decision
])

# Define the sequential execution order
root_unfair.order.add_edge(receive_app, loop_completeness)
root_unfair.order.add_edge(loop_completeness, skill_assess)
root_unfair.order.add_edge(skill_assess, xor_cultural_bias)
root_unfair.order.add_edge(xor_cultural_bias, managerial_review)
root_unfair.order.add_edge(managerial_review, final_decision)
```

---

### 2. POWL Model Without Unfairness

This model eliminates the biased branch. All applicants proceed directly through the standardized `CulturalFitCheck` assessment, ensuring fairness and uniformity across the evaluation pipeline.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# --- Define Activities ---
receive_app = Transition(label="ReceiveApplication")
data_check = Transition(label="DataCompletenessCheck")
request_info = Transition(label="RequestMoreInfo")
skill_assess = Transition(label="SkillAssessment")

# Standardized cultural fit (no biased branch)
cultural_fit = Transition(label="CulturalFitCheck")

managerial_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# --- Define Control Flow Structures ---

# Loop: Data Completeness Check
loop_completeness = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# --- Construct the Workflow Graph (Partial Order) ---
root_fair = StrictPartialOrder(nodes=[
    receive_app, 
    loop_completeness, 
    skill_assess, 
    cultural_fit, 
    managerial_review, 
    final_decision
])

# Define the sequential execution order
root_fair.order.add_edge(receive_app, loop_completeness)
root_fair.order.add_edge(loop_completeness, skill_assess)
root_fair.order.add_edge(skill_assess, cultural_fit)
root_fair.order.add_edge(cultural_fit, managerial_review)
root_fair.order.add_edge(managerial_review, final_decision)
```
