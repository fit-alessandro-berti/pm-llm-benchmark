# Integrated Event Log for Process Mining

Below is the comprehensive event log derived from the provided ERP tables. Each row represents a single event in the lifecycle of an order, sorted by **Case ID** and **Timestamp**.

## Assumptions Made
1. **Case ID** = `order_id` as instructed.
2. Each **picked line item** generates a separate "Items Picked" event (since `picker_id` and timestamps differ per line).
3. **Delivery** generates two events: "Order Dispatched" (using `delivery_dispatch_date`) and "Order Arrived" (using `delivery_arrival_date`). For split shipments, both events are generated per delivery record.
4. **Invoice** generates one "Invoice Issued" event per invoice record.
5. **Payment** generates one "Payment Received" event per payment record. The `order_id` is resolved via the `invoice_id  order_id` link.
6. **Resource names** are resolved from the Resources table for clarity; the raw resource ID is also retained.
7. Events are sorted chronologically within each case.
