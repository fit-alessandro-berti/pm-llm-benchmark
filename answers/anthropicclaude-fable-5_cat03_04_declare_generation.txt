# DECLARE Model for the Product Design & Launch Process

Below is a Python dictionary representing the DECLARE model for this scenario. Each constraint is annotated with its business meaning. Unary templates (`existence`, `absence`, `exactly_one`, `init`) use **activities** as keys; binary templates use **tuples of activities** `(A, B)` as keys. Each value is a dictionary with the **support** (1.0) and the **confidence** of the rule.

```python
declare_model = {

    # ----- UNARY CONSTRAINTS -----

    # Activities that must occur in every product case
    'existence': {
        'Idea Generation':    {'support': 1.0, 'confidence': 1.0},
        'Design Draft':       {'support': 1.0, 'confidence': 1.0},
        'Approval Gate':      {'support': 1.0, 'confidence': 0.95},
        'Final Launch':       {'support': 1.0, 'confidence': 0.90},
    },

    # Activities that must NOT occur (none in a successfully launched product)
    'absence': {},

    # Activities that must occur exactly once per case
    'exactly_one': {
        'Idea Generation':    {'support': 1.0, 'confidence': 1.0},
        'Approval Gate':      {'support': 1.0, 'confidence': 0.90},
        'Final Launch':       {'support': 1.0, 'confidence': 1.0},
    },

    # The process always starts with brainstorming an idea
    'init': {
        'Idea Generation':    {'support': 1.0, 'confidence': 1.0},
    },

    # ----- BINARY CONSTRAINTS -----

    # If A occurs, B must occur somewhere in the trace (any order)
    'responded_existence': {
        ('Prototype Creation', 'Laboratory Testing'): {'support': 1.0, 'confidence': 0.98},
        ('Design Draft', 'Cost Evaluation'):          {'support': 1.0, 'confidence': 0.95},
        ('Marketing Plan', 'Approval Gate'):          {'support': 1.0, 'confidence': 1.0},
    },

    # A and B always occur together (or neither occurs)
    'coexistence': {
        ('Technical Feasibility Check', 'Cost Evaluation'): {'support': 1.0, 'confidence': 0.92},
        ('Laboratory Testing', 'User Testing'):             {'support': 1.0, 'confidence': 0.88},
    },

    # If A occurs, B must eventually occur AFTER A
    'response': {
        ('Idea Generation', 'Design Draft'):       {'support': 1.0, 'confidence': 1.0},
        ('Design Draft', 'Technical Feasibility Check'): {'support': 1.0, 'confidence': 0.97},
        ('Prototype Creation', 'Laboratory Testing'):    {'support': 1.0, 'confidence': 0.98},
        ('Approval Gate', 'Marketing Plan'):       {'support': 1.0, 'confidence': 0.93},
        ('Marketing Plan', 'Final Launch'):        {'support': 1.0, 'confidence': 0.90},
    },

    # B can occur only if A occurred BEFORE it
    'precedence': {
        ('Design Draft', 'Prototype Creation'):    {'support': 1.0, 'confidence': 1.0},
        ('Technical Feasibility Check', 'Prototype Creation'): {'support': 1.0, 'confidence': 0.96},
        ('Cost Evaluation', 'Approval Gate'):      {'support': 1.0, 'confidence': 0.94},
        ('User Testing', 'Approval Gate'):         {'support': 1.0, 'confidence': 0.91},
        ('Approval Gate', 'Final Launch'):         {'support': 1.0, 'confidence': 1.0},
    },

    # Both response(A,B) and precedence(A,B) hold
    'succession': {
        ('Idea Generation', 'Design Draft'):       {'support': 1.0, 'confidence': 1.0},
        ('Prototype Creation', 'Laboratory Testing'): {'support': 1.0, 'confidence': 0.95},
        ('Approval Gate', 'Final Launch'):         {'support': 1.0, 'confidence': 0.90},
    },

    # After A, B must occur before A can occur again
    'altresponse': {
        ('Design Draft', 'Technical Feasibility Check'): {'support': 1.0, 'confidence': 0.93},
        ('Prototype Creation', 'Laboratory Testing'):    {'support': 1.0, 'confidence': 0.92},
    },

    # Before each B, A must have occurred without another B in between
    'altprecedence': {
        ('Laboratory Testing', 'User Testing'):    {'support': 1.0, 'confidence': 0.89},
        ('Design Draft', 'Prototype Creation'):    {'support': 1.0, 'confidence': 0.94},
    },

    # Both altresponse and altprecedence hold
    'altsuccession': {
        ('Prototype Creation', 'Laboratory Testing'): {'support': 1.0, 'confidence': 0.90},
    },

    # If A occurs, B must occur IMMEDIATELY after
    'chainresponse': {
        ('Approval Gate', 'Marketing Plan'):       {'support': 1.0, 'confidence': 0.85},
        ('Marketing Plan', 'Final Launch'):        {'support': 1.0, 'confidence': 0.87},
    },

    # If B occurs, A must have occurred IMMEDIATELY before
    'chainprecedence': {
        ('Marketing Plan', 'Final Launch'):        {'support': 1.0, 'confidence': 0.87},
        ('Idea Generation', 'Design Draft'):       {'support': 1.0, 'confidence': 0.82},
    },

    # Both chainresponse and chainprecedence hold
    'chainsuccession': {
        ('Marketing Plan', 'Final Launch'):        {'support': 1.0, 'confidence': 0.85},
    },

    # A and B must NEVER occur together in the same trace
    'noncoexistence': {
        # A product cannot be launched if the feasibility check is missing —
        # modeled here as: Final Launch never coexists with a rejected/aborted state.
        # (No strict non-coexistence pairs in a successful launch process.)
    },

    # If A occurs, B must NOT occur afterwards
    'nonsuccession': {
        # Once the product is launched, no new design drafts in the same case
        ('Final Launch', 'Design Draft'):          {'support': 1.0, 'confidence': 0.96},
        # After final launch, no further prototype creation
        ('Final Launch', 'Prototype Creation'):    {'support': 1.0, 'confidence': 0.97},

    # B must never IMMEDIATELY follow A
    'nonchainsuccession': {
        # A prototype is never built immediately after idea generation
        # (design and checks must happen in between)
        ('Idea Generation', 'Prototype Creation'): {'support': 1.0, 'confidence': 1.0},
        # Launch never directly follows lab testing (user testing/approval needed)
        ('Laboratory Testing', 'Final Launch'):    {'support': 1.0, 'confidence': 0.98},
    },
}
```

## Key Modeling Decisions

| Constraint | Rationale |
|---|---|
| `init: Idea Generation` | Every product case starts with an idea. |
| `exactly_one: Final Launch` | A product is launched at most/exactly once. |
| `precedence: (Approval Gate, Final Launch)` | No launch without management approval — confidence 1.0 (hard rule). |
| `response: (Design Draft, TFC)` | Every design draft triggers a feasibility check; confidence < 1.0 allows rare deviations (e.g., abandoned drafts). |
| `nonsuccession: (Final Launch, Design Draft)` | Post-launch redesign belongs to a new case, not the current one. |
| `chainresponse: (Approval Gate, Marketing Plan)` | Marketing typically starts directly after approval, but occasionally other steps intervene (confidence 0.85). |

This model is compatible with pm4py's conformance checking, e.g.:

```python
import pm4py
conf_result = pm4py.conformance_declare(event_log, declare_model)
```
