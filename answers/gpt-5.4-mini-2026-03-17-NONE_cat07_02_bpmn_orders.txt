The visualization is a **BPMN-style business process diagram** with four horizontal swimlanes:

- **Sales**
- **Production**
- **Procurement**
- **Warehouse**

It shows how an order moves through the organization, with dashed message flows connecting the lanes.

### High-level flow
1. **Sales** starts the process on **Monday morning**.
2. It checks the customer’s order for completeness and verifies whether the customer already exists in the database.
   - If the customer is not in the database, it is added.
3. Sales then **sends order details** to Production and creates a **draft invoice**.
4. **Production** receives the order details, plans production, and manufactures the product.
   - If materials are not faulty / not available as expected, the process interacts with Procurement.
5. **Procurement** receives order details and runs a **material compensation subprocess**:
   - order materials
   - return material
   - check materials
   - possibly cancel the order if needed
6. Once the product is complete, **Warehouse** receives the product completion message and **sends the product**.
7. The process ends after the product is sent, with message events indicating completion and handoffs between departments.

### Visual features
- **Rounded rectangles** = activities/tasks
- **Diamonds** = gateways/decision points
- **Envelope icons** = message events
- **Dashed arrows** = message exchanges between departments
- **Circles** = start/end events
- A **database symbol** appears in the Sales lane for customer data

Overall, the diagram describes an **order fulfillment and production coordination workflow** spanning sales, production, procurement, and warehouse operations.
