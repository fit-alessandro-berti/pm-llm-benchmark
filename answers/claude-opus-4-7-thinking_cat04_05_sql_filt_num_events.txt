Here's a DuckDB SQL query that accomplishes this task:

```sql
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
ORDER BY case_id, timestamp;
```

### Explanation

1. **Inner query (subquery):**
   - `GROUP BY case_id` groups all events by their case.
   - `COUNT(*)` counts the number of events per case.
   - `HAVING COUNT(*) <= 6` keeps only those cases whose total event count is six or fewer.

2. **Outer query:**
   - Selects all columns (`*`) from `event_log`.
   - The `WHERE case_id IN (...)` clause restricts the result to events belonging only to the filtered cases.
   - `ORDER BY case_id, timestamp` ensures the events are returned in a sensible, chronological order per case (optional but helpful for process analysis).

### Alternative using a Window Function

If you prefer a single-pass approach using a window function, here's an equivalent version:

```sql
SELECT case_id, activity, timestamp
FROM (
    SELECT *,
           COUNT(*) OVER (PARTITION BY case_id) AS case_event_count
    FROM event_log
) t
WHERE case_event_count <= 6
ORDER BY case_id, timestamp;
```

This version:
- Uses `COUNT(*) OVER (PARTITION BY case_id)` to compute the event count per case inline.
- Filters out any rows where the case contains more than six events.

Both queries produce the same result; the choice depends on readability and performance characteristics of your dataset. For very large event logs, the window-function version may be more efficient, as it avoids the self-referential subquery.
