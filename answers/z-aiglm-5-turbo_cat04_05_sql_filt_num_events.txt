Here are two ways to write this query in DuckDB. 

### Option 1: Using a Subquery with `JOIN` (Standard SQL)
This is the most standard and widely compatible approach. It groups the events by `case_id` to find the valid cases, and then joins that list back to the original table to retrieve the full event details.

```sql
SELECT e.*
FROM event_log e
JOIN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
) valid_cases ON e.case_id = valid_cases.case_id;
```

### Option 2: Using a Window Function with `EXCLUDE` (DuckDB-specific)
DuckDB supports the `EXCLUDE` clause, which makes window functions incredibly clean for this type of task. You can count the events per case and then select everything *except* the temporary count column.

```sql
SELECT *
EXCLUDE (event_count)
FROM (
    SELECT *,
           COUNT(*) OVER (PARTITION BY case_id) AS event_count
    FROM event_log
)
WHERE event_count <= 6;
```

### How they work:
* **Option 1** calculates the total number of events *before* retrieving the rows. By using `HAVING COUNT(*) <= 6`, it completely discards any `case_id` that exceeds the threshold, and the `JOIN` ensures only the original rows belonging to those valid cases are returned. Using `e.*` guarantees all original columns are returned.
* **Option 2** calculates the total number of events on the fly for every single row using `OVER (PARTITION BY case_id)`. It then filters out the rows where that count is greater than 6, and uses `EXCLUDE (event_count)` to drop the helper column so the output matches the exact schema of your input table.
