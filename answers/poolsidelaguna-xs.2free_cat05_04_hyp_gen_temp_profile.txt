### Verification Approaches Using SQL Queries

#### 1. **Identify Claims with Outliers in R to P Timing**
```sql
WITH r_p_times AS (
  SELECT 
    c.claim_id,
    c.claim_type,
    EXTRACT(EPOCH FROM (p.timestamp - r.timestamp)) AS time_r_to_p
  FROM claim_events r
  JOIN claim_events p ON r.claim_id = p.claim_id
  JOIN claims c ON r.claim_id = c.claim_id
  WHERE r.activity = 'R' AND p.activity = 'P'
)
SELECT 
  claim_id,
  claim_type,
  time_r_to_p,
  CASE 
    WHEN time_r_to_p < 82800 THEN 'Too Fast'
    WHEN time_r_to_p > 97200 THEN 'Too Slow'
    ELSE 'Normal'
  END AS anomaly_status
FROM r_p_times
WHERE time_r_to_p < 82800 OR time_r_to_p > 97200;
```
- **Goal:** Find claims where the time between "R" and "P" falls outside the expected range (25 hours ± 2 hours).

#### 2. **Detect Claims Skipping Intermediate Steps (A to C)**
```sql
SELECT 
  a.claim_id,
  c.claim_type,
  a.timestamp AS assign_time,
  c_event.timestamp AS close_time
FROM claim_events a
JOIN claim_events c_event ON a.claim_id = c_event.claim_id
JOIN claims c ON a.claim_id = c.claim_id
WHERE a.activity = 'A' AND c_event.activity = 'C'
  AND NOT EXISTS (
    SELECT 1 FROM claim_events e 
    WHERE e.claim_id = a.claim_id 
      AND e.activity IN ('E', 'P', 'N')
  );
```
- **Goal:** Find claims where "C" follows "A" without any "E," "P," or "N" steps.

#### 3. **Analyze P to N Delays by Claim Type and Adjuster**
```sql
WITH p_n_times AS (
  SELECT 
    p.claim_id,
    c.claim_type,
    a.adjuster_id,
    EXTRACT(EPOCH FROM (n.timestamp - p.timestamp)) AS time_p_to_n
  FROM claim_events p
  JOIN claim_events n ON p.claim_id = n.claim_id
  JOIN claims c ON p.claim_id = c.claim_id
  JOIN claim_events a ON p.claim_id = a.claim_id AND a.activity = 'A'
  WHERE p.activity = 'P' AND n.activity = 'N'
)
SELECT 
  claim_type,
  adjuster_id,
  AVG(time_p_to_n) AS avg_delay_seconds,
  STDDEV(time_p_to_n) AS stdev_delay_seconds
FROM p_n_times
GROUP BY claim_type, adjuster_id
HAVING AVG(time_p_to_n) > 604800 OR STDDEV(time_p_to_n) > 172800;
```
- **Goal:** Identify claim types or adjusters with excessively long or inconsistent P-to-N delays.

#### 4. **Flag Claims with Suspiciously Fast E to N Transitions**
```sql
SELECT 
  e.claim_id,
  c.claim_type,
  EXTRACT(EPOCH FROM (n.timestamp - e.timestamp)) AS time_e_to_n

JOIN claim_events n ON e.claim_id = n.claim_id
JOIN claims c ON e.claim_id = c.claim_id

```

#### 5. **Correlate Anomalies with Resource Availability (Adjuster Region)**
```sql
SELECT 
  adj.region,
  COUNT(*) AS anomaly_count,
  AVG(time_r_to_p) AS avg_r_to_p
FROM (
  SELECT 
    r.claim_id,
    EXTRACT(EPOCH FROM (p.timestamp - r.timestamp)) AS time_r_to_p
  FROM claim_events r
  JOIN claim_events p ON r.claim_id = p.claim_id
  WHERE r.activity = 'R' AND p.activity = 'P'
    AND EXTRACT(EPOCH FROM (p.timestamp - r.timestamp)) < 82800
) AS anomalies
JOIN claims c ON anomalies.claim_id = c.claim_id
JOIN claim_events a ON anomalies.claim_id = a.claim_id AND a.activity = 'A'
JOIN adjusters adj ON a.resource = adj.adjuster_id
GROUP BY adj.region
ORDER BY anomaly_count DESC;
```
- **Goal:** Determine if anomalies cluster in specific regions, indicating resource constraints or policy inconsistencies.
