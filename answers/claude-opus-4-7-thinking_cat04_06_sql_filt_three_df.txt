```sql
WITH seq AS (
    SELECT 
        case_id,
        activity,
        timestamp,
        LEAD(activity, 1) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_activity,
        LEAD(activity, 2) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_next_activity,
        LEAD(timestamp, 1) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_timestamp,
        LEAD(timestamp, 2) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_next_timestamp
    FROM event_log
),
bad_cases AS (
    SELECT DISTINCT case_id
    FROM seq
    WHERE activity = 'Approve Purchase Requisition'
      AND next_activity = 'Create Order'
      AND next_next_activity = 'Confirm Order'
      AND (next_next_timestamp - next_timestamp) > INTERVAL '5 days'
)
SELECT *
FROM event_log
WHERE case_id NOT IN (SELECT case_id FROM bad_cases)
ORDER BY case_id, timestamp;
```

**How it works:**

1. **`seq` CTE** — For each event, uses `LEAD()` to look at the next 2 events in the same case (ordered by time). This lets us check for 3 consecutive events in direct succession.

2. **`bad_cases` CTE** — Finds cases where the triplet (`Approve Purchase Requisition`  `Create Order`  `Confirm Order`) occurs in direct succession **and** the elapsed time between `Create Order` (`next_timestamp`) and `Confirm Order` (`next_next_timestamp`) exceeds 5 days.

3. **Final `SELECT`** — Returns all events from `event_log` whose `case_id` is not in the set of offending cases, ordered by case and timestamp for readability.
