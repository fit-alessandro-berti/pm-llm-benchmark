  - **Process Mining:** Use conformance checking to track delays between "Item Picking" completion and "Packing" start for cold-packing orders. Identify gaps between `Timestamp (START)` and the previous activity’s `COMPLETE` time.  
  - **Metrics:**  
    - *Waiting Time Due to Resource Contention:* Calculate the time between order readiness (e.g., after Item Picking) and resource allocation for cold-packing.  
    - *Queue Length Analysis:* Count orders waiting for cold-packing stations during peak times.  
  - **Differentiation:** Compare waiting time gaps to the average inter-arrival rate of orders. If gaps exceed the average, it indicates between-instance contention.  

- **Shipping Batches:**  
  - **Process Mining:** Use case-based analysis to identify delays in "Shipping Label Generation" caused by waiting for batch formation. Track batch IDs and the time from last order in a batch to batch completion.  
  - **Metrics:**  
    - *Batch Wait Time:* Time between an order’s completion of "Quality Check" and batch processing.

- **Priority Handling:**  
  - **Process Mining:** Identify instances where a standard order’s activity (e.g., Packing) is paused to accommodate an express order. Track delays in standard orders caused by priority interruptions.  
  - **Metrics:**  
    - *Priority Interruption Delays:* Time standard orders were paused for express orders.  
    - *Express Order Throughput vs. Standard Orders:* Compare processing times of express vs. standard orders.  

- **Hazardous Material Limits:**  
  - **Process Mining:** Count concurrent hazardous orders in "Packing" or "Quality Check" using the "Hazardous Material" flag.  
  - **Metrics:**  
    - *Hazardous Capacity Utilization:* Percentage of time exceeding the 10-simultaneous-activity limit.  
    - *Hazardous Order Throughput:* Time taken to process hazardous orders while respecting the limit.  

#### **Differentiating Within- vs. Between-Instance Delays**  
- Use *time distance analysis* in process mining to distinguish delays caused by:  
  1. **Within-Instance Factors:** Calculate the "expected" processing time for an activity (e.g., average Packing duration). If an order’s actual duration exceeds this, it’s within-instance.  
  2. **Between-Instance Factors:** Use resource logs to see if the delay aligns with resource availability (e.g., cold-packing station was busy with another order).
