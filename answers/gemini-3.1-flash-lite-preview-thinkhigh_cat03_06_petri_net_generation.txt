To model the insurance claim management process, we will define places as the states between the activities and transitions as the activities themselves. We will implement loops for the re-submission and in-depth investigation stages.

Here is the implementation using `pm4py`:

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# 1. Initialize Petri Net
net = PetriNet("Insurance_Claim_Process")

# 2. Define Places
p_start = PetriNet.Place("start")
p_reg = PetriNet.Place("registered")
p_doc = PetriNet.Place("doc_verified")
p_fraud = PetriNet.Place("fraud_checked")
p_assess = PetriNet.Place("assessed")
p_app = PetriNet.Place("approved")
p_pay = PetriNet.Place("paid")
p_notif = PetriNet.Place("notified")
p_end = PetriNet.Place("closed")

# Add places to net
for p in [p_start, p_reg, p_doc, p_fraud, p_assess, p_app, p_pay, p_notif, p_end]:
    net.places.add(p)

# 3. Define Transitions
t_cr = PetriNet.Transition("CR", "Claim Registration")
t_dv = PetriNet.Transition("DV", "Document Verification")
t_rds = PetriNet.Transition("RDS", "Re-Document Submission")
t_fc = PetriNet.Transition("FC", "Fraud Check")
t_ii = PetriNet.Transition("II", "In-depth Investigation")
t_ca = PetriNet.Transition("CA", "Claim Assessment")
t_ap = PetriNet.Transition("AP", "Approval")
t_p = PetriNet.Transition("P", "Payment")
t_n = PetriNet.Transition("N", "Notification")
t_cl = PetriNet.Transition("CL", "Closure")

# Add transitions to net
for t in [t_cr, t_dv, t_rds, t_fc, t_ii, t_ca, t_ap, t_p, t_n, t_cl]:
    net.transitions.add(t)

# 4. Define Arcs (Logic sequence + Loops)
# Sequence
petri_utils.add_arc_from_to(p_start, t_cr, net)
petri_utils.add_arc_from_to(t_cr, p_reg, net)
petri_utils.add_arc_from_to(p_reg, t_dv, net)
petri_utils.add_arc_from_to(t_dv, p_doc, net)

# Loop: Document Verification -> Re-Submission
petri_utils.add_arc_from_to(p_doc, t_rds, net)
petri_utils.add_arc_from_to(t_rds, p_reg, net)

# Move forward from Doc verified to Fraud Check
petri_utils.add_arc_from_to(p_doc, t_fc, net)
petri_utils.add_arc_from_to(t_fc, p_fraud, net)

# Loop: Fraud Check -> In-depth Investigation
petri_utils.add_arc_from_to(p_fraud, t_ii, net)
petri_utils.add_arc_from_to(t_ii, p_fraud, net)

# Proceed to Assessment and beyond
petri_utils.add_arc_from_to(p_fraud, t_ca, net)
petri_utils.add_arc_from_to(t_ca, p_assess, net)
petri_utils.add_arc_from_to(p_assess, t_ap, net)
petri_utils.add_arc_from_to(t_ap, p_pay, net)
petri_utils.add_arc_from_to(p_pay, t_p, net) # Payment transition
petri_utils.add_arc_from_to(t_p, p_notif, net)
petri_utils.add_arc_from_to(p_notif, t_n, net)
petri_utils.add_arc_from_to(t_n, p_end, net)
petri_utils.add_arc_from_to(p_end, t_cl, net)

# 5. Define Markings
im = Marking()
im[p_start] = 1

fm = Marking()
# The process ends when the claim is closed/archived
# We add a dummy place or target the final state place
# Let's assume a place 'archived' follows closure
p_archived = PetriNet.Place("archived")
net.places.add(p_archived)
petri_utils.add_arc_from_to(t_cl, p_archived, net)
fm[p_archived] = 1
```

### Explanation:
*   **Sequencing:** Arcs are created between the logical steps defined in your scenario.
*   **Loops:** 
    *   **RDS:** An arc is added from `p_doc` back to the `t_rds` transition, which leads back to `p_reg`, allowing the registration/verification loop.
    *   **II:** An arc is added from `p_fraud` back to the `t_ii` transition, which loops back to the same place (`p_fraud`), allowing multiple investigations before proceeding to Assessment.
*   **Markings:** The `im` (Initial Marking) starts with one token at the `start` place. The `fm` (Final Marking) is defined at a post-closure `archived` place to signify the process completion.
