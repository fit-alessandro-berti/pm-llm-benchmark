### Comprehensive Event Log for Process Mining

The following event log integrates data from the provided ERP tables. Each row represents a single event in the process lifecycle. Data has been enriched with resource names, product descriptions, and relevant attributes. Cases with multiple activities (e.g., split deliveries or invoices) are represented by multiple events.

**Key Design Decisions & Assumptions:**
1.  **Granularity:** "Item Picked" is logged per order line to capture picking granularity and resource assignment per item. "Order Dispatched" and "Order Arrived" are logged per delivery record to handle split shipments correctly.
2.  **Resources:** Resource IDs were mapped to names using the `Resources Table`. For payments, no specific resource ID exists; "Payment System" is used as the actor.
3.  **Missing Data:** Order `1004` has no corresponding delivery, invoice, or payment records. Only "Order Created" and "Item Picked" events are included. This may indicate a cancelled order or data truncation.
4.  **Attributes:** All available attributes are included as columns. Null values are represented as empty strings where an attribute does not apply to a specific event type.
5.  **Chronology:** Events are sorted by Case ID and Timestamp to reflect the process flow.

### Notes on Data Integration
*   **Case 1002:** This case demonstrates parallelism and splitting. It has two items picked concurrently, two dispatch/arrival events for split shipments, two invoices, and two payments. The log captures this complexity accurately.
*   **Case 1004:** This case is incomplete. While the order was created and items were picked, there is no evidence of delivery, invoicing, or payment. In a process mining analysis, this would appear as a "dead end" case, suggesting potential cancellation or processing failure.
*   **Resources:** The `Resource` column uses human-readable names derived from the `Resources Table`. For example, `WH-001` maps to `David Lin`.
*   **Products:** `Item Description` and `Category` are populated from the `Products Table` where available.
*   **Timestamps:** All timestamps are in ISO 8601 format. Chronological order is preserved within each case.
