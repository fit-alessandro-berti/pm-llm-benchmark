# Business Process Model and Notation (BPMN) Diagram Description

This visualization is a **BPMN (Business Process Model and Notation) diagram** depicting an order fulfillment process across four organizational departments or swim lanes.

## Swim Lanes (Departments)

### 1. **Sales** (Top Lane)
- **Trigger**: Monday morning (timer start event)
- **Process Flow**: 
  - Check order for completeness
  - Check if customer is already in the database
  - If not in database  Add customer to the database (connected to database storage)
  - If in database  Continue
  - Send order details
  - Draft invoice (after product is sent)
  - End event

### 2. **Production** (Second Lane)
- **Trigger**: Order details arrive (message event)
- **Process Flow**:
  - Plan production (parallel gateway splits process)
  - Material verification check (material not faulty)
  - Manufacture product
  - Product complete (end event)

### 3. **Procurement** (Third Lane)
- **Trigger**: Order details arrive
- **Material Compensation Subprocess** (dashed box):
  - Order materials
  - Check materials
  - Return material (compensation activity if needed)
  - Order cancellation handling
  - Includes error/exception handling symbols

### 4. **Warehouse** (Bottom Lane)
- **Trigger**: Product complete (message event)
- **Process**: Send product  Product sent (end event)

## Key Notation Elements
- **Circles**: Start/End events
- **Rectangles**: Tasks/Activities
- **Diamonds**: Gateways (decision points)
- **Dashed lines**: Message flows between lanes
- **Database symbol**: Data storage
