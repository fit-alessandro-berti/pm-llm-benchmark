Here is a DuckDB SQL query that excludes all events from cases containing that exact consecutive 3-step pattern, where `Confirm Order` happens more than 5 days after `Create Order`:

```sql
WITH ordered_events AS (
    SELECT
        case_id,
        activity,
        "timestamp" AS event_ts,
        ROW_NUMBER() OVER (
            PARTITION BY case_id
            ORDER BY "timestamp"
        ) AS rn
    FROM event_log
),
cases_to_exclude AS (
    SELECT DISTINCT a.case_id
    FROM ordered_events a
    JOIN ordered_events b
      ON b.case_id = a.case_id
     AND b.rn = a.rn + 1
    JOIN ordered_events c
      ON c.case_id = a.case_id
     AND c.rn = a.rn + 2
    WHERE a.activity = 'Approve Purchase Requisition'
      AND b.activity = 'Create Order'
      AND c.activity = 'Confirm Order'
      AND c.event_ts > b.event_ts + INTERVAL '5 days'
)
SELECT e.*
FROM event_log e
WHERE NOT EXISTS (
    SELECT 1
    FROM cases_to_exclude x
    WHERE x.case_id = e.case_id
)
ORDER BY e.case_id, e."timestamp";
```

Notes:
- `ROW_NUMBER()` defines the event order within each `case_id`, so `rn`, `rn+1`, and `rn+2` guarantee direct succession.
- `cases_to_exclude` contains any case with at least one occurrence of the exact sequence:
  `Approve Purchase Requisition -> Create Order -> Confirm Order`
  where `Confirm Order` is more than 5 days after `Create Order`.
- The final `NOT EXISTS` removes all events from those cases.

If your log can have multiple events with the exact same timestamp inside a case, add a stable tie-breaker (for example `event_id`) to the `ORDER BY` inside `ROW_NUMBER()`.
