# Process Tree Model for the Application Handling Process

## Analysis of the Event Log

Looking at the traces from the event log:

| Case | Trace |
|------|-------|
| 1 | RA  VF  AE  RMI  VF  AD  SC  AC |
| 2 | RA  VF  AE  AD  SC  AC |
| 3 | RA  VF  AE  RMI  RMI  VF  AD  SC  AC |

**Key observations:**
- All cases start with: **RA  VF  AE**
- All cases end with: **AD  SC  AC**
- The middle section is optional and involves:
  - Zero or more requests for missing info (RMI)
  - If RMI occurs, it's followed by re-validation (VF)
  - Case 3 shows multiple RMIs can occur before re-validation

## Final Process Tree Model

```
-> (
    'RA',
    'VF',
    'AE',
    * (
        tau,
        -> (
            * ( 'RMI', tau ),
            'VF'
        )
    ),
    'AD',
    'SC',
    'AC'
)
```

## Visual Representation

```
                            -> (SEQUENCE)
                                  
    
                                                  
  'RA'  'VF'  'AE'   * (LOOP)             'AD'  'SC'  'AC'
                      
                
                           
               tau    -> (SEQUENCE)
                            
                      
                                 
                 * (LOOP)       'VF'
                      
                
                           
              'RMI'        tau
```

## Explanation

| Component | Purpose |
|-----------|---------|
| `-> ('RA', 'VF', 'AE', ..., 'AD', 'SC', 'AC')` | Main sequence capturing the mandatory flow |
| `* (tau, ...)` | Outer loop allowing **zero or more** rounds of re-validation |
| `* ('RMI', tau)` | Inner loop allowing **one or more** RMI requests per round |
| `-> (* ('RMI', tau), 'VF')` | Each re-validation round: multiple RMIs followed by VF |

## Traces Allowed by This Model

-  **Case 2**: RA  VF  AE  AD  SC  AC *(loop executes 0 times)*
-  **Case 1**: RA  VF  AE  RMI  VF  AD  SC  AC *(1 round, 1 RMI)*
-  **Case 3**: RA  VF  AE  RMI  RMI  VF  AD  SC  AC *(1 round, 2 RMIs)*
-  Allows multiple rounds: RA  VF  AE  RMI  VF  RMI  VF  AD  SC  AC *(generalizes for real-world scenarios)*
