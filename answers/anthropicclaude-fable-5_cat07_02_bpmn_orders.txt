# BPMN Process Diagram: Order Fulfillment Process

This is a **Business Process Model and Notation (BPMN)** diagram showing an order fulfillment workflow across **four swimlanes (pools)**: Sales, Production, Procurement, and Warehouse. The pools communicate via message flows (dashed arrows).

## Sales Lane (top)
- The process begins with a **timer start event** labeled "Monday morning."
- It enters an expanded subprocess called **"Check customer's order"**, which contains:
  - **Check order for completeness**
  - **Check if customer is already in the database**
  - An **exclusive (XOR) gateway** that branches:
    - *Customer not in the database*  **Add customer to the database** (linked to a **database data store** via a dotted association)
    - *Customer in the database*  flows directly onward
  - The branches merge at a second XOR gateway, ending the subprocess.
- After the subprocess: **Send order details** task (which sends "Order details" messages to both Production and Procurement).
- A **message intermediate event** "Product sent" (receiving a message from the Warehouse) precedes the final task **Draft invoice**, then an **end event**.

## Production Lane
- Starts with a **message start event** "Order details arrive."
- A **parallel/event gateway** leads to **Plan production**, then another gateway, then **Manufacture product**.
- A **message intermediate event** "Material not faulty" (receiving from Procurement) loops back into the flow.
- Ends with a **message end event** "Product complete" (notifying the Warehouse).

## Procurement Lane
- Starts with a **message start event** "Order details arrive."
- Contains an expanded **"Material Compensation Subprocess"** with:
  - **Order materials** task
  - A **compensation boundary event** linked (dotted line) to a **Return material** compensation task
- Then a **Check materials** task with an **error boundary event**:
  - Success path  **message end event** (material OK, notifying Production)
  - Error path  **compensation throw event** "Order cancelation"  end event

## Warehouse Lane (bottom)
- Starts with a **message start event** "Product complete" (from Production).
- **Send product** task follows.
- Ends with a **message end event** "Product sent," which messages back to the Sales lane to trigger invoicing.

## Overall Flow
The diagram models a complete order-to-delivery process: Sales validates the order, Production and Procurement work in coordination (with compensation/error handling for faulty materials), the Warehouse ships the product, and Sales finalizes by drafting the invoice.
