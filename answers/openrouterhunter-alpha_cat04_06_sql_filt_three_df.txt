Here's a DuckDB SQL query that meets all the specified requirements:

```sql
WITH case_sequences AS (
    -- Step 1: Identify cases with the exact 3-event sequence in direct succession
    SELECT DISTINCT e1.case_id,
           e2.timestamp AS create_order_time,
           e3.timestamp AS confirm_order_time
    FROM event_log e1
    JOIN event_log e2 ON e1.case_id = e2.case_id
                     AND e2.activity = 'Create Order'
    JOIN event_log e3 ON e2.case_id = e3.case_id
                     AND e3.activity = 'Confirm Order'
    WHERE e1.activity = 'Approve Purchase Requisition'
      -- Ensure e2 is the immediate next event after e1
      AND NOT EXISTS (
          SELECT 1 FROM event_log e_between
          WHERE e_between.case_id = e1.case_id
            AND e_between.timestamp > e1.timestamp
            AND e_between.timestamp < e2.timestamp
      )
      -- Ensure e3 is the immediate next event after e2
      AND NOT EXISTS (
          SELECT 1 FROM event_log e_between
          WHERE e_between.case_id = e2.case_id
            AND e_between.timestamp > e2.timestamp
            AND e_between.timestamp < e3.timestamp
      )
),

excluded_cases AS (
    -- Step 2: Further refine to only cases where Confirm Order - Create Order > 5 days
    SELECT case_id
    FROM case_sequences
    WHERE DATEDIFF('day', create_order_time, confirm_order_time) > 5
)

-- Step 3: Return all events except those from excluded cases
SELECT *
FROM event_log
WHERE case_id NOT IN (SELECT case_id FROM excluded_cases)
ORDER BY case_id, timestamp;
```

**How this query works:**

1. **Identifying the sequence:** The CTE `case_sequences` finds cases where:
   - 'Approve Purchase Requisition'  'Create Order'  'Confirm Order' appear in exactly that order
   - No other events from the same case occur between them (verified with NOT EXISTS subqueries)

2. **Time condition:** The second CTE `excluded_cases` filters to only those cases where:
   - The time between 'Create Order' and 'Confirm Order' exceeds 5 days
   - Using DATEDIFF('day', ...) to calculate day difference

3. **Final filtering:** The main query returns all events except those belonging to cases identified in `excluded_cases`

**Key features:**
- Uses explicit joins with NOT EXISTS to ensure events are truly consecutive
- Handles edge cases where a case might have multiple occurrences of the sequence
- Properly handles datetime arithmetic for the 5-day condition
- Returns results ordered by case_id and timestamp for readability

**Note:** If you're using a version of DuckDB that doesn't support DATEDIFF with 'day', you can replace it with:
```sql
WHERE (confirm_order_time - create_order_time) > INTERVAL 5 DAYS
```
