## 1. Process Discovery and Conformance Checking

### 1.1 Data Integration and Preprocessing
**Integration approach**  
The four data sources must be fused into a single case-centric event log. I define a case as a *Vehicle-Day* (e.g., V12-20241205), capturing one complete delivery run. The integration steps are:

1. **Temporal alignment and identifier mapping**  
   - Unify timestamps (UTC).  
   - Link events via Vehicle ID, Driver ID, and Package ID to the Vehicle-Day case.  
   - Map GPS traces to scanner events by vehicle and time, and enrich with dispatch data (planned stops, time windows).

2. **GPS abstraction**  
   - High-frequency GPS is turned into discrete activities: `Depart Depot`, `Travel`, `Low-Speed/Traffic`, `Parked`, `Unscheduled Stop`, `Arrive Depot`.  
   - Geofencing the depot and customer locations (using dispatch addresses) allows automatic detection of `Arrive Customer` and `Depart Customer` even when scanner data is missed.  
   - Idle detection (ignition on, speed = 0) distinguishes genuine service time from traffic stops.

3. **Scanner event harmonisation**  
   - Standardise milestones: `Arrive Customer`, `Delivery Success`, `Delivery Failed (Attempted)`, `Depart Customer`.  
   - Link each delivery attempt to a planned stop (Stop #) from dispatch.

4. **Maintenance log incorporation**  
   - Insert `Unscheduled Maintenance` or `Scheduled Maintenance` events into the timeline, noting the start and end timestamps, vehicle ID, and nature of the issue.

5. **Dispatch plan as reference model**  
   - Create a “planned” process trace per case: an ordered sequence of depot departure, scheduled stops (with expected time windows), and depot return. This serves as the target for conformance checking.

**Challenges**  
- **Data quality:** GPS signal loss in urban canyons, scanner usage inconsistency, mismatched timestamps, missing package scans.  
- **Semantic mapping:** Distinguishing a “stop for parking” from a “traffic jam” requires contextual rules (proximity to customer, duration).  
- **Failed deliveries:** May cause multiple visits to the same customer; these must be recognised as reattempts, not new stops.

**Method**  
I convert the dispatch-planned route into a normative process model (a strict sequence of depot departure, ordered stops, and depot return). Using **alignmentbased conformance checking**, I compute optimal alignments between each actual trace and the planned model, identifying:
