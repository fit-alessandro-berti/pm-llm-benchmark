Below are the two POWL models representing the hiring process at GlobalTeq Solutions, with and without the potential unfairness introduced by the community affiliation check.

---

### **1. POWL Model with Potential Unfairness**
This model includes an XOR branch where applicants may either undergo a standard **CulturalFitCheck** or a **CommunityAffiliationCheck**, which introduces bias by giving preferential treatment to certain applicants.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Define the loop for data completeness
loop_body = StrictPartialOrder(nodes=[DataCompletenessCheck, RequestMoreInfo])
loop_body.order.add_edge(DataCompletenessCheck, RequestMoreInfo)
data_completeness_loop = OperatorPOWL(operator=Operator.LOOP, children=[ReceiveApplication, loop_body])
loop_body.order.add_edge(ReceiveApplication, DataCompletenessCheck)

# Define the XOR branch for cultural fit vs. community affiliation
cultural_fit_xor = OperatorPOWL(operator=Operator.XOR, children=[CulturalFitCheck, CommunityAffiliationCheck])

# Define the main partial order
root = StrictPartialOrder(nodes=[data_completeness_loop, SkillAssessment, cultural_fit_xor, ManagerialReview, FinalDecision])
root.order.add_edge(data_completeness_loop, SkillAssessment)
root.order.add_edge(SkillAssessment, cultural_fit_xor)
root.order.add_edge(cultural_fit_xor, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)
```

#### **Explanation:**
- **Loop:** Ensures that if the application is incomplete, the applicant is asked to provide more information (`RequestMoreInfo`) before proceeding.
- **XOR Branch:** After the `SkillAssessment`, applicants are either evaluated via a standard `CulturalFitCheck` or a `CommunityAffiliationCheck`, which introduces bias.
- **Sequential Flow:** The process proceeds from `ManagerialReview` to `FinalDecision`.

---

### **2. POWL Model Without Unfairness**
This model removes the XOR branch, ensuring all applicants undergo the same **CulturalFitCheck** without preferential treatment.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Define the loop for data completeness
loop_body = StrictPartialOrder(nodes=[DataCompletenessCheck, RequestMoreInfo])
loop_body.order.add_edge(DataCompletenessCheck, RequestMoreInfo)
data_completeness_loop = OperatorPOWL(operator=Operator.LOOP, children=[ReceiveApplication, loop_body])
loop_body.order.add_edge(ReceiveApplication, DataCompletenessCheck)

# Define the main partial order (no XOR branch)
root = StrictPartialOrder(nodes=[data_completeness_loop, SkillAssessment, CulturalFitCheck, ManagerialReview, FinalDecision])
root.order.add_edge(data_completeness_loop, SkillAssessment)
root.order.add_edge(SkillAssessment, CulturalFitCheck)
root.order.add_edge(CulturalFitCheck, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)
```

#### **Explanation:**
- **Loop:** Same as before, ensuring data completeness.
- **No XOR Branch:** All applicants undergo the same `CulturalFitCheck` without any special treatment.
- **Sequential Flow:** The process remains fair and transparent, with no preferential paths.

---

### **Key Differences:**
| **Feature**               | **Model with Unfairness** | **Model Without Unfairness** |
|---------------------------|--------------------------|-----------------------------|
| **XOR Branch**            | Yes (CulturalFitCheck vs. CommunityAffiliationCheck) | No (only CulturalFitCheck) |
| **Potential Bias**        | Present (community affiliation advantage) | Absent (equal treatment) |
| **Fairness**              | Compromised | Ensured |

These models clearly illustrate how the introduction of an XOR branch can lead to unfairness, while a linear process ensures equal treatment.
