## Proposed high-level process model

| Step order | High-level activity name           | Low-level events included                                                                                  | Main purpose                                                                                      |
| ---------: | ---------------------------------- | ---------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
|          1 | **Material Preparation and Setup** | Retrieve raw metal sheet; Scan barcode; Place metal sheet on table; Align metal sheet; Preheat metal sheet | Get the correct raw material identified, positioned, aligned, and thermally prepared for welding. |
|          2 | **Welding Assembly**               | Pick up welding tool; Weld corner A; Weld corner B                                                         | Perform the physical joining operation on the prepared metal sheet.                               |
|          3 | **Weld Integrity Inspection**      | Measure weld integrity                                                                                     | Verify that the welding operation produced an acceptable joint.                                   |
|          4 | **Protective Coating and Curing**  | Apply protective coating; Dry coating                                                                      | Add the protective surface layer and cure/dry it so the coating becomes stable.                   |
|          5 | **Final Visual Inspection**        | Visual check                                                                                               | Confirm the finished item passes the final human/operator inspection.                             |

---

## Rationale for each grouping

### 1. **Material Preparation and Setup**

**Included events:**

```text
Retrieve raw metal sheet
Scan barcode
Place metal sheet on table
Align metal sheet
Preheat metal sheet
```

These events all happen before any assembly work begins. Together, they transform the item from an unprocessed raw sheet into a work-ready component.

The barcode scan is grouped here because it is part of material intake and traceability. The placement, alignment, and preheating steps prepare the material for the next operation. Although multiple resources are involved — Operator A, Robot Arm #2, and Heating Unit #1 — the shared objective is the same: prepare the raw material for welding.

**Proposed high-level name:**
**Material Preparation and Setup**

---

### 2. **Welding Assembly**

**Included events:**

```text
Pick up welding tool
Weld corner A
Weld corner B
```

The tool pickup is grouped with the actual welding because it is the immediate setup action for the welding task. The following two events are the core joining operations. These events are also performed by the same resource, Operator B, and occur close together in time.

**Proposed high-level name:**
**Welding Assembly**

---

### 3. **Weld Integrity Inspection**

**Included event:**

```text
Measure weld integrity
```

This is a quality-control checkpoint immediately after welding. It uses a quality sensor and produces a measurement value, such as `IntegrityScore: 95` or `IntegrityScore: 93`.

It should remain separate from the welding step because it does not perform assembly; it evaluates whether the assembly was successful.

**Proposed high-level name:**
**Weld Integrity Inspection**

---

### 4. **Protective Coating and Curing**

**Included events:**

```text
Apply protective coating
Dry coating
```

These two events form a single finishing stage. Applying the coating adds the protective layer, while drying cures or stabilizes that layer. The drying step is dependent on the coating step, so they belong together as one higher-level finishing activity.

**Proposed high-level name:**
**Protective Coating and Curing**

---

### 5. **Final Visual Inspection**

**Included event:**

```text
Visual check
```

This is the final acceptance check after coating and drying. It is distinct from the earlier weld integrity inspection because it evaluates the finished item rather than just the weld quality.

**Proposed high-level name:**
**Final Visual Inspection**

---

## Suggested grouping rules

A practical abstraction rule set would look like this:

```text
IF Activity IN {
    "Retrieve raw metal sheet",
    "Scan barcode",
    "Place metal sheet on table",
    "Align metal sheet",
    "Preheat metal sheet"
}
THEN HighLevelActivity = "Material Preparation and Setup"

IF Activity IN {
    "Pick up welding tool",
    "Weld corner A",
    "Weld corner B"
}
THEN HighLevelActivity = "Welding Assembly"

IF Activity IN {
    "Measure weld integrity"
}
THEN HighLevelActivity = "Weld Integrity Inspection"

IF Activity IN {
    "Apply protective coating",
    "Dry coating"
}
THEN HighLevelActivity = "Protective Coating and Curing"

IF Activity IN {
    "Visual check"
}
THEN HighLevelActivity = "Final Visual Inspection"
```

---

## Resulting high-level process flow

```text
Material Preparation and Setup
        ↓
Welding Assembly
        ↓
Weld Integrity Inspection
        ↓
Protective Coating and Curing
        ↓
Final Visual Inspection
```

This abstraction preserves the logical manufacturing flow while reducing the original twelve low-level events per case into five meaningful process stages.
