## 2. Root Cause Analysis

### Root Cause #1: Complexity drives Document Request loops
There is a clear correlation between **Complexity** and the number of "Request Additional Documents" events:

| Complexity | Avg # Doc Requests | Avg Duration |
|------------|-------------------|--------------|
| Low        | 0                 | ~1.5 hours   |
| Medium     | 1                 | ~26 hours    |
| High       | 2–3               | 48–77 hours  |

High-complexity claims (2003, 2005) loop repeatedly through the **Request Additional Documents** step, creating multi-day delays between each iteration (often >24 hours of waiting per request).

### Root Cause #2: Adjuster_Lisa + Region B bottleneck
- Case 2005 (the slowest) involves **Adjuster_Lisa** issuing 3 sequential document requests over 3 days, with ~24-30 hours between each request.
- Case 2003 (Adjuster_Mike, Region A) also has multiple requests but fewer (2) and completes faster than 2005.
- This suggests **Adjuster_Lisa in Region B** is slower at handling high-complexity claims than Adjuster_Mike.

### Root Cause #3: Manager_Bill approval delay
- **Manager_Ann** approves Low/Medium cases relatively quickly.
- **Manager_Bill** handles high-complexity approvals (2003, 2005) and appears only after long document-gathering phases. The approval step itself follows the final document request by ~17–19 hours, indicating a possible approval queue bottleneck for complex cases.

### Root Cause #4: Overnight/Weekend idle time
- Case 2003: gap from 04-01 17:00  04-02 16:00 (23 hours between doc requests).
- Case 2005: gap from 04-02 17:00  04-03 15:00 (22 hours).
- This suggests document requests made late in the day are not worked on until the next day — creating handoff inefficiency.

## 3. Explanations & Mitigation Suggestions

| Root Cause | Explanation | Suggested Mitigation |
|-----------|-------------|----------------------|
| **Repeated doc requests on High-complexity claims** | Requests are made iteratively instead of upfront, each adding ~1 day. | Create a **complexity-based checklist** at intake so all required documents are requested at once. |
| **Adjuster_Lisa slow on complex claims in Region B** | Skill/training gap or overload. | Provide advanced training, rebalance workload, or route high-complexity Region B claims to a specialist. |
| **Manager_Bill approval delay** | Complex approvals bottleneck on a single manager. | Add secondary approvers for High complexity or set SLAs for approval turnaround. |
| **End-of-day requests sit overnight** | Idle time between process steps dominates lead time. | Implement cut-off times, automated reminders, or parallelize document collection with evaluation. |
| **Customer response latency (implicit)** | Customer may be slow to respond to doc requests. | Offer a digital self-service portal with reminders to reduce response time. |

---

## Summary
The primary driver of long lead times is **High-complexity claims requiring multiple sequential document requests**, compounded by **resource-specific inefficiencies (Adjuster_Lisa in Region B, Manager_Bill approvals)** and **overnight idle times**. Addressing document collection upfront and rebalancing complex-case resources would most directly reduce case durations.
