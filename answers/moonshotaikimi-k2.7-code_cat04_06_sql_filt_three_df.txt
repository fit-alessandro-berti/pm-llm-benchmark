Here is a DuckDB query that uses window functions to find the exact three-activity direct succession, checks the 5day timing condition, and then excludes the matching `case_id`s:

```sql
WITH ordered_events AS (
    SELECT
        *,
        LEAD(activity) OVER (
            PARTITION BY case_id
            ORDER BY timestamp, rowid
        )   AS next_activity,
        LEAD(activity, 2) OVER (
            PARTITION BY case_id
            ORDER BY timestamp, rowid
        )   AS next_next_activity,
        LEAD(timestamp) OVER (
            PARTITION BY case_id
            ORDER BY timestamp, rowid
        )   AS create_order_time,
        LEAD(timestamp, 2) OVER (
            PARTITION BY case_id
            ORDER BY timestamp, rowid
        )   AS confirm_order_time
    FROM event_log
),
excluded_cases AS (
    SELECT DISTINCT case_id
    FROM ordered_events
    WHERE activity                 = 'Approve Purchase Requisition'
      AND next_activity            = 'Create Order'
      AND next_next_activity       = 'Confirm Order'
      AND confirm_order_time - create_order_time > INTERVAL '5 days'
)
SELECT e.*
FROM event_log e
WHERE e.case_id NOT IN (SELECT case_id FROM excluded_cases)
ORDER BY e.case_id, e.timestamp;
```

**How it works**

1. **`ordered_events`** – for each event in a case, looks ahead one and two rows (ordered by `timestamp`, using `rowid` to break ties) to capture the next activities and their timestamps.
2. **`excluded_cases`** – identifies cases where the exact direct sequence  
   `Approve Purchase Requisition  Create Order  Confirm Order`  
   occurs **and** the time between `Create Order` and `Confirm Order` is more than 5 days.
3. The final `SELECT` returns every event from `event_log` **except** those belonging to an excluded case.
