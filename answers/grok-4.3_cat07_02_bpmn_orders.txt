This is a **BPMN (Business Process Model and Notation)** diagram that visualizes an **order fulfillment and invoicing process** across four organizational departments (swimlanes). It shows how an order moves from receipt through production, material handling, shipping, and invoicing, with decision points, message flows, and exception handling.

### High-Level Overview
The process starts in **Sales** when an order arrives and flows through **Production**, **Procurement**, and **Warehouse**. Key interactions occur via message flows (dashed arrows) between departments. The process includes:
- Customer management
- Material quality checks and compensation
- Production planning and manufacturing
- Final product shipment and invoicing

### Breakdown by Swimlane

**1. Sales (Top Lane)**
- **Start**: Triggered by a "Monday morning" timer event (order arrival).
- **Main Flow**:
  - Check order for completeness.
  - Check if customer exists in the database.
  - **Decision point**: If customer not found  "Add customer to the database."
  - Send order details (message flow) to Production and Procurement.
- **End**: After receiving confirmation that the product was sent, it drafts the invoice.
- **Note**: There's a database icon connected to customer-related activities.

**2. Production (Second Lane)**
- **Start**: Triggered by "Order details arrive" message from Sales.
- **Main Flow**:
  - Plan production.
  - Manufacture product.
- **Exception Handling**:
  - Includes a loop/feedback for "Material not faulty" check.
  - If materials are faulty, it can trigger actions back to Procurement.
- **End**: "Product complete" event (message sent to Warehouse).

**3. Procurement (Third Lane)**
- **Start**: Triggered by "Order details arrive" from Sales.
- **Main Flow**:
  - Contains a **Material Compensation Subprocess**:
    - Order materials.
    - Return material (if needed).
  - Check materials (with possible order cancellation).
- **Key Elements**:
  - Timer events and message flows.
  - Decision points for material quality.
  - Connection to Production for faulty material handling.

**4. Warehouse (Bottom Lane)**
- **Start**: Triggered by "Product complete" message from Production.
- **Main Flow**:
  - Send product.
- **End**: "Product sent" event (which triggers the invoice drafting in Sales).

### Key Features of the Diagram
- **Gateways**: Multiple exclusive gateways (X symbols) for decisions (e.g., customer in database? Material faulty?).
- **Message Flows**: Dashed lines show communication between departments (e.g., Sales  Production, Production  Warehouse).
- **Subprocess**: The "Material Compensation Subprocess" in Procurement handles returns and quality issues.
- **Events**: Includes start events, end events, message events, and timer events.
- **Data Objects**: References to "Order details," "Product complete," and "Customer in the database."

This diagram appears to model a **manufacturing order-to-cash process** with built-in quality control and customer onboarding steps. It highlights how exceptions (like faulty materials or new customers) are handled across departments.
