### 1. POWL Model With Potential Unfairness

In this model, the workflow follows the exact description where an **XOR (Exclusive Choice)** operator is introduced at the cultural fit stage. This creates two distinct paths: one for a standard evaluation and another for applicants who trigger the "Community Affiliation" path (the source of bias).

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# --- Define Activities (Transitions) ---
# Initial Step
t_receive_app = Transition(label="ReceiveApplication")

# Stage 1: Loop for Data Completeness (Resume Parsing)
t_data_check = Transition(label="DataCompletenessCheck")
t_request_info = Transition(label="RequestMoreInfo")

# Stage 2: Skill Assessment
t_skill_assess = Transition(label="SkillAssessment")

# Stage 3: Cultural Fit (Biased XOR Split)
t_standard_culture = Transition(label="StandardCulturalFit")
t_community_affil = Transition(label="CommunityAffiliationCheck") # Bias Source

# Stage 4: Final Steps
t_mgr_review = Transition(label="ManagerialReview")
t_final_decision = Transition(label="FinalDecision")


# --- Construct Operators ---

# 1. Loop for Data Completeness: 
# Executes DataCheck, if fails executes RequestInfo, then repeats DataCheck.
loop_data_check = OperatorPOWL(operator=Operator.LOOP, children=[t_data_check, t_request_info])

# 2. XOR Choice for Cultural Fit:
# Applicants are routed either to a Standard check OR a Community Affiliation check.
# This represents the potential unfairness described.
biased_choice = OperatorPOWL(operator=Operator.XOR, children=[t_standard_culture, t_community_affil])


# --- Construct Main Workflow (Partial Order) ---

# We define the set of nodes that constitute the process root.
# Note: The 'biased_choice' node encapsulates the branching logic.
nodes_unfair = [
    t_receive_app, 
    loop_data_check, 
    t_skill_assess, 
    biased_choice, 
    t_mgr_review, 
    t_final_decision
]

model_unfair = StrictPartialOrder(nodes=nodes_unfair)

# Define the sequence of execution (Edges)
model_unfair.order.add_edge(t_receive_app, loop_data_check)
model_unfair.order.add_edge(loop_data_check, t_skill_assess)
model_unfair.order.add_edge(t_skill_assess, biased_choice)
model_unfair.order.add_edge(biased_choice, t_mgr_review)
model_unfair.order.add_edge(t_mgr_review, t_final_decision)

# model_unfair now represents the process with potential bias.
```

### 2. POWL Model Without Unfairness

# Stage 2: Skill Assessment

# Stage 3: FAIR Cultural Fit (Single Path Only)
# We replace the previous XOR node with a single unified transition.

# Stage 4: Final Steps
t_mgr_review = Transition(label="ManagerialReview")
t_final_decision = Transition(label="FinalDecision")


# --- Construct Operators ---

# NOTE: No XOR operator is created here.


# --- Construct Main Workflow (Partial Order) ---

nodes_fair = [
    t_receive_app, 
    loop_data_check, 
    t_skill_assess,
