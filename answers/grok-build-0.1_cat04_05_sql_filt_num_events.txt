**Recommended DuckDB SQL Query:**

```sql
SELECT 
    case_id,
    activity,
    timestamp,
    -- Add any other columns from the event_log table here as needed
FROM (
    SELECT 
        *,
        COUNT(*) OVER (PARTITION BY case_id) AS case_length
    FROM event_log
) AS filtered_events
WHERE case_length <= 6;
```

### Alternative (CTE + JOIN) version:

```sql
WITH case_counts AS (
    SELECT 
        case_id,
        COUNT(*) AS case_length
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT el.*
FROM event_log el
INNER JOIN case_counts cc 
    ON el.case_id = cc.case_id;
```

---

### Recommendation:

- Use the **first query** (window function) if you want a clean, single-pass solution.
- Use the **second query** (CTE + JOIN) if you prefer more explicit logic or if you're working in a context where window functions are less desirable.

Both queries correctly:
1. Calculate the number of events per `case_id`
2. Filter out cases with more than 6 events
3. Return all events belonging to the remaining cases
