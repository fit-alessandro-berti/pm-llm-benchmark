To achieve the tasks outlined in the prompt using DuckDB, we'll follow these steps in our SQL query:

1. **Extract the ordered sequence of activities per `case_id`** by grouping events within each case and ordering them by `timestamp`.
2. **Identify unique process variants** by grouping these sequences.
3. **Count the occurrences of each variant** to determine their frequency.
4. **Select the top K variants** based on frequency.
5. **Filter the original event log** to return only those events belonging to cases that match these top K variants.

Here is the DuckDB SQL query that accomplishes these tasks:

```sql
-- Step 1: Extract ordered activity sequences for each case
WITH ordered_sequences AS (
    SELECT
        case_id,
        LIST_CONCAT(activity, ORDER BY timestamp) AS activity_sequence
    FROM
        event_log
    GROUP BY
        case_id
),

-- Step 2: Group by activity sequence to identify unique variants and count them
variant_counts AS (
    SELECT
        activity_sequence,
        COUNT(*) AS variant_count
    FROM
        ordered_sequences
    GROUP BY
        activity_sequence
),

-- Step 3: Determine the top K variants by frequency
top_k_variants AS (
    SELECT
        activity_sequence
    FROM
        variant_counts
    ORDER BY
        variant_count DESC
    LIMIT K
)

-- Step 4: Filter the original event log to include only events from cases in the top K variants
SELECT
    *
FROM
    event_log
WHERE
    case_id IN (
        SELECT
            case_id
        FROM
            ordered_sequences
        WHERE
            activity_sequence IN (
                SELECT
                    activity_sequence
                FROM
                    top_k_variants
            )
    );
```

### Explanation:

- **`ordered_sequences` CTE**: This Common Table Expression (CTE) groups the `event_log` by `case_id` and constructs a concatenated, ordered list of activities (`activity_sequence`) for each case using `LIST_CONCAT` with an order by `timestamp`.

- **`variant_counts` CTE**: This CTE groups the `activity_sequence` from `ordered_sequences` to count how many cases follow each unique sequence, thereby identifying unique process variants and their frequencies.

- **`top_k_variants` CTE**: This CTE selects the top K activity sequences based on their frequency (`variant_count DESC`) by ordering the `variant_counts` and limiting the result to K rows.

- **Final SELECT**: The main query filters the `event_log` to return all events where the `case_id` belongs to the cases identified in the top K variants. It does this by checking if the `activity_sequence` of each case is in the list of top K sequences.

### Note:
- Replace `K` with the desired number of top variants you wish to retrieve.
- Ensure that DuckDB's version supports `LIST_CONCAT` with an `ORDER BY` clause. If not, you might need to use a different approach to maintain order, such as generating a subsequence of timestamps and activities and then concatenating them accordingly.
