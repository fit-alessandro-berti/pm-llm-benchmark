### Assumptions / Mapping Used

- `order_id` is used as the **Case ID**.
- One **Item Picked** event is created per order line because each line can have its own picking timestamp and picker.
- One **Order Dispatched** and one **Order Arrived** event is created per delivery record.
- Payments are linked to orders through `invoice_id`.
- Resource names and roles are enriched using the Resources table.
- Payment events do not have an explicit resource in the source data, so the resource is marked as “Not provided”.
- Order `1004` has no delivery, invoice, or payment records in the provided data, so its trace ends after item picking.
- Timestamps are kept in UTC and events are ordered chronologically within each case.
