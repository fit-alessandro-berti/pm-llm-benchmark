# Accepting Petri Net for Insurance Claim Management Process

Below is the Python code to construct an accepting Petri net for the insurance claim management process with all the specified activities, including the conditional loops for document re-submission and in-depth investigation.

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# Create the Petri net
net = PetriNet(name="Insurance_Claim_Management")

# Create places (states between activities)
start = PetriNet.Place(name="Start")
after_registration = PetriNet.Place(name="After_Registration")
after_verification = PetriNet.Place(name="After_Verification")
after_redoc_submission = PetriNet.Place(name="After_ReDoc_Submission")
after_fraud_check = PetriNet.Place(name="After_Fraud_Check")
after_investigation = PetriNet.Place(name="After_Investigation")
after_assessment = PetriNet.Place(name="After_Assessment")
after_approval = PetriNet.Place(name="After_Approval")
after_payment = PetriNet.Place(name="After_Payment")
after_notification = PetriNet.Place(name="After_Notification")
end = PetriNet.Place(name="End")

# Add places to the net
net.places.add(start)
net.places.add(after_registration)
net.places.add(after_verification)
net.places.add(after_redoc_submission)
net.places.add(after_fraud_check)
net.places.add(after_investigation)
net.places.add(after_assessment)
net.places.add(after_approval)
net.places.add(after_payment)
net.places.add(after_notification)
net.places.add(end)

# Create transitions (activities)
claim_registration = PetriNet.Transition(name="CR", label="Claim Registration")
document_verification = PetriNet.Transition(name="DV", label="Document Verification")
redoc_submission = PetriNet.Transition(name="RDS", label="Re-Document Submission")
fraud_check = PetriNet.Transition(name="FC", label="Fraud Check")
investigation = PetriNet.Transition(name="II", label="In-depth Investigation")
claim_assessment = PetriNet.Transition(name="CA", label="Claim Assessment")
approval = PetriNet.Transition(name="AP", label="Approval")
payment = PetriNet.Transition(name="P", label="Payment")
notification = PetriNet.Transition(name="N", label="Notification")
closure = PetriNet.Transition(name="CL", label="Closure")

# Add transitions to the net
net.transitions.add(claim_registration)
net.transitions.add(document_verification)
net.transitions.add(redoc_submission)
net.transitions.add(fraud_check)
net.transitions.add(investigation)
net.transitions.add(claim_assessment)
net.transitions.add(approval)
net.transitions.add(payment)
net.transitions.add(notification)
net.transitions.add(closure)

# Add arcs between places and transitions to form the process flow

# Start -> Claim Registration -> After_Registration
petri_utils.add_arc_from_to(start, claim_registration, net)
petri_utils.add_arc_from_to(claim_registration, after_registration, net)

# After_Registration -> Document Verification -> After_Verification
petri_utils.add_arc_from_to(after_registration, document_verification, net)
petri_utils.add_arc_from_to(document_verification, after_verification, net)

# After_Verification can go to either:
# 1. Re-Document Submission -> After_ReDoc_Submission -> Document Verification (loop)
petri_utils.add_arc_from_to(after_verification, redoc_submission, net)
petri_utils.add_arc_from_to(redoc_submission, after_redoc_submission, net)
petri_utils.add_arc_from_to(after_redoc_submission, document_verification, net)

# 2. Fraud Check -> After_Fraud_Check (normal path)
petri_utils.add_arc_from_to(after_verification, fraud_check, net)
petri_utils.add_arc_from_to(fraud_check, after_fraud_check, net)

# After_Fraud_Check can go to either:
# 1. In-depth Investigation -> After_Investigation -> Fraud Check (loop)
petri_utils.add_arc_from_to(after_fraud_check, investigation, net)
petri_utils.add_arc_from_to(investigation, after_investigation, net)
petri_utils.add_arc_from_to(after_investigation, fraud_check, net)

# 2. Claim Assessment -> After_Assessment (normal path)
petri_utils.add_arc_from_to(after_fraud_check, claim_assessment, net)
petri_utils.add_arc_from_to(claim_assessment, after_assessment, net)

# After_Assessment -> Approval -> After_Approval
petri_utils.add_arc_from_to(after_assessment, approval, net)
petri_utils.add_arc_from_to(approval, after_approval, net)

# After_Approval -> Payment -> After_Payment
petri_utils.add_arc_from_to(after_approval, payment, net)
petri_utils.add_arc_from_to(payment, after_payment, net)

# After_Payment -> Notification -> After_Notification
petri_utils.add_arc_from_to(after_payment, notification, net)
petri_utils.add_arc_from_to(notification, after_notification, net)

# After_Notification -> Closure -> End
petri_utils.add_arc_from_to(after_notification, closure, net)
petri_utils.add_arc_from_to(closure, end, net)

# Define initial marking (token in Start place)
im = Marking()
im[start] = 1

# Define final marking (token in End place)
fm = Marking()
fm[end] = 1
```

## Explanation of the Petri Net Structure:

1. **Main Flow**:
   - Start  Claim Registration  Document Verification  Fraud Check  Claim Assessment  Approval  Payment  Notification  Closure  End

2. **Document Verification Loop**:
   - After Document Verification, if documents are incomplete, the process goes to Re-Document Submission and then back to Document Verification.

3. **Fraud Check Loop**:
   - After Fraud Check, if doubts are raised, the process goes to In-depth Investigation and then back to Fraud Check (can repeat multiple times).

4. **Initial and Final Markings**:
   - Initial marking has a token in the "Start" place (representing a new claim)
   - Final marking has a token in the "End" place (representing a closed claim)

This Petri net accurately models the insurance claim management process with all the specified activities and conditional loops, providing a complete representation of the process flow.
