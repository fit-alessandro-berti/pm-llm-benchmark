Below are two POWL models constructed with pm4py. Both share the same intake, data-completeness loop, and sequencing. They differ only in how they handle the cultural/affiliation step after SkillAssessment.

Model 1 (with potential unfairness) introduces an XOR branch where CommunityAffiliationCheck can subtly tilt outcomes. Model 2 removes that branch so everyone goes through the same CulturalFitCheck.

Python (pm4py) definitions:

import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# ----------------------------
# Model 1: POWL with potential unfairness (XOR branch includes CommunityAffiliationCheck)
# ----------------------------

# Activities
ReceiveApplication_u = Transition(label="ReceiveApplication")
ResumeParsing_u = Transition(label="ResumeParsing")
DataCompletenessCheck_u = Transition(label="DataCompletenessCheck")
RequestMoreInfo_u = Transition(label="RequestMoreInfo")
SkillAssessment_u = Transition(label="SkillAssessment")
CulturalFitCheck_u = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck_u = Transition(label="CommunityAffiliationCheck")  # Potential bias path
ManagerialReview_u = Transition(label="ManagerialReview")
FinalDecision_u = Transition(label="FinalDecision")

# Loop for data completeness: execute DataCompletenessCheck, then optionally RequestMoreInfo and re-check
completeness_loop_u = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck_u, RequestMoreInfo_u])

# XOR after skill assessment: either standard CulturalFitCheck OR CommunityAffiliationCheck (biased uplift)
fit_or_affiliation_xor = OperatorPOWL(operator=Operator.XOR, children=[CulturalFitCheck_u, CommunityAffiliationCheck_u])

# Assemble the process as a strict partial order (sequence)
root_unfair = StrictPartialOrder(
    nodes=[ReceiveApplication_u, ResumeParsing_u, completeness_loop_u, SkillAssessment_u, fit_or_affiliation_xor, ManagerialReview_u, FinalDecision_u]
)
root_unfair.order.add_edge(ReceiveApplication_u, ResumeParsing_u)
root_unfair.order.add_edge(ResumeParsing_u, completeness_loop_u)
root_unfair.order.add_edge(completeness_loop_u, SkillAssessment_u)
root_unfair.order.add_edge(SkillAssessment_u, fit_or_affiliation_xor)  # Potential unfairness introduced here
root_unfair.order.add_edge(fit_or_affiliation_xor, ManagerialReview_u)
root_unfair.order.add_edge(ManagerialReview_u, FinalDecision_u)

# ----------------------------
# Model 2: POWL without unfairness (no special community branch; everyone gets CulturalFitCheck)
# ----------------------------

# Activities (separate instances to avoid node sharing across models)
ReceiveApplication_f = Transition(label="ReceiveApplication")
ResumeParsing_f = Transition(label="ResumeParsing")
DataCompletenessCheck_f = Transition(label="DataCompletenessCheck")
RequestMoreInfo_f = Transition(label="RequestMoreInfo")
SkillAssessment_f = Transition(label="SkillAssessment")
CulturalFitCheck_f = Transition(label="CulturalFitCheck")  # Single, uniform path
ManagerialReview_f = Transition(label="ManagerialReview")
FinalDecision_f = Transition(label="FinalDecision")

# Same loop for data completeness
completeness_loop_f = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck_f, RequestMoreInfo_f])

# Assemble the process as a strict partial order (linear sequence without XOR bias)
root_fair = StrictPartialOrder(
    nodes=[ReceiveApplication_f, ResumeParsing_f, completeness_loop_f, SkillAssessment_f, CulturalFitCheck_f, ManagerialReview_f, FinalDecision_f]
)
root_fair.order.add_edge(ReceiveApplication_f, ResumeParsing_f)
root_fair.order.add_edge(ResumeParsing_f, completeness_loop_f)
root_fair.order.add_edge(completeness_loop_f, SkillAssessment_f)
root_fair.order.add_edge(SkillAssessment_f, CulturalFitCheck_f)  # No affiliation-based branch
root_fair.order.add_edge(CulturalFitCheck_f, ManagerialReview_f)
root_fair.order.add_edge(ManagerialReview_f, FinalDecision_f)

Notes:
- In Model 1, the XOR node fit_or_affiliation_xor explicitly marks where subtle unfairness can enter via CommunityAffiliationCheck.
- In Model 2, that XOR is removed; all applicants go through the same CulturalFitCheck before ManagerialReview and FinalDecision.
