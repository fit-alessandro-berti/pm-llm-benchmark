Here is the DuckDB SQL query that fulfills the requirements described in your task:

```sql
WITH valid_cases AS (
    -- Identify case_ids that have 6 or fewer events
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
-- Return all event rows for those valid cases
SELECT el.*
FROM event_log el
JOIN valid_cases vc ON el.case_id = vc.case_id;
```

### Explanation of the Logic:
1.  **CTE (`valid_cases`)**: This Common Table Expression first aggregates the data to count events per `case_id`. It uses `GROUP BY case_id` and filters using `HAVING COUNT(*) <= 6` to isolate only the valid process instances.
2.  **Main Query**: The main query selects all columns (`el.*`) from the original `event_log` table where the `case_id` matches one of the IDs identified in the CTE.
3.  **JOIN**: Using an `INNER JOIN` ensures that only rows belonging to the filtered cases are returned, effectively excluding any case with more than six events.

**Alternative (Window Function):**
If you prefer a single-pass approach using window functions, DuckDB supports this syntax as well:

```sql
SELECT *
FROM (
    SELECT *, COUNT(*) OVER (PARTITION BY case_id) AS event_count
    FROM event_log
) t
WHERE event_count <= 6;
```
