Here is a DuckDB SQL solution that does exactly that.

```sql
-- Replace ? with the desired K value (or use a parameter from your client)
WITH case_variants AS (
    -- 1) Build the ordered activity sequence for each case
    SELECT
        case_id,
        string_agg(activity, ' -> ' ORDER BY "timestamp") AS variant_sequence
    FROM event_log
    GROUP BY case_id
),

variant_counts AS (
    -- 2) Count how many cases belong to each unique variant
    SELECT
        variant_sequence,
        COUNT(*) AS case_count
    FROM case_variants
    GROUP BY variant_sequence
),

top_k_variants AS (
    -- 3) Keep only the top K variants by frequency
    SELECT
        variant_sequence,
        case_count
    FROM variant_counts
    ORDER BY case_count DESC, variant_sequence
    LIMIT ?
),

filtered_cases AS (
    -- 4) Find all case_ids whose variant is in the top K
    SELECT
        cv.case_id,
        cv.variant_sequence
    FROM case_variants cv
    INNER JOIN top_k_variants tkv
        ON cv.variant_sequence = tkv.variant_sequence
)

-- 5) Return all original events for those cases only
SELECT
    e.case_id,
    e.activity,
    e."timestamp",
    fc.variant_sequence,
    tkv.case_count AS variant_frequency
FROM event_log e
INNER JOIN filtered_cases fc
    ON e.case_id = fc.case_id
INNER JOIN top_k_variants tkv
    ON fc.variant_sequence = tkv.variant_sequence
ORDER BY
    tkv.case_count DESC,
    fc.variant_sequence,
    e.case_id,
    e."timestamp";
```

### What this does
- `case_variants`: creates one ordered activity sequence per `case_id`
- `variant_counts`: groups identical sequences into process variants and counts cases
- `top_k_variants`: selects the most frequent `K` variants
- `filtered_cases`: identifies the cases belonging to those top variants
- final `SELECT`: returns all original events for only those cases

### Note
If multiple events in the same case can share the exact same `timestamp`, the ordering may be ambiguous. In that situation, add a secondary sort key, e.g.:

```sql
string_agg(activity, ' -> ' ORDER BY "timestamp", event_id)
```

if such a column exists.
