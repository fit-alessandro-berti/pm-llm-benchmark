Here's a DuckDB SQL query that accomplishes exactly what you described:

```sql
-- First approach: Using a subquery in the WHERE clause
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
ORDER BY case_id, timestamp;
```

**Alternative approach using a CTE (Common Table Expression):**
```sql
-- Second approach: Using a CTE for better readability
WITH cases_to_keep AS (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT e.*
FROM event_log e
JOIN cases_to_keep c ON e.case_id = c.case_id
ORDER BY e.case_id, e.timestamp;
```

**How it works:**
1. The subquery/CTE groups events by `case_id` and counts the events in each case
2. The `HAVING COUNT(*) <= 6` filters to keep only cases with 6 or fewer events
3. The outer query selects all events from the original table where the `case_id` matches one of the filtered cases
4. Results are ordered by `case_id` and `timestamp` for better readability

**Key points:**
- This efficiently excludes entire cases that have more than 6 events
- It returns all columns for the remaining events
- The grouping happens at the case level, not the event level
- Both approaches are functionally equivalent - use whichever you find more readable

The query will return all events from cases containing exactly 1, 2, 3, 4, 5, or 6 events, while completely excluding any cases with 7 or more events.
