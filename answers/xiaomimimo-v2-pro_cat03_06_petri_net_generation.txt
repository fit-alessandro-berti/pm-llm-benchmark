from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# Create places for different states in the process
start_place = PetriNet.Place(name="start")
after_cr = PetriNet.Place(name="after_cr")
after_dv = PetriNet.Place(name="after_dv")

after_ca = PetriNet.Place(name="after_ca")
after_ap = PetriNet.Place(name="after_ap")
after_payment = PetriNet.Place(name="after_payment")
after_notification = PetriNet.Place(name="after_notification")
end_place = PetriNet.Place(name="end")

# Add all places to the net
for place in [start_place, after_cr, after_dv, after_rds, after_fc, 
              after_ii, after_ca, after_ap, after_payment, 
              after_notification, end_place]:
    net.places.add(place)

# Create transitions for each activity
cr_transition = PetriNet.Transition(name="cr", label="CR")
dv_transition = PetriNet.Transition(name="dv", label="DV")
rds_transition = PetriNet.Transition(name="rds", label="RDS")
fc_transition = PetriNet.Transition(name="fc", label="FC")
ii_transition = PetriNet.Transition(name="ii", label="II")
ca_transition = PetriNet.Transition(name="ca", label="CA")
ap_transition = PetriNet.Transition(name="ap", label="AP")
payment_transition = PetriNet.Transition(name="payment", label="P")
notification_transition = PetriNet.Transition(name="notification", label="N")
closure_transition = PetriNet.Transition(name="closure", label="CL")

# Add all transitions to the net
for transition in [cr_transition, dv_transition, rds_transition, fc_transition, 
                   ii_transition, ca_transition, ap_transition, payment_transition, 
                   notification_transition, closure_transition]:
    net.transitions.add(transition)

# Create arcs to connect places and transitions

# 1. Start to CR
petri_utils.add_arc_from_to(start_place, cr_transition, net)
petri_utils.add_arc_from_to(cr_transition, after_cr, net)

# 2. CR to DV
petri_utils.add_arc_from_to(after_cr, dv_transition, net)
petri_utils.add_arc_from_to(dv_transition, after_dv, net)

# 3. DV to RDS loop (if documents incomplete)
petri_utils.add_arc_from_to(after_dv, rds_transition, net)

This Petri net models the insurance claim process with:

1. **Main flow**: Start  CR  DV  FC  CA  (AP if needed)  Payment  Notification  Closure  End

2. **Document resubmission loop**: DV  RDS (if documents incomplete)

3. **Fraud investigation loop**: FC  II (if fraud doubts arise)

4. **Conditional approval**: After CA, claims can go either directly to Payment (low amount) or through AP then Payment (high amount)

The initial marking starts with one token in the `start` place (representing a new claim), and the final marking has one token in the `end` place (representing a closed claim).

Key features of this model:
- The RDS loop allows documents to be resubmitted multiple times
- The II loop allows multiple rounds of in-depth investigation
- Both approval paths (with or without AP) converge at the Payment transition
- All paths eventually lead to Notification and Closure
