## 3) How these factors increase cycle times + recommendations

### How they increase cycle times
- **Escalations** add processing time because tickets move to another queue/team.
- **Waiting times** between steps mean the ticket is idle even though it is not closed.
- **Delayed investigation** postpones diagnosis and resolution.
- **Rework / repeated investigation** implies the first attempt did not solve the issue, extending the lifecycle.
- **Overnight pauses** suggest limited staffing or batching behavior, which can dramatically stretch lead time.

---

## Recommendations

### 1. Reduce unnecessary escalations
- Improve triage rules so tickets are routed correctly earlier.
- Provide Level-1 agents with better knowledge bases or decision support.
- Track escalation reasons to identify common patterns that could be resolved at Level-1.

### 2. Shorten time-to-investigation
- Set service targets for ōAssign to Investigateö latency.
- Use queue prioritization for aging tickets.
- Monitor tickets that sit too long after assignment.

### 3. Improve specialist availability
- If Level-2 is a bottleneck, increase staffing or define faster handoff procedures.
- Introduce clear SLAs for escalated cases.

### 4. Reduce rework
- Ensure the first investigation captures enough information.
- Standardize diagnostic checklists to reduce back-and-forth.

### 5. Address overnight delays
- If possible, add cross-shift coverage or automated next-step assignment.
- If overnight handling is expected, separate ōbusiness hours delayö from actual process inefficiency so performance is measured fairly.
