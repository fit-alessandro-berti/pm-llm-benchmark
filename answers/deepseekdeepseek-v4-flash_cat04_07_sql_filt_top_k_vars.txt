Below is a DuckDB SQL query that solves the given problem step by step. It constructs the ordered activity sequence per case, counts cases per variant, selects the top `K` variants (using `ROW_NUMBER` to handle ties deterministically), and finally returns all events from cases belonging to those top variants.

```sql
-- Parameter: replace :K with the desired number of top variants (e.g., 3)
WITH
-- 1. Build the ordered activity sequence per case
case_sequences AS (
    SELECT
        case_id,
        STRING_AGG(activity, ',' ORDER BY timestamp) AS variant_seq
    FROM event_log
    GROUP BY case_id
),
-- 2. Count the number of cases per variant
variant_counts AS (
    SELECT
        variant_seq,
        COUNT(*) AS num_cases
    FROM case_sequences
    GROUP BY variant_seq
),
-- 3. Select exactly the top K variants (ties broken by variant_seq alphabetically)
top_variants AS (
    SELECT variant_seq
    FROM (
        SELECT
            variant_seq,
            num_cases,
            ROW_NUMBER() OVER (ORDER BY num_cases DESC, variant_seq) AS rn
        FROM variant_counts
    ) ranked
    WHERE rn <= :K   -- Replace :K with integer (e.g., 3, 5, 10)
)
-- 4. Return all events from only those cases that belong to the top K variants
SELECT el.*
FROM event_log el
INNER JOIN case_sequences cs
    ON el.case_id = cs.case_id
INNER JOIN top_variants tv
    ON cs.variant_seq = tv.variant_seq
ORDER BY el.case_id, el.timestamp;
```

### Explanation of the query:

- **`case_sequences`** – Aggregates all activities per `case_id` in the order of `timestamp`, creating a commaseparated sequence. This defines the process variant for that case.
- **`variant_counts`** – Counts how many unique cases share each variant sequence.
- **`top_variants`** – Uses `ROW_NUMBER()` to assign a unique rank to each variant, ordered by `num_cases` descending (most frequent first). In case of ties, `variant_seq` is used as a deterministic tiebreaker. Only the first `:K` rows are kept.
- **Final `SELECT`** – Joins the original `event_log` back to the tables `case_sequences` and `top_variants` to filter only those cases whose variant is among the top `K`. The result includes all original columns and rows from those cases.

### Usage:
Replace `:K` with an actual integer (e.g., `3`) or use DuckDB’s parameter syntax (e.g., `PREPARE` + `EXECUTE`) if the value is provided at runtime. The query returns the filtered event log as required.
