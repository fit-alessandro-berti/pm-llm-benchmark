# BPMN Process Diagram: Dispatch of Goods — Computer Hardware Shop

This is a **Business Process Model and Notation (BPMN) diagram** showing how a computer hardware shop dispatches goods. The process is organized into a pool titled "Dispatch of goods / Computer Hardware Shop" with three **swimlanes** representing different roles:

## Lanes and Flow

###  Start (Secretary lane)
- The process begins with the start event **"Ship goods"**
- A **parallel gateway (+)** splits the work into two simultaneous branches: one handled by the Secretary, one by the Warehouse

### Secretary Lane (middle)
1. **Clarify shipment method** (with an attached "Order details" document artifact)
2. An **exclusive gateway (X)** asks: *"Special handling?"*
   - **No**  **Check if insurance is required** (this task reads from a **"Customer information" data store**, shown as a database cylinder)
     - An **inclusive/event-based gateway** then branches:
       - *"If insurance necessary"*  goes up to the Logistics lane
       - *"Always"*  **Write package label**
     - These paths merge back at another gateway
   - **Yes**  **Get 3 offers from logistic companies**  **Select logistic company and place order**
3. Both paths converge at an **exclusive merge gateway (X)**

### Logistics Lane (top)
- **Insure parcel** — executed only when insurance is deemed necessary, then rejoins the main flow

### Warehouse Lane (bottom)
1. **Package goods** — this task has an attached **timer boundary event (48 hours)**:
   - If packaging takes longer than 48 hours  **Notify customer about delay**
   - Both paths merge at a gateway
2. A **parallel join gateway (+)** synchronizes the Secretary's branch and the Warehouse branch
3. **Fill out shipment paperwork**  **Move package to pickup area**
4. The process concludes with the end event **"Shipment prepared"** 

## Key BPMN Elements Used
- **Start/End events** (circles)
- **Tasks** (rounded rectangles)
- **Parallel gateways** (+) for concurrent work
- **Exclusive gateways** (X) for decisions
- **Timer boundary event** for escalation/delay handling
- **Data store and data object** artifacts
