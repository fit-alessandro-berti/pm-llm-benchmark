**DuckDB SQL Query:**

```sql
WITH filtered_cases AS (
    SELECT case_id
    FROM event_log
    WHERE activity IN ('Approve Purchase Requisition', 'Create Order', 'Confirm Order')
      AND timestamp BETWEEN (
          (SELECT MIN(timestamp)
           FROM event_log e2
           WHERE e2.case_id = event_log.case_id
             AND e2.activity IN ('Approve Purchase Requisition', 'Create Order')
         ) + interval '1 day'
          AND (SELECT MAX(timestamp)
               FROM event_log e3
               WHERE e3.case_id = event_log.case_id
                 AND e3.activity = 'Confirm Order')
      )
    GROUP BY case_id
    HAVING COUNT(*) = 3
)

SELECT *
FROM event_log
WHERE case_id NOT IN (SELECT case_id FROM filtered_cases);
```

**Explanation:**

1. **Identify Cases with the Specific Sequence:**  
   The `filtered_cases` common table expression (CTE) uses a subquery to find cases where all three activities ('Approve Purchase Requisition', 'Create Order', and 'Confirm Order') appear in direct succession within that case. It does this by checking if there is only one set of such sequential activities per `case_id`.

2. **Check the Time Elapsed Between Events:**  
   Within the subquery, we calculate whether the time difference between the `'Create Order'` and `'Confirm Order'` events exceeds 5 days for each case that contains all three activities in sequence.

3. **Exclude Such Cases from Final Result:**  
   Finally, the main query selects all rows from `event_log` except those where `case_id` is present in the `filtered_cases` CTE, effectively returning only events from cases that do not meet both conditions (i.e., they either lack the sequence or have a time gap of less than 5 days between `'Create Order'` and `'Confirm Order'`).
