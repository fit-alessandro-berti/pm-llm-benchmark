Here's a DuckDB SQL query that meets all the specified requirements:

```sql
WITH ordered_events AS (
    -- Assign sequential row numbers to events within each case
    SELECT 
        *,
        ROW_NUMBER() OVER (
            PARTITION BY case_id 
            ORDER BY timestamp, activity
        ) as row_num
    FROM event_log
),
-- Identify cases that meet both conditions
cases_to_exclude AS (
    SELECT DISTINCT o1.case_id
    FROM ordered_events o1
    JOIN ordered_events o2 ON o1.case_id = o2.case_id AND o2.row_num = o1.row_num + 1
    JOIN ordered_events o3 ON o1.case_id = o3.case_id AND o3.row_num = o1.row_num + 2
    WHERE 
        o1.activity = 'Approve Purchase Requisition'
        AND o2.activity = 'Create Order'
        AND o3.activity = 'Confirm Order'
        -- Check that there are no intermediate events (which is guaranteed by the row numbering)
        AND DATEDIFF('day', o2.timestamp, o3.timestamp) > 5
)
-- Return all events except those from excluded cases
SELECT el.*
FROM event_log el
WHERE el.case_id NOT IN (SELECT case_id FROM cases_to_exclude)
ORDER BY el.case_id, el.timestamp;
```

### How this query works:

1. **`ordered_events` CTE**: Assigns sequential row numbers to each event within a case, ordered by timestamp and activity (to handle potential ties).

2. **`cases_to_exclude` CTE**: 
   - Joins three consecutive events within each case using the row numbers
   - Filters for the exact activity sequence: 'Approve Purchase Requisition'  'Create Order'  'Confirm Order'
   - Applies the timing condition: checks if more than 5 days elapsed between 'Create Order' and 'Confirm Order'
   - Returns only `case_id`s that satisfy both conditions

3. **Final SELECT**: Returns all events from the original `event_log` table except those belonging to cases identified in the exclusion list.

### Key assumptions:
- The `ROW_NUMBER()` with `ORDER BY timestamp, activity` ensures we're looking at truly consecutive events (no other events between them)
- The `DATEDIFF('day', ...)` function calculates the difference in days between the two timestamps
- If multiple instances of this sequence exist in a case, at least one meeting the timing condition is sufficient to exclude the entire case

This approach efficiently identifies and excludes the specified cases while returning all events from the remaining cases.
