### **1. POWL Model with Potential Unfairness**
Here the XOR after `SkillAssessment` creates two paths: standard cultural fit vs. community affiliation uplift. That’s where the bias enters.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# Define activities
receive_app = Transition(label="ReceiveApplication")
data_check = Transition(label="DataCompletenessCheck")
request_info = Transition(label="RequestMoreInfo")
skill_assess = Transition(label="SkillAssessment")
cultural_fit = Transition(label="CulturalFitCheck")
community_check = Transition(label="CommunityAffiliationCheck")
managerial_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# Loop: check data completeness, request more info until complete
completeness_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# XOR: biased branching - standard path vs community affiliation uplift path
cultural_xor = OperatorPOWL(
    operator=Operator.XOR, 
    children=[cultural_fit, community_check]
)

# Overall partial order: sequential flow with XOR branch
root_unfair = StrictPartialOrder(nodes=[
    receive_app, 
    completeness_loop, 
    skill_assess, 
    cultural_xor, 
    managerial_review, 
    final_decision
])

# Define execution order
root_unfair.order.add_edge(receive_app, completeness_loop)
root_unfair.order.add_edge(completeness_loop, skill_assess)
root_unfair.order.add_edge(skill_assess, cultural_xor)
root_unfair.order.add_edge(cultural_xor, managerial_review)
root_unfair.order.add_edge(managerial_review, final_decision)
```

**Where unfairness appears:** The `cultural_xor` node. Applicants routed through `CommunityAffiliationCheck` get a subjective uplift if they’re in “Global Business Leaders Club” or flagged as local, while others only get `CulturalFitCheck`. Same skill score, different evaluation path.

### **2. POWL Model Without Unfairness**
Same overall flow, but we remove the XOR. Every applicant goes through the identical `CulturalFitCheck`. The loop for data completeness stays, as does the sequence.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# Define activities - no CommunityAffiliationCheck needed
receive_app = Transition(label="ReceiveApplication")
data_check = Transition(label="DataCompletenessCheck")
request_info = Transition(label="RequestMoreInfo")
skill_assess = Transition(label="SkillAssessment")
cultural_fit = Transition(label="CulturalFitCheck")
managerial_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# Loop: check data completeness, request more info until complete
completeness_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# Overall partial order: strict sequence, no biased XOR
root_fair = StrictPartialOrder(nodes=[
    receive_app, 
    completeness_loop, 
    skill_assess, 
    cultural_fit,  # no XOR - everyone takes same path
    managerial_review, 
    final_decision
])

# Define execution order - fully sequential
root_fair.order.add_edge(receive_app, completeness_loop)
root_fair.order.add_edge(completeness_loop, skill_assess)
root_fair.order.add_edge(skill_assess, cultural_fit)
root_fair.order.add_edge(cultural_fit, managerial_review)
root_fair.order.add_edge(managerial_review, final_decision)
```

**Key differences:**

| Aspect | Unfair Model | Fair Model |
| --- | --- | --- |
| **Loop** | `* (DataCompletenessCheck, RequestMoreInfo)` | Same |
| **Skill → Culture** | `X (CulturalFitCheck, CommunityAffiliationCheck)` | Direct `CulturalFitCheck` |
| **Bias source** | XOR gives selective advantage via `CommunityAffiliationCheck` | Removed - single evaluation path |
| **Sequence** | Receive → Loop → Skill → XOR → Review → Decision | Receive → Loop → Skill → Culture → Review → Decision |

Both models keep your loop for data completeness and the sequential ordering. The only structural change is collapsing the XOR choice into a single activity, which removes the mechanism that enabled affiliation-based score adjustments.
