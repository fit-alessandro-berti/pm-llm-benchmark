This is a **BPMN diagram** — Business Process Model and Notation. It maps out how an order moves through 4 departments: Sales, Production, Procurement, and Warehouse. Each horizontal lane is one department, and the arrows show how work and messages flow between them.

### **Overall flow**
The process starts Monday morning in Sales and ends when the Warehouse ships the product and Sales drafts the invoice.

### **Lane by lane breakdown**

**Sales**
- **Trigger**: Monday morning, start event kicks off.
- **Check customer's order**: First check the order for completeness, then check if the customer is already in the database.
- **Gateway decision**: If customer not in database → Add customer to the database. If already in → skip that step.
- **Send order details**: Once customer is confirmed, order details are sent to Production and Procurement via message flows.
- **Wait + Invoice**: After Production/Warehouse finish, Sales receives a "Product sent" message, then drafts the invoice and the process ends.

**Production**
- **Receive**: Gets "Order details arrive" message from Sales.
- **Plan production**: Parallel gateway splits the flow.
- **Manufacture product**: Build the item.
- **Loop for quality**: If material is faulty, it loops back before manufacturing. If material not faulty, it continues.
- **Complete**: Sends "Product complete" message to Warehouse.

**Procurement**
- **Receive**: Also gets "Order details arrive" from Sales.
- **Material Compensation Subprocess**: 
  - Order materials
  - If material is faulty: trigger "Return material" compensation event
- **Check materials**: After ordering, check if materials are OK.
- **Exception path**: If there’s an "Order cancellation" error event, the process can terminate early.

**Warehouse**
- **Receive**: Gets "Product complete" message from Production.
- **Send product**: Ship the item to the customer.
- **Notify**: Sends "Product sent" message back to Sales to trigger invoicing.

### **Key BPMN symbols you’re seeing**

| Symbol | Meaning |
| --- | --- |
| Circle | Start, intermediate, or end event |
| Rounded rectangle | Task/activity someone does |
| Diamond with X | Exclusive gateway: pick one path |
| Diamond with + | Parallel gateway: do all paths |
| Dashed arrow | Message flow between lanes/departments |
| Solid arrow | Sequence flow within a lane |
| Envelope icon | Message event: send or receive |
| Double border event | Boundary event attached to a task, like error or compensation |

### **The big picture**
This diagram enforces a clear handoff: Sales validates and kicks off the order, Production and Procurement work in parallel, Warehouse ships, and Sales closes the loop with invoicing. It also bakes in error handling for faulty materials and cancellations, so the process doesn’t just break when something goes wrong.
