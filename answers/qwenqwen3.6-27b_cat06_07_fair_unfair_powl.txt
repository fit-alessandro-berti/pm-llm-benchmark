###  Model 1: POWL with Potential Unfairness
This model explicitly includes an `XOR` choice after the skill assessment. One path follows a standard cultural fit evaluation, while the other routes applicants with local/community affiliations to a `CommunityAffiliationCheck` activity that implicitly boosts their score. This branching structure is where the bias is introduced.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# ==================== ACTIVITY DEFINITIONS ====================
receive_app = Transition(label="ReceiveApplication")
data_check = Transition(label="DataCompletenessCheck")
request_info = Transition(label="RequestMoreInfo")
skill_assess = Transition(label="SkillAssessment")
cultural_fit = Transition(label="CulturalFitCheck")
community_affil = Transition(label="CommunityAffiliationCheck")
manager_review = Transition(label="ManagerialReview")
final_dec = Transition(label="FinalDecision")

# ==================== CONTROL FLOW OPERATORS ====================
# Loop: Check completeness -> if missing, request info -> loop back to check
completeness_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# XOR Choice: Standard cultural fit VS Community affiliation check (Bias injection point)
eval_branch = OperatorPOWL(operator=Operator.XOR, children=[cultural_fit, community_affil])

# ==================== PARTIAL ORDER CONSTRUCTION ====================
unfair_nodes = [receive_app, completeness_loop, skill_assess, eval_branch, manager_review, final_dec]
unfair_powl = StrictPartialOrder(nodes=unfair_nodes)

# Define precedence constraints (execution order)
unfair_powl.order.add_edge(receive_app, completeness_loop)
unfair_powl.order.add_edge(completeness_loop, skill_assess)
unfair_powl.order.add_edge(skill_assess, eval_branch)
unfair_powl.order.add_edge(eval_branch, manager_review)
unfair_powl.order.add_edge(manager_review, final_dec)

# unfair_powl is now ready for analysis, visualization, or execution in pm4py
```

** How Unfairness Appears:**
- The `eval_branch` XOR node splits the workflow. Applicants flagged for local/community ties go to `CommunityAffiliationCheck`, which structurally bypasses the standard `CulturalFitCheck`.
- In process mining or fairness auditing, this exclusive branch creates a divergent evaluation path that can be traced to disparate outcomes, even if the underlying scoring logic is subtle.
