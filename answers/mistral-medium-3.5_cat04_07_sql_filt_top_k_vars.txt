# DuckDB SQL Query for Process Variant Analysis

Here's a comprehensive DuckDB SQL query that solves the problem by:
1. Extracting ordered activity sequences per case
2. Identifying unique variants and their frequencies
3. Filtering for top K variants
4. Returning all events from cases in those top K variants

```sql
WITH
-- Step 1: Get ordered activity sequences for each case
case_sequences AS (
    SELECT
        case_id,
        LIST(activity ORDER BY timestamp) AS activity_sequence
    FROM event_log
    GROUP BY case_id
),

-- Step 2: Count frequency of each variant
variant_counts AS (
    SELECT
        activity_sequence,
        COUNT(*) AS case_count,
        -- Generate a unique variant ID for easier joining
        ROW_NUMBER() OVER (ORDER BY COUNT(*) DESC) AS variant_rank
    FROM case_sequences
    GROUP BY activity_sequence
),

-- Step 3: Get the top K variants (parameterized with K)
top_variants AS (
    SELECT activity_sequence
    FROM variant_counts
    WHERE variant_rank <= K  -- Replace K with your desired number of top variants
),

-- Step 4: Get all case_ids that belong to top K variants
top_case_ids AS (
    SELECT DISTINCT e.case_id
    FROM event_log e
    JOIN case_sequences cs ON e.case_id = cs.case_id
    JOIN top_variants tv ON cs.activity_sequence = tv.activity_sequence
)

-- Final result: All events from cases in top K variants
SELECT e.*
FROM event_log e
JOIN top_case_ids t ON e.case_id = t.case_id
ORDER BY e.case_id, e.timestamp;
```

## Explanation:

1. **case_sequences CTE**:
   - Groups events by `case_id`
   - Uses DuckDB's `LIST` aggregate function with `ORDER BY timestamp` to create the ordered activity sequence for each case

2. **variant_counts CTE**:
   - Groups the sequences to count how many cases follow each variant
   - Uses `ROW_NUMBER()` to rank variants by frequency (most common first)

3. **top_variants CTE**:
   - Filters to only the top K variants by their rank
   - Replace `K` with your desired number (e.g., 5 for top 5 variants)

4. **top_case_ids CTE**:
   - Identifies all case_ids that belong to the top K variants

5. **Final query**:
   - Joins back to the original event log to get all events
   - Only includes events from cases that are in the top K variants
   - Orders results by case_id and timestamp for readability

## Usage Notes:

- Replace `K` with your desired number of top variants (e.g., 5, 10, etc.)
- The query uses DuckDB-specific functions like `LIST` which creates an array of values
- For very large datasets, you might want to add appropriate indexes on `case_id` and `timestamp`
- The final result includes all original columns from the event log for the filtered cases

## Alternative for DuckDB versions without LIST:

If your DuckDB version doesn't support the `LIST` function, you can use `STRING_AGG` instead:

```sql
case_sequences AS (
    SELECT
        case_id,
        STRING_AGG(activity, '|' ORDER BY timestamp) AS activity_sequence
    FROM event_log
    GROUP BY case_id
)
```

This will create a pipe-delimited string of activities for each case. The rest of the query would remain the same.
