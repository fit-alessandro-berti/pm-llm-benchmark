This visualization is a **BPMN diagram** (Business Process Model and Notation) that maps out the "Dispatch of goods" workflow for a Computer Hardware Shop. It shows who does what, in what order, and how decisions branch.

### The three swimlanes
The diagram is split into three horizontal lanes that represent the different roles/departments involved:

| Lane | What happens here |
| --- | --- |
| **Warehouse** | Physical handling: packaging the goods, dealing with delays |
| **Secretary** | Admin + coordination: clarifying shipment, checking insurance, selecting logistics, writing labels |
| **Logistics** | External action: insuring the parcel when needed |

### How the process flows

**1. Process start: "Ship goods"** 
The flow kicks off with a start event, then immediately hits a **parallel gateway (+)**. This means two things start at the same time:
- **Warehouse track**: `Package goods` begins. There's a timer boundary event on this task: if packaging takes longer than **48 hours**, the process triggers `Notify customer about delay`. Both paths rejoin before moving on.
- **Secretary track**: `Clarify shipment method` begins using the `Order details` document.

**2. Secretary decisions**
After clarifying the method, there's an **exclusive gateway (X)** for `Special handling?`
- **No** → Check if insurance is required using `Customer information`. Another gateway splits this: if insurance is necessary, the task goes to the **Logistics** lane to `Insure parcel`. Either way, the Secretary then does `Write package label`.
- **Yes** → The Secretary has to `Get 3 offers from logistic companies`, then `Select logistic company and place order`.

**3. Convergence and finish**
All the separate paths come back together at another gateway before hitting a second **parallel gateway (+)**. This is a sync point: the process waits until *both* the packaging work in the Warehouse AND all the Secretary/Logistics admin work is done. 

Once both are complete, the Warehouse does `Fill out Shipment paperwork` → `Move package to pickup area` → end event `Shipment prepared`.

### Key BPMN symbols used here

- **Circle**: Start/end events. Thin circle = start, thick circle = end.
- **Rounded rectangle**: Task or activity someone performs.
- **Diamond with +**: Parallel gateway. All paths happen.
- **Diamond with X**: Exclusive gateway. Only one path is chosen based on yes/no condition.
- **Diamond with O**: Inclusive gateway. One or more paths can happen. Used here for the insurance branch.
- **Document icon**: Data object, like `Order details`.
- **Database icon**: Data store, like `Customer information`.
- **Circle on task border**: Boundary event. The 48-hour timer interrupts packaging if it runs too long.

**The big picture**: This process is designed so physical packaging and admin work happen in parallel to save time. It has built-in checks for special handling, insurance, and delays. The only way a shipment moves to the pickup area is after paperwork is done AND the goods are packaged AND any required insurance/logistics selection is complete.
