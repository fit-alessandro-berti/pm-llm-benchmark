**BPMNlike Process Model (Textual Description)**  

```
Process: Invoice Approval Workflow

Start Event: Invoice Receipt
  -> Activity: Accounts Payable Clerk – Mary checks for PO number.
      - If NO PO Number:
          -> Task: Request More Details from Supplier
          -> End Event (Rejected / Resend to Supplier)
      - If YES PO Number:
          -> Task: Log Invoice into Accounting System

Intermediate Event: Purchase Order Validation
  -> Gate: IF PO Matches Inventory Receipt?
      - Yes:
          -> Activity: Purchasing Officer – Dan verifies invoice details against PO.
              * Check quantities, prices, charges.
              * Resolve discrepancies (contact supplier or hold for correction).
          -> End Event (Proceed to Manager Approval)
      - No:
          -> Task: Escalate Issue to Accounts Payable Supervisor – Rita
            (to investigate missing/incorrect PO references)

Intermediate Event: Manager Approval Triggered by Validation Success
  -> Gate: IF All Matching Conditions Met? (PO, Goods Received, Price Correct)
      - Yes:
          -> Activity: Department Manager – Carlos reviews invoice.
              * Approve if all looks good and budgetary constraints are satisfied.
              * Reject or Request Further Investigation for anomalies.
          -> End Event (Proceed to Payment Scheduling)
      - No:
          -> Task: Return to Accounts Payable Clerk – Mary with Rework Note

Intermediate Exception Handling by Accounts Payable Supervisor – Rita
  -> Task: Resolve Exceptions Identified During Approval Steps
      * Handle invoices without PO, informal services contracts.
      * Create retrospective PO or decide on rejection if cannot be reconciled.

Final End Event: Payment Scheduling
  -> Activity: Accounts Payable schedules the invoice for payment in next cycle.
```

**Key Elements Explained**

- **Activities / Tasks**:  
  - “Log Invoice” (Mary), “Request More Details” (Karen), “Verify PO & Goods Received” (Dan).  

- **Sequence Flow**:  
  - Begins with receipt  checks PO  logging into system.  
  - Followed by purchaseorder validation, then manager approval if everything aligns.  

- **Gateways / Decision Points**:  
  - PO number missing?  request details from supplier.  
  - PO matches inventory & all fields match?  proceed to manager.  
  - Manager rejects/approves or flags anomalies  escalation path to Rita.  

- **Roles / Participants**:  
  - Accounts Payable Clerk (Mary), Purchasing Officer (Dan), Department Manager (Carlos), Accounts Payable Supervisor (Rita), Supplier Relationship Manager (Karen).  

- **Artifacts / Inputs/Outputs**:  
  - Invoice, PO number, purchase order details, supplier contact information, approval emails.  

- **Exceptions / Variations**:  
  - Handling of missing PO numbers, price or quantity discrepancies, informal contracts requiring retrospective PO creation, escalation to Rita for unresolved issues.  

**Ambiguities Noted**

- Some interviewees mention “sometimes” a step (e.g., escalations are rare) but do not quantify frequency.  
- The exact timing of the automatic email from Dan’s side to Carlos is not specified in all accounts; assume it follows successful PO & goods verification.
