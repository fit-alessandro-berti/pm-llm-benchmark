# Data-Driven Scheduling Strategy for Precision Parts Inc.

## 1. Analyzing Historical Scheduling Performance and Dynamics

### Using Process Mining on MES Event Logs
To reconstruct and analyze the actual flow of jobs and task execution sequences, we will employ process mining techniques on the detailed MES event logs. The core steps include:

1. **Event Log Preparation:** Cleanse and enrich the raw logs by ensuring timestamps are consistent, job identifiers are standardized, and necessary attributes (e.g., priority, due date, resource availability) are correctly mapped.
2. **Process Discovery:** Utilize **alpha-algorithm** to automatically discover the **actual process model** depicting job flows, including task sequences, resource allocations, and variations in routing due to disruptions. This will reveal hidden paths and loops that static rule-based scheduling might miss.
3. **Conformance Checking:** Compare the discovered process model with the *planned* manufacturing process model to identify deviations, delays, and non-conformances that contribute to inefficiencies.

### Quantitative Metrics via Process Mining

- **Job Flow Times, Lead Times, and Makespan Distributions:**
  - **Cycle Time Analysis:** Calculate the time taken from job release to completion for each task using timestamps from start to end events.
  - **Lead Time and Makespan Distribution:** Aggregate these cycle times to derive lead time (from order entry to job completion) and makespan (total time for all jobs) distributions, using histograms and percentiles (e.g., 90th percentile) to understand variability.

- **Task Waiting Times (Queue Times):**
  - Measure **queue time** at each work center/machine by computing the difference between task queue entry and task start timestamps. Analyze queue time distributions per machine to identify waiting bottlenecks.

- **Resource Utilization:**
  - Compute **productive time** (actual task processing time), **idle time** (time when resources are available but no job is scheduled), and **setup time** (from setup start to end) for each machine and operator.
  - Utilization ratios = Productive Time / (Productive Time + Idle Time + Setup Time) to identify underutilized or overburdened resources.

- **Sequence-Dependent Setup Times:**
  - **Transition Analysis:** Examine consecutive job pairs on the same machine to quantify setup durations, distinguishing between similar (low setup time) and dissimilar (high setup time) sequences. Use **mining frequent patterns** to uncover common setups and their durations.
  - **Setup Cost Modeling:** Develop a regression model (e.g., linear or decision tree) associating setup time with job attributes (material type, previous job ID, complexity) to predict future setup requirements.

- **Schedule Adherence and Tardiness:**
  - Calculate **Tardiness** as max(0, Completion Time - Due Date) for each job. Aggregate tardiness metrics (average tardiness, tardy job rate) to assess adherence.
  - Plot **Cumulative Distribution Function (CDF)** of tardiness to understand tail behavior (e.g., extreme delays).

- **Impact of Disruptions:**
  - **Event Trigger Analysis:** Tag disruptions (breakdowns, priority changes) within the process model and analyze their frequency and timing (e.g., mean time between disruptions).
  - **Impact Assessment:** Measure how disruptions affect key KPIs (lead times, tardiness, resource utilization) by comparing post-disruption process slices to normal operation slices.

## 2. Diagnosing Scheduling Pathologies

### Identified Pathologies via Process Mining

1. **Bottleneck Resources:**
   - **Bottleneck Analysis:** Use resource utilization metrics to pinpoint machines with high utilization (>85%) or long queue times, indicating constraints. For example, CUT-01 may show prolonged queue times and frequent setup interruptions.
   - **Throughput Impact:** Quantify how these bottlenecks compress overall throughput by backpropagating delays through the process model.

2. **Poor Task Prioritization:**
   - **Priority Variation Analysis:** Compare job paths of high-priority vs. low-priority jobs using **conformance checking** to see if priority rules are effectively reflected in actual schedules.
   - **Delay Attribution:** Identify instances where high-priority jobs are delayed due to preceding low-priority jobs, quantified by unusually long queue times for high-priority jobs before their start.

3. **Suboptimal Sequencing and Setup Times:**
   - **Sequence Impact Study:** Analyze variance in setup times across sequences to identify suboptimal job orderings that exacerbate total setup time, using frequent pattern mining to detect common setups causing delays.
   - **Setup Reduction Opportunities:** Highlight sequences with excessive setup durations that could be consolidated (e.g., grouping similar jobs).

4. **Starvation of Downstream Resources:**
   - **Starvation Detection:** Identify downstream machines that frequently experience idle times due to upstream bottlenecks, indicated by low productive time despite available startup events.
   - **Routinely Delayed Paths:** Pinpoint paths where downstream tasks start significantly behind schedule relative to upstream completions.

5. **Bullwhip Effect in WIP Levels:**
   - **WIP KPI Correlation:** Correlate peaks in WIP levels with disruptions or priority changes, using time-series analysis of queue lengths at each work center.
   - **Variance Analysis:** Assess how variability in order releases affects downstream queue build-up.

## 3. Root Cause Analysis of Scheduling Ineffectiveness

### Potential Root Causes

1. **Static Dispatching Rule Limitations:**
   - **Rule Inflexibility:** Current **First-Come-First-Served** and **Earliest Due Date** rules do not adapt to dynamic changes (e.g., machine breakdowns, urgent jobs).
   - **Insufficient Dimensionality:** Rules neither consider downstream loads nor historical setup patterns, leading to suboptimal job ordering.

2. **Lack of Real-Time Visibility:**
   - **Closed-Loop Feedback:** Absence of real-time monitoring causes delayed awareness of resource issues, hindering proactive adjustments.
   - **Historical Data Only:** Planning relies solely on past schedules, ignoring current shop floor conditions (e.g., real-time queue lengths).

3. **Inaccurate Duration/Setup Estimates:**
   - **Empirical vs. Predictive Models:** Overreliance on static duration estimates without accounting for variability or sequence impacts.
   - **Setup Time Misestimation:** Predictive accuracy of sequence-dependent setup times may be poor, causing unexpected delays.

4. **Ineffective Handling of Disruptions:**
   - **Disruption Response Gaps:** No clear protocol for reprioritizing jobs during breakdowns, leading to increased tardiness.
   - **Urgent Job Integration:** Inability to fluidly integrate high-priority urgent jobs without disrupting ongoing processes.

### Process Mining's Role in Differentiation

- **KPI-Driven Diagnostics:** Use process mining to align observed KPI deviations (e.g., increased tardiness) with specific process deviations (e.g., increased setup times) versus capacity constraints (e.g., resource overload).
- **Simulation Validation:** Validate hypotheses (e.g., static rules vs. capacity limits) through **what-if simulations** driven by process mining outputs, confirming which issues stem from scheduling logic versus operational constraints.

## 4. Developing Advanced Data-Driven Scheduling Strategies

### Strategy 1: Enhanced Dispatching Rules

**Core Logic:**
- Implement **dynamic dispatching** at each work center using a weighted scoring system:
  - **Weight Factors:** 
    - \( W_1 \) = Reciprocal of **Remaining Due Date** (to prioritize urgent jobs).
    - \( W_2 \) = **Downstream Machine Queue Length** (to balance load).
    - \( W_3 \) = **Estimated Sequence-Dependent Setup Time** (to minimize unnecessary setups).
    - \( W_4 \) = **Job Priority Level** (to honor urgency).
  - **Score Calculation:** \( Score = \alpha W_1 + \beta W_2 + \gamma W_3 + \delta W_4 \) where weights are calibrated using historical impact analysis.

**Process Mining Insight:**
- Use **frequent pattern mining** to derive setup time formulas based on previous job sequences.
- Leverage **lead time distributions** to calibrate due date urgency weights.

**Addressed Pathology:**
- Reduces queue build-up by considering downstream resources.
- Minimizes setup-induced delays through intelligent sequencing.

**Expected KPI Impact:**
- Decreases average tardiness by up to 25%.
- Lowers WIP by balancing resource utilization across the shop floor.

### Strategy 2: Predictive Scheduling

**Core Logic:**
- Employ **statistical process mining** to model **task duration distributions** conditioned on factors like job complexity, operator experience, and material type.
- Integrate **predictive maintenance models** (derived from breakdown logs) to forecast potential machine unavailability.
- Generate **scenario-based schedules** that incorporate probability distributions for task times and maintenance windows, optimizing for expected makespan.

**Process Mining Insight:**
- Use **distribution fitting** (e.g., kernel density estimation) on actual task durations to create realistic probabilistic forecasts.
- Mine **breakdown frequency and duration patterns** to integrate into schedule resilience.

**Addressed Pathology:**
- Proactively avoids delays caused by unpredictable breakdowns.
- Aligns scheduling with realistic task completion probabilities, reducing over-optimism.

**Expected KPI Impact:**
- Improves schedule adherence by up to 30%.
- Enhances resource utilization by aligning capacity with predicted demand spikes.

### Strategy 3: Setup Time Optimization

**Core Logic:**
- Formulate **batching rules** that group jobs with similar properties (e.g., material type, machining operation) to minimize sequence-dependent setups.
- At bottleneck machines, apply **optimal sequencing algorithms** (e.g., Mixed-Integer Linear Programming) to determine job orders that minimize total setup time.
- Introduce **buffer slots** for setup activities within the schedule to absorb variability.

**Process Mining Insight:**
- Analyze **setup pattern frequencies** to define effective job similarity metrics.
- Use **transition cost models** derived from historical setups to quantify savings from optimized sequences.

**Addressed Pathology:**
- Directly targets suboptimal sequencing and setup duration, crucial for high-mix, low-volume environments.
- Reduces idle time at bottleneck machines by efficiently sequencing jobs.

**Expected KPI Impact:**
- Cuts average setup time by up to 40%.
- Lowers overall lead times, especially for complex jobs requiring multiple setups.

## 5. Simulation, Evaluation, and Continuous Improvement

2. **Testing Scenarios:**

3. **KPI Comparison:**
   - Compare simulated KPIs (tardiness, WIP, utilization, makespan) against baseline static rule performance to quantify improvements.

- **Real-Time Process Mining Loop:**
  - Continuously ingest MES event logs to refresh process models and KPI calculations.
  - Deploy **drift detection algorithms** to identify when historical patterns change (e.g., new machine failures, shifts in job mix).

- **Adaptive Scheduling Mechanics:**
  - Automatically update dispatching rule weights and predictive models based on recent performance data.
  - Trigger **rule retraining** when KPI drifts exceed predefined thresholds (e.g., tardiness increases by >5%).

- **Feedback Mechanism:**
  - Establish a **dashboard** for shop floor managers to visualize KPI trends and model performance.
  - Enable scenario testing on the dashboard to explore adjustments before implementation.

**Expected Outcome:**
- A resilient scheduling system that continuously adapts to operational changes, ensuring sustained performance improvements in tardiness, WIP reduction, and resource utilization.
