**LogSkeleton for the orderfulfilment process**

```python
log_skeleton = {
    # -----------------------------------------------------------------
    # 1  Equivalence – activities that must appear the same number of times
    # -----------------------------------------------------------------
    # All activities are required exactly once, therefore we can declare a
    # few “equivalence” pairs (any pair with the same frequency would be
    # admissible).
    "equivalence": {
        ("RO",  "CA"),   # Receive Order    Check Availability
        ("PO",  "GSL"),  # Pack Order      Generate Shipping Label
        ("IC",  "RP")    # Invoice Customer   Record Payment
    },

    # -----------------------------------------------------------------
    # 2  AlwaysBefore – if the first activity occurs, the second must
    #     have already occurred earlier in the same case.
    # -----------------------------------------------------------------
    "always_before": {
        ("CA",  "RO"),   # Check Availability after Receive Order
        ("RI",  "CA"),   # Reserve Items after Check Availability
        ("PI",  "RI"),   # Pick Items after Reserve Items
        ("QC",  "PI"),   # Quality Check after Pick Items
        ("PO",  "QC"),   # Pack Order after Quality Check
        ("GSL", "PO"),   # Generate Shipping Label after Pack Order
        ("D",   "GSL"),  # Dispatch after Generate Shipping Label
        ("IC",  "D"),    # Invoice after Dispatch
        ("RP",  "IC")    # Record Payment after Invoice
    },

    # -----------------------------------------------------------------
    # 3  AlwaysAfter – if the first activity occurs, the second must
    #     occur later (not necessarily immediately).
    # -----------------------------------------------------------------
    "always_after": {
        ("RO",  "CA"),   # Receive Order before Check Availability
        ("CA",  "RI"),   # Check Availability before Reserve Items
        ("RI",  "PI"),   # Reserve Items before Pick Items
        ("PI",  "QC"),   # Pick Items before Quality Check
        ("QC",  "PO"),   # Quality Check before Pack Order
        ("PO",  "GSL"),  # Pack Order before Generate Shipping Label
        ("GSL", "D"),    # Generate Shipping Label before Dispatch
        ("D",   "IC"),   # Dispatch before Invoice
        ("IC",  "RP")    # Invoice before Record Payment
    },

    # -----------------------------------------------------------------
    # 4  NeverTogether – activities that must not coexist in the same case
    # -----------------------------------------------------------------
    # In this straightforward fulfillment flow there is no pair that is
    # mutually exclusive, therefore the set is empty.
    "never_together": set(),

    # -----------------------------------------------------------------
    # 5  Activity Frequency – allowed number of occurrences per case
    # -----------------------------------------------------------------
    # Every activity is mandatory and occurs exactly once.
    "activ_freq": {
        "RO":  {1},
        "CA":  {1},
        "RI":  {1},
        "PI":  {1},
        "QC":  {1},
        "PO":  {1},
        "GSL": {1},
        "D":   {1},
        "IC":  {1},
        "RP":  {1}
    },

    # -----------------------------------------------------------------
    # 6  DirectlyFollows – the second activity must occur immediately
    #     after the first one.
    # -----------------------------------------------------------------
    "directly_follows": {
        ("RO",  "CA"),
        ("CA",  "RI"),
        ("RI",  "PI"),
        ("PI",  "QC"),
        ("QC",  "PO"),
        ("PO",  "GSL"),
        ("GSL", "D"),
        ("D",   "IC"),
        ("IC",  "RP")
    }
}
```

**Explanation of the chosen constraints**

| Constraint | Rationale for the chosen pairs |
|------------|--------------------------------|
| **Equivalence** | All three selected pairs occur the same number of times (once). |
| **AlwaysBefore** | Guarantees that a later step cannot start before its predecessor has finished. |
| **AlwaysAfter** | The mirror of *alwaysbefore*; it makes the ordering explicit from the predecessor’s point of view. |
| **NeverTogether** | No mutually exclusive activities exist in this linear fulfillment flow, so the set is empty. |
| **Activity Frequency** | Every activity is mandatory and appears exactly once per order case. |
| **DirectlyFollows** | The process is essentially a straightline workflow; each activity must be executed immediately after its predecessor. |

The dictionary `log_skeleton` can be fed directly into a LogSkeletonbased conformancechecking tool or used as a declarative specification for the orderfulfilment process.
