Assumptions made (to keep a meaningful event granularity):
- **Case ID** = `order_id`.
- For **picking**, one event is created **per order line** using `item_picking_date` (so each picker/date is preserved).
- For **delivery**, one event is created **per delivery record** for both **Dispatch** and **Arrival** (including carrier info).
- For **payments**, one event is created **per payment record** (can be multiple installments).
- If an order has multiple invoices, each invoice and its payment(s) are included in chronological order.
