## 4. Justification of naming choices

- **Initial Document Verification** – Used for the first document check right after the request is initiated. The context clearly describes an identity/document check, and it occurs before any problem or correction.

- **Re-Verify Documents after Additional Upload** – In C001, this verification happens only after the customer uploaded the missing ID scan (`Submit Additional`). The context says the agent “re-checks documents now that ID scan is provided,” so it is a re-verification triggered by new evidence.

- **Re-Verify Documents after Account Adjustment** – In C003, the second verification follows an `Adjust Account` event, and the context states “Re-verify docs after account adjustment.” This distinguishes it from both the initial check and the re-verification in C001.

- **Call Customer to Request Missing Documents** – The C001 call occurs because the initial verification was incomplete; the context explicitly says the agent calls “to request missing docs.”

- **Call Customer to Clarify Account Discrepancy** – The first C002 call happens before any account change and the context states it is to “clarify account discrepancy.”

- **Call Customer to Confirm Adjustments** – The second C002 call follows `Adjust Account` and the context says it is a “follow-up call to confirm adjustments made.”

- **Call Customer to Return to Branch** – The C003 call is not about information or confirmation; the customer physically left the branch, so the officer calls to ask them to come back.

By renaming the recurring activities this way, the process flow for each case becomes explicit: it is now clear which verification is an initial check versus a re-check, and which call is for information-gathering versus follow-up.
