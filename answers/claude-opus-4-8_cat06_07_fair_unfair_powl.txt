# POWL Models for GlobalTeq Solutions Hiring Process

## Model 1: POWL Model with Potential Unfairness

This model includes the XOR branch where applicants can either go through a standard cultural fit evaluation **or** take the community-affiliation path that introduces subtle bias.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# --- Activities ---
receive_application = Transition(label="ReceiveApplication")
data_check          = Transition(label="DataCompletenessCheck")
request_more_info   = Transition(label="RequestMoreInfo")
skill_assessment    = Transition(label="SkillAssessment")
cultural_fit        = Transition(label="CulturalFitCheck")
community_check     = Transition(label="CommunityAffiliationCheck")
managerial_review   = Transition(label="ManagerialReview")
final_decision      = Transition(label="FinalDecision")

# --- Loop for data completeness ---
# Execute DataCompletenessCheck; if info is missing, RequestMoreInfo and recheck.
data_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[data_check, request_more_info]
)

# --- XOR branch (BIAS POINT) ---
# Either a standard cultural fit evaluation, OR a community affiliation check
# that grants an implicit advantage.
cultural_choice = OperatorPOWL(
    operator=Operator.XOR,
    children=[cultural_fit, community_check]
)

# --- Root partial order (sequential ordering of stages) ---
root_unfair = StrictPartialOrder(
    nodes=[
        receive_application,
        data_loop,
        skill_assessment,
        cultural_choice,
        managerial_review,
        final_decision
    ]
)
root_unfair.order.add_edge(receive_application, data_loop)
root_unfair.order.add_edge(data_loop, skill_assessment)
root_unfair.order.add_edge(skill_assessment, cultural_choice)
root_unfair.order.add_edge(cultural_choice, managerial_review)
root_unfair.order.add_edge(managerial_review, final_decision)
```

**Where the unfairness appears:** The `cultural_choice` XOR node lets some applicants be routed to `CommunityAffiliationCheck` instead of the neutral `CulturalFitCheck`. This branch is the structural source of bias, because affiliated/local candidates receive a subjective uplift that other candidates never get.

---

## Model 2: POWL Model Without Unfairness

This model removes the XOR branch entirely. **Every** applicant undergoes the same `CulturalFitCheck`, so there is no selective advantage path.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# --- Activities ---
receive_application = Transition(label="ReceiveApplication")
data_check          = Transition(label="DataCompletenessCheck")
request_more_info   = Transition(label="RequestMoreInfo")
skill_assessment    = Transition(label="SkillAssessment")
cultural_fit        = Transition(label="CulturalFitCheck")
managerial_review   = Transition(label="ManagerialReview")
final_decision      = Transition(label="FinalDecision")

# --- Loop for data completeness (unchanged) ---
data_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[data_check, request_more_info]
)

# --- Root partial order ---
# No XOR branch: all applicants follow the SAME cultural fit evaluation.
root_fair = StrictPartialOrder(
    nodes=[
        receive_application,
        data_loop,
        skill_assessment,
        cultural_fit,          # single, uniform path for everyone
        managerial_review,
        final_decision
    ]
)
root_fair.order.add_edge(receive_application, data_loop)
root_fair.order.add_edge(data_loop, skill_assessment)
root_fair.order.add_edge(skill_assessment, cultural_fit)
root_fair.order.add_edge(cultural_fit, managerial_review)
root_fair.order.add_edge(managerial_review, final_decision)
```

---

## Summary of the Difference

| Feature | Model 1 (Unfair) | Model 2 (Fair) |
|---|---|---|
| Data completeness loop |  `*(DataCompletenessCheck, RequestMoreInfo)` |  Same |
| Skill assessment (sequential) |  |  |
| Cultural evaluation |  **XOR** between `CulturalFitCheck` and `CommunityAffiliationCheck` |  Single uniform `CulturalFitCheck` |
| Source of bias | The XOR branch routes affiliated/local candidates to a favorable path | None — same path for all |
| Managerial review  Final decision |  Sequential |  Sequential |

**Key structural change:** Model 1 contains an `OperatorPOWL(Operator.XOR, [CulturalFitCheck, CommunityAffiliationCheck])`. Model 2 replaces this choice with a single deterministic activity, eliminating the discriminatory branch while preserving the rest of the process (the loop, the sequential ordering, and the human review stage).
