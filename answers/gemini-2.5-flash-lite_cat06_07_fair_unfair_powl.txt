Here are the two POWL models representing the GlobalTeq Solutions hiring process, one with potential unfairness and one without.

We will use the following POWL components:
-   `Transition` for individual activities.
-   `OperatorPOWL` with `Operator.LOOP` for loops.
-   `OperatorPOWL` with `Operator.XOR` for exclusive choices.
-   `StrictPartialOrder` to define the overall workflow and ordering.

First, we define common activities that appear in both models:

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Common Activities
InitialApplication = Transition(label="InitialApplication")
ResumeParsingAndCheck = Transition(label="ResumeParsingAndCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Loop for Data Completeness (common to both models)
# *(A, B) means execute A, then if A indicates continuation, execute B, then go back to A.
# If A indicates completion, exit loop.
DataCompletenessLoop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[ResumeParsingAndCheck, RequestMoreInfo]
)
```

---

### 1. POWL Model with Potential Unfairness

This model includes an XOR split after the skill assessment. One branch leads to a standard cultural fit evaluation, while the other involves a `CommunityAffiliationCheck` which can introduce bias.

```python
# --- POWL Model 1: With Potential Unfairness ---

# Activities specific to the unfairness branch
StandardCulturalFitEvaluation = Transition(label="StandardCulturalFitEvaluation")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")

# XOR split for cultural fit/affiliation check
# One branch is standard, the other includes affiliation check
CulturalBiasXOR = OperatorPOWL(
    operator=Operator.XOR,
    children=[StandardCulturalFitEvaluation, CommunityAffiliationCheck]
)

# Define the overall workflow for Model 1
root1_nodes = [
    InitialApplication,
    DataCompletenessLoop,
    SkillAssessment,
    CulturalBiasXOR,
    ManagerialReview,
    FinalDecision
]
root1 = StrictPartialOrder(nodes=root1_nodes)

# Define the sequential and branching order
root1.order.add_edge(root1_nodes[0], root1_nodes[1]) # InitialApplication --> DataCompletenessLoop
root1.order.add_edge(root1_nodes[1], root1_nodes[2]) # DataCompletenessLoop --> SkillAssessment
root1.order.add_edge(root1_nodes[2], root1_nodes[3]) # SkillAssessment --> CulturalBiasXOR
root1.order.add_edge(root1_nodes[3], root1_nodes[4]) # CulturalBiasXOR --> ManagerialReview
root1.order.add_edge(root1_nodes[4], root1_nodes[5]) # ManagerialReview --> FinalDecision

print("--- POWL Model 1: With Potential Unfairness ---")
# In a real scenario, you might visualize or serialize this model.
# For demonstration, we print a representation of the structure.
print("Model Structure:")
print("InitialApplication --> DataCompletenessLoop --> SkillAssessment --> CulturalBiasXOR")
print("  CulturalBiasXOR branches into:")
print("    -> StandardCulturalFitEvaluation")
print("    -> CommunityAffiliationCheck")
print("  (Both branches converge before ManagerialReview)")
print("--> ManagerialReview --> FinalDecision")
print("\nPotential unfairness point: The 'CommunityAffiliationCheck' branch in the XOR.")
```

**Explanation for Model 1:**
The model starts with `InitialApplication`, followed by the `DataCompletenessLoop` which ensures all necessary information is gathered via `ResumeParsingAndCheck` and `RequestMoreInfo`. After passing the `SkillAssessment`, applicants enter an `XOR` split (`CulturalBiasXOR`). One path is the `StandardCulturalFitEvaluation`, while the other is the `CommunityAffiliationCheck`. This `CommunityAffiliationCheck` activity is where preferential treatment can occur, making the process potentially unfair. Both paths then converge before `ManagerialReview` and `FinalDecision`.

---

### 2. POWL Model Without Unfairness

This model ensures all applicants undergo the same standardized cultural fit evaluation, removing the selective XOR branch present in the first model.

```python
# --- POWL Model 2: Without Unfairness ---

# Activity for a uniform cultural fit evaluation
UniformCulturalFitEvaluation = Transition(label="UniformCulturalFitEvaluation")

# Define the overall workflow for Model 2
root2_nodes = [
    InitialApplication,
    DataCompletenessLoop, # Re-used from common activities
    SkillAssessment,      # Re-used from common activities
    UniformCulturalFitEvaluation,
    ManagerialReview,     # Re-used from common activities
    FinalDecision         # Re-used from common activities
]
root2 = StrictPartialOrder(nodes=root2_nodes)

# Define the sequential order
root2.order.add_edge(root2_nodes[0], root2_nodes[1]) # InitialApplication --> DataCompletenessLoop
root2.order.add_edge(root2_nodes[1], root2_nodes[2]) # DataCompletenessLoop --> SkillAssessment
root2.order.add_edge(root2_nodes[2], root2_nodes[3]) # SkillAssessment --> UniformCulturalFitEvaluation
root2.order.add_edge(root2_nodes[3], root2_nodes[4]) # UniformCulturalFitEvaluation --> ManagerialReview
root2.order.add_edge(root2_nodes[4], root2_nodes[5]) # ManagerialReview --> FinalDecision

print("\n--- POWL Model 2: Without Unfairness ---")
# For demonstration, we print a representation of the structure.
print("Model Structure:")
print("InitialApplication --> DataCompletenessLoop --> SkillAssessment --> UniformCulturalFitEvaluation --> ManagerialReview --> FinalDecision")
print("\nFairness ensured: All applicants undergo the same 'UniformCulturalFitEvaluation'.")
```

**Explanation for Model 2:**
This model follows the same initial steps: `InitialApplication`, `DataCompletenessLoop`, and `SkillAssessment`. However, instead of an XOR split, all applicants proceed directly to a single, standardized `UniformCulturalFitEvaluation` activity. This eliminates the possibility of bias introduced by a separate community affiliation check path. The process then continues sequentially to `ManagerialReview` and `FinalDecision`, ensuring a fairer assessment.
