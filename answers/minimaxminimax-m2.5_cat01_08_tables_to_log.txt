### Assumptions and Notes:
- **Resource Mapping**: Resource IDs (e.g., `SP-001`, `WH-001`) were mapped to their corresponding names and roles using the Resources table.
- **Payment Events**: Since the Payments table does not specify a company resource responsible for receiving payment, the Resource column is set to "N/A" (not applicable). In practice, this might be automated or recorded by a billing system.
- **Missing Data**: Order 1004 has no delivery, invoice, or payment records, so only the order creation and item picking events are included.
- **Chronological Order**: Events are sorted by `Case ID` (order_id) and then by `Timestamp` to reflect the actual process flow.
- **Multiple Deliveries**: For order 1002, which had two deliveries (DLV-5002 and DLV-5003), both dispatch and arrival events are included in the order they occurred.
- **Item Picking**: Each line item in the Order Lines table generates a separate "Item Picked" event to capture the granular picking times and pickers.
