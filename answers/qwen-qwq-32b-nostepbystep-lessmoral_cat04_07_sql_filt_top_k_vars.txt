```sql
WITH case_sequences AS (
    SELECT 
        case_id,
        ARRAY_AGG(activity ORDER BY timestamp) AS activity_sequence
    FROM event_log
    GROUP BY case_id
),
variant_counts AS (
    SELECT 
        activity_sequence AS variant,
        COUNT(*) AS frequency
    FROM case_sequences
    GROUP BY variant
),
top_variants AS (
    SELECT variant
    FROM variant_counts
    ORDER BY frequency DESC
    LIMIT K
)
SELECT el.*
FROM event_log el
JOIN case_sequences cs ON el.case_id = cs.case_id
WHERE cs.activity_sequence IN (SELECT variant FROM top_variants);
```

**Explanation:**

1. **case_sequences:** For each `case_id`, it creates an ordered array of activities (`activity_sequence`) based on `timestamp`.

2. **variant_counts:** Groups all cases by their `activity_sequence` (now considered as a `variant`) and counts the occurrences (frequency) of each variant.

3. **top_variants:** Selects the top K variants ordered by their frequency (descending). The `LIMIT K` restricts to the top K variants.

4. **Final SELECT:** Retrieves all events from `event_log` where the corresponding `case_id` has an `activity_sequence` that matches one of the variants in `top_variants`.

This approach ensures only events from cases belonging to the top K variants are returned.
