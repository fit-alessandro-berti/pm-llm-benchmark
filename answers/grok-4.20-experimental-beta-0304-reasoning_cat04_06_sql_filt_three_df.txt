```sql
WITH ordered_events AS (
    SELECT 
        *,
        ROW_NUMBER() OVER (PARTITION BY case_id ORDER BY timestamp) AS seq
    FROM event_log
),
bad_sequences AS (
    SELECT DISTINCT o1.case_id
    FROM ordered_events o1
    JOIN ordered_events o2 
        ON o1.case_id = o2.case_id 
        AND o1.seq + 1 = o2.seq
    JOIN ordered_events o3 
        ON o2.case_id = o3.case_id 
        AND o2.seq + 1 = o3.seq
    WHERE o1.activity = 'Approve Purchase Requisition'
      AND o2.activity = 'Create Order'
      AND o3.activity = 'Confirm Order'
      AND (o3.timestamp - o2.timestamp) > INTERVAL '5 days'
)
SELECT * 
FROM event_log
WHERE case_id NOT IN (SELECT case_id FROM bad_sequences)
ORDER BY case_id, timestamp;
```

This query:
- Uses `ROW_NUMBER()` to establish strict event order per case.
- Self-joins on consecutive sequence numbers to enforce *direct succession* with no intervening events.
- Applies the exact activity sequence and the >5-day condition between "Create Order" and "Confirm Order".
- Collects distinct `case_id`s that match the bad pattern.
- Returns **all columns** for every event belonging to any case *not* in the bad set (i.e. excludes the entire case once a violating pattern is found).
