Here's a DuckDB SQL query that accomplishes the task:

```sql
-- Step 1-3: Identify top K variants by frequency
WITH case_variants AS (
    SELECT 
        case_id,
        STRING_AGG(activity, ' -> ' ORDER BY timestamp) AS variant,
        COUNT(*) OVER () AS total_cases
    FROM event_log
    GROUP BY case_id
),
variant_counts AS (
    SELECT 
        variant,
        COUNT(*) AS variant_frequency,
        ROUND(COUNT(*) * 100.0 / MAX(total_cases), 2) AS percentage
    FROM case_variants
    GROUP BY variant
    ORDER BY variant_frequency DESC
    LIMIT 5  -- Top K variants (change 5 to your desired K)
),
top_variant_cases AS (
    SELECT DISTINCT case_id
    FROM case_variants
    WHERE variant IN (SELECT variant FROM variant_counts)
)

-- Step 4: Return events for cases in top K variants
SELECT 
    e.*,
    cv.variant AS case_variant,
    vc.variant_frequency,
    vc.percentage
FROM event_log e
INNER JOIN case_variants cv ON e.case_id = cv.case_id
INNER JOIN variant_counts vc ON cv.variant = vc.variant
ORDER BY 
    vc.variant_frequency DESC,
    e.case_id,
    e.timestamp;
```

## Explanation:

### **CTE 1: `case_variants`**
- Groups events by `case_id`
- Uses `STRING_AGG` with `ORDER BY timestamp` to create the ordered activity sequence
- Example result: `'A -> B -> C'` represents activities A, then B, then C
- `COUNT(*) OVER ()` counts total cases across all variants

### **CTE 2: `variant_counts`**
- Groups by variant sequence to count unique variants
- Calculates frequency (how many cases follow each variant)
- Calculates percentage of total cases
- `LIMIT 5` returns top 5 variants (change this number to your desired K)
- Orders by frequency (most common first)

### **CTE 3: `top_variant_cases`**
- Extracts distinct `case_id`s that belong to top K variants
- Uses `WHERE variant IN (SELECT variant FROM variant_counts)` to filter

### **Final SELECT**
- Joins back to original event log
- Only includes events from cases in top K variants
- Adds variant details and statistics for context
- Orders results by variant frequency, case_id, and timestamp

## Key Features:

1. **Correct Variant Creation**: `STRING_AGG(activity, ' -> ' ORDER BY timestamp)` ensures activities are ordered by timestamp within each case
2. **Frequency Analysis**: Counts cases per variant and calculates percentages
3. **Top K Selection**: `LIMIT 5` in `variant_counts` CTE (adjust K as needed)
4. **Complete Filtering**: Returns all original events only for cases matching top K variants
5. **Contextual Information**: Includes variant details and statistics in output
