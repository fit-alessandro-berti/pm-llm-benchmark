Below is an integrated event log constructed from the given tables. Each row represents a single event in the lifecycle of an order. The **Case ID** is the `order_id`. Activities have been derived logically from the data:

- **Order Created** (from Orders)
- **Items Picked** (from Order Lines, one event per item line)
- **Order Dispatched** and **Order Arrived** (from Delivery)
- **Invoice Issued** (from Invoice)
- **Payment Received** (from Payments)

**Assumptions & Notes:**

- If multiple items were picked at different times, each item line generates a separate "Items Picked" event.
- If multiple deliveries or invoices exist for one order, each corresponds to separate events.
- The "Resource" column is enriched by looking up resource IDs in the Resources table (Sales Person, Warehouse Staff, Carrier, Billing Clerk). For payments, no corresponding resource from the resource table was found, so the resource is left blank. Payment method details are recorded in "Other Attributes".
- Additional attributes such as `customer_id`, `customer_type`, `sales_person_id`, `order_priority`, `item_id`, `invoice_id`, `delivery_id`, `quantity`, `invoice_amount`, and `payment_reference` are included where relevant.
- Events are listed by case (order_id) in chronological order.
