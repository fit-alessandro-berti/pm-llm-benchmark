The visualization illustrates **inter-organizational workflow coordination** using the **proclet approach** (a method for modeling flexible cross-organizational business processes). It consists of three complementary components:

## (a) Process Diagram: Two Connected Proclet Classes

The left side depicts a **"Lab Visit"** proclet (patient-facing), and the right side shows a **"Lab Test"** proclet (laboratory-side). They interact through two **communication channels**:

**Lab Visit Proclet flow:**
- Starts with **Order Entry** (doctor role)
- **Take blood sample** (lab assistant)
- **Receive result**  **Judge results** (decision gateway):
  - *Path A*: "Everything fine"  **Send report** (lab assistant)  End
  - *Path B*: "Do tests again"  **Receive updated result**  loop back for re-judgment

**Lab Test Proclet flow:**
- **Perform test**  **Make report** (chemical analyst roles)
- Can branch to: **Perform test again** / **Provide updated report** (for re-testing scenarios)
- Ends with **Finish lab test**

**Channels (dashed boxes):**
- **Order system**: Connects order initiation to test execution (multiplicity *..1 : 1..*)
- **HIS (Hospital Information System)**: Coordinates result delivery and status updates between the two processes (multiple connection points with 1..1 and 1..* multiplicities)

## (b) Class Diagram: Structural Relationship

Shows the static relationship between the two proclet types:
- **Lab visit** (1 instance) **requires** **1..\*** Lab tests
- This captures that a single patient visit may necessitate multiple laboratory analyses

## (c) Communication Example: Performative Table

Illustrates a concrete message exchanged at **11:00** via the **Order system** channel:

| Field | Value |
|-------|-------|
| **Sender** | Lab visit - John |
| **Receiver** | Lab test - HGB (Hemoglobin test) |
| **Action** | Create |
| **Content** | "Can you perform a HGB test for John?" |
| **Scope** | Private |
| **Direction** | OUT (from Lab visit perspective) |

---

**Key Insight:** The figure demonstrates how **proclets** allow independent organizations (clinic vs. lab) to maintain autonomous internal workflows while coordinating through well-defined message channels—enabling flexibility (e.g., handling re-tests) without tight coupling. The swimlane-like annotations show human roles (doctor, lab assistant, chemical analyst) responsible for each activity.
