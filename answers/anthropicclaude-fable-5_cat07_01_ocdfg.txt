# Process Mining Visualization — Object-Centric Process Model

This is an **object-centric process map**, typical of process mining tools (e.g., Celonis-style), showing how multiple business object types flow through a procure-to-pay / order-to-cash style process.

## Object Types (Entry Points)

Five object types enter the process, each shown with its volume and a distinct color-coded path:

| Object Type | Volume | Color |
|---|---|---|
| PurchaseRequisitionItem | 2.0M objects | Purple |
| SalesOrderItem | 3.5M objects | Orange |
| PurchaseOrderItem | 5.0M objects | Blue |
| VendorInvoiceItem | 1.2M objects | Green/Teal |
| ReturnOrderItem | 0.8M objects | Red |

Each object type begins at its own **Start** node.

## Activities (Nodes)

Each activity box shows the activity name, the overall event count, and colored dots indicating which object types participate:

- **Create Requisition** (3.0M) — purple
- **Create Order** (8.5M) — blue, orange
- **Approve Requisition** (5.7M) — purple, blue, green, red (a major convergence point)
- **Send Order** (5.2M) — purple, blue
- **Check Stock** (1.0M) — purple
- **Process Return** (0.8M) — green, red
- **Pack Goods** (2.0M)  **Ship Goods** (1.8M)  **Confirm Delivery** (1.7M) — orange (fulfillment chain)
- **Approve Payment** (1.5M) — blue
- **Create Invoice** (2.0M) — blue, green
- **Close Order** (1.7M) — orange, red

## Edges and Timing

Arrows between activities are labeled with **average durations** (e.g., 300ms, 450ms, 600ms), and edge colors trace each object type's path through the process.

## Key Observations

- **Approve Requisition** is the busiest hub, with four object types converging on it.
- The **orange (SalesOrderItem)** path follows a clean linear fulfillment flow: Create Order  Pack Goods  Ship Goods  Confirm Delivery  Close Order.
- The process has **five End nodes**, indicating different termination points depending on object type and path taken.
- Some loops/feedback exist (e.g., flows returning toward Approve Requisition and Send Order).

Overall, the diagram illustrates how interconnected business objects (orders, requisitions, invoices, returns) interact across a shared end-to-end process.
