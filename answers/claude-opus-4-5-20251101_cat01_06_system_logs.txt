## Explanation of Transformation Logic

### Case Identification Strategy

I grouped events into **5 distinct cases** based on the **document/artifact-centric** approach, where each case represents a coherent unit of work on a specific object:

| Case ID | Description | Rationale |
|---------|-------------|-----------|
| **DOC-001** | Editing Document1.docx | All activities related to creating and editing this new Word document, including the return to add budget references |
| **DOC-002** | Editing Quarterly_Report.docx | Brief initial focus followed by later substantive editing session (Executive Summary work) |
| **EMAIL-001** | Handling Annual Meeting Email | Complete email workflow from opening inbox to sending reply |
| **PDF-001** | Reviewing Report_Draft.pdf | PDF review and annotation session |
| **EXCEL-001** | Updating Budget_2024.xlsx | Budget spreadsheet update session |

### Activity Naming Logic

I transformed low-level system actions into **meaningful business activities**:

| Raw Action(s) | Transformed Activity | Logic |
|--------------|---------------------|-------|
| FOCUS (first occurrence) | Open Document/Spreadsheet/etc. | Initial access to a resource |
| FOCUS (return after switch) | Resume Editing | Returning to previously opened work |
| TYPING (initial) | Draft Content | Creating new content |
| TYPING (subsequent) | Edit Content / Update Data | Modifying existing content |
| TYPING (specific context) | Add Reference / Compose Reply | Context-aware naming based on Keys field |
| SAVE | Save Document/Spreadsheet | Persisting changes |
| CLOSE | Close Document | Completing work session |
| SWITCH (to new app) | Open [Application Type] | Beginning new work context |
| SCROLL | Review Document/Email Content | Reading/scanning activity |
| CLICK (Open) | Open Email | Accessing specific item |
| CLICK (Reply) | Initiate Reply | Starting response workflow |
| CLICK (Send) | Send Email | Completing communication |
| HIGHLIGHT | Annotate Document | Active document markup |

### Key Design Decisions

1. **Document-centric cases**: Rather than session-based or time-window grouping, I chose document/artifact-centric cases because:
   - They align with how knowledge workers conceptualize their tasks
   - They enable analysis of document lifecycle patterns
   - They allow cross-reference analysis (e.g., DOC-001 references EXCEL-001)

2. **Handling interrupted work**: The user's return to Document1.docx after working in Excel is captured as "Resume Editing" rather than a new case, preserving the complete document editing lifecycle.

3. **Initial FOCUS on Quarterly_Report.docx**: The very first event (08:59:50) shows brief focus on Quarterly_Report before switching to Document1. This is treated as the "Open Document" for DOC-002, with substantive work occurring later—reflecting a common pattern where users briefly view one document before working on another.

4. **Standardized activity vocabulary**: Activities use consistent verbs (Open, Edit, Save, Close, Review, etc.) to enable meaningful process mining analysis such as:
   - Conformance checking against expected workflows
   - Discovery of common activity patterns
   - Performance analysis of document handling times
