Here is a DuckDB SQL pattern that does exactly that. Replace `5` in `params` with your desired `K`.

```sql
WITH params AS (
    SELECT 5 AS k   -- top K variants
),

-- 1) Build the ordered activity sequence for each case
case_variants AS (
    SELECT
        case_id,
        string_agg(activity, ' -> ' ORDER BY "timestamp") AS variant
    FROM event_log
    GROUP BY case_id
),

-- 2) Count how many cases belong to each variant
variant_counts AS (
    SELECT
        variant,
        COUNT(*) AS case_count
    FROM case_variants
    GROUP BY variant
),

-- 3) Keep only the top K variants by frequency
top_k_variants AS (
    SELECT
        variant,
        case_count
    FROM (
        SELECT
            variant,
            case_count,
            ROW_NUMBER() OVER (
                ORDER BY case_count DESC, variant
            ) AS rn
        FROM variant_counts
    ) ranked
    WHERE rn <= (SELECT k FROM params)
),

-- 4) Find the cases that belong to those top K variants
filtered_cases AS (
    SELECT
        cv.case_id,
        cv.variant,
        tkv.case_count AS variant_frequency
    FROM case_variants cv
    JOIN top_k_variants tkv
      ON cv.variant = tkv.variant
)

-- 5) Return only events from cases in the top K variants
SELECT
    e.case_id,
    e.activity,
    e."timestamp",
    fc.variant,
    fc.variant_frequency
FROM event_log e
JOIN filtered_cases fc
  ON e.case_id = fc.case_id
ORDER BY
    fc.variant_frequency DESC,
    fc.variant,
    e.case_id,
    e."timestamp";
```

Notes:
- `string_agg(activity, ' -> ' ORDER BY "timestamp")` creates the ordered sequence per case.
- `variant_counts` aggregates cases by identical full sequences.
- `top_k_variants` picks the most frequent variants.
- The final join returns only the original events whose `case_id` belongs to those top variants.

If you want **exactly the original event log columns only**, change the final `SELECT` to:

```sql
SELECT e.*
FROM event_log e
JOIN filtered_cases fc
  ON e.case_id = fc.case_id
ORDER BY e.case_id, e."timestamp";
```

If ties at the K-th rank should all be included, use `DENSE_RANK()` instead of `ROW_NUMBER()`.
