Here's a DuckDB SQL query that meets your requirements:

```sql
WITH case_counts AS (
    SELECT 
        case_id,
        COUNT(*) AS event_count
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT 
    el.case_id,
    el.activity,
    el.timestamp
FROM event_log el
INNER JOIN case_counts cc ON el.case_id = cc.case_id
ORDER BY el.case_id, el.timestamp;
```

**How it works:**

1. **CTE (Common Table Expression)**: The `case_counts` CTE first identifies all `case_id` values with 6 or fewer events by:
   - Grouping by `case_id`
   - Counting events per case
   - Filtering to keep only cases with 6 events using `HAVING COUNT(*) <= 6`

2. **Main Query**: Joins the original `event_log` table with our filtered case list to return all events (columns) for the qualifying cases.

3. **Sorting**: Results are ordered by `case_id` and `timestamp` for consistent output, though you can adjust or remove this based on your needs.

**Alternative approach using a window function:**
```sql
SELECT 
    case_id,
    activity,
    timestamp
FROM (
    SELECT *,
        COUNT(*) OVER (PARTITION BY case_id) AS event_count
    FROM event_log
) AS subquery
WHERE event_count <= 6
ORDER BY case_id, timestamp;
```

Both queries will efficiently filter out cases with more than 6 events and return all events from cases with 6 or fewer events. The window function approach might be slightly more efficient for very large datasets as it avoids a separate aggregation step.
