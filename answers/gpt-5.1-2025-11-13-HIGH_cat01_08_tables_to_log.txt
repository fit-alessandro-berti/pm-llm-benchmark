Below is an integrated event log built from all provided tables.  
- **Case ID** = `order_id`  
- One row = one event.  
- Activities: `Order Created`, `Item Picked`, `Delivery Dispatched`, `Delivery Arrived`, `Invoice Issued`, `Payment Received`.

**Main assumptions & mappings**

- **Order Created**: from `Orders.order_date`, actor = `sales_person_id` (Sales Person).  
- **Item Picked**: one event per row in `Order Lines`, actor = `picker_id` (Warehouse Staff), enriched with product description from `Products`.  
- **Delivery Dispatched / Arrived**: two events per `Delivery` record (dispatch & arrival), actor = `carrier_id` (Carrier), includes `delivery_note`.  
- **Invoice Issued**: one per `Invoice` record, actor = `billing_person_id` (Billing Clerk).  
- **Payment Received**: one per `Payments` record, linked to the order via `invoice_id  order_id`; actor not specified in data, so `resource_*` left blank.  
- All timestamps are used as-is (UTC, ISO 8601).  
- Order-level attributes (`customer_id`, `customer_type`, `order_priority`, `sales_person_*`) are repeated on all events of the case.
