### 1. Analyzing Historical Scheduling Performance and Dynamics  
#### Process Mining Techniques and Metrics  
**Job Flow Reconstruction & KPIs:**  
- **Process Discovery:** Use *process trees or Petri nets* to visualize the actual job routing paths and identify deviations from planned workflows.  
- **Case Duration Analysis:** Calculate **lead time** (time from job release to completion) and **makespan** (total time from first to last job completion) distributions using *case tables*.  
- **Task Waiting Times:** For each machine/resource, compute the time between a job𠏋 queue entry and task start using *lifecycle diagrams*.  
- **Resource Utilization:**  
  - **Productive Time** = Task execution time.  
  - **Idle Time** = Time when a machine is available but no job is assigned.  
  - **Setup Time** = Duration between setup start and end events.  
  - Metrics: Utilization = (Productive Time + Setup Time) / Total Available Time.  

**Sequence-Dependent Setup Times:**  
- **Setup Time Modeling:** Extract sequences of consecutive jobs on each machine, aggregating setup durations based on the *previous job𠏋 ID* and *current job𠏋 attributes* (e.g., material type). Use *case tables* or *parallel coordinates* to identify pairs of jobs with abnormally long setups.  

**Schedule Adherence & Disruptions:**  
- **Tardiness Calculation:** Compare `Due Date` vs. `Actual Completion Time` for each job.  
- **Disruption Impact Analysis:** Trace delays caused by breakdowns or priority changes using *conformance checking* (e.g., identify delays post-breakdown via *event log correlations*).

### 3. Root Cause Analysis  
**Scheduling Logic Limitations:**  
- **Static Rules:** First-Come-First-Served (FCFS) or Earliest Due Date (EDD) fail to account for setup times, resource dependencies, or disruptions.  
- **Real-Time Visibility:** Lack of machine availability or queue status updates leads to suboptimal dispatching.  
- **Setup Ignorance:** Setup times are not accounted for in scheduling, worsening sequence-dependent delays.  
- **Coordination Gaps:** Work centers operate in silos, leading to unbalanced resource loads.  

**Separating Causes with Process Mining:**  
- **Capacity vs. Scheduling Issues:**  
  - If a machine𠏋 utilization is near 100% *and* its queue length is stable, the issue is capacity.  
  - If utilization is low but lead times are high, poor scheduling is the culprit.  
- **Sequence-Dependent Setup Costs:** If similar jobs are not batched, setups dominate delays.
