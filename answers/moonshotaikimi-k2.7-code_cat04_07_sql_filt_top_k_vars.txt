Below is a single DuckDB query that:

1. Builds the ordered activity sequence per `case_id`.
2. Counts how many cases follow each sequence (variant).
3. Keeps only the **top K** most frequent variants.
4. Returns **all original events** belonging to cases in those top K variants.

```sql
WITH case_variants AS (
    -- 1. Ordered activity sequence for each case
    SELECT
        case_id,
        string_agg(activity, '->' ORDER BY timestamp, activity) AS variant
    FROM event_log
    GROUP BY case_id
),

variant_counts AS (
    -- 2. Number of cases per variant
    SELECT
        variant,
        count(*) AS case_count
    FROM case_variants
    GROUP BY variant
),

top_k_variants AS (
    -- 3. Top K variants by frequency
    SELECT variant
    FROM variant_counts
    ORDER BY case_count DESC, variant
    LIMIT 3  -- replace 3 with the desired K
),

filtered_cases AS (
    -- 4. Cases whose variant is in the top K
    SELECT case_id
    FROM case_variants
    WHERE variant IN (SELECT variant FROM top_k_variants)
)

-- 5. Return all original events from those cases
SELECT e.*
FROM event_log e
WHERE e.case_id IN (SELECT case_id FROM filtered_cases)
ORDER BY e.case_id, e.timestamp;
```

### Notes

- `string_agg(activity, '->' ORDER BY timestamp, activity)` produces a deterministic ordered sequence per case. The secondary sort on `activity` breaks ties when two events have the same `timestamp`.
- Replace `LIMIT 3` with whatever `K` you need.
- If you want to include ties at the K-th rank, replace `top_k_variants` with a `DENSE_RANK()` version:

```sql
top_k_variants AS (
    SELECT variant
    FROM (
        SELECT
            variant,
            dense_rank() OVER (ORDER BY case_count DESC) AS rnk
        FROM variant_counts
    )
    WHERE rnk <= 3  -- replace 3 with K
)
```
