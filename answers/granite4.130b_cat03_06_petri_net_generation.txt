Below is a complete **Python** script that builds an accepting Petri net for the insuranceclaim workflow you described.  
The net uses places to represent “states” between activities, transitions labelled with the activity names, and arcs that enforce the required order while allowing the two possible loops:

* **DV  RDS (resubmit documents)** – if the verification fails.  
* **FC  II (indepth investigation)** – which may repeat any number of times before moving to *Claim Assessment*.

The initial marking puts a single token on the start place (`C`), and the final marking places that token on the `CL` (Closure) place, indicating a closed claim.

```python
# --------------------------------------------------------------
# Accepting Petri Net for Insurance Claim Management Process
# --------------------------------------------------------------

from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# ------------------------------------------------------------------
# 1. Create the Petri net object
# ------------------------------------------------------------------
net = PetriNet(name="InsuranceClaimManagement")

# ------------------------------------------------------------------
# 2. Define places (states) -------------------------------------------------
# ------------------------------------------------------------------
places = {
    # Start of the process – a new claim arrives
    "C":   PetriNet.Place(name="C", label="Insurance Claim"),
    
    # After each activity we keep a token in the corresponding place
    "CR":  PetriNet.Place(name="CR", label="Claim Registered"),
    "DV":  PetriNet.Place(name="DV", label="Document Verified"),
    "RDS": PetriNet.Place(name="RDS", label="ReDocument Submitted"),
    
    "FC":  PetriNet.Place(name="FC", label="Fraud Checked"),
    "II":  PetriNet.Place(name="II", label="Indepth Investigation"),
    
    "CA":  PetriNet.Place(name="CA", label="Claim Assessed"),
    "AP":  PetriNet.Place(name="AP", label="Approved (if needed)"),
    "P":   PetriNet.Place(name="P", label="Payment Made"),
    "N":   PetriNet.Place(name="N", label="Notification Sent"),
    
    # End of the process – claim closed and archived
    "CL":  PetriNet.Place(name="CL", label="Claim Closed")
}

# Add all places to the net
for p in places.values():
    net.places.add(p)

# ------------------------------------------------------------------
# 3. Define transitions (activities) ------------------------------------
# ------------------------------------------------------------------
transitions = {
    # Stepbystep activities
    "CR":   PetriNet.Transition(name="CR", label="Claim Registration"),
    "DV":   PetriNet.Transition(name="DV", label="Document Verification"),
    "RDS":  PetriNet.Transition(name="RDS", label="ReDocument Submission"),
    
    "FC":   PetriNet.Transition(name="FC", label="Fraud Check"),
    "II":   PetriNet.Transition(name="II", label="Indepth Investigation"),
    
    "CA":   PetriNet.Transition(name="CA", label="Claim Assessment"),
    "AP":   PetriNet.Transition(name="AP", label="Approval (if needed)"),
    "P":    PetriNet.Transition(name="P",  label="Payment"),
    "N":    PetriNet.Transition(name="N",  label="Notification"),
    "CL":   PetriNet.Transition(name="CL", label="Closure")
}

# Add all transitions to the net
for t in transitions.values():
    net.transitions.add(t)

# ------------------------------------------------------------------
# 4. Connect places and activities with arcs -------------------------
# ------------------------------------------------------------------
# Helper function (optional) – adds an arc from a place  transition  place
def add_seq(src_place, trans, tgt_place):
    petri_utils.add_arc_from_to(src_place, trans, net)
    petri_utils.add_arc_from_to(trans, tgt_place, net)

# Linear flow with the two possible loops

# 1. Claim arrives  Registration
add_seq(places["C"], transitions["CR"], places["CR"])

# 2. Registration  Document Verification
add_seq(places["CR"], transitions["DV"], places["DV"])

# -------------------------------------------------
# Loop 1: If verification fails, resubmit docs
# -------------------------------------------------
petri_utils.add_arc_from_to(places["DV"], transitions["RDS"], net)   # DV -> RDS
petri_utils.add_arc_from_to(transitions["RDS"], places["RDS"], net)  # RDS output place
petri_utils.add_arc_from_to(places["RDS"], transitions["DV"], net)  # back to DV for recheck

# -------------------------------------------------
# Continue after successful verification (or after last RDS)
# -------------------------------------------------
add_seq(places["DV"], transitions["FC"], places["FC"])

# -------------------------------------------------
# Loop 2: Fraud doubts  Indepth Investigation (may repeat)
# -------------------------------------------------
petri_utils.add_arc_from_to(places["FC"], transitions["II"], net)   # FC -> II
petri_utils.add_arc_from_to(transitions["II"], places["II"], net)   # II output place
petri_utils.add_arc_from_to(places["II"], transitions["FC"], net)  # back to Fraud Check

# -------------------------------------------------
# After fraud cleared  Assessment, then Approval (optional)
# -------------------------------------------------
add_seq(places["FC"], transitions["CA"], places["CA"])
# Optional approval only when amount > threshold – we model it as a mandatory step
# for simplicity; in a real net you could make it conditional.
add_seq(places["CA"], transitions["AP"], places["AP"])

# Continue to payment, notification and closure
add_seq(places["AP"], transitions["P"], places["P"])
add_seq(places["P"],  transitions["N"], places["N"])
add_seq(places["N"],  transitions["CL"], places["CL"])

# ------------------------------------------------------------------
# 5. Define initial and final markings -------------------------------
# ------------------------------------------------------------------
im = Marking()          # Initial marking
im[places["C"]] = 1    # One token representing a newly filed claim

fm = Marking()          # Final marking
fm[places["CL"]] = 1   # Token ends in the Closure place (claim closed)

# ------------------------------------------------------------------
# 6. (Optional) Validation / inspection -----------------------------
# ------------------------------------------------------------------
# You can print places, transitions and arcs to verify correctness:
print("Places:", [p.name for p in net.places])
print("Transitions:", [t.name for t in net.transitions])

# If you need to visualise the net with pm4py's builtin viewer:
# from pm4py.visualization.petrinet import visualizer as pn_visualizer
# gviz = pn_visualizer.apply(net, im, fm)
# pn_visualizer.view(gviz)

```

### Explanation of the Net Structure

| Place (state) | Meaning |
|---------------|---------|
| **C** | New insurance claim arrives. |
| **CR** | Claim has been registered in the system. |
| **DV** | Documents have passed verification. |
| **RDS** | Resubmission of documents after a failure at DV. |
| **FC** | Fraud check completed without suspicion. |
| **II** | Indepth investigation triggered by FC; can repeat. |
| **CA** | Claim assessed and compensation amount decided. |
| **AP** | Senior manager’s approval (if the amount exceeds threshold). |
| **P** | Payment executed to the customer. |
| **N** | Customer notified of resolution & payment details. |
| **CL** | Claim closed and archived – final state. |

- **Arcs** follow a classic “place  transition  place” pattern, guaranteeing that a token can only move forward once the corresponding activity fires.
- The **DVRDSDV** loop allows resubmission of documents any number of times until verification succeeds.
- The **FCIIFC** loop permits an arbitrary (including zero) number of indepth investigations before proceeding to assessment.

The initial marking (`im`) places a single token on `C`, and the final marking (`fm`) expects that token to reside on `CL` after the whole workflow finishes, fulfilling the definition of an **accepting Petri net**.
