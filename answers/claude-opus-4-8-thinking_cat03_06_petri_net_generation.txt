Here's the Python code to construct the accepting Petri net for the insurance claim management process:

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# Create the Petri net
net = PetriNet(name="insurance_claim_process")

# ---------------------------------------------------------------------------
# PLACES (states between activities)
# ---------------------------------------------------------------------------
source   = PetriNet.Place(name="source")        # a new claim arrives
p_c      = PetriNet.Place(name="p_after_C")      # after claim filed
p_cr     = PetriNet.Place(name="p_after_CR")     # after claim registration
p_dv     = PetriNet.Place(name="p_after_DV")     # after document verification
p_fc     = PetriNet.Place(name="p_after_FC")     # after fraud check
p_ca     = PetriNet.Place(name="p_after_CA")     # after claim assessment
p_ap     = PetriNet.Place(name="p_after_AP")     # after approval
p_p      = PetriNet.Place(name="p_after_P")      # after payment
p_n      = PetriNet.Place(name="p_after_N")      # after notification
sink     = PetriNet.Place(name="sink")           # claim closed

for p in [source, p_c, p_cr, p_dv, p_fc, p_ca, p_ap, p_p, p_n, sink]:
    net.places.add(p)

# ---------------------------------------------------------------------------
# TRANSITIONS (activities)
# ---------------------------------------------------------------------------
t_C   = PetriNet.Transition(name="C",   label="Insurance Claim")
t_CR  = PetriNet.Transition(name="CR",  label="Claim Registration")
t_DV  = PetriNet.Transition(name="DV",  label="Document Verification")
t_RDS = PetriNet.Transition(name="RDS", label="Re-Document Submission")
t_FC  = PetriNet.Transition(name="FC",  label="Fraud Check")
t_II  = PetriNet.Transition(name="II",  label="In-depth Investigation")
t_CA  = PetriNet.Transition(name="CA",  label="Claim Assessment")
t_AP  = PetriNet.Transition(name="AP",  label="Approval")
t_P   = PetriNet.Transition(name="P",   label="Payment")
t_N   = PetriNet.Transition(name="N",   label="Notification")
t_CL  = PetriNet.Transition(name="CL",  label="Closure")

for t in [t_C, t_CR, t_DV, t_RDS, t_FC, t_II, t_CA, t_AP, t_P, t_N, t_CL]:
    net.transitions.add(t)

# ---------------------------------------------------------------------------
# ARCS (control flow)
# ---------------------------------------------------------------------------
# Main sequential flow
petri_utils.add_arc_from_to(source, t_C,  net)
petri_utils.add_arc_from_to(t_C,    p_c,  net)

petri_utils.add_arc_from_to(p_c,    t_CR, net)
petri_utils.add_arc_from_to(t_CR,   p_cr, net)

petri_utils.add_arc_from_to(p_cr,   t_DV, net)
petri_utils.add_arc_from_to(t_DV,   p_dv, net)

# Document re-submission loop (between DV and RDS):
# If documents are incomplete, RDS sends the flow back so DV can run again.
petri_utils.add_arc_from_to(p_dv,   t_RDS, net)
petri_utils.add_arc_from_to(t_RDS,  p_cr,  net)   # back to before DV

# Continue to Fraud Check
petri_utils.add_arc_from_to(p_dv,   t_FC, net)
petri_utils.add_arc_from_to(t_FC,   p_fc, net)

# In-depth investigation loop (between FC and II):
# Can happen multiple times before proceeding; II sends the flow back so FC runs again.
petri_utils.add_arc_from_to(p_fc,   t_II, net)
petri_utils.add_arc_from_to(t_II,   p_dv, net)    # back to before FC

# Continue to Claim Assessment
petri_utils.add_arc_from_to(p_fc,   t_CA, net)
petri_utils.add_arc_from_to(t_CA,   p_ca, net)

# Approval
petri_utils.add_arc_from_to(p_ca,   t_AP, net)
petri_utils.add_arc_from_to(t_AP,   p_ap, net)

# Payment
petri_utils.add_arc_from_to(p_ap,   t_P,  net)
petri_utils.add_arc_from_to(t_P,    p_p,  net)

# Notification
petri_utils.add_arc_from_to(p_p,    t_N,  net)
petri_utils.add_arc_from_to(t_N,    p_n,  net)

# Closure (final step)
petri_utils.add_arc_from_to(p_n,    t_CL, net)
petri_utils.add_arc_from_to(t_CL,   sink, net)

# ---------------------------------------------------------------------------
# INITIAL & FINAL MARKINGS
# ---------------------------------------------------------------------------
im = Marking()
im[source] = 1          # a new claim arriving

fm = Marking()
fm[sink] = 1            # a closed/archived claim

# `net`, `im`, `fm` now together form the accepting Petri net.
```

### Explanation of the model

**Main sequence:** `source  C  CR  DV  FC  CA  AP  P  N  CL  sink`. Each activity is a transition, and each place encodes the state *between* two activities.

**Re-Document Submission loop (DV  RDS):**
- After `DV` puts a token in `p_after_DV`, the process has a choice:
  - proceed to `FC`, **or**
  - fire `RDS`, which returns the token to `p_after_CR` so that `DV` is executed again. This naturally allows multiple re-submission rounds.

**In-depth Investigation loop (FC  II):**
- After `FC` puts a token in `p_after_FC`, the process has a choice:
  - proceed to `CA`, **or**
  - fire `II`, which returns the token to `p_after_DV` so that `FC` runs again. Because the token cycles back, **In-depth Investigation can happen multiple times** before assessment continues.

**Markings:**
- `im` places one token in `source`, representing a freshly arrived claim.
- `fm` places one token in `sink`, reached only after `Closure (CL)`, representing a fully closed and archived claim.

This produces a sound, executable accepting Petri net with the two required loops and the correct start/end semantics.
