Using the normative text as a **mostly strict model**, the allowed paths are essentially:

1) **Fulfillment path:**  
Receive  Check inventory  Authorize payment  Package items  Ship order  Close order

2) **Cancellation path:**  
Receive  Check inventory  Cancel order  Close order

So the cases that violate are: **Case 2, Case 3, and Case 5**.

### Case-by-case assessment

| Case | Conformant? | Reason |
|------|-------------|--------|
| **1** |  Yes | Exactly matches the fulfillment path. |
| **2** |  No | **Package items occurs before Authorize payment**. Violates rule that payment authorization should happen before packaging. |
| **3** |  No | Multiple violations: (a) **Ship order occurs after Cancel order** (explicitly forbidden), (b) cancel branch should go to close, not back to fulfillment, (c) shipping also happens **without Package items**. |
| **4** |  Yes | Matches cancellation path: Receive  Check  Cancel  Close. |
| **5** |  No | **Ship order without Package items**. Violates “Package before Ship.” |

## Summary of violations found
- **Out-of-order authorization/packaging:** Case 2  
- **Shipping after cancellation / mixed branches:** Case 3  
- **Missing required packaging before shipping:** Case 5

So, the deviating cases are **2, 3, and 5**.
