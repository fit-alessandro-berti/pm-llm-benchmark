# Comprehensive Process Mining Approach for Speedy Parcels

## 1. Process Discovery and Conformance Checking

### Data Preprocessing and Integration
To create a cohesive event log suitable for process mining:
- **Standardize Timestamps:** Convert all timestamps to a common format (e.g., ISO 8601) across GPS, scanner, and dispatch systems.
- **Unify Identifiers:** Map vehicle IDs, driver IDs, and package IDs consistently across systems (e.g., V12 in GPS matches V12 in dispatch).
- **Event Correlation:** Link related events (e.g., "Arrive Customer" from scanner with GPS coordinates at the same timestamp).
- **Handle Missing Data:** Use interpolation or flag missing events (e.g., if a "Depart Customer" is missing but GPS shows movement).
- **Filter Noise:** Remove irrelevant events (e.g., GPS pings during driver breaks) while preserving key milestones.

**Challenges:**
- **Data Granularity:** GPS data is high-frequency, while scanner events are sparse. We'll aggregate GPS data into meaningful states (e.g., "traveling," "idle").
- **Semantic Alignment:** Terms like "Unscheduled Stop" in GPS vs. "Engine Warning Light" in maintenance logs need harmonization.
- **Case Definition:** Define a "case" as a vehicle's daily shift (e.g., V12-20241205) and link package-level events as sub-processes.

### Process Discovery
- **Algorithms:** Use Inductive Miner (for structured processes) or Heuristics Miner (for noisy logs) to generate a process model from the event log.
- **Visualization:** Create a process map showing:
  - Core sequence: Depot  Travel  Customer Stop (Success/Failure)  Travel  Depot.
  - Deviations: Unplanned stops (e.g., maintenance, traffic), failed deliveries, and reattempts.
  - Parallel paths: Concurrent deliveries vs. sequential stops.

### Conformance Checking
- **Planned vs. Actual:** Overlay the discovered model with planned routes from the dispatch system.
- **Deviations to Identify:**
  - **Sequence Deviations:** Deliveries out of planned order (e.g., Stop 5 before Stop 3).
  - **Unplanned Stops:** Maintenance, traffic delays, or driver breaks not in the plan.
  - **Timing Differences:** Excessive dwell time at stops or travel segments.
  - **Missing Activities:** Planned deliveries not attempted or logged.

**Tools:** Use alignment algorithms (e.g., in Disco or Celonis) to highlight mismatches.

## 2. Performance Analysis and Bottleneck Identification

### Key Performance Indicators (KPIs)
| KPI                          | Calculation Method                                                                 | Relevance to Goals                     |
|------------------------------|-----------------------------------------------------------------------------------|----------------------------------------|
| On-Time Delivery Rate        | (Successful deliveries within window) / (Total deliveries)                         | Customer satisfaction, punctuality     |
| Avg. Time per Delivery Stop  | (Total service time at stops) / (Number of stops)                                 | Operational efficiency                 |
| Travel Time Ratio            | (Total travel time) / (Total service time)                                        | Balance of travel vs. service work     |
| Fuel Consumption per km      | (Total fuel used) / (Total km driven)                                             | Cost reduction                         |
| Vehicle Utilization Rate     | (Active delivery hours) / (Total shift hours)                                     | Asset efficiency                       |
| Traffic Delay Frequency      | (Count of "Low Speed Detected" events) / (Total travel segments)                  | External factor impact                 |
| Failed Delivery Rate         | (Failed attempts) / (Total deliveries)                                            | Rework and cost driver                 |

### Bottleneck Identification Techniques
- **Variant Analysis:** Compare process variants (e.g., routes with vs. without traffic delays) to identify patterns.
- **Bottleneck Miner:** Use frequency and duration metrics to highlight slow activities (e.g., "Arrive Customer" to "Depart Customer").
- **Geospatial Analysis:** Overlay GPS data with traffic patterns to pinpoint congestion hotspots.
- **Temporal Analysis:** Identify peak delay times (e.g., 8–9 AM rush hour) or days (e.g., Fridays).

**Quantifying Impact:**
- For a bottleneck like "high dwell time at Stop X," calculate:
  - Extra time per stop vs. average.
  - Total daily delay across all routes.
  - Correlation with failed deliveries (e.g., longer dwell  higher failure rate).
