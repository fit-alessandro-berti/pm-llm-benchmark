Looking at this complex scenario, I need to identify the main structure:

**Overall flow:** A sequential pipeline of 7 major stages (ArrivalTriageInitial TestLoopConcurrent Imaging/CounselingFinal SequenceConcurrent Admin).

**Key modeling decisions:**
- Stages 1, 2, 6 are pure **sequences**
- Stage 3 is an **XOR** (one diagnostic test)
- Stage 4 is a **LOOP** (SC is the always-executed part; the treatment+re-test is the "B" body)
- Stages 5 and 7 are **PARALLEL** (true concurrency)

    # STAGE 1: Arrival -> Registration -> Intake Assessment
    # ---------------------------------------------------------------

    A   = ProcessTree(label="A",  parent=stage1)
    RG  = ProcessTree(label="RG", parent=stage1)
    IA  = ProcessTree(label="IA", parent=stage1)
    stage1.children.extend([A, RG, IA])

    # ---------------------------------------------------------------
    # STAGE 2: Triage Interview -> Triage Classification
    # ---------------------------------------------------------------
    stage2 = ProcessTree(operator=Operator.SEQUENCE, parent=root)
    TI = ProcessTree(label="TI", parent=stage2)
    TC = ProcessTree(label="TC", parent=stage2)

# Helper: print the tree in a readable form
# ---------------------------------------------------------------------
def print_tree(node, depth=0):
    indent = "  " * depth
    if node.label is not None:
        print(f"{indent}- {node.label}")
    else:
        op_name = {
            Operator.SEQUENCE: "->",
            Operator.XOR:      "X",
            Operator.PARALLEL: "+",
            Operator.LOOP:     "*",
        }[node.operator]
        print(f"{indent}{op_name}(")
        for child in node.children:
            print_tree(child, depth + 1)
        print(f"{indent})")


if __name__ == "__main__":
    tree = build_hospital_process_tree()
    print_tree(tree)

## Why each operator was chosen

| Stage | Operator | Reason |
|-------|----------|--------|
| Root | `->` | The 7 stages strictly follow one another |
| 1, 2, 6 | `->` | All listed activities are mandatory and ordered |
| 3 | `X` | Exactly one initial test is performed (cardiac vs. general) |
| 4 | `*` | A * (SC, body) — the Specialist Consultation always happens first; then either exit (normal results) or run the treatment+re-test body, then SC again. Repeats until normal. |
| 5, 7 | `+` | All activities in these blocks run in true concurrency with no imposed order |

The loop body `->( X(TM, SPT) , X(FCE, FBA) )` captures that **within each iteration** a treatment is chosen exclusively and then a re-diagnostic test is chosen exclusively, and this whole body is what gets re-run between successive Specialist Consultations until results normalize.
