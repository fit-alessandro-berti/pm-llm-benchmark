### **General Rules for Grouping – To Apply to Full Log**

1. **Functional Cohesion Rule**  
   Group events that contribute to the same process objective. E.g., all actions that change material state for input readiness → Material Preparation. All actions that join/fix components → Assembly.

2. **Resource + Temporal Proximity Rule**  
   Events performed by the same resource within short time gaps often belong together. Operator B’s 3 events in 10s are clearly one task. But don’t rely on resource alone: handoffs between Operator A → Robot Arm #2 still belong to Material Prep because the goal persists.

3. **Stage Boundary Heuristics**  
   A new high-level step starts when:  
   - **Material state changes phase**: Raw → Heated → Assembled → Coated  
   - **Tool/resource type changes purpose**: From handling/prep tools to welding tools to inspection sensors  
   - **Time gap > typical intra-step gap**: 08:00:20 to 08:01:00 is a 40s gap, signaling phase transition

4. **Quality Events Handling**  
   Inspections can be “in-line” or “final”. If timestamps are interleaved with production steps, consider two QA stages:  
   - **In-Process QA**: Measure weld integrity right after welding  
   - **Final QA**: Visual check after all processing
