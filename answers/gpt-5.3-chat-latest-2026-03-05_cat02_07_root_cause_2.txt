- High complexity: Cases 2003 and 2005  ~48–77 hours.

- High complexity cases trigger multiple "Request Additional Documents" events.

- Case 2005: 3 document requests.

3. Root causes of performance issues

Primary root cause: Claim complexity
- High-complexity claims require additional investigation and documentation.
- These cases trigger multiple "Request Additional Documents" steps.

Secondary root cause: Iterative document requests
- Instead of requesting all required information at once, adjusters ask for documents repeatedly.
- This creates several waiting cycles.

Example:
Case 2005
- 3 document requests over 3 days.
- Each request introduces large idle periods.

Operational root cause: Waiting for external responses
- After document requests, the process pauses until customers provide information.
- These waiting times dominate total case duration.

Possible workload specialization
- Manager_Bill and certain adjusters may handle more complex cases, concentrating delays in their workload.

4. Suggestions to mitigate performance issues

Improve initial information collection
- At the "Submit Claim" stage, require a more complete document checklist.
- Provide customers with clear guidance on required evidence.

Bundle document requests
- Train adjusters to request all missing documents in a single step instead of multiple rounds.

Complexity-based routing
- Automatically classify claims by complexity.
- Assign high-complexity cases to specialized teams with standardized investigation procedures.

Customer response acceleration
- Implement automated reminders for missing documents.
- Provide digital upload portals.

Predictive checks
- Use historical patterns to predict which claims will likely require additional documentation.
- Request those documents proactively during submission.

Process monitoring
- Track metrics such as:
  - number of document request loops
  - waiting time after document requests
  - case duration by complexity

Summary

The major performance bottlenecks are driven primarily by high claim complexity and repeated document-request loops rather than specific resources or regions. Reducing iterative document requests and improving upfront information collection would significantly shorten overall case duration.
