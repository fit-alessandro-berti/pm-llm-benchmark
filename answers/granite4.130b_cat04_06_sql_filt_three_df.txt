**DuckDB SQL solution**

```sql
/* 1  Find all “CreateOrder  ConfirmOrder” pairs that belong to a case
    which also contains the preceding “Approve Purchase Requisition” activity.
   2  Keep only those pairs whose elapsed time is >5 days.               */
WITH seq_cases AS (
    SELECT create_case.case_id
    FROM   event_log      AS approve
           JOIN event_log AS create_order
                ON  approve.case_id = create_order.case_id
                AND create_order.activity = 'Create Order'
                AND create_order.timestamp > approve.timestamp
           JOIN event_log AS confirm_order
                ON  create_order.case_id = confirm_order.case_id
                AND confirm_order.activity = 'Confirm Order'
                AND confirm_order.timestamp > create_order.timestamp
    /* 3  The three activities must be consecutive for the same case.
        We ensure there is no other event (with a different activity) between
        “Approve Purchase Requisition” and “Create Order”, and likewise
        between “Create Order” and “Confirm Order”.                         */
    WHERE  approve.activity = 'Approve Purchase Requisition'
           -- no intervening event after Approve but before Create Order
           AND NOT EXISTS (
               SELECT 1
               FROM   event_log e_mid
               WHERE  e_mid.case_id = approve.case_id
                     AND e_mid.timestamp > approve.timestamp
                     AND e_mid.timestamp < create_order.timestamp
           )
           -- no intervening event after Create Order but before Confirm Order
           AND NOT EXISTS (
               SELECT 1
               FROM   event_log e_mid2
               WHERE  e_mid2.case_id = create_order.case_id
                     AND e_mid2.timestamp > create_order.timestamp
                     AND e_mid2.timestamp < confirm_order.timestamp
           )
    /* 4  Keep only those cases where the interval between Create Order
        and Confirm Order exceeds five days.                                 */
    AND (confirm_order.timestamp - create_order.timestamp) > INTERVAL '5' DAY
)

/* 5  Return every event **except** those that belong to a case listed in seq_cases.*/
SELECT *
FROM   event_log
WHERE  case_id NOT IN (SELECT case_id FROM seq_cases)
ORDER BY case_id, timestamp;
```

### How the query works

| Step | What it does |
|------|--------------|
| **CTE `seq_cases`** | • Joins three rows from the same `case_id` to enforce the exact activity order: <br>`Approve Purchase Requisition`  `Create Order`  `Confirm Order`. <br>• Uses two `NOT EXISTS` subqueries to guarantee that **no other event** (with any activity) occurs between each consecutive pair, i.e., the three activities are *direct successors*. <br>• Filters on the time gap: `(confirm_order.timestamp - create_order.timestamp) > INTERVAL '5' DAY`. The resulting `case_id`s are those that satisfy both conditions. |
| **Final SELECT** | Retrieves **all** rows from `event_log` whose `case_id` is **not** in the set produced by `seq_cases`. This effectively excludes every event belonging to a case that contains the forbidden sequence with a >5day interval between “Create Order” and “Confirm Order”. <br>• The `ORDER BY` clause simply makes the output deterministic (grouped by case and chronological order). |

The query therefore returns exactly the events you asked for: **all events from cases that do *not* contain the specified threeactivity pattern with a >5day gap between “Create Order” and “Confirm Order”.**
