1) Identifying InstanceSpanning Constraints and Their Impact

Approach using process mining

• Perform process discovery first (Inductive Miner or Heuristics Miner) to establish the baseline fulfillment model: Order Received  Item Picking  Packing  Quality Check  Shipping Label Generation.

• Then perform performance analysis with start/complete timestamps to compute:
  - Activity duration
  - Waiting time between activities
  - Queue length per resource
  - Resource utilization

• Enrich the log using case attributes (Order Type, Region, Cold Packing, Hazardous Material) to segment traces and detect crossinstance dependencies.

Detecting each constraint

Shared ColdPacking Stations

Method:
• Filter cases where Requires Cold Packing = TRUE.
• Analyze resource perspective for stations C1–C5.
• Calculate queue formation before Packing START.

Metrics:
• Average waiting time before Packing (cold orders).
• Station utilization rate.
• Queue length over time.
• Percentage of time all cold stations are occupied.

Impact indicator:
If waiting time before packing spikes when all cold stations are occupied, the delay is instancespanning (resource contention).

Shipping Batch Dependency

Method:
• Identify Shipping Label Generation events referencing the same batch ID.
• Measure time between:
  - Quality Check COMPLETE
  - Shipping Label Generation COMPLETE

Metrics:
• Batch formation delay.
• Average batch size by region.
• Idle time after QC waiting for batch completion.
• Throughput per region batch.

Impact indicator:
If QC completes but shipping label generation occurs significantly later due to waiting for other cases with same region, this indicates batch dependency.

Priority Handling (Express Orders)

Method:
• Compare resource usage patterns between Express and Standard orders.
• Detect interruptions or preemption events when a standard order is paused or delayed while an express order starts.

Metrics:
• Delay added to standard orders due to express arrivals.
• Resource preemption frequency.
• Cycle time difference between express and standard orders.
• Queue reshuffling events.

Impact indicator:
If standard orders already waiting or processing experience increased wait immediately after express arrival, this is crossinstance priority interference.

Hazardous Material Regulatory Limit

Method:
• Track concurrency of hazardous orders in Packing and Quality Check.
• Use timestamp overlap analysis to compute active hazardous instances.

Metrics:
• Number of simultaneous hazardous orders.
• Waiting time for hazardous orders before Packing or QC.
• Throughput of hazardous orders.
• Frequency of reaching the threshold of 10.

Impact indicator:
If hazardous orders must wait even when resources are available because the facility already processes 10 hazardous cases, the delay originates from regulatory instancespanning constraints.

Distinguishing withininstance vs betweeninstance waiting

Withininstance delays:
• Long activity duration
• Worker inefficiency
• Complex order handling

Betweeninstance delays:
• Resource queueing
• Batch waiting
• Priority interruptions
• Regulatory concurrency limits

Analytical techniques

• Queue analysis: Waiting time increases when resource utilization approaches capacity.
• Resource occupancy timeline: Detect if another case occupies the required station.
• Concurrency analysis: Detect regulatory limits or batching dependencies.
• Attribute correlation: Waiting time correlated with cold packing flag, region, express priority, or hazardous flag.

If waiting time correlates with other cases’ attributes or resource saturation, it indicates instancespanning causes.

2) Analyzing Constraint Interactions

ColdPacking + Priority Orders

Express orders requiring cold packing can preempt standard coldpacking jobs. With only 5 cold stations, this can create cascading delays where:

• Express orders jump the queue
• Standard cold orders accumulate
• Cold station utilization remains high but fairness drops

ColdPacking + Hazardous Limit

If a cold order is also hazardous:
• It consumes both a cold station and one of the 10 hazardous slots.
• When the hazardous threshold is reached, even available cold stations may remain idle because hazardous capacity is saturated.

Batching + Hazardous Orders

If several hazardous orders belong to the same region:
• They may all wait for batch formation.
• Meanwhile they occupy hazardous processing capacity earlier in the process.
• This can block other hazardous orders waiting for packing.

Batching + Express Orders

Express orders ideally should not wait for batch formation. If batching logic ignores priority:
• Express orders finish QC but wait for batch grouping
• Delivery time promises are violated.

ColdPacking + Batching

Coldpacked orders may require faster shipping to maintain temperature integrity. If batching delays them:
• Coldchain risk increases
• Station capacity becomes constrained because downstream steps slow.

Why interaction analysis matters

Optimizing a single constraint independently can worsen another. Examples:

• Adding priority to express orders improves express SLA but increases coldpacking queues.
• Larger batches reduce shipping cost but increase waiting time and hazardous occupancy.
• Increasing cold capacity without addressing batching might shift the bottleneck downstream.

Therefore optimization must consider the system as an interconnected network of instancespanning constraints.

3) ConstraintAware Optimization Strategies

Strategy 1: Predictive Resource Allocation for ColdPacking

Constraints addressed:
• Shared ColdPacking Stations
• Express Priority Interference

Proposed changes

• Implement predictive queue management for coldpacking stations.
• Reserve dynamic capacity for express orders.

Example rule:
• 3 stations  standard orders
• 2 stations  flexible pool prioritized for express

Data leverage

Use historical logs to predict:

• arrival rates of cold orders
• express order probability
• peak demand windows

Machine learning or timeseries forecasting can predict coldpacking demand per hour.

Expected outcomes

• Reduced waiting time for express orders
• Controlled queue growth for standard cold orders
• Higher utilization without chaotic preemption

Strategy 2: Dynamic Regional Batch Formation

Constraints addressed:
• Shipping batching delays
• Express order waiting

Proposed changes

Replace fixed batch formation with dynamic triggers:

Batch closes when:
• N orders reached, OR
• Maximum waiting time reached, OR
• Express order arrives (immediate release)

Example:

North region batch closes if:
• 12 orders collected OR
• 8 minutes passed since first order

Data leverage

Use event log analysis to determine optimal batch sizes minimizing:

Total waiting time + transport cost.

Expected outcomes

• Reduced idle time after QC
• Faster express shipping
• Balanced logistics efficiency.

Strategy 3: Hazardous Material Flow Control

Constraints addressed:
• Regulatory concurrency limit

Proposed changes

Introduce upstream throttling and scheduling.

Instead of allowing hazardous orders to reach packing simultaneously:

• Limit hazardous orders released from picking.
• Maintain a controlled queue before packing.

Example rule:

If 8 hazardous orders already active in packing/QC:
• temporarily delay new hazardous picking completion.

Data leverage

Use concurrency analysis from logs to model safe throughput levels.

Expected outcomes

• Avoid hitting regulatory cap frequently
• Reduce blocking situations
• Smooth hazardous processing flow.

Strategy 4: Decoupled Quality Check for Express Orders

Constraints addressed:
• Priority delays
• Batch interaction

Proposed changes

Create a fasttrack QC lane for express orders:

• Dedicated QC staff or automated QC tools
• Express orders skip batch queue and immediately generate labels.

Expected outcomes

• Reduced cycle time for express orders
• Lower interference with standard orders.

4) Simulation and Validation

Use a discreteevent simulation model built from mined process parameters.

Model inputs from event logs

• Activity duration distributions
• Arrival rate distributions
• Resource pools
• Priority rules
• Batch size rules
• Hazardous concurrency limit

Key components to model accurately

Resource contention

• Packing stations
• Coldpacking stations
• QC staff

Batching logic

• Regional grouping
• Batch closure rules
• Waiting thresholds

Priority scheduling

• Express preemption rules
• Queue reordering.

Regulatory constraints

• Maximum 10 concurrent hazardous orders across packing + QC.

KPIs evaluated in simulation

• Endtoend order cycle time
• Express SLA compliance
• Average queue length
• Resource utilization
• Throughput per hour
• Hazardous capacity utilization.

Run multiple scenarios

• Baseline system
• Each strategy individually
• Combined strategies

MonteCarlo runs capture variability during peak seasons.
