To filter out cases with more than six events and return all events from eligible cases, use a subquery to identify valid `case_id`s based on their event count. Here's the SQL query:

```sql
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
);
```

### Explanation:
1. **Subquery**  
   The inner query (`SELECT case_id FROM event_log GROUP BY case_id HAVING COUNT(*) <= 6`) calculates the number of events per `case_id` and retains only those with 6 events.

2. **Main Query**  
   The outer query retrieves all events from the original `event_log` table where the `case_id` appears in the filtered list of valid cases from the subquery.

This approach ensures only events from cases with six or fewer events are returned.
