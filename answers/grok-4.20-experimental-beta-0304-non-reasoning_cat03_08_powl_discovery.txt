**POWL Model for the Manufacturing & Distribution Process**

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Basic activities
OR = Transition("OR")
MS = Transition("MS")
QS = Transition("QS")
AS = Transition("AS")
IQC = Transition("IQC")
TST = Transition("TST")
RT = Transition("RT")
PK = Transition("PK")
LB = Transition("LB")
DP = Transition("DP")
D = Transition("D")
skip = SilentTransition()
```

### Core Control-Flow Fragments

```python
# 1. IQC + Assembly rework loop
# (IQC -> AS) repeated until IQC succeeds
iqc_as_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[
        IQC,                                      # do IQC
        OperatorPOWL(Operator.XOR, [AS, skip])    # then either redo AS or exit loop
    ]
)

# 2. Testing + Retest loop
# TST -> (RT -> TST)*  (i.e. do TST, then optionally loop through RT+TST)
test_loop_body = OperatorPOWL(
    operator=Operator.LOOP,
    children=[
        TST,
        OperatorPOWL(Operator.XOR, [RT, skip])
    ]
)

test_section = OperatorPOWL(
    operator=Operator.LOOP,
    children=[
        test_loop_body,
        skip   # exit after successful test
    ]
)
```

### Main Process Structure

```python
# Root is a StrictPartialOrder capturing the overall process dependencies
root = StrictPartialOrder(nodes=[
    OR, MS, QS, AS, iqc_as_loop, test_section, PK, LB, DP, D
])

# Define the partial order (precedence constraints)
root.order.add_edge(OR, MS)
root.order.add_edge(MS, QS)
root.order.add_edge(QS, AS)
root.order.add_edge(AS, iqc_as_loop)
root.order.add_edge(iqc_as_loop, test_section)
root.order.add_edge(test_section, PK)
root.order.add_edge(PK, LB)
root.order.add_edge(PK, DP)      # LB can be skipped  concurrency with DP after PK
root.order.add_edge(LB, DP)
root.order.add_edge(DP, D)
```

### Final POWL Model

The complete model is represented by the `root` object above.

**Interpretation:**
- OR  MS  QS  AS is strictly sequential.
- After AS, an **IQC-AS rework loop** occurs (can repeat multiple times, matching cases 3 and 5).
- After successful IQC, the **Testing loop** (TST + optional RT  TST) is executed.
- After testing, PK is mandatory.
- LB is **optional** (modeled by the concurrency `PK  LB` and `PK  DP`).
- DP  D is always last.

This structure faithfully captures all observed behavior in the event log, including:
- Optional LB
- Repeated IQC/AS cycles
- Retesting loops (RT)
- Strict precedence where required
- Concurrency between LB and DP when LB is skipped

The resulting `root` POWL object can be used directly with PM4Py for further analysis or visualization.
