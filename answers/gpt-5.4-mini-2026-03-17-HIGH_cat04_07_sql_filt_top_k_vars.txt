Here is a DuckDB SQL query that does what you asked:

```sql
WITH case_variants AS (
    -- Build the ordered activity sequence for each case
    SELECT
        case_id,
        string_agg(activity, ' -> ' ORDER BY timestamp) AS variant
    FROM event_log
    GROUP BY case_id
),
variant_counts AS (
    -- Count how many cases share the same variant
    SELECT
        variant,
        COUNT(*) AS case_count
    FROM case_variants
    GROUP BY variant
),
top_variants AS (
    -- Pick the top K variants by case frequency
    SELECT variant
    FROM variant_counts
    ORDER BY case_count DESC, variant
    LIMIT K
),
top_cases AS (
    -- Find all cases that belong to the top K variants
    SELECT cv.case_id
    FROM case_variants cv
    JOIN top_variants tv
      ON cv.variant = tv.variant
)
-- Return all events belonging to those cases
SELECT e.*
FROM event_log e
JOIN top_cases tc
  ON e.case_id = tc.case_id
ORDER BY e.case_id, e.timestamp;
```

### Notes
- Replace `K` with the desired number of variants, e.g. `5`.
- `string_agg(activity, ' -> ' ORDER BY timestamp)` creates the ordered process variant per case.
- The query:
  1. derives each case’s sequence,
  2. groups identical sequences,
  3. counts their frequencies,
  4. selects the top K variants,
  5. returns all events from cases matching those variants.
