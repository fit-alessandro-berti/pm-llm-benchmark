# DuckDB Query: Filter Events by Case Count

Here's the query, along with a couple of equivalent approaches:

## Approach 1: CTE with JOIN (Most Readable)

```sql
WITH case_counts AS (
    SELECT case_id, COUNT(*) AS event_count
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT el.*
FROM event_log el
JOIN case_counts cc ON el.case_id = cc.case_id
ORDER BY el.case_id, el.timestamp;
```

**How it works:**
- The CTE groups by `case_id`, counts events, and keeps only cases with 6 events.
- The main query joins back to `event_log` to retrieve the full rows for those qualifying cases.

---

## Approach 2: Window Function (More Concise)

```sql
SELECT *
FROM (
    SELECT *,
           COUNT(*) OVER (PARTITION BY case_id) AS case_event_count
    FROM event_log
) sub
WHERE case_event_count <= 6
ORDER BY case_id, timestamp;
```

**How it works:**
- The inner query attaches a count of events *per case* to every row using a window function.
- The outer query simply filters where that count is 6.

---

## Performance Note

For large event logs, **Approach 1** often performs better because the aggregation step drastically reduces the rows before joining back. However, if the table has many columns but relatively few rows per case, **Approach 2** can be faster since it avoids the join entirely.

Both will give you the same result — pick whichever fits your readability preferences or performance needs.
