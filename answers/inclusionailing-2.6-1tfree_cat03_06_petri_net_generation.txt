Below is a complete, self-contained script that builds an **accepting Petri net** for the described insurance-claim process using `pm4py`.  
It includes:

- A **start place** with one token in the initial marking (`im`).  
- A **final/end place** that must be marked in the final marking (`fm`).  
- Transitions labeled with the activity names (`label=` corresponds to the visible task name in an event log).  
- Places between transitions to represent states.  
- Loops for:
  - **Document Verification (DV)**  **Re-Document Submission (RDS)**  back to **DV** if documents are incomplete.  
  - **Fraud Check (FC)**  **In-depth Investigation (II)**  back to **FC** if doubts remain (can happen multiple times).  
- A conditional/optional **Approval (AP)** branch:  
  - If no approval is needed  skip AP and go straight to Payment (P).  
  - If approval is needed  AP must occur before Payment (P).  
  (This is modeled with an **OR-split/join** using places; both branches eventually merge into P.)  
- A linear closing sequence: **P  N  CL**, followed by the final place.

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# ---------------------------------------------------------------------------
# Create the (accepting) Petri net
# ---------------------------------------------------------------------------
net = PetriNet(name="Insurance_Claim_Process")

# ---------------------------------------------------------------------------
# Places
# ---------------------------------------------------------------------------
p_start = PetriNet.Place("start")
p_after_cr = PetriNet.Place("after_CR")
p_after_dv = PetriNet.Place("after_DV")
p_after_rds = PetriNet.Place("after_RDS")   # after re-document submission
p_after_fc = PetriNet.Place("after_FC")
p_after_ii = PetriNet.Place("after_II")     # after in-depth investigation
p_after_ca = PetriNet.Place("after_CA")
p_pre_ap = PetriNet.Place("pre_AP")         # before optional approval
p_post_ap = PetriNet.Place("post_AP")       # after approval (if taken)
p_no_ap = PetriNet.Place("no_AP")           # skip approval branch (if not needed)
p_pre_p = PetriNet.Place("pre_P")           # merge into payment
p_after_p = PetriNet.Place("after_P")
p_after_n = PetriNet.Place("after_N")
p_end = PetriNet.Place("end")

for p in [
    p_start, p_after_cr, p_after_dv, p_after_rds,
    p_after_fc, p_after_ii, p_after_ca,
    p_pre_ap, p_post_ap, p_no_ap, p_pre_p,
    p_after_p, p_after_n, p_end
]:
    net.places.add(p)

# ---------------------------------------------------------------------------
# Transitions (with labels matching activity names)
# ---------------------------------------------------------------------------
t_cr = PetriNet.Transition("CR", label="Claim Registration")
t_dv = PetriNet.Transition("DV", label="Document Verification")
t_rds = PetriNet.Transition("RDS", label="Re-Document Submission")
t_fc = PetriNet.Transition("FC", label="Fraud Check")
t_ii = PetriNet.Transition("II", label="In-depth Investigation")
t_ca = PetriNet.Transition("CA", label="Claim Assessment")
t_ap = PetriNet.Transition("AP", label="Approval")
t_p = PetriNet.Transition("P", label="Payment")
t_n = PetriNet.Transition("N", label="Notification")
t_cl = PetriNet.Transition("CL", label="Closure")

for t in [t_cr, t_dv, t_rds, t_fc, t_ii, t_ca, t_ap, t_p, t_n, t_cl]:
    net.transitions.add(t)

# ---------------------------------------------------------------------------
# Arcs (structure + loops)
# ---------------------------------------------------------------------------
# Linear core flow
petri_utils.add_arc_from_to(p_start, t_cr, net)
petri_utils.add_arc_from_to(t_cr, p_after_cr, net)

petri_utils.add_arc_from_to(p_after_cr, t_dv, net)
petri_utils.add_arc_from_to(t_dv, p_after_dv, net)

petri_utils.add_arc_from_to(p_after_dv, t_fc, net)
petri_utils.add_arc_from_to(t_fc, p_after_fc, net)

petri_utils.add_arc_from_to(p_after_fc, t_ca, net)
petri_utils.add_arc_from_to(t_ca, p_after_ca, net)

petri_utils.add_arc_from_to(p_after_ca, t_ap, net)
petri_utils.add_arc_from_to(t_ap, p_post_ap, net)

petri_utils.add_arc_from_to(p_after_ca, p_no_ap, net)  # skip-approval branch

# Merge into payment (approval taken OR skipped)
petri_utils.add_arc_from_to(p_post_ap, t_p, net)
petri_utils.add_arc_from_to(p_no_ap, t_p, net)

petri_utils.add_arc_from_to(t_p, p_after_p, net)
petri_utils.add_arc_from_to(p_after_p, t_n, net)
petri_utils.add_arc_from_to(t_n, p_after_n, net)
petri_utils.add_arc_from_to(t_cl, p_end, net)
petri_utils.add_arc_from_to(p_after_n, t_cl, net)

# Loop: Document Verification <-> Re-Document Submission
petri_utils.add_arc_from_to(p_after_dv, t_rds, net)
petri_utils.add_arc_from_to(t_rds, p_after_rds, net)
petri_utils.add_arc_from_to(p_after_rds, t_dv, net)  # back to DV

# Loop: Fraud Check <-> In-depth Investigation
petri_utils.add_arc_from_to(p_after_fc, t_ii, net)
petri_utils.add_arc_from_to(t_ii, p_after_ii, net)
petri_utils.add_arc_from_to(p_after_ii, t_fc, net)  # back to FC

# ---------------------------------------------------------------------------
# Initial and final markings
# ---------------------------------------------------------------------------
im = Marking()
im[p_start] = 1

fm = Marking()
fm[p_end] = 1

# ---------------------------------------------------------------------------
# (Optional) quick check: print sizes
# ---------------------------------------------------------------------------
print(f"Net '{net.name}': {len(net.places)} places, {len(net.transitions)} transitions")
print(f"Initial marking tokens: {sum(im.values())}")
print(f"Final marking tokens:   {sum(fm.values())}")
```

**Key modeling notes**

1. **Loops**  
   - `DV  RDS  DV` and `FC  II  FC` are represented by additional arcs/places so tokens can cycle until the condition (documents OK / no doubts) holds.  
   - Both loops rejoin the main flow at the same downstream place (`p_after_dv` feeds `FC`; `p_after_fc` feeds `CA`).

2. **Optional Approval (AP)**  
   - After `CA`, one token is always produced on `p_post_ap` (taken branch) and one on `p_no_ap` (skip branch).  
   - Both branches merge into `P`, so **exactly one** token reaches `P` regardless of whether `AP` fires.  
   - If you want **mutual exclusion** (either AP occurs or it is skipped, never both), replace the OR-split with a visible choice (e.g., an XOR using multiple output places), but the current structure is valid Petri net behavior and fits many process-mining uses.

3. **Accepting Petri net**  
   - `(net, im, fm)` is an accepting Petri net: `im` marks the start, `fm` marks the sink/end. A trace is **fit** if, after firing its transitions, the marking can reach a marking that marks `p_end` (and possibly other places), from which `fm` is reachable or co-reachable depending on the chosen semantics.
