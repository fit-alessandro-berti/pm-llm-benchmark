### 4. Justification of Naming Choices

The renames were driven by the **trigger** for the activity and the **goal** of the interaction, as described in the "Additional Context" column:

**1. Disambiguating "Verify Documents"**
*   **Initial Document Screening:** In all three cases (C001, C002, C003), the first instance occurs immediately after initiation. The context ("checks identity," "identity verification," "physical verification") indicates this is the baseline validation step.
*   **Re-verify After Correction:** In C001 and C003, the second instance occurs only after a loop (submitting extra docs or adjusting the account). The context explicitly uses terms like "re-checks" and "re-verify," distinguishing this as a secondary quality control step rather than a duplicate of the first.

**2. Disambiguating "Call Customer"**
*   **Outbound Request for Info (C001):** The context states the call is because verification was incomplete. The goal is input gathering.
*   **Outbound Discrepancy Inquiry (C002 - 1st):** The context highlights "clarify account discrepancy." This is investigative work, distinct from simply asking for a file.
*   **Outbound Confirmation (C002 - 2nd):** The context is "follow-up to confirm adjustments." This is a closing action, signaling the end of the active work phase.
*   **Customer Retrieval Call (C003):** The context is unique ("customer stepped out"). This is a logistical interruption handling rather than a procedural inquiry.
