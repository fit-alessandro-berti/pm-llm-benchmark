### Justification for Renaming Choices

#### **1. `Verify Documents`  Context-Specific Verifications**
- **`Initial Document Verification`**: Used for the **first document check** in all cases (e.g., initial identity verification in C001/C002, physical ID check in C003). This distinguishes it from later re-verifications.
- **`Re-Verification After Submission`**: Used in **C001** where documents were re-checked *after the customer submitted additional ID scans*. The trigger ("Submission") is explicitly named.
- **`Re-Verification After Adjustment`**: Used in **C003** where documents were re-checked *after an account adjustment*. The trigger ("Adjustment") clarifies the purpose.

> *Why?* The original "Verify Documents" was ambiguous. Renaming reflects *why* verification occurred (e.g., initial check vs. post-submission recheck), avoiding confusion between identical activity names in the same case.

---

#### **2. `Call Customer`  Purpose-Specific Call Types**
- **`Request Missing Documents Call`**: Used in **C001** for calls to **request missing docs** (e.g., "Verification incomplete, agent calls to request missing docs").
- **`Clarification Call`**: Used in **C002** for calls to **resolve discrepancies** (e.g., "Outbound call to clarify account discrepancy").
- **`Confirmation Call`**: Used in **C002** for **follow-up calls after adjustments** (e.g., "Follow-up call to confirm adjustments made").
- **`Return Request Call`**: Used in **C003** for calls to **ask customers to return** (e.g., "Customer stepped out. Officer calls to have them return").

> *Why?* All "Call Customer" instances had distinct purposes. Renaming explicitly ties the call type to its *business objective* (e.g., clarification vs. confirmation), eliminating ambiguity.

---

#### **3. Other Activity Renames**
- **`Submit Additional`  `Submit Additional Documents`**: Clarifies the *type of submission* (documents), avoiding vagueness.
- **`Adjust Account`  `Account Adjustment`**: Standardizes terminology for consistency and professionalism (e.g., "Internal correction due to missing info" is an *adjustment*, not just "adjust").

> *Why?* These names now clearly describe the *action* (submitting documents) and *outcome* (account adjustment), aligning with standard financial services terminology.

---

### Key Improvement Summary
- **Eliminated ambiguity** for activities repeated in the same case (e.g., "Verify Documents" in C001/C003, "Call Customer" in C002).
- **Preserved traceability**: Each renamed activity now directly reflects its *business purpose* and *triggering event* (e.g., "Re-Verification After Submission").
- **Standardized terminology**: Consistent naming across all cases (e.g., "Account Adjustment" vs. "Adjust Account").
- **No loss of context**: All original details (e.g., "Physical verification of ID" in C003) are retained in the "Additional Context" column, while the activity name itself now explicitly defines *what the step is for*.
