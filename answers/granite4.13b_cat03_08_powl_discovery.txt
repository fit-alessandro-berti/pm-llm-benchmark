**POWL Model for the Global Manufacturing and Distribution Process**

Below is a textual representation of the POWL model that captures the described workflow. The structure uses:

* **Transitions (activities)** with labels corresponding to each step in the manufacturing/distribution process.
* **Silent transitions** where appropriate (e.g., between IQC steps, labeling LB skipping in Case2).
* **Exclusive choice operator X** for steps that have alternative branches (RT after TST failure; LB may or may not be present depending on the case).
* **Loop operator *** to model repeated Quality Screening (QIC) and ReTesting (RT) loops.
* A **StrictPartialOrder** to define the overall execution order between major stages.

```python
# Define activity objects for each step in the workflow
OR = Transition(label="OR")          # Order Received
MS = Transition(label="MS")          # Material Sourcing
QS = Transition(label="QS")          # Quality Screening (initial)
AS = Transition(label="AS")          # Assembly
IQC = Transition(label="IQC")        # In-line Quality Check
TST = Transition(label="TST")       # Testing
RT = Transition(label="RT")          # Re-Testing (if product fails TST)
PK = Transition(label="PK")          # Packaging
LB = Transition(label="LB")          # Labeling (may be skipped)

# Define silent transitions where steps are implicitly repeated without a user action
IQC_repeat = SilentTransition()     # Additional IQC loop for Case3 & 5 (repeated quality checks)
skip_lb_2   = SilentTransition()    # Implicitly skipping LB in Case2

# Build the choice operator X for RT after TST failure
RT_choice = OperatorPOWL(operator=Operator.XOR, children=[TST, RT])

# Build loop structures using * operator
IQCLoop = OperatorPOWL(operator=Operator.LOOP, children=[IQC, IQC_repeat])
RTLoop = OperatorPOWL(operator=Operator.LOOP, children=[RT, TST])   # Loop for re-testing

# Define the overall partial order of major activities (nodes)
nodes = [
    OR,
    MS,
    QS,
    AS,
    IQCLoop,          # Repeated quality screening loop
    PK,
    LB                # Labeling step
]

# Partial ordering between nodes to reflect concurrency and dependencies:
order = {
    OR: MQ,            # Order must precede Material Sourcing & Quality Screening
    MS: QS,           # Material sourcing must occur before initial quality screening
    QS: AS,           # Initial quality screening must be completed before assembly
    AS: IQCLoop,      # Assembly cannot happen until the loop of quality checks is finished (implicit)
    IQCLoop: PK,      # Only after repeated IQC can packaging proceed
    PK: LB,           # Packaging follows labeling; otherwise LB may be skipped
    LB: DP            # Labeling precedes Distribution Planning & Dispatch
}
# Additional order for RT in cases where TST fails:
RT_order = {
    TST: RT_choice   # Testing must lead to either proceeding (TST) or retesting (RT)
}

# Construct the StrictPartialOrder object with nodes and orders
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition

powl_model = StrictPartialOrder(nodes=nodes, order=order)

# Add RT choice to the overall model as a separate branch (exclusive XOR)
powl_model.order.add_edge(TST_choice, RT_choice)   # This captures RT after TST failure
```

### Explanation of Structure:

1. **Activities**: Each activity is represented by a `Transition` object with its respective label.
2. **Silent Transitions**:
   - `IQC_repeat` models the repeated IQC checks for Cases3 and5.
   - `skip_lb_2` captures that Labeling (LB) is skipped in Case2, indicating a branching where LB may not occur after PK.
3. **Choice Operator `X`**:
   - `RT_choice` (XOR) models the conditional branch after TST failure for Cases1 and5 where RT is executed if TST fails.
4. **Loop Operator `*`**:
   - `IQCLoop` ensures that IQC can be repeated as needed, especially evident in Cases3 and5.
   - `RTLoop` captures the retesting loop triggered by a failed test outcome (TST failure).
5. **Partial Order**: The nodes are ordered to reflect dependencies:
   - OR  MS  QS  AS  IQCLoop  PK  LB  DP
   - Additional edge for RT_choice linking TST  RT, capturing the conditional re-testing after a failed test.

This POWL model faithfully represents the described manufacturing and distribution process, including variations such as skipped labeling (Case2) and repeated quality checks (Cases3 &5).
