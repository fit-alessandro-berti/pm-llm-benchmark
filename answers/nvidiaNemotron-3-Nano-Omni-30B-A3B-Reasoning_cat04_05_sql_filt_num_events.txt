Here’s a concise DuckDB SQL query that accomplishes the described task:

```sql
WITH case_event_counts AS (
    SELECT
        case_id,
        COUNT(*) AS event_count
    FROM event_log
    GROUP BY case_id
)
SELECT
    el.case_id,
    el.activity,
    el.timestamp
FROM event_log AS el
JOIN case_event_counts AS cec
    ON el.case_id = cec.case_id
WHERE cec.event_count <= 6
ORDER BY el.case_id, el.timestamp;
```

**Explanation**

1. **`case_event_counts` CTE**  
   - Groups rows by `case_id` and counts how many events each case contains (`event_count`).  
   - This gives us the total number of events per case.

2. **Main query**  
   - Joins the original `event_log` with the CTE on `case_id` so we can filter based on the computed count.  
   - The `WHERE` clause keeps only rows where `event_count` is **six or fewer**.  
   - The `ORDER BY` ensures the output is sorted by case and timestamp, which is often useful for readability.

This query returns all events from cases that have at most six events, effectively excluding any cases that exceed that limit.
