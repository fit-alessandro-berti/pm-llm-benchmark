Here's a Python implementation of the **accepting Petri net** for the **Insurance Claim Management Process** using `pm4py`. The Petri net includes all the required places, transitions, and arcs, including loops for **Re-Document Submission (RDS)** and **In-depth Investigation (II)**. The **initial marking** starts with a token in the **Start** place, and the **final marking** ends with a token in the **Closed** place.

---

###  **Key Features of the Petri Net:**
- **Places** represent stages in the process.
- **Transitions** represent activities (e.g., Claim Registration, Document Verification, etc.).
- **Arcs** define the flow between places.
- **Loops** allow for re-submission of documents or repeated investigations.
- **Initial marking** starts with a new claim.
- **Final marking** ends with a closed claim.

---

###  **Design Notes:**
- **Start Place**: `Start` (initial token).
- **End Place**: `Closed` (final token).
- **Re-Document Submission (RDS)**: Loop between **Document Verification (DV)** and **Re-Document Submission (RDS)**.
- **In-depth Investigation (II)**: Loop between **Fraud Check (FC)** and **In-depth Investigation (II)**.
- **Approval (AP)**: Conditional transition based on claim amount (simulated here as a loop or conditional logic).
- **Payment (P)**: Final step before **Notification (N)** and **Closure (CL)**.

---

###  **Python Code:**

```
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# Create the Petri net
net = PetriNet(name="InsuranceClaimProcess")

# Define places
start = net.places.add(net.Place(name="Start"))
claim_registration = net.places.add(net.Place(name="Claim Registration"))
document_verification = net.places.add(net.Place(name="Document Verification"))
re_document_submission = net.places.add(net.Place(name="Re-Document Submission"))
fraud_check = net.places.add(net.Place(name="Fraud Check"))
in_depth_investigation = net.places.add(net.Place(name="In-depth Investigation"))
claim_assessment = net.places.add(net.Place(name="Claim Assessment"))
approval = net.places.add(net.Place(name="Approval"))
payment = net.places.add(net.Place(name="Payment"))
notification = net.places.add(net.Place(name="Notification"))
closure = net.places.add(net.Place(name="Closed"))

# Define transitions
cr = net.transitions.add(net.Transition(name="Claim Registration", label="CR"))
dv = net.transitions.add(net.Transition(name="Document Verification", label="DV"))
rds = net.transitions.add(net.Transition(name="Re-Document Submission", label="RDS"))
fc = net.transitions.add(net.Transition(name="Fraud Check", label="FC"))
ii = net.transitions.add(net.Transition(name="In-depth Investigation", label="II"))
ca = net.transitions.add(net.Transition(name="Claim Assessment", label="CA"))
ap = net.transitions.add(net.Transition(name="Approval", label="AP"))
p = net.transitions.add(net.Transition(name="Payment", label="P"))
n = net.transitions.add(net.Transition(name="Notification", label="N"))
cl = net.transitions.add(net.Transition(name="Closure", label="CL"))

# Define arcs (flow between places)
# Start -> Claim Registration
petri_utils.add_arc_from_to(start, cr, claim_registration)

# Claim Registration -> Document Verification
petri_utils.add_arc_from_to(cr, dv, document_verification)

# Document Verification -> Re-Document Submission (loop)
petri_utils.add_arc_from_to(dv, rds, document_verification, net)

# Re-Document Submission -> Document Verification (loop)
petri_utils.add_arc_from_to(rds, dv, document_verification, net)

# Document Verification -> Fraud Check
petri_utils.add_arc_from_to(dv, fc, fraud_check)

# Fraud Check -> In-depth Investigation (loop)
petri_utils.add_arc_from_to(fc, ii, in_depth_investigation, net)

# In-depth Investigation -> Fraud Check (loop)
petri_utils.add_arc_from_to(ii, fc, fraud_check, net)

# In-depth Investigation -> Claim Assessment
petri_utils.add_arc_from_to(ii, ca, claim_assessment)

# Claim Assessment -> Approval (conditional)
# For simplicity, assume approval is always granted (can be extended with conditions)
petri_utils.add_arc_from_to(ca, ap, claim_assessment)

# Approval -> Payment
petri_utils.add_arc_from_to(ap, p, claim_assessment)

# Payment -> Notification
petri_utils.add_arc_from_to(p, n, payment)

# Notification -> Closure
petri_utils.add_arc_from_to(n, cl, notification)

# Closure -> End (final state)
# (Optional: Add a final place if needed)

# Initial marking: Start with a token in Start place
im = Marking()
im[start] = 1

# Final marking: Closed place has a token
fm = Marking()
fm[cl] = 1

# Optional: Remove places or tokens if needed (e.g., remove Start or Close)
# petri_utils.remove_place(net, start)
# del im[start]
# petri_utils.remove_place(net, cl)
# del fm[cl]

# Print the Petri net structure (optional)
print("Initial Marking:", im)
print("Final Marking:", fm)
```

---

###  **Explanation of Key Components:**

- **Places**:
  - `Start`: Initial state.
  - `Claim Registration`, `Document Verification`, etc.: Represent each step.
  - `Closed`: Final state after payment and notification.

- **Transitions**:
  - Each activity is a transition with a label.
  - Loops allow for re-submission and repeated investigations.

- **Arcs**:
  - Define the flow between places.
  - Loops are implemented using `petri_utils.add_arc_from_to(..., net)` to allow multiple executions.

- **Markings**:
  - `im` starts with a token in `Start`.
  - `fm` ends with a token in `Closed`.
