Here's a DuckDB SQL query that accomplishes the task:

```sql
WITH event_counts AS (
    SELECT case_id, COUNT(*) AS event_count
    FROM event_log
    GROUP BY case_id
)
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_counts
    WHERE event_count <= 6
);
```

Explanation:

1. **CTE (Common Table Expression) `event_counts`:** This part aggregates the number of events (`COUNT(*)`) per `case_id`. It groups all rows with the same `case_id` and counts how many events each case has.

2. **Main Query:** The main query selects all columns from the `event_log` table, but only for those rows where the `case_id` is present in the list of cases that have 6 or fewer events (as determined by the CTE).

This ensures we filter out any `case_id` with more than six events and return only the corresponding events.
