Here is a clean, efficient DuckDB SQL query that fulfills the requirement:

```sql
WITH case_event_counts AS (
    SELECT case_id, COUNT(*) AS event_count
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT el.*
FROM event_log el
INNER JOIN case_event_counts cec
    ON el.case_id = cec.case_id;
```

### Explanation:

1. **CTE `case_event_counts`**:
   - Groups events by `case_id`.
   - Counts the number of events per case using `COUNT(*)`.
   - Filters out cases with more than 6 events using `HAVING COUNT(*) <= 6`.

2. **Main Query**:
   - Joins the original `event_log` table with the filtered `case_event_counts` CTE on `case_id`.
   - Returns all columns (`*`) from `event_log` for only those cases that have 6 or fewer events.

### Why this approach?
- **Efficient**: Uses a CTE to pre-filter case IDs, avoiding repeated aggregation.
- **Readable**: Clear separation of logic (counting vs. retrieving).
- **Correct**: Ensures that *all events* from qualifying cases are returned — not just aggregated results.
- **DuckDB Compatible**: Uses standard SQL supported by DuckDB (CTEs, `GROUP BY`, `HAVING`, `INNER JOIN`).

This query ensures no case with more than six events appears in the result, while preserving all original event data for cases that meet the threshold.
