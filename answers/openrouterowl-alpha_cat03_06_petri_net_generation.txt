

# Accepting Petri Net for Insurance Claim Management Process

## Analysis of the Process

Let me first map out the control flow:

```
Start  CR  DV  [if docs incomplete: RDS  DV]  FC  [if fraud suspected: II  FC]  CA  
[if amount > threshold: AP]  P  N  CL  End
```

Key features:
1. Documents may need **multiple re-submissions** (loop between DV and RDS)
2. In-depth investigation can happen **multiple times** (loop between FC and II)
3. Approval is **conditional** (only if amount exceeds threshold) — modeled as a skip path for simplicity

## Python Code

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# ---------------------------------------------------------------------------
# Create the Petri net
# ---------------------------------------------------------------------------
net = PetriNet(name="InsuranceClaimManagement")

# ---- Places (each represents a state in the process) ----
p_start = PetriNet.Place(name="start")
p_after_reg_verif   = PetriNet.Place(name="after_claim_registr_and_verif")      # after CR, before DV
p_after_verif       = PetriNet.Place(name="after_document_verification")        # after DV
p_after_fraud       = PetriNet.Place(name="after_fraud_check")                 # after FC
p_after_assess      = PetriNet.Place(name="after_claim_assessment")            # after CA
p_after_approval    = PetriNet.Place(name="after_approval")                    # after conditional Approval
p_after_payment     = PetriNet.Place(name="after_payment")                     # after Payment
p_after_notification = PetriNet.Place(name="after_notification")                # after Notification
p_end               = PetriNet.Place(name="end")

for pl in [p_start, p_after_reg_verif, p_after_verif, p_after_fraud,
           p_after_assess, p_after_approval, p_after_payment,
           p_after_notification, p_end]:
    net.places.add(pl)

# ---------------------------------------------------------------------------
# Transitions – visible (labelled) and silent (tau)
# ---------------------------------------------------------------------------

# -- Visible transitions (activity labels) --
t_CR  = PetriNet.Transition(name="t_CR",   label="Claim Registration")
t_DV  = PetriNet.Transition(name="t_DV",   label="Document Verification")
t_RDS = PetriNet.Transition(name="t_RDS",  label="Re-Document Submission")
t_FC  = PetriNet.Transition(name="t_FC",   label="Fraud Check")
t_II  = PetriNet.Transition(name="t_II",   label="In-depth Investigation")
t_CA  = PetriNet.Transition(name="t_CA",   label="Claim Assessment")
t_AP  = PetriNet.Transition(name="t_AP",   label="Approval")
t_P   = PetriNet.Transition(name="t_P",    label="Payment")
t_N   = PetriNet.Transition(name="t_N",    label="Notification")
t_CL  = PetriNet.Transition(name="t_CL",   label="Closure")

# -- Silent (invisible) transitions for routing logic --
t_skip_approval = PetriNet.Transition(name="t_skip_approval", label=None)   # skip AP

for tr in [t_CR, t_DV, t_RDS, t_FC, t_II, t_CA, t_AP,
           t_P, t_N, t_CL, t_skip_approval]:
    net.transitions.add(tr)

# ---------------------------------------------------------------------------
# Arcs – build the control flow
# ---------------------------------------------------------------------------

# 1. Start  Claim Registration  after_reg_verif  Document Verification  after_verif
petri_utils.add_arc_from_to(p_start,          t_CR, net)
petri_utils.add_arc_from_to(t_CR,              p_after_reg_verif, net)
petri_utils.add_arc_from_to(p_after_reg_verif, t_DV, net)
petri_utils.add_arc_from_to(t_DV,              p_after_verif, net)

# 2. Document Verification possible loop-back:  after_verif  RDS  after_verif
#    (documents incomplete  customer re-submits, verification repeats)
petri_utils.add_arc_from_to(p_after_verif, t_RDS, net)
petri_utils.add_arc_from_to(t_RDS,         t_DV, net)
# DV output again goes to p_after_verif (already added above), so the RDSDV(loop) works.
# Note: RDS connects to DV, allowing RDS  DV  p_after_verif  RDS  DV  ...

# 3. after_verif  Fraud Check  after_fraud
petri_utils.add_arc_from_to(p_after_verif,  t_FC, net)
petri_utils.add_arc_from_to(t_FC,           p_after_fraud, net)

# 4. Fraud Check possible loop-back: after_fraud  In-depth Investigation  fraud check again
#    (investigation may need to happen multiple times)
petri_utils.add_arc_from_to(p_after_fraud, t_II, net)
petri_utils.add_arc_from_to(t_II,          t_FC, net)
# FC output goes to p_after_fraud (already added), so loop: FCp_after_fraudIIFC...

# 5. after_fraud  Claim Assessment  after_assess
petri_utils.add_arc_from_to(p_after_fraud,  t_CA, net)
petri_utils.add_arc_from_to(t_CA,           p_after_assess, net)

# 6. Conditional Approval branch:
#    Path A (skip): after_assess  skip_approval (silent)  after_approval
#    Path B (execute): after_assess  Approval  after_approval
petri_utils.add_arc_from_to(p_after_assess,      t_skip_approval, net)
petri_utils.add_arc_from_to(t_skip_approval,     p_after_approval, net)
petri_utils.add_arc_from_to(p_after_assess,      t_AP, net)
petri_utils.add_arc_from_to(t_AP,                p_after_approval, net)

# 7. after_approval  Payment  after_payment  Notification  after_notification
#     Closure  end
petri_utils.add_arc_from_to(p_after_approval,    t_P,  net)
petri_utils.add_arc_from_to(t_P,                 p_after_payment, net)
petri_utils.add_arc_from_to(p_after_payment,     t_N,  net)
petri_utils.add_arc_from_to(t_N,                 p_after_notification, net)
petri_utils.add_arc_from_to(p_after_notification, t_CL, net)
petri_utils.add_arc_from_to(t_CL,               p_end, net)

# ---------------------------------------------------------------------------
# Initial and Final Markings
# ---------------------------------------------------------------------------
im = Marking()
im[p_start] = 1            # one token in the start place

fm = Marking()
fm[p_end] = 1              # one token in the end place

# ---------------------------------------------------------------------------
# Summary print (optional verification)
# ---------------------------------------------------------------------------
print("=== Petri Net Summary ===")
print(f"  Places  : {len(net.places)}")
print(f"  Transitions: {len(net.transitions)}")
print(f"  Initial marking : { {p.name: c for p, c in im.items()} }")
print(f"  Final marking   : { {p.name: c for p, c in fm.items()} }")
```

## Process Flow Diagram (Conceptual)

```
                          
                                   DV                    
                          
                                                     (docs OK)
              (incomplete)      (re-submit)         
                            
   start CR p1 DV   p2   FC  
                                      
                                           RDS              
                                             (OK)
                                                          
                                                      p3  CA  p4
                                                 II
                                                     (suspect)
                                                          
                                                  
                                                   skip_approval () 
                                                  
                                                           
                                                           
                                                      p5  AP  p6
                                                           
                                                           
                                                      P  p7  N  p8  CL  end
                    
```

## Key Design Decisions

| Aspect | Design Choice |
|---|---|
| **Re-documentation loop** | `p_after_verif  RDS  DV  p_after_verif` allows *any number* of re-submissions |
| **Investigation loop** | `p_after_fraud  II  FC  p_after_fraud` allows repeated in-depth investigations |
| **Conditional Approval** | Used a silent -transition `skip_approval` as a bypass; the model supports both paths |
| **FIFO semantics** | Each place feeds exactly one transition (except decision points), ensuring clean token flow |
