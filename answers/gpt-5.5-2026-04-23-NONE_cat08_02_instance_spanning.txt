## 1. Identifying Instance-Spanning Constraints and Their Impact

The key issue in this warehouse is that an order’s performance is not determined only by its own process path. It is also affected by other orders competing for the same resources, waiting for batch partners, being preempted by express orders, or being limited by facility-wide hazardous-material rules.

A standard process-mining analysis should therefore be extended from a purely case-centric view to an object-centric, resource-aware, and constraint-aware analysis.

---

### 1.1 Shared Cold-Packing Stations

#### Constraint

Orders requiring cold packing can only use one of a limited number of specialized Cold-Packing stations, for example five stations. Multiple cold-chain orders compete for those stations.

#### How to identify it from the event log

Use the following fields:

- `Activity = Packing`
- `Requires Cold Packing = TRUE`
- `Resource = Station C1, C2, ..., C5`
- `Timestamp Type = START / COMPLETE`

Process-mining techniques:

1. **Resource utilization mining**
   - Reconstruct occupancy intervals for each cold-packing station:
     - Packing start timestamp
     - Packing completion timestamp
     - Station ID
   - Build a timeline showing when each cold-packing station was occupied.

2. **Queue reconstruction**
   - For each cold-packing order, estimate when the order became ready for packing.
   - Usually this is the completion of `Item Picking`.
   - Waiting time before packing is:

   \[
   \text{Cold Packing Queue Time} = \text{Packing Start Time} - \text{Item Picking Complete Time}
   \]

3. **Contention detection**
   - At the moment an order becomes ready for cold packing, check how many cold-packing stations are occupied.
   - If all cold stations are occupied and the order waits, classify the delay as likely resource-contention-driven.

4. **Bottleneck analysis**
   - Use performance mining to compare:
     - Cold-packing orders versus standard-packing orders
     - Express cold-packing orders versus standard cold-packing orders
     - Peak versus non-peak periods

#### Metrics

Key metrics include:

| Metric | Purpose |
|---|---|
| Average and median waiting time before cold packing | Measures resource contention |
| 90th/95th percentile cold-packing wait time | Captures peak congestion |
| Cold station utilization | Identifies overloading |
| Cold station queue length over time | Shows congestion patterns |
| Number of orders waiting while all cold stations are occupied | Direct evidence of capacity constraint |
| SLA breach rate for cold-packing orders | Business impact |
| Throughput of cold-packing stations per hour/day | Capacity measurement |

#### Impact quantification

Example findings might be:

- Cold-packing orders wait 18 minutes on average before packing, versus 5 minutes for normal packing.
- During 13:00–16:00, all cold-packing stations are occupied 82% of the time.
- 65% of delayed express cold orders occur when cold station utilization exceeds 90%.

This would demonstrate that the shared specialized station constraint materially affects delivery performance.

---

### 1.2 Shipping Batches

#### Constraint

Orders going to the same geographical region are grouped before `Shipping Label Generation` to optimize delivery routes. An order that has completed quality check may wait for other orders in the same region before labels are generated.

#### How to identify it from the event log

Use:

- `Destination Region`
- `Quality Check COMPLETE`
- `Shipping Label Gen. COMPLETE`
- `Resource`, especially entries such as `System (Batch B1)`

If batch IDs are explicitly present, use them directly. If not, infer batches by grouping orders with:

- Same destination region
- Same or very close shipping label generation timestamp
- Same system batch reference, if embedded in resource or metadata

Process-mining techniques:

1. **Batch mining**
   - Detect groups of cases that move together through `Shipping Label Generation`.
   - Identify batch size distribution by region and time of day.

2. **Synchronization delay analysis**
   - For each order:

   \[
   \text{Batch Waiting Time} = \text{Shipping Label Generation Time} - \text{Quality Check Complete Time}
   \]

3. **First-complete versus last-complete analysis**
   - Within each batch:
     - First order ready time
     - Last order ready time
     - Label generation time

   The first completed orders often suffer the largest batching delay.

4. **Region-specific analysis**
   - Compare batch waiting times across North, South, West, East, etc.

#### Metrics

| Metric | Purpose |
|---|---|
| Average waiting time after QC before label generation | Measures batch-induced delay |
| Waiting time by destination region | Detects problematic regions |
| Batch size distribution | Shows batching behavior |
| Time to form batch | Measures synchronization delay |
| Percentage of orders waiting more than threshold for batch | SLA risk |
| Batch utilization versus delay trade-off | Measures logistics efficiency versus service time |
| First-order waiting time in batch | Captures penalty imposed on early-completed orders |

#### Impact quantification

Potential finding:

- North-region batches average 12 orders but cause a median post-QC waiting time of 22 minutes.
- West-region express orders are frequently delayed because they wait for regional batch formation.
- Orders that are first ready in a batch wait 3–5 times longer than the last ready order.

This would confirm that batching is a significant instance-spanning synchronization constraint.

---

### 1.3 Priority Handling for Express Orders

#### Constraint

Express orders are expedited. They may receive preferential access to resources and can delay standard orders. In some cases, processing of a standard order may be paused or a standard order may be forced to wait longer because an express order receives priority.

#### How to identify it from the event log

Use:

- `Order Type = Express / Standard`
- Activity start and complete timestamps
- Resource assignments
- Resource handover sequences
- Waiting times before key activities

Process-mining techniques:

1. **Priority-aware queue analysis**
   - For each resource or station, reconstruct the queue.
   - Compare the order in which cases become ready for an activity with the order in which they start the activity.
   - If an express order starts before earlier-ready standard orders, that is evidence of priority overtaking.

2. **Preemption detection**
   - True preemption is easier to detect if the log records `SUSPEND`, `RESUME`, or multiple start/complete cycles.
   - If not explicitly recorded, infer possible interruptions from:
     - Unusually long activity durations for standard orders
     - Resource switching away from a standard order to an express order
     - Gaps between standard order start and complete while the same resource handles express work

3. **Comparative performance mining**
   - Compare waiting times and cycle times by order type:
     - Express
     - Standard
   - Identify whether express acceleration causes disproportionate standard-order delays.

4. **Causal analysis**
   - Use regression, survival analysis, or queueing-based models to estimate whether the arrival of express orders increases waiting or completion times for standard orders, controlling for time of day, workload, resource pool, and order attributes.

#### Metrics

| Metric | Purpose |
|---|---|
| Express average waiting time by activity | Confirms expedited treatment |
| Standard average waiting time when express queue exists | Measures impact on standard orders |
| Number of priority overtakes | Direct evidence of priority handling |
| Delay added to standard orders due to express arrivals | Quantifies externality |
| Preemption frequency | Measures operational disruption |
| Standard SLA breach rate during high express volume | Measures business impact |
| Express SLA achievement | Confirms whether priority policy is effective |

#### Impact quantification

Potential finding:

- Express orders have 70% lower packing wait time than standard orders.
- Standard orders waiting for cold packing experience an additional 9 minutes of delay when express cold orders are in queue.
- During peak hours, 35% of standard cold-packing orders are overtaken by express orders.

This would quantify the trade-off: express orders improve, but standard orders may suffer.

---

### 1.4 Hazardous Material Simultaneity Limit

#### Constraint

No more than 10 hazardous-material orders may be in `Packing` or `Quality Check` simultaneously across the whole facility.

This is a facility-wide concurrency constraint, not a local resource constraint.

#### How to identify it from the event log

Use:

- `Hazardous Material = TRUE`
- `Activity in {Packing, Quality Check}`
- Start and complete timestamps

Process-mining techniques:

1. **Concurrency analysis**
   - Build intervals for hazardous orders in restricted activities:
     - Packing start to packing complete
     - Quality Check start to Quality Check complete
   - Count how many hazardous orders are active at each point in time.

2. **Regulatory limit monitoring**
   - Create a time series of hazardous WIP in restricted zones.
   - Check whether hazardous WIP ever exceeds 10.

3. **Queue analysis for hazardous orders**
   - Determine when hazardous orders become ready for Packing or QC.
   - Identify whether they wait even when local resource capacity is available.
   - If they wait while hazardous WIP is already at 10, classify the delay as caused by the regulatory limit.

4. **Bottleneck impact analysis**
   - Compare hazardous versus non-hazardous cycle times.
   - Compare hazardous waiting times during periods when hazardous WIP is near the threshold.

#### Metrics

| Metric | Purpose |
|---|---|
| Hazardous WIP count over time | Compliance and flow control |
| Percentage of time hazardous WIP is at 10 | Measures bindingness of constraint |
| Waiting time before Packing/QC for hazardous orders | Measures delay |
| Waiting time while regulatory limit is saturated | Direct impact measure |
| Hazardous order throughput | Measures capacity under regulation |
| Compliance violations | Must be zero |
| Non-hazardous impact during hazardous congestion | Detects indirect effects |

#### Impact quantification

Potential finding:

- Hazardous WIP reaches the limit of 10 during 28% of peak-season working hours.
- Hazardous orders wait an average of 14 extra minutes before QC when the facility is at the regulatory cap.
- No compliance breaches occur, but throughput is constrained.

This shows whether the regulation is a binding throughput limiter.

---

### 1.5 Differentiating Within-Instance Waiting from Between-Instance Waiting

A central analytical task is separating delays caused by the order itself from delays caused by other orders.

#### Within-instance factors

These include:

- Long picking duration due to many items
- Rework after failed quality check
- Missing product information
- Order-specific complexity
- Manual exception handling

They affect the case internally.

#### Between-instance factors

These include:

- Waiting for a cold-packing station occupied by another order
- Waiting for a shipping batch to form
- Waiting because express orders overtake standard orders
- Waiting because the hazardous-material concurrency cap has been reached

They arise from interactions between cases.

#### Practical method to distinguish them

For every transition between activities, define:

\[
\text{Inter-Activity Waiting Time} = \text{Next Activity Start} - \text{Previous Activity Complete}
\]

Then classify the reason using system state at the time the order became ready.

Example:

| Situation | Likely Cause |
|---|---|
| Order ready for cold packing, all cold stations occupied | Shared cold-packing contention |
| Order completed QC but shipping label delayed until other same-region orders complete | Batch waiting |
| Standard order ready earlier but express order starts first on same resource | Priority overtaking |
| Hazardous order ready for Packing/QC while hazardous WIP = 10 | Regulatory concurrency limit |
| Activity itself takes unusually long after start | Within-instance processing duration |
| Order waits despite available station and no regulatory cap | Possible staffing, missing data, unlogged rule, or operational idle time |

To support this, create a “state snapshot” at each event timestamp, including:

- Available resources
- Occupied cold-packing stations
- Queue lengths by activity
- Number of hazardous orders in restricted activities
- Express orders waiting
- Batch formation state by region
- Workload level by hour

Then use:

- Queue mining
- Object-centric process mining
- Resource-aware process mining
- Event interval analysis
- Statistical modeling
- Counterfactual simulation

This allows attribution of waiting time to specific instance-spanning constraints.

## 2. Analyzing Constraint Interactions

The constraints do not operate independently. They can amplify or offset each other.

---

### 2.1 Express Orders and Cold-Packing Stations

An express order requiring cold packing competes for the same limited cold-packing stations as standard cold orders.

If express orders receive priority, they may:

- Overtake standard cold orders
- Increase standard cold-packing queue time
- Cause starvation of standard cold-chain orders during peak express periods
- Increase cold-station switching or setup overhead if stations need to adapt to urgent orders

Example interaction:

- A standard cold order is waiting for Station C2.
- An express cold order arrives later.
- The express order is assigned to the next available cold station first.
- The standard cold order waits longer, possibly breaching its SLA.

This interaction means that adding a simple priority rule can improve express performance while degrading total system stability.

---

### 2.2 Express Orders and Hazardous Material Limit

If an express order contains hazardous material, priority handling may conflict with the facility-wide hazardous WIP cap.

For example:

- There are already 10 hazardous orders in Packing/QC.
- A hazardous express order completes picking and is ready for packing.
- It cannot legally enter Packing until another hazardous order leaves Packing or QC.

In this situation, priority cannot override compliance. The system needs a hierarchy of rules:

1. Regulatory compliance
2. Safety and facility constraints
3. Express-priority handling
4. Standard FIFO or optimized sequencing

Without this rule hierarchy, operators may either breach compliance or create confusion and inconsistent prioritization.

---

### 2.3 Batching and Express Orders

Batching can undermine express handling.

For example:

- An express order completes QC quickly.
- It then waits for other orders in the same destination region before label generation.
- The earlier gains from priority packing and QC are lost during batch synchronization.

This suggests that express orders may need:

- Smaller batch thresholds
- Time-limited batching
- Separate express batches
- Immediate label generation under SLA risk

Otherwise, priority handling upstream may not translate into end-to-end delivery improvement.

---

### 2.4 Batching and Hazardous Material Orders

Hazardous orders going to the same region may be batched together for shipping, but they are constrained earlier by the Packing/QC hazardous WIP cap.

Potential effects:

- Hazardous orders may reach label generation unevenly because the cap throttles their flow.
- Batches containing hazardous orders may form more slowly.
- If the batching rule waits for multiple hazardous orders from the same region, post-QC waiting can increase significantly.
- If hazardous goods require special transport documentation or carrier capacity, shipping label generation may become another regulatory-sensitive batching point.

This interaction can produce hidden delay: the hazardous cap delays QC completion, and batching adds another wait after QC.

---

### 2.5 Cold Packing and Batching

Cold-packed perishable goods may have stricter freshness or temperature constraints. If they wait too long after packing or QC for regional batch formation, the process may create quality risks.

Thus, batching logic should treat cold-chain orders differently:

- Maximum waiting time after cold packing
- Smaller or faster batch release
- Route optimization that respects perishability
- Potential separation from standard regional batches

---

### 2.6 Why Interaction Analysis Matters

If each constraint is optimized independently, the company may shift bottlenecks rather than solve them.

Examples:

- Adding cold-packing stations may reduce packing wait, but if regional batching is unchanged, end-to-end lead time may not improve.
- Prioritizing express orders may improve express SLA but increase standard-order backlog and worsen average throughput.
- Increasing hazardous order release to Packing may be impossible due to the regulatory cap.
- Reducing batch size may improve speed but increase shipping cost.

Therefore, optimization must be system-wide, not activity-local.

The correct approach is to evaluate:

- End-to-end order cycle time
- SLA adherence by order class
- Resource utilization
- Queue lengths
- Regulatory compliance
- Shipping cost or route efficiency
- Fairness between standard and express orders
- Bottleneck migration after changes

---

## 3. Developing Constraint-Aware Optimization Strategies

Below are several concrete strategies. In practice, I would test combinations of them through simulation before implementation.

---

### Strategy 1: Dynamic Cold-Packing Scheduling with Priority and Fairness Rules

#### Constraints addressed

Primarily:

- Shared Cold-Packing stations
- Express priority handling

Secondarily:

- Hazardous material limit, if hazardous cold orders exist

#### Proposed changes

Replace simple FIFO or ad hoc station assignment with a dynamic scheduling rule for cold-packing stations.

The scheduler should consider:

- Order type: Express versus Standard
- SLA remaining time
- Cold-chain requirement
- Time already spent waiting
- Hazardous material status
- Station availability
- Predicted downstream congestion
- Risk of standard-order starvation

A possible rule:

1. Do not release hazardous orders into Packing if hazardous Packing/QC WIP is already 10.
2. Prioritize orders at high SLA breach risk.
3. Give express orders priority, but cap the number of consecutive express overtakes.
4. Use aging rules so that standard orders gain priority the longer they wait.
5. Assign cold orders to the first available cold station based on minimizing predicted completion lateness.

#### Data and analysis used

Use historical logs to estimate:

- Arrival rate of cold orders by hour/day
- Express cold-order frequency
- Average and percentile cold-packing durations
- Queue length patterns
- SLA breach probability
- Impact of express overtaking on standard orders

Potential analytical methods:

- Predictive process monitoring
- Queueing analysis
- Resource utilization analysis
- Machine-learning models for remaining cycle time
- Discrete-event simulation

#### Expected outcomes

- Reduced cold-packing waiting time
- Fewer SLA breaches for express cold orders
- Lower starvation risk for standard cold orders
- More stable use of cold-packing stations
- Better trade-off between priority service and overall throughput

---

### Strategy 2: Dynamic Batch Formation for Shipping Label Generation

#### Constraints addressed

Primarily:

- Shipping batching delays

Secondarily:

- Express priority
- Cold-chain freshness
- Hazardous-order flow variation

#### Proposed changes

Move from a fixed regional batch size rule to a dynamic batching policy.

Instead of waiting until a fixed batch size is reached, use a hybrid trigger:

- Release a batch when target batch size is reached, or
- Release a batch when the oldest order in the batch has waited longer than a maximum threshold, or
- Release immediately if an express order is approaching SLA risk, or
- Use smaller batch thresholds for low-volume regions, express orders, hazardous orders, or cold-chain orders.

Example policy:

| Order/Region Type | Batch Rule |
|---|---|
| High-volume standard region | Batch size target, e.g. 15, with max wait 10 minutes |
| Low-volume region | Smaller batch size, e.g. 5, with max wait 8 minutes |
| Express orders | Immediate or micro-batch with max wait 2 minutes |
| Cold-chain orders | Max post-QC waiting time based on freshness constraint |
| Hazardous orders | Separate documentation-aware batching with strict max wait |

#### Data and analysis used

Use event logs to estimate:

- Batch size distribution by region
- Time to form batch by region and hour
- Post-QC waiting time
- SLA impact of batch waiting
- Shipping cost impact of smaller batches
- Volume forecasts by region

Use optimization to balance:

\[
\text{Total Cost} = \text{Delay Penalty} + \text{SLA Penalty} + \text{Shipping Inefficiency Cost}
\]

#### Expected outcomes

- Reduced post-QC waiting time
- Faster label generation
- Improved express SLA
- Less unnecessary waiting for early-completed orders in a batch
- Controlled shipping-cost increase through data-driven thresholds

---

### Strategy 3: Hazardous Material WIP Control with Predictive Release Management

#### Constraints addressed

Primarily:

- Hazardous material regulatory limit

Secondarily:

- Packing/QC resource utilization
- Express hazardous handling
- Batch formation variability for hazardous orders

#### Proposed changes

Introduce a hazardous-order release control mechanism before Packing and Quality Check.

Rather than allowing hazardous orders to accumulate randomly at Packing/QC, manage them using a regulatory-aware release queue.

Rules:

1. Continuously monitor the number of hazardous orders active in Packing and QC.
2. Do not start a new hazardous Packing or QC activity if the active count is 10.
3. Prioritize hazardous orders based on:
   - SLA risk
   - Express status
   - Waiting time
   - Destination batch readiness
   - Downstream QC availability
4. Reserve limited hazardous WIP slots for urgent express hazardous orders if legally and operationally feasible.
5. Smooth hazardous order entry into Packing/QC to avoid bursts.

#### Data and analysis used

Use logs to calculate:

- Hazardous WIP over time
- Time spent at cap
- Hazardous order arrival patterns
- Average Packing and QC duration for hazardous orders
- Hazardous order SLA breach causes
- Whether delays occur before Packing, before QC, or both

Predictive models can estimate:

- When hazardous WIP slots will free up
- Which order should be released next
- Whether releasing a hazardous order now will block downstream QC

#### Expected outcomes

- Zero regulatory violations
- Reduced hazardous-order waiting time
- Better use of the allowed hazardous capacity
- Lower congestion caused by uncontrolled hazardous bursts
- Improved predictability for hazardous-order completion

---

### Strategy 4: Cross-Trained Flexible Packing Capacity During Peaks

#### Constraints addressed

Primarily:

- Cold-packing station contention
- Standard packing congestion

Secondarily:

- Express priority side effects

#### Proposed changes

If feasible, add flexible capacity rather than permanent capacity.

Options include:

- Cross-train staff to operate cold-packing stations.
- Convert some standard stations into temporary cold-capable stations during peak windows.
- Use mobile cold-packing equipment.
- Schedule additional cold-packing staff during predictable peak periods.
- Reserve one cold station for express or SLA-critical orders during extreme peaks, while using aging rules to prevent standard backlog.

#### Data and analysis used

Use process-mining findings to identify:

- Peak hours where cold utilization exceeds threshold, e.g. 85–90%
- Queue length spikes
- Seasonal patterns
- Express cold demand
- Cost per additional temporary station versus delay reduction

#### Expected outcomes

- Reduced queueing at cold-packing bottleneck
- Better handling of peak-season volume
- Lower express-standard conflict
- Improved throughput without overinvesting in permanent capacity

---

### Strategy 5: Decoupling Upstream Picking from Downstream Constraints

#### Constraints addressed

Primarily:

- Cold-packing congestion
- Hazardous WIP limit
- Packing/QC congestion

#### Proposed changes

Currently, picking may release work into constrained downstream steps without regard for capacity. This creates congestion after picking.

Implement workload-aware release from Picking to Packing.

Examples:

- Delay picking of cold orders if cold-packing queue is already excessive, unless the order is express.
- Sequence hazardous orders so they do not all become ready for Packing/QC simultaneously.
- Pick standard non-hazardous orders during periods when hazardous WIP is capped, keeping labor productive.
- Use a “ready-to-pack buffer” with controlled release rules.

#### Data and analysis used

Use event logs to model:

- Downstream queue lengths
- Picking completion rates
- Packing capacity
- Hazardous WIP levels
- Cold-station availability
- SLA remaining time

#### Expected outcomes

- Less downstream congestion
- Better flow synchronization
- Reduced excessive waiting after picking
- Improved labor and station utilization
- More predictable end-to-end cycle times

---

## 4. Simulation and Validation

Before implementing changes, I would build a constraint-aware discrete-event simulation model based on the process-mining analysis.

A normal process model is insufficient because the critical behavior comes from interactions between cases. Therefore, the simulation must include shared resources, queues, priority rules, batching, and global WIP constraints.

---

### 4.1 Simulation Approach

Use process mining to discover the baseline process and enrich it with:

- Activity duration distributions
- Arrival patterns by hour, day, and season
- Resource calendars
- Resource capacities
- Routing probabilities
- Order attributes:
  - Express/Standard
  - Destination region
  - Requires cold packing
  - Hazardous material
- Batch formation rules
- Priority rules
- Regulatory limits

Then implement the model as a discrete-event simulation or agent-based simulation.

---

### 4.2 What the Simulation Must Capture

#### 1. Resource contention

The model should include:

- Number of standard packing stations
- Number of cold-packing stations
- Staff availability
- QC staff availability
- Station-specific service times
- Queues per resource type

For cold-packing:

- Cold orders can only use cold-capable stations.
- Standard non-cold orders cannot use cold-only stations unless policy allows it.
- Express cold orders may have higher priority.

#### 2. Batching behavior

The simulation should represent:

- Destination-region batch queues
- Batch size thresholds
- Maximum waiting-time thresholds
- Different rules for express, cold-chain, hazardous, and standard orders
- Label generation timing
- Shipping cost or route efficiency if available

It should measure how long each order waits for batch formation.

#### 3. Priority interruptions and overtaking

The model should include:

- Express priority queues
- Overtaking logic
- Optional preemption logic if operationally realistic
- Setup or interruption penalties if pausing standard work causes inefficiency
- Aging rules for standard orders

If the actual operation rarely preempts mid-activity but prioritizes at queue boundaries, the model should reflect that. It is important not to simulate preemption unless the process actually supports it.

#### 4. Hazardous material regulatory limit

The model must enforce:

\[
\text{Hazardous orders in Packing + Hazardous orders in QC} \leq 10
\]

At every simulation time point.

It should also track:

- Hazardous WIP
- Blocked hazardous orders
- Time spent at cap
- Compliance violations, which should be zero
- Impact on hazardous and non-hazardous throughput

#### 5. Interactions between constraints

The simulation should explicitly allow combined cases:

- Express + cold
- Express + hazardous
- Cold + hazardous
- Hazardous + same-region batch
- Express + cold + hazardous

These combined order types are often where delays or policy conflicts are most severe.

---

### 4.3 Experiments to Run

Run baseline simulation first and calibrate it against historical logs.

Baseline validation should compare:

- End-to-end cycle time distribution
- Activity duration distributions
- Queue waiting times
- Resource utilization
- Batch sizes
- Hazardous WIP profiles
- SLA breach rates

Then test scenarios such as:

1. Dynamic cold-packing scheduling
2. Dynamic batching with maximum waiting time
3. Hazardous WIP release control
4. Additional temporary cold-packing capacity
5. Express micro-batching
6. Combined policy scenarios

---

### 4.4 KPIs to Evaluate in Simulation

| KPI | Why It Matters |
|---|---|
| End-to-end order cycle time | Overall performance |
| SLA achievement rate | Business outcome |
| Express SLA rate | Priority policy success |
| Standard SLA rate | Detects unfair degradation |
| Cold-packing queue time | Bottleneck reduction |
| Post-QC batch waiting time | Batch improvement |
| Hazardous WIP compliance | Regulatory requirement |
| Time at hazardous cap | Binding constraint indicator |
| Resource utilization | Efficiency |
| Queue lengths by activity | Congestion |
| Throughput per day | Capacity |
| Shipping batch size and cost proxy | Logistics efficiency |
| Number of priority overtakes | Express impact on standard orders |

The best policy is not necessarily the one with the fastest average time. It should improve SLA performance and throughput while maintaining compliance and controlling shipping cost.

The dashboards should combine process-mining views, resource dashboards, compliance dashboards, and predictive alerts.

- End-to-end cycle time by:

  - Region

- SLA compliance rate
- Throughput per hour/day
- Work-in-progress by process stage
- Average and percentile waiting time between activities
- Rework or exception rates

Visualizations:

- Process map with performance overlays
- Bottleneck heatmap
- Variant analysis
- Cycle-time distribution
- SLA breach root-cause breakdown

---

### 5.2 Cold-Packing Constraint Dashboard

Metrics:

- Cold-packing station utilization

- Average and 95th percentile cold-packing wait time

- SLA risk among cold orders

Useful dashboard views:

- Timeline of cold station occupancy
- Queue length by hour
- Utilization heatmap

Success indicators:

- Reduced cold-packing queue length

- Alerts at thresholds, e.g. 8, 9, and 10 active hazardous orders

Success indicators:

- Zero violations
- Reduced waiting time while at cap
- Better utilization of allowed hazardous capacity
- Improved hazardous-order SLA adherence

### 5.7 Continuous Improvement Loop

Post-implementation monitoring should follow a closed-loop approach:

1. **Monitor**
   - Use dashboards to track KPIs and constraints daily.

2. **Detect**
   - Identify deviations, new bottlenecks, or policy failures.

3. **Diagnose**
   - Use process mining to determine whether delays are caused by resources, batches, priority rules, or compliance caps.

4. **Predict**
   - Use predictive monitoring to forecast congestion and SLA risk.

5. **Adjust**
   - Modify staffing, station allocation, batch thresholds, or release rules.

6. **Validate**
   - Compare before/after performance using statistically sound A/B or phased rollout analysis.

## Summary

The fulfillment center’s performance problems are driven not only by individual order processing times but by instance-spanning constraints: limited cold-packing stations, regional shipping batches, express priority handling, and hazardous-material concurrency limits.

A suitable improvement strategy should therefore combine:

- Resource-aware process mining
- Queue and concurrency analysis
- Batch mining
- Priority-overtaking analysis
- Regulatory WIP monitoring
- Constraint-aware simulation
- Continuous dashboard-based control

The most important recommendations are:

1. Implement dynamic cold-packing scheduling with priority and fairness controls.
2. Replace fixed shipping batches with dynamic batch triggers based on size, time, SLA risk, region, and order type.
3. Introduce predictive hazardous-material WIP release control.
4. Add flexible cold-packing capacity during predictable peaks.
5. Synchronize upstream picking with downstream capacity to avoid uncontrolled queue buildup.

These strategies directly address the dependencies between orders and should improve cycle time, SLA performance, throughput, and operational predictability while maintaining regulatory compliance.
