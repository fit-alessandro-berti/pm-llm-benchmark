```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define all activities
ReceiveApplication = Transition(label="ReceiveApplication")
ResumeParsing = Transition(label="ResumeParsing")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")
skip = SilentTransition()

# Create the data completeness loop: check -> if incomplete -> request more info -> check again
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck, RequestMoreInfo])

# Create the XOR choice for cultural fit evaluation
# One branch leads to standard cultural fit, the other to community affiliation check
cultural_fit_xor = OperatorPOWL(operator=Operator.XOR, children=[CulturalFitCheck, CommunityAffiliationCheck])

# Build the main workflow
root = StrictPartialOrder(nodes=[
    ReceiveApplication, 
    ResumeParsing, 
    data_loop, 
    SkillAssessment, 
    cultural_fit_xor, 
    ManagerialReview, 
    FinalDecision
])

# Define the sequence/dependencies
root.order.add_edge(ReceiveApplication, ResumeParsing)
root.order.add_edge(ResumeParsing, data_loop)
root.order.add_edge(data_loop, SkillAssessment)
root.order.add_edge(SkillAssessment, cultural_fit_xor)
root.order.add_edge(cultural_fit_xor, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)

# Print the model structure
print("Model 1 - With Potential Unfairness:")

print("Order edges:", list(root.order.edges))
```

## Key Differences Analysis

### Model 1 (With Potential Unfairness):
```
ReceiveApplication  ResumeParsing  [DataCompletenessLoop]  SkillAssessment  XOR(CulturalFitCheck, CommunityAffiliationCheck)  ManagerialReview  FinalDecision
```
**Bias Mechanism**: The XOR choice creates two parallel evaluation paths. Applicants with community affiliations get the "CommunityAffiliationCheck" path, which gives them preferential treatment, while others get the standard "CulturalFitCheck". This creates unfair advantage based on non-merit factors.

### Model 2 (Without Unfairness):
```
ReceiveApplication  ResumeParsing  [DataCompletenessLoop]  SkillAssessment  CulturalFitCheck  ManagerialReview  FinalDecision
```
**Fairness Mechanism**: All applicants follow the same sequential path. Every candidate undergoes the identical "CulturalFitCheck" evaluation process, ensuring merit-based assessment without special advantages for community affiliations.

## Summary

The first model explicitly captures the bias point described in the process: the XOR branch allows for differential treatment based on community affiliations. The second model eliminates this unfairness by ensuring all applicants go through the same evaluation process, making the cultural fit assessment standardized and bias-free.

Both models maintain the essential structure of the hiring process (application receipt, data validation loop, skill assessment, cultural evaluation, managerial review, and final decision), but differ fundamentally in how they handle the cultural fit evaluation stage.
