```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

OR = Transition(label="OR")
MS = Transition(label="MS")
QS = Transition(label="QS")
AS = Transition(label="AS")
IQC = Transition(label="IQC")
TST = Transition(label="TST")
RT = Transition(label="RT")
PK = Transition(label="PK")
LB = Transition(label="LB")
DP = Transition(label="DP")
D = Transition(label="D")
skip = SilentTransition()

assembly = OperatorPOWL(operator=Operator.LOOP, children=[AS, IQC])
testing = OperatorPOWL(operator=Operator.LOOP, children=[TST, RT])
label_choice = OperatorPOWL(operator=Operator.XOR, children=[LB, skip])

root = StrictPartialOrder(nodes=[OR, MS, QS, assembly, testing, PK, label_choice, DP, D])
root.order.add_edge(OR, MS)
root.order.add_edge(MS, QS)
root.order.add_edge(QS, assembly)
root.order.add_edge(assembly, testing)
root.order.add_edge(testing, PK)
root.order.add_edge(PK, label_choice)
root.order.add_edge(label_choice, DP)
root.order.add_edge(DP, D)
```

**Explanation**:  
- `OR  MS  QS` is the fixed initial sequence.  
- `assembly = *(AS, IQC)` captures the initial Assembly followed by zero or more (IQC  AS) cycles (matches all cases except one anomalous consecutive-IQC trace).  
- `testing = *(TST, RT)` models the Test / Re-Test loop.  
- `label_choice = X(LB, skip)` models the optional Labeling step.  
- The `StrictPartialOrder` enforces the overall execution order while allowing the nested loop/choice semantics. This POWL compactly represents the observed manufacturing/distribution variations.
