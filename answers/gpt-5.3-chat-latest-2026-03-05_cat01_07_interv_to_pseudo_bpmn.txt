BPMN-like Textual Process Model: Invoice Approval Process

Participants / Roles (suggested swimlanes):
- Accounts Payable Clerk (Mary)
- Purchasing Department (Dan)
- Department Manager (Carlos)
- Accounts Payable Supervisor (Rita)
- Supplier Relationship Manager (Karen)
- Accounting System (automated notifications)

Start Event:
- Invoice Received (typically via email by Accounts Payable)

Main Process Flow:

1. Task (Accounts Payable Clerk): Receive Invoice
   Input: Supplier Invoice

2. Task (Accounts Payable Clerk): Check for Purchase Order (PO) Number

3. Exclusive Gateway: PO Number Present?
   - If NO:
       3a. Task (Accounts Payable Clerk): Request Missing PO Information from Supplier
       3b. Intermediate Event: Await Supplier Response
       3c. If PO Provided  Return to "Check for Purchase Order Number"
       3d. If PO Cannot Be Provided  Escalate to Supervisor (Exception Path)

   - If YES:
       Continue to next step.

4. Task (Accounts Payable Clerk): Log Invoice in Accounting System
   Artifact: Invoice Record Created in Accounting System

5. Task (Accounts Payable Clerk): Send Invoice to Purchasing for Goods/Service Confirmation
   Output: Invoice + PO reference forwarded to Purchasing

6. Task (Purchasing Officer): Verify Invoice Against Purchase Order
   Activities include:
   - Check quantities
   - Check pricing
   - Check delivery charges
   - Confirm goods/services received

7. Exclusive Gateway: Invoice Consistent with PO and Delivery?
   - If YES:
       7a. Task (Purchasing Officer): Send Confirmation Email to Accounts Payable
       Flow returns to Accounts Payable.

   - If NO:
       7b. Task (Purchasing Officer): Investigate Discrepancy
           Possible actions:
           - Contact Supplier directly
           - Ask Accounts Payable to place invoice on hold
           - Request corrected invoice
       7c. Intermediate Event: Await Supplier Clarification or Corrected Invoice
       7d. If resolved  Send confirmation to Accounts Payable
       7e. If unresolved or complex  Escalate to Supervisor (Exception Path)

8. Task (Accounts Payable Clerk): Match Invoice Line Items with Purchase Order in System

9. Exclusive Gateway: Invoice Matches PO?
   - If YES:
       Continue to Manager Approval.
   - If NO:
       9a. Task (Accounts Payable Clerk): Contact Purchasing for Clarification
       9b. If discrepancy persists  Escalate to Supervisor (Exception Path)
       9c. Once resolved  Return to "Match Invoice Line Items with PO"

10. Task (Accounting System): Send Automatic Approval Request to Department Manager
    Trigger: Confirmation of receipt + successful invoice–PO match

11. Task (Department Manager): Review Invoice, PO, and Confirmation Details

12. Exclusive Gateway: Manager Approval Decision
   - If APPROVED:
       12a. Task (Department Manager): Approve Invoice in System
       12b. Flow returns to Accounts Payable.

   - If REJECTED or QUESTIONED:
       12c. Task (Department Manager): Request Investigation / Reject Invoice
       12d. Task (Accounts Payable Clerk): Place Invoice on Hold and Investigate
       12e. Investigation may involve Purchasing or Supplier Relationship Manager
       12f. Once clarified  Return to Manager Review or Resume Normal Flow

13. Task (Accounts Payable Clerk): Schedule Invoice for Payment Run
    Artifact: Payment Schedule Entry

End Event:
- Invoice Included in Payment Cycle

Exception Handling / Alternative Paths:

Exception Path A: Missing PO that cannot be obtained
- Task (Accounts Payable Clerk): Escalate to AP Supervisor
- Task (AP Supervisor): Investigate Missing PO
    Possible actions:
    - Contact department heads
    - Create retrospective PO for informal service purchases
    - Reject invoice
- Gateway: Resolution Outcome
    - Retrospective PO created  Return to PO verification stage
    - Invoice rejected  End Event: Invoice Rejected

Exception Path B: Unresolved Invoice Discrepancy
- Escalation Trigger: Pricing mismatch, incorrect quantities, or unclear terms
- Task (AP Supervisor): Coordinate with departments and/or suppliers
- Task (Supplier Relationship Manager): Contact supplier and verify contract terms
- Intermediate Event: Await supplier correction or clarification
- Gateway: Issue Resolved?
    - YES  Resume normal flow at Purchasing confirmation
    - NO  Possible invoice rejection

Supporting Artifacts / Data Objects:
- Supplier Invoice
- Purchase Order (PO)
- Corrected Invoice (if issued)
- Confirmation Email from Purchasing
- Contract Information (used by Supplier Relationship Manager)
- Payment Schedule Entry in Accounting System

Noted Variations / Uncertainty:
- Purchasing may either contact the supplier directly or request Accounts Payable to hold the invoice during discrepancy resolution.
- Supplier Relationship Manager becomes involved mainly in cases involving pricing disputes or supplier communication needs.
- The exact point where the invoice is formally “held” during discrepancy resolution varies between Accounts Payable and Purchasing depending on the situation.
- Manager rejection occurs infrequently but results in additional investigation before re-entering the approval step or rejecting the invoice.
