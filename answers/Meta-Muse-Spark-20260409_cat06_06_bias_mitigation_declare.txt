### Rationale for Added Constraints

| Constraint Added | Type | Why It Reduces Bias |
| --- | --- | --- |
| `existence: BiasMitigationCheck, ManualReview` | Unary | Forces every trace to include explicit fairness activities. You cannot complete a loan application without invoking the mitigation steps, so “fast-path” discriminatory outcomes are ruled out. |
| `responded_existence: CheckApplicant* -> BiasMitigationCheck` | Binary | If any sensitive attribute is examined, the model demands that a `BiasMitigationCheck` also occurs somewhere in the trace. This prevents silent use of sensitive data. |
| `response: CheckApplicant* -> BiasMitigationCheck` | Binary | Stronger than `responded_existence`: the mitigation check must occur *after* the sensitive attribute is accessed, ensuring it can actually influence the downstream decision. |
| `precedence: BiasMitigationCheck -> Approve/Reject/FinalDecision` | Binary | No decision is allowed unless a mitigation check has already occurred. This blocks decisions that are based solely on pre-mitigation information. |
| `coexistence: FinalDecision/Approve/Reject <-> ManualReview` | Binary | Couples every decision with `ManualReview`. Automated decisions on sensitive groups must be reviewed by a human, reducing algorithmic bias risk. |
| `nonsuccession: CheckApplicant* -> Approve/Reject/FinalDecision` | Binary | Prohibits any path where a sensitive attribute check is eventually followed by a decision, unless other activities intervene. Breaks direct causal chains. |
| `nonchainsuccession: CheckApplicant* -> Approve/Reject` | Binary | Strictest form: forbids *immediate* succession from sensitive attribute to decision. This eliminates “look at race -> instant reject” patterns entirely. |

**Overall effect**: The updated model enforces a mandatory firewall between sensitive data access and outcomes. Any trace that reads `ApplicantRace`, `ApplicantGender`, or `ApplicantAge` must run `BiasMitigationCheck` and `ManualReview` before reaching `Approve`, `Reject`, or `FinalDecision`. Direct and immediate paths from sensitive attributes to decisions are explicitly banned, which limits the process’s ability to encode or execute biased decision rules.
