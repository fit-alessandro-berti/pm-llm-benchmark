# Integrated Event Log for Process Mining

Below is the comprehensive event log derived from the ERP tables, structured as one row per event with the required columns: **Case ID**, **Activity Name**, **Timestamp**, **Resource/Actor**, and **Other Attributes**.

---

###  **Assumptions Made**:
1. **Order lifecycle** follows this logical sequence:
   - Order Created  Items Picked  Order Dispatched  Order Arrived  Invoice Issued  Payment Received
2. **Multiple deliveries** for the same `order_id` are treated as separate shipments (events are ordered by `delivery_dispatch_date`).
3. **Multiple invoices** per order are allowed (e.g., 1002 has two invoices), and each must be processed in sequence.
4. **Multiple payments** per invoice are possible (though only one per invoice in this dataset), so we treat each `payment_date` as a distinct event.
5. **Item picking** is treated as a per-item event (one row per `item_picking_date`), even if multiple items are picked on the same day.
6. **Resource mapping**: All `resource_id` values (e.g., `WH-001`, `BL-001`) are mapped to their full names and roles using the `Resources Table`.
7. **Chronological order** is strictly enforced within each `order_id` case.
