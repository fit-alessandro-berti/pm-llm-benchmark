Below are two POWL models representing the hiring process at GlobalTeq Solutions, as requested.

---

## **1. POWL Model with Potential Unfairness**

This model includes an **XOR branch** after the skill assessment:
- One path leads to a **standard CulturalFitCheck**.
- The other path leads to a **CommunityAffiliationCheck**, which may give an implicit advantage to local-affiliated applicants.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Silent transitions for flow control
tau = SilentTransition()

# Loop for incomplete data: if data missing, request more info, then re-check
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck, RequestMoreInfo])

# XOR branch after skill assessment
unfair_xor = OperatorPOWL(operator=Operator.XOR, children=[CulturalFitCheck, CommunityAffiliationCheck])

# Partial order: sequence of main activities with the XOR in between
root = StrictPartialOrder(nodes=[
    ReceiveApplication,
    data_loop,
    SkillAssessment,
    unfair_xor,
    ManagerialReview,
    FinalDecision
])

# Define ordering constraints
root.order.add_edge(ReceiveApplication, data_loop)
root.order.add_edge(data_loop, SkillAssessment)
root.order.add_edge(SkillAssessment, unfair_xor)
root.order.add_edge(unfair_xor, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)
```

> **Where Unfairness Appears:**  
> The **XOR choice** between `CulturalFitCheck` and `CommunityAffiliationCheck` introduces a subtle bias. Applicants with local community affiliations may take the `CommunityAffiliationCheck` path and receive an implicit score uplift, giving them an unfair advantage over others who follow the standard evaluation path.

---

## **2. POWL Model Without Unfairness**

This version removes the preferential branch. All applicants follow the **same cultural fit evaluation process**, ensuring fairness.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Silent transition for flow control
tau = SilentTransition()

# Loop for incomplete data
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck, RequestMoreInfo])

# No XOR Ś all applicants go through the same cultural fit check
root = StrictPartialOrder(nodes=[
    ReceiveApplication,
    data_loop,
    SkillAssessment,
    CulturalFitCheck,
    ManagerialReview,
    FinalDecision
])

# Define ordering constraints
root.order.add_edge(ReceiveApplication, data_loop)
root.order.add_edge(data_loop, SkillAssessment)
root.order.add_edge(SkillAssessment, CulturalFitCheck)
root.order.add_edge(CulturalFitCheck, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)
```

> **How Unfairness Is Removed:**  
> By replacing the XOR branch with a single **CulturalFitCheck** for all applicants, the model ensures equal treatment. No group receives special consideration based on community affiliation or educational background.
