Here is a DuckDB SQL query that keeps only the events belonging to cases with six or fewer events:

```sql
WITH case_counts AS (
    SELECT
        case_id,
        COUNT(*) AS event_cnt
    FROM event_log
    GROUP BY case_id
)
SELECT
    e.*
FROM event_log AS e
JOIN case_counts AS cc
    ON e.case_id = cc.case_id
WHERE cc.event_cnt <= 6
ORDER BY e.case_id, e.timestamp;
```

**Explanation**

1. **`case_counts` CTE** – computes the number of events per `case_id`.
2. **Main query** – joins the original `event_log` to the counts and retains only rows where the counted events (`event_cnt`) are 6.
3. The final `ORDER BY` is optional but helps to view the results grouped by case and chronological order.
