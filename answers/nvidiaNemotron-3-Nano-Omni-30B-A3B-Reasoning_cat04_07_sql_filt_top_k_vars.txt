Below is a **singlestatement** DuckDB query that fulfils every requirement.  
Replace `:K` with the desired number of top variants (e.g. `5` for the top5 variants).

```sql
WITH
-- 1  Ordered activity sequence per case
ordered_seq AS (
    SELECT
        case_id,
        LISTAGG(activity, '  ' ORDER BY timestamp) AS activity_seq          -- ordered list
    FROM event_log
    GROUP BY case_id
),

-- 2  Count how many cases follow each distinct variant
variant_counts AS (
    SELECT
        activity_seq,
        COUNT(*) AS case_cnt
    FROM ordered_seq
    GROUP BY activity_seq
),

-- 3  Determine the topK variants (by frequency)
top_k_variants AS (
    SELECT
        activity_seq
    FROM variant_counts
    ORDER BY case_cnt DESC
    LIMIT :K
),

-- 4  Keep only cases that belong to those topK variants
cases_in_top_k AS (
    SELECT DISTINCT case_id
    FROM ordered_seq
    WHERE activity_seq IN (SELECT activity_seq FROM top_k_variants)
)

-- 5  Return **all** original events that belong to the selected cases
SELECT
    el.case_id,
    el.activity,
    el.timestamp
FROM event_log AS el
JOIN cases_in_top_k AS c ON el.case_id = c.case_id
ORDER BY el.case_id, el.timestamp;
```

### How the query satisfies the specification

| Step | What the query does | Relevant part of the query |
|------|---------------------|----------------------------|
| **1** | Builds the ordered activity sequence for each `case_id` by sorting events on `timestamp` and concatenating the `activity` values. | `ordered_seq` CTE (`LISTAGG(activity, '  ' ORDER BY timestamp)`) |
| **2** | Groups the sequences and counts how many cases share each exact variant. | `variant_counts` CTE (`COUNT(*) AS case_cnt`) |
| **3** | Picks the *K* most frequent variants. | `top_k_variants` CTE (`ORDER BY case_cnt DESC LIMIT :K`) |
| **4** | Identifies the `case_id`s that belong to those topK variants. | `cases_in_top_k` CTE (`WHERE activity_seq IN (…)`) |
| **5** | Returns **all** original rows (events) from the selected cases, preserving the original order. | Final `SELECT … FROM event_log … JOIN cases_in_top_k` |

You can run the query asis in DuckDB (e.g., via `duckdb.sql(...)` in Python, the DuckDB CLI, or any client). Just substitute `:K` with the integer you need. The result set contains every event from the event log that belongs to a case whose activity sequence is among the topK most common variants.
