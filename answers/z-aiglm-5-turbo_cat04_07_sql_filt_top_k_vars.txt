Here is the DuckDB SQL query to achieve this. 

To ensure accurate matching of variants, this query uses `array_agg` instead of `string_agg` (which can fail if an activity name coincidentally contains your chosen delimiter). It also uses a secondary sort key (`activity ASC`) to guarantee deterministic variant creation in edge cases where two events happen at the exact same millisecond.

```sql
WITH CaseVariants AS (
    -- Step 1: Extract the ordered sequence of activities per case_id.
    -- Sorting by timestamp and activity ensures deterministic variant creation.
    SELECT 
        case_id,
        array_agg(activity ORDER BY timestamp ASC, activity ASC) AS variant
    FROM event_log
    GROUP BY case_id
),
TopKVariants AS (
    -- Steps 2 & 3: Group by variant, count the cases, and get the top K.
    SELECT 
        variant
    FROM (
        SELECT 
            variant,
            COUNT(*) AS case_frequency
        FROM CaseVariants
        GROUP BY variant
        ORDER BY case_frequency DESC
        LIMIT 5  -- Replace '5' with your desired K value
    ) AS ranked_variants
)
-- Step 4: Return original events filtered down to only the top K variant cases.
SELECT 
    e.case_id,
    e.activity,
    e.timestamp
FROM event_log e
INNER JOIN CaseVariants cv 
    ON e.case_id = cv.case_id
INNER JOIN TopKVariants tkv 
    ON cv.variant = tkv.variant
ORDER BY 
    e.case_id, 
    e.timestamp ASC;
```

### How it works:
1. **`CaseVariants` CTE**: Groups the log by `case_id` and utilizes DuckDB's native `array_agg` to build an array representing the exact path of activities. Using an array protects against delimiter collisions (e.g., if an activity is named `"A->B"`).
2. **`TopKVariants` CTE**: Takes the arrays from the previous step, counts how many distinct `case_id`s share that exact array, sorts them descending by frequency, and applies `LIMIT K` to isolate only the top sequences.
3. **Final `SELECT`**: Joins the original `event_log` back to the `CaseVariants` CTE, but filters it using an `INNER JOIN` against the `TopKVariants` CTE. Because it's an inner join, any case that does not belong to one of the top K variants is instantly dropped, and all remaining original event rows for the qualifying cases are returned.
