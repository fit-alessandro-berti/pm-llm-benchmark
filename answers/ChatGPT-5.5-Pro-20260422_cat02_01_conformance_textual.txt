## Expected process paths

The description implies two valid variants:

**Fulfillment path, when inventory is sufficient:**

> Receive order → Check inventory → Authorize payment → Package items → Ship order → Close order

**Cancellation path, when inventory is insufficient:**

> Receive order → Check inventory → Cancel order → Close order

The key constraints are:

* **Receive order** must start the case.
* **Check inventory** must happen before payment, packaging, shipping, or cancellation decisions.
* **Authorize payment** must happen after inventory is checked and before packaging.
* **Package items** must happen after payment authorization and before shipping.
* **Ship order** must happen only after packaging and never after cancellation.
* **Close order** must be the final activity.

## Case-by-case assessment

### Case 1 — Correct

Sequence:

> Receive order → Check inventory → Authorize payment → Package items → Ship order → Close order

This exactly follows the normal fulfillment path for sufficient inventory. No violation.

---

### Case 2 — Violating

Sequence:

> Receive order → Check inventory → Package items → Authorize payment → Ship order → Close order

Violation:

**Package items occurs before Authorize payment.**

The normative description says that after inventory is confirmed, payment should be authorized before items are prepared for shipment. It explicitly states that:

> “Package items” should only occur after authorization and before shipping.

In this case, the items are packaged at 10:00, but payment is authorized only at 10:30. That reverses the expected order of those two activities.

So **Case 2 violates the rule that payment authorization must precede packaging**.

---

### Case 3 — Violating

Sequence:

> Receive order → Check inventory → Authorize payment → Cancel order → Ship order → Close order

There are several problems here.

First, **Cancel order is followed by Ship order**. The description states that once an order is canceled, it is illogical to continue with fulfillment activities such as packaging or shipping. It also says:

> “Ship order” should only occur after packaging and never after a cancellation.

This case directly violates that rule because the order is shipped after it has been canceled.

Second, **Ship order occurs without Package items**. In the normal fulfillment path, shipping must be preceded by packaging:

> Authorize payment → Package items → Ship order

Here, the case goes from cancellation directly to shipping, with no packaging step.

Third, the presence of both **Authorize payment** and **Cancel order** is suspicious under the described branching logic. The text implies that after checking inventory, the process should either continue down the fulfillment path or directly cancel the order. If the inventory is insufficient, the expected path is:

> Receive order → Check inventory → Cancel order → Close order

So authorizing payment before cancellation does not fit the intended cancellation path. This is somewhat less explicit than the shipping-after-cancellation violation, but it is still inconsistent with the described branching logic.

Therefore, **Case 3 violates the cancellation logic, ships after cancellation, and ships without packaging**.

---

### Case 4 — Correct

Sequence:

> Receive order → Check inventory → Cancel order → Close order

This matches the expected cancellation path for insufficient inventory. No violation.

---

### Case 5 — Violating

Sequence:

> Receive order → Check inventory → Authorize payment → Ship order → Close order

Violation:

**Ship order occurs without Package items.**

The normative fulfillment path requires packaging before shipping:

> Receive order → Check inventory → Authorize payment → Package items → Ship order → Close order

The description explicitly says that items must be prepared before they can be shipped. In Case 5, the process skips **Package items** entirely and goes directly from **Authorize payment** to **Ship order**.

So **Case 5 violates the rule that shipping must only occur after packaging**.

## Final classification

**Conforming cases:**

* **Case 1**
* **Case 4**

**Violating cases:**

* **Case 2** — packaging before payment authorization.
* **Case 3** — cancellation followed by shipping; shipping without packaging; inconsistent branching after inventory check.
* **Case 5** — shipping without packaging.
