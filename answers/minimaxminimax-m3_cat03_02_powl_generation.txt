# POWL Model for Telecom Corporate Network Provisioning

Looking at the scenario, I need to model:
1. A linear start (SR  CS)
2. Two parallel tasks after CS (DA and ACI)
3. A loop (BNC  D&R) after both parallel tasks
4. An exclusive choice (CD vs MD) after the loop
5. A partial order with concurrent QA/SA, both preceding GLA

Here is the complete POWL model construction:

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# ----------------------------------------------------------------------
# 1. Define all individual activities (transitions)
# ----------------------------------------------------------------------
SR  = Transition(label="SR")   # Service Request verification
CS  = Transition(label="CS")   # Contract Setup
DA  = Transition(label="DA")   # Datacenter Allocation
ACI = Transition(label="ACI")  # Access Control Initialization
BNC = Transition(label="BNC")  # Base Network Configuration
DR  = Transition(label="D&R")  # Debug & Reconfigure
CD  = Transition(label="CD")   # Containerized Deployment
MD  = Transition(label="MD")   # Monolithic Deployment
QA  = Transition(label="QA")   # Quality Assurance
SA  = Transition(label="SA")   # Security Audit
GLA = Transition(label="GLA")  # Go-Live Approval

# ----------------------------------------------------------------------
# 2. Loop: BNC followed by optional D&R (executed repeatedly until stable)
#    The LOOP operator means: execute A, then either exit or execute B
#    and then A again.
# ----------------------------------------------------------------------
config_loop = OperatorPOWL(operator=Operator.LOOP, children=[BNC, DR])

# ----------------------------------------------------------------------
# 3. Exclusive choice: either Containerized Deployment OR Monolithic Deployment
# ----------------------------------------------------------------------
deployment_choice = OperatorPOWL(operator=Operator.XOR, children=[CD, MD])

# ----------------------------------------------------------------------
# 4. Final phase as a partial order:
#    - QA and SA are concurrent (no edge between them)
#    - Both must be completed before GLA (edges QA -> GLA, SA -> GLA)
# ----------------------------------------------------------------------
final_phase = StrictPartialOrder(nodes=[QA, SA, GLA])
final_phase.order.add_edge(QA,  GLA)   # QA must complete before GLA
final_phase.order.add_edge(SA,  GLA)   # SA must complete before GLA
# (No edge between QA and SA -> they are concurrent)

# ----------------------------------------------------------------------
# 5. Root partial order wiring everything together
# ----------------------------------------------------------------------
root = StrictPartialOrder(
    nodes=[SR, CS, DA, ACI, config_loop, deployment_choice, final_phase]
)

# Linear start: SR must precede CS
root.order.add_edge(SR, CS)

# After CS, DA and ACI can run in parallel
root.order.add_edge(CS, DA)
root.order.add_edge(CS, ACI)

# Loop can start only when BOTH DA and ACI are finished
root.order.add_edge(DA, config_loop)
root.order.add_edge(ACI, config_loop)

# After the configuration loop ends, choose deployment method
root.order.add_edge(config_loop, deployment_choice)

# Final phase (QA/SA/GLA) can start only after the deployment choice completes
root.order.add_edge(deployment_choice, final_phase)
```

## How the model captures the scenario

| Step in scenario | POWL construct |
|---|---|
| Service Request  Contract Setup | `SR  CS` edge in root partial order |
| Contract Setup precedes both DA and ACI (parallel) | `CS  DA` and `CS  ACI` edges (no order between DA and ACI) |
| DA and ACI both gate the loop | `DA  config_loop` and `ACI  config_loop` |
| Base config  Debug & Reconfigure loop | `OperatorPOWL(LOOP, [BNC, DR])` |
| Choose CD **or** MD | `OperatorPOWL(XOR, [CD, MD])` |
| QA and SA concurrent, both before GLA | Inner `StrictPartialOrder` with `QA  GLA`, `SA  GLA` and no edge between QA and SA |
| Final phase starts after deployment | `deployment_choice  final_phase` |

The model faithfully encodes: **mandatory ordering** (edges in partial order), **concurrency** (unconnected nodes), **iteration** (LOOP operator on BNC/DR), and **exclusive branching** (XOR operator on CD/MD) — exactly as described in the provisioning workflow.
