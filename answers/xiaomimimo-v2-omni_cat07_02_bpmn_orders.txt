This is a **BPMN, Business Process Model and Notation, diagram** that maps an end-to-end order fulfillment process across four departments: Sales, Production, Procurement, Warehouse, with built-in error/compensation handling for material issues. Here's a breakdown:

1.  **Sales Department (Top Lane): Order Initiation**
    - Starts with a *timer event* (`Monday morning`) to begin the order process.
    - Steps:
      - First, `Check order for completeness`, then `Check if customer is already in the database`
      - A decision gateway splits here:
        - If the customer is **not in the database**: run `Add customer to the database` (connected to a database data object)
        - If the customer is already in the database: proceed directly
      - Paths rejoin, then `Send order details`: this triggers the **Production** department via a cross-lane message flow
      - Finally, a message event `Product sent` feeds into `Draft invoice`, then the process ends.

2.  **Production Department (Second Lane): Manufacturing**
    - Triggered by `Order details arrive`: a message start event from Sales
    - Steps:
      - Starts with a gateway, then `Plan production`, then a second gateway leading to `Manufacture product`
      - If materials are faulty: the `Material not faulty` compensation event triggers the **Procurement** department's process for material correction
      - When manufacturing finishes: `Product complete`: a message event sends a signal to the **Warehouse** department, and also feeds back to Sales' invoicing step.

3.  **Procurement Department (Third Lane): Material Management**
    - Triggered by `Order details arrive`: a message start event from Sales
    - Steps:
      - Includes a `Material Compensation Subprocess`: `Order materials` for initial sourcing, with a compensation activity `Return material` if materials need to be sent back
      - Then `Check materials`:
        - If materials can't be sourced: `Order cancellation`: an end event to terminate the process
        - If materials are valid: a signal end event triggers Production to continue manufacturing

4.  **Warehouse Department (Bottom Lane): Product Delivery**
    - Triggered by `Product complete`: a message start event from Production
    - Executes `Send product`, then a signal end event `Product sent`: sends a message back to Sales, to trigger the invoicing step and close out the customer order.

Solid lines are *sequence flows* within a department, dashed lines are *message flows* cross-department communication, with clear error/compensation pathways for handling material issues.
