## Solution

```sql
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
);
```

## Explanation

Here's how the query works:

1. **Inner Query (Subquery)**: 
   - Groups events by `case_id`
   - Uses `HAVING COUNT(*) <= 6` to identify only those cases with 6 or fewer events

2. **Outer Query**:
   - Selects all columns from `event_log`
   - Filters rows using `WHERE case_id IN (...)` to include only events from the qualifying cases

This approach efficiently:
- Calculates the event count per case
- Excludes cases with more than 6 events
- Returns the complete set of events for the remaining cases

## Alternative Using CTE

For better readability, you could also use a Common Table Expression:

```sql
WITH valid_cases AS (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT e.*
FROM event_log e
JOIN valid_cases v ON e.case_id = v.case_id;
```

Both versions will give you the same result — all events from cases that have 6 or fewer total events.
