This is a **BPMN Business Process Model and Notation diagram** for the **Dispatch of goods - Computer Hardware Shop** process, organized into three swimlanes representing roles: *Secretary*, *Logistics*, and *Warehouse*:

1.  **Process Initiation**:
    The flow starts with the *Ship goods* start event, which triggers a parallel gateway: this splits the work into two simultaneous streams: **Secretary coordination tasks** and **Warehouse physical tasks**.

2.  **Secretary Workflow**:
    -   First, complete *Clarify shipment method*, then a decision gateway *Special handling?*:
        -   If **Yes**: Move to *Get 3 offers from logistic companies*  *Select logistic company and place order*, then proceed to sync with the insurance/label path.
        -   If **No**: Proceed to *Check if insurance is required*, which uses "Order details" and "Customer information" data objects:
            -   Then an event gateway *If insurance necessary*:
                -   If **Yes**: The task *Insure parcel* in the *Logistics* swimlane at the top is triggered, then flow continues to a merge gateway.
                -   If **No**: Directly complete *Write package label*, then proceed to the merge gateway.
    -   Both paths merge at an exclusive gateway, then connect to a parallel gateway to sync with the Warehouse's work.

3.  **Warehouse Workflow**:
    -   Parallel to the Secretary's work: *Package goods*, with a 48-hour timer attached: if the 48-hour mark is hit, trigger *Notify customer about delay*, then sync back to the main packaging flow.
    -   After packaging, flow proceeds to an exclusive gateway, then syncs with the Secretary's process at a parallel gateway.
    -   Final steps: *Fill out Shipment paperwork*  *Move package to pickup area*, ending at the *Shipment prepared* end event.

This diagram maps a coordinated process for shipping hardware goods, combining administrative logistics planning and physical warehouse preparation, with built-in delay handling, parallel work, and conditional insurance/label steps.
