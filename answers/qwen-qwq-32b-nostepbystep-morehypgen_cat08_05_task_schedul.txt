### 1. Analyzing Historical Scheduling Performance and Dynamics  
#### Process Mining Techniques and Metrics  
**Reconstructing Job Flows:**  
- **Process Discovery:** Use process mining tools (e.g., ProM, Disco) to reconstruct the actual job routing paths and task sequences. This reveals deviations from ideal workflows (e.g., unexpected delays at certain machines).  
- **Conformance Checking:** Compare actual execution against "ideal" process models to identify systemic deviations (e.g., frequent task rework or unexpected waiting times).  

**Key Metrics and Techniques:**  
1. **Job Flow & Lead Times:**  
   - Calculate **makespan** (total job duration from release to completion) and **lead time distribution** using timestamps. Identify outliers and trends (e.g., jobs with >50% longer lead times than average).  
   - Cluster jobs by industry/complexity to assess variability in lead times.  

2. **Task Waiting Times:**  
   - Track **queue times** at each machine by analyzing the interval between a job’s "queue entry" and "task start" events. Identify machines with consistently high waiting times (bottlenecks).  

3. **Resource Utilization:**  
   - Compute **machine uptime** (productive time / total time), **idle time** (downtime between tasks), and **setup time variability**. For example:  
     - `Idle time = (Time between task end of Job A and setup start for Job B)`  
     - `Setup duration = Actual setup time vs. planned time for job transitions (e.g., from Job X to Job Y on Machine Z).`  

4. **Sequence-Dependent Setup Times:**  
   - Create a **setup time matrix** for each machine, cross-referencing previous and next jobs. For example, if switching from Job A to Job B on Machine 1 consistently takes 25% longer than average, flag this pair as a high-setup-cost transition.  

5. **Schedule Adherence & Tardiness:**  
   - Calculate **tardiness** as `max(0, completion time - due date)` for each job.  
   - **Critical Path Analysis:** Identify tasks that disproportionately delay downstream operations (e.g., a delayed CNC milling step causing WIP backups).  

6. **Disruption Impact:**  
   - Use **event log correlation** to quantify how disruptions (e.g., breakdowns, hot job insertions) propagate delays. For example, analyze how an hour-long breakdown on `MILL-02` (as in the log snippet) cascades into downstream delays.

### 3. Root Cause Analysis  
**Limitations of Current Approach:**  
- **Static Dispatching Rules:** FCFS/EDD ignore dynamic factors like setup dependencies and machine availability. For example, prioritizing a job with a short due date might require a lengthy setup, delaying others.  
- **Lack of Real-Time Visibility:** Operators can’t re-optimize schedules when breakdowns occur (e.g., `MILL-02` breakdown in the log caused JOB-7001 to wait longer).  
- **Setup Time Ignorance:** The current system likely uses fixed setup estimates, not historical data (e.g., Job A  Job B might have a 20-minute setup, but if the log shows it takes 25 minutes, this error compounds delays.  
- **Isolated Work Centers:** Each machine uses local rules, leading to suboptimal global outcomes (e.g., one machine overloads while others idle).  

**Differentiating Root Causes:**  
- **Capacity Limitations:** If a machine’s bottleneck status persists even with optimized schedules, it may need capacity expansion.  
- **Scheduling Logic Flaws:** If bottlenecks disappear when applying dynamic rules in simulation, the issue is algorithmic.
