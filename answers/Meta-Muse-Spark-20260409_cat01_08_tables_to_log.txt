**Assumptions made:**
1. "Items Picked" is one event per `item_picking_date` in Order Lines. If multiple items were picked at the exact same timestamp, they’re still separate events since they have different `item_id` and `picker_id`.
2. Delivery split into two activities: "Order Dispatched" and "Order Delivered" using `delivery_dispatch_date` and `delivery_arrival_date`.
3. Payment events are linked back to the `order_id` via `invoice_id` to keep the case consistent.
4. Resource names pulled from `Resources` table where available. If no resource is applicable, field left blank.
5. Order 1004 has no delivery, invoice, or payment records, so the case ends at "Items Picked". This reflects incomplete data, not an error.
