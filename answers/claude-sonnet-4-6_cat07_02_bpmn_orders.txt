# Business Process Model and Notation (BPMN) Diagram Description

This is a **collaborative BPMN diagram** showing an order fulfillment process across **four swim lanes**:

---

## Swim Lanes

### 1. **Sales**
- Starts with a **Monday morning** timer event
- Checks order for completeness
- Checks if customer is already in the database
- **Exclusive gateway**: Routes to either:
  - "Add customer to the database" (if not found)
  - Continue flow (if found)
- Ends with **Send order details**  **Draft Invoice**  End event
- Sends "Product sent" notification

### 2. **Production**
- Receives **Order details arrive** message
- **Parallel gateway**  **Plan production**  another parallel gateway
- Leads to **Manufacture product**  "Product complete" event
- Has a loop back for **"Material not faulty"** handling

### 3. **Procurement**
- Receives **Order details arrive** message
- Contains a **Material Compensation Subprocess**:
  - **Order materials**  intermediate event
  - **Return material** (with compensation marker)
- **Check materials** task
- Handles **Order cancellation** via terminate event

### 4. **Warehouse**
- Receives **Product complete** signal
- Executes **Send product** task
- Emits **Product sent** signal back to Sales

---

## Key Features
- **Dashed lines** indicate message flows between pools
- Uses **compensation**, **signal**, and **message events**
- Subprocess notation in Procurement lane
