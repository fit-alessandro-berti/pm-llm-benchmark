###  Reference Solution (DuckDB)
**Approach 1: Window Function (Recommended for readability & performance)**
```sql
SELECT *
FROM (
    SELECT *, COUNT(*) OVER (PARTITION BY case_id) AS case_count
    FROM event_log
)
WHERE case_count <= 6
ORDER BY case_id, timestamp;
```

**Approach 2: `JOIN` with `GROUP BY` + `HAVING`**
```sql
SELECT e.*
FROM event_log e
INNER JOIN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
) c ON e.case_id = c.case_id
ORDER BY e.case_id, e.timestamp;
```
*Both are fully DuckDB-compatible. The window approach avoids a second table scan and is generally preferred in modern SQL.*
