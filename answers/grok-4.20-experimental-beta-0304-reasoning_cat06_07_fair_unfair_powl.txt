```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# ===================================================================
# Common activities
# ===================================================================
receive = Transition(label="ReceiveApplication")
data_check = Transition(label="DataCompletenessCheck")
request_info = Transition(label="RequestMoreInfo")
skill = Transition(label="SkillAssessment")
cultural = Transition(label="CulturalFitCheck")
community = Transition(label="CommunityAffiliationCheck")
managerial = Transition(label="ManagerialReview")
final = Transition(label="FinalDecision")

# ===================================================================
# 1. POWL Model WITH Potential Unfairness
#    After the skill assessment an XOR splits into a normal cultural-fit
#    path OR a privileged CommunityAffiliationCheck path.
# ===================================================================
data_loop_unfair = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])
xor_cultural = OperatorPOWL(operator=Operator.XOR, children=[cultural, community])

unfair_nodes = [receive, data_loop_unfair, skill, xor_cultural, managerial, final]
unfair_model = StrictPartialOrder(nodes=unfair_nodes)

unfair_model.order.add_edge(receive, data_loop_unfair)
unfair_model.order.add_edge(data_loop_unfair, skill)
unfair_model.order.add_edge(skill, xor_cultural)
unfair_model.order.add_edge(xor_cultural, managerial)
unfair_model.order.add_edge(managerial, final)

# ===================================================================
# 2. POWL Model WITHOUT Unfairness
#    All applicants follow the identical CulturalFitCheck path;
#    the discriminatory XOR branch has been removed.
# ===================================================================
data_loop_fair = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

fair_nodes = [receive, data_loop_fair, skill, cultural, managerial, final]
fair_model = StrictPartialOrder(nodes=fair_nodes)

fair_model.order.add_edge(receive, data_loop_fair)
fair_model.order.add_edge(data_loop_fair, skill)
fair_model.order.add_edge(skill, cultural)
fair_model.order.add_edge(cultural, managerial)
fair_model.order.add_edge(managerial, final)
```

**Explanation of the two models**

* **Unfair model**: After `SkillAssessment` an `XOR` operator routes the case either to the ordinary `CulturalFitCheck` or to the `CommunityAffiliationCheck` activity. The latter path can introduce the subtle preferential treatment described in the scenario. The data-completeness loop (`* (DataCompletenessCheck, RequestMoreInfo)`) and the overall sequential order are expressed with a `StrictPartialOrder`.
* **Fair model**: The same overall structure is kept (loop for data completeness, skill assessment, managerial review, final decision) but the discriminatory `XOR` has been replaced by a single `CulturalFitCheck` activity that every applicant traverses. Consequently the model contains no control-flow construct that can give an advantage based on community affiliation. 

Both models are valid POWL objects that can be used with the `pm4py` library exactly as the example in the query.
