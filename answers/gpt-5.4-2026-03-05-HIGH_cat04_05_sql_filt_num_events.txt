```sql
WITH valid_cases AS (
    SELECT
        case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT
    e.*
FROM event_log AS e
JOIN valid_cases AS v
    ON e.case_id = v.case_id
ORDER BY e.case_id, e.timestamp;
```

This works as follows:

- `GROUP BY case_id` collects events by case.
- `HAVING COUNT(*) <= 6` keeps only cases with six or fewer events.
- The `JOIN` brings back all original event rows for those valid cases.

If you want a shorter DuckDB-specific version, you can also use a window function:

```sql
SELECT *
FROM (
    SELECT
        *,
        COUNT(*) OVER (PARTITION BY case_id) AS case_event_count
    FROM event_log
) t
WHERE case_event_count <= 6
ORDER BY case_id, timestamp;
```

This returns the same result, though the first version is usually cleaner if you do not need the count in the output.
