# Accepting Petri Net for Insurance Claim Management Process

Here's the Python code to construct the accepting Petri net for this insurance claim management process:

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# Create the Petri net
net = PetriNet(name="insurance_claim_management")

# -------------------------------------------------------------------
# PLACES (states between activities)
# -------------------------------------------------------------------
p_start = PetriNet.Place(name="p_start")        # new claim arrives
p_after_C = PetriNet.Place(name="p_after_C")    # after claim filed
p_after_CR = PetriNet.Place(name="p_after_CR")  # after claim registration
p_after_DV = PetriNet.Place(name="p_after_DV")  # after document verification
p_after_FC = PetriNet.Place(name="p_after_FC")  # after fraud check
p_after_CA = PetriNet.Place(name="p_after_CA")  # after claim assessment
p_after_AP = PetriNet.Place(name="p_after_AP")  # after approval
p_after_P = PetriNet.Place(name="p_after_P")    # after payment
p_after_N = PetriNet.Place(name="p_after_N")    # after notification
p_end = PetriNet.Place(name="p_end")            # claim closed (final)

# Add places to the net
for p in [p_start, p_after_C, p_after_CR, p_after_DV, p_after_FC,
          p_after_CA, p_after_AP, p_after_P, p_after_N, p_end]:
    net.places.add(p)

# -------------------------------------------------------------------
# TRANSITIONS (activities)
# -------------------------------------------------------------------
t_C = PetriNet.Transition(name="C", label="Insurance Claim")
t_CR = PetriNet.Transition(name="CR", label="Claim Registration")
t_DV = PetriNet.Transition(name="DV", label="Document Verification")
t_RDS = PetriNet.Transition(name="RDS", label="Re-Document Submission")
t_FC = PetriNet.Transition(name="FC", label="Fraud Check")
t_II = PetriNet.Transition(name="II", label="In-depth Investigation")
t_CA = PetriNet.Transition(name="CA", label="Claim Assessment")
t_AP = PetriNet.Transition(name="AP", label="Approval")
t_P = PetriNet.Transition(name="P", label="Payment")
t_N = PetriNet.Transition(name="N", label="Notification")
t_CL = PetriNet.Transition(name="CL", label="Closure")

# Add transitions to the net
for t in [t_C, t_CR, t_DV, t_RDS, t_FC, t_II, t_CA, t_AP, t_P, t_N, t_CL]:
    net.transitions.add(t)

# -------------------------------------------------------------------
# ARCS (logical sequence + loops)
# -------------------------------------------------------------------
# Main sequential flow
petri_utils.add_arc_from_to(p_start, t_C, net)
petri_utils.add_arc_from_to(t_C, p_after_C, net)

petri_utils.add_arc_from_to(p_after_C, t_CR, net)
petri_utils.add_arc_from_to(t_CR, p_after_CR, net)

petri_utils.add_arc_from_to(p_after_CR, t_DV, net)
petri_utils.add_arc_from_to(t_DV, p_after_DV, net)

# --- Loop 1: Re-Document Submission (DV <-> RDS) ---
# If documents incomplete after DV, perform RDS, then go back to DV
petri_utils.add_arc_from_to(p_after_DV, t_RDS, net)
petri_utils.add_arc_from_to(t_RDS, p_after_CR, net)   # back to redo DV

# Continue: from DV result to Fraud Check
petri_utils.add_arc_from_to(p_after_DV, t_FC, net)
petri_utils.add_arc_from_to(t_FC, p_after_FC, net)

# --- Loop 2: In-depth Investigation (FC <-> II, possibly multiple times) ---
# If fraud check raises doubts, perform II, then re-run FC
petri_utils.add_arc_from_to(p_after_FC, t_II, net)
petri_utils.add_arc_from_to(t_II, p_after_DV, net)    # back to redo FC

# Continue: Claim Assessment
petri_utils.add_arc_from_to(p_after_FC, t_CA, net)
petri_utils.add_arc_from_to(t_CA, p_after_CA, net)

# Approval
petri_utils.add_arc_from_to(p_after_CA, t_AP, net)
petri_utils.add_arc_from_to(t_AP, p_after_AP, net)

# Payment
petri_utils.add_arc_from_to(p_after_AP, t_P, net)
petri_utils.add_arc_from_to(t_P, p_after_P, net)

# Notification
petri_utils.add_arc_from_to(p_after_P, t_N, net)
petri_utils.add_arc_from_to(t_N, p_after_N, net)

# Closure -> final
petri_utils.add_arc_from_to(p_after_N, t_CL, net)
petri_utils.add_arc_from_to(t_CL, p_end, net)

# -------------------------------------------------------------------
# INITIAL AND FINAL MARKINGS
# -------------------------------------------------------------------
im = Marking()
im[p_start] = 1     # a new claim arrives

fm = Marking()
fm[p_end] = 1       # the claim is closed

# 'net', 'im', 'fm' together form the accepting Petri net
```

## Explanation of the Design

**Sequential flow:**  
`C  CR  DV  FC  CA  AP  P  N  CL`, with one place between each consecutive pair of activities, encoding the logical ordering.

**Loop 1 — Re-Document Submission (RDS):**  
After `DV` deposits a token in `p_after_DV`, two transitions compete:
- `FC` continues the process, OR
- `RDS` fires (documents incomplete), returning the token to `p_after_CR` so that `DV` runs again. This naturally allows **repeated re-submissions**.

**Loop 2 — In-depth Investigation (II):**  
After `FC` deposits a token in `p_after_FC`, two transitions compete:
- `CA` continues the process, OR
- `II` fires (fraud doubts), returning the token to `p_after_DV` so that `FC` runs again. This allows **multiple in-depth investigations** before proceeding to assessment.

**Markings:**
- `im` places one token in `p_start` (a new claim arriving).
- `fm` requires one token in `p_end` (a closed/archived claim).

This makes the model an **accepting Petri net** that recognizes valid execution sequences of the insurance claim process, including the optional repeatable loops.
