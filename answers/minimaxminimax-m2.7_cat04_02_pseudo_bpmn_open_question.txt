**3.  Predictive analytics ¢ ōwho needs a custom lane?ö**

* **Model training**  
  *Historical data*: request attributes (product category, price, quantity, customer tier, past customrequest flag), external signals (seasonality, market trends), and outcome (custom needed? approval needed?).  
  *Algorithms*: gradientboosted trees (XGBoost) for interpretability + a lightweight LSTM if temporal patterns matter.  

* **Score usage**  
  * At intake (TaskA) the model outputs a *complexity score*.  
  * If score0.35, the request is routed **directly** to the ōDynamic Customö lane, bypassing the standard validation loop.  
  * If 0.2score<0.35, the ōHybridö lane runs both streams in parallel, merging when the first finishes.  

* **Feedback loop**  
  * Each completed requestÆs *actual* customflag and approvaloutcome are fed back to the training set (daily batch).  
  * Model performance (AUC, precision) is monitored; a drift >5% triggers a retraining job.
