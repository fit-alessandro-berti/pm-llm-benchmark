**Event Log for Process Mining**

**Assumptions made:**
- Case ID = `order_id`.
- Activities derived logically: "Order Created" (from Orders), "Item Picked" (one event per order line), "Order Dispatched" / "Order Arrived" (one pair per delivery record), "Invoice Issued" (per invoice), "Payment Received" (per payment).
- Resources use full names from the Resources table where available (sales persons, warehouse staff, carriers for dispatch/arrival, billing clerks). Payments have no specific actor  marked as "-".
- One row per granular event (multiple picks/deliveries/invoices per case are kept separate because they have distinct timestamps/resources/items).
- Events are listed for all orders; order 1004 has no delivery/invoice/payment data.
- Table is sorted by Case ID, then by Timestamp (chronological within each case).
- Additional identifiers (Item ID, Delivery ID, etc.) are included for traceability.
