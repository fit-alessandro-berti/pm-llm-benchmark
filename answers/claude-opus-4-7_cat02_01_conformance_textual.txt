# Analysis of Event Log Deviations

## Case-by-Case Evaluation

### Case 1:  Compliant
Sequence: Receive  Check  Authorize  Package  Ship  Close
This matches the normative "sufficient inventory" flow exactly.

### Case 2:  Violation
Sequence: Receive  Check  **Package  Authorize**  Ship  Close

**Violation:** "Package items" occurs *before* "Authorize payment." The normative description explicitly states:
- "Authorize payment" should occur "before packaging"
- "Package items" should "only occur after authorization"

The packaging and authorization steps are swapped, which violates the required ordering.

### Case 3:  Violation
Sequence: Receive  Check  Authorize  **Cancel  Ship**  Close

**Multiple violations:**
1. **Authorize payment before Cancel:** If inventory was insufficient (leading to cancellation), the process should have gone directly from "Check inventory"  "Cancel order." Authorizing payment in a cancellation branch is not part of the normative flow (the cancellation path is Receive  Check  Cancel  Close).
2. **Ship order after Cancel order:** The description explicitly states that "Ship order" should "never [occur] after a cancellation." Shipping a canceled order is one of the clearest violations possible.

### Case 4:  Compliant
Sequence: Receive  Check  Cancel  Close
This matches the normative "insufficient inventory" flow exactly.

### Case 5:  Violation
Sequence: Receive  Check  Authorize  **Ship**  Close (no "Package items")

**Violation:** "Package items" is missing entirely. The normative description requires that:
- "Package items" must occur after authorization and before shipping
- "Ship order" should only occur *after packaging*

Shipping without first packaging the items violates the mandatory precedence relationship between these two activities.
