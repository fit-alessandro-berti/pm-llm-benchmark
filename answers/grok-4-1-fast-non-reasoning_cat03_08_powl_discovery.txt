```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Basic activities
OR = Transition(label="OR")
MS = Transition(label="MS") 
QS = Transition(label="QS")
AS = Transition(label="AS")
IQC = Transition(label="IQC")
TST = Transition(label="TST")
RT = Transition(label="RT")
PK = Transition(label="PK")
LB = Transition(label="LB")
DP = Transition(label="DP")
D = Transition(label="D")

skip = SilentTransition()

# IQC loop: AS -> IQC -> (skip AS or AS again) - models repeated IQC-AS pairs
iqc_loop = OperatorPOWL(operator=Operator.LOOP, children=[IQC, AS])

# Testing loop: TST -> (skip or RT -> TST again)
test_loop = OperatorPOWL(operator=Operator.LOOP, children=[TST, OperatorPOWL(operator=Operator.XOR, children=[RT, skip])])

# Optional Labeling: choice between LB or skip
labeling_choice = OperatorPOWL(operator=Operator.XOR, children=[LB, skip])

# AS-IQC sequence with IQC loop (captures case 4 where IQC is skipped by just doing AS once)
as_iqc_seq = StrictPartialOrder(nodes=[AS, iqc_loop])
as_iqc_seq.order.add_edge(AS, iqc_loop)

# Assembly-QC-Testing partial order (AS can lead to testing directly or through IQC loop)
assembly_testing_po = StrictPartialOrder(nodes=[as_iqc_seq, test_loop])
assembly_testing_po.order.add_edge(as_iqc_seq, test_loop)

# Final packaging-delivery partial order (PK || labeling_choice -> DP -> D)
final_po = StrictPartialOrder(nodes=[PK, labeling_choice, DP, D])
final_po.order.add_edge(PK, DP)
final_po.order.add_edge(labeling_choice, DP)
final_po.order.add_edge(DP, D)

# Root structure: sequential backbone with concurrent/ordered substructures
root = StrictPartialOrder(nodes=[OR, MS, QS, assembly_testing_po, final_po])

# Define the strict partial order relationships
root.order.add_edge(OR, MS)
root.order.add_edge(MS, QS) 
root.order.add_edge(QS, assembly_testing_po)
root.order.add_edge(assembly_testing_po, final_po)

print("POWL model constructed successfully!")
print("Structure summary:")
print("- Sequential backbone: OR  MS  QS  [AS-IQC-Testing]  [PK||LBDPD]")
print("- IQC loop handles repeated quality checks with AS")
print("- Testing loop handles re-testing via RT")
print("- Labeling is optional via XOR with silent transition")
print("- Case 4 captured by single AS execution (skipping IQC loop body)")
```

This POWL model captures all observed behaviors:

1. **Sequential backbone**: ORMSQSassemblyfinal_delivery
2. **IQC loop**: Models repeated `ASIQC` pairs (cases 1,3,5), with case 4 captured by single initial AS
3. **Testing loop**: Models `TST(RTTST)*` retesting (cases 1,3,5), with case 2 captured by single TST
4. **Optional labeling**: XOR(LB, ) captures presence/absence of LB
5. **Concurrency**: PK and LB can occur in either order (both before DP)
6. **Strict ordering**: All causal dependencies preserved

The model handles all 5 traces exactly while remaining compact and generalizable.
