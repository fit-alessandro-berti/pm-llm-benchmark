Below is the Python code to construct the accepting Petri net for the insurance claim management process, including the loops for re-submission and in-depth investigation. The net includes all the described activities, places, and transitions with appropriate arcs and markings.

### Petri Net Construction:
```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# Create the Petri net
net = PetriNet(name="Insurance Claim Management Process")

# Define places
start = PetriNet.Place(name="start")
insurance_claim = PetriNet.Place(name="insurance_claim")
claim_registered = PetriNet.Place(name="claim_registered")
documents_verified = PetriNet.Place(name="documents_verified")
fraud_check_passed = PetriNet.Place(name="fraud_check_passed")
fraud_check_failed = PetriNet.Place(name="fraud_check_failed")
claim_assessed = PetriNet.Place(name="claim_assessed")
approval_required = PetriNet.Place(name="approval_required")
payment_processed = PetriNet.Place(name="payment_processed")
notification_sent = PetriNet.Place(name="notification_sent")
closed = PetriNet.Place(name="closed")

# Re-submission loop places
documents_incomplete = PetriNet.Place(name="documents_incomplete")
re_document_submission = PetriNet.Place(name="re_document_submission")

# In-depth investigation loop places
investigation_required = PetriNet.Place(name="investigation_required")
in_depth_investigation = PetriNet.Place(name="in_depth_investigation")

# Add all places to the net
net.places.add(start)
net.places.add(insurance_claim)
net.places.add(claim_registered)
net.places.add(documents_verified)
net.places.add(documents_incomplete)
net.places.add(re_document_submission)
net.places.add(fraud_check_passed)
net.places.add(fraud_check_failed)
net.places.add(investigation_required)
net.places.add(in_depth_investigation)
net.places.add(claim_assessed)
net.places.add(approval_required)
net.places.add(payment_processed)
net.places.add(notification_sent)
net.places.add(closed)

# Define transitions
transitions = [
    PetriNet.Transition(name="Insurance Claim (C)", label="Insurance Claim"),
    PetriNet.Transition(name="Claim Registration (CR)", label="Claim Registration"),
    PetriNet.Transition(name="Document Verification (DV)", label="Document Verification"),
    PetriNet.Transition(name="Re-Document Submission (RDS)", label="Re-Document Submission"),
    PetriNet.Transition(name="Fraud Check (FC)", label="Fraud Check"),
    PetriNet.Transition(name="In-depth Investigation (II)", label="In-depth Investigation"),
    PetriNet.Transition(name="Claim Assessment (CA)", label="Claim Assessment"),
    PetriNet.Transition(name="Approval (AP)", label="Approval"),
    PetriNet.Transition(name="Payment (P)", label="Payment"),
    PetriNet.Transition(name="Notification (N)", label="Notification"),
]

net.transitions.add(*transitions)

# Add arcs for the main process flow
petri_utils.add_arc_from_to(start, insurance_claim, net)
petri_utils.add_arc_from_to(insurance_claim, claim_registered, net)
petri_utils.add_arc_from_to(claim_registered, documents_verified, net)
petri_utils.add_arc_from_to(documents_verified, documents_incomplete, net)  # Loop for RDS
petri_utils.add_arc_from_to(documents_incomplete, re_document_submission, net)
petri_utils.add_arc_from_to(re_document_submission, documents_verified, net)  # Re-submission loop
petri_utils.add_arc_from_to(documents_verified, fraud_check_passed, net)
petri_utils.add_arc_from_to(fraud_check_passed, claim_assessed, net)
petri_utils.add_arc_from_to(fraud_check_failed, investigation_required, net)  # Loop for II
petri_utils.add_arc_from_to(investigation_required, in_depth_investigation, net)
petri_utils.add_arc_from_to(in_depth_investigation, fraud_check_passed, net)  # Investigation loop
petri_utils.add_arc_from_to(claim_assessed, approval_required, net)
petri_utils.add_arc_from_to(approval_required, payment_processed, net)
petri_utils.add_arc_from_to(payment_processed, notification_sent, net)
petri_utils.add_arc_from_to(notification_sent, closed, net)

# Initial marking (im): Start with a token in 'start' and 'insurance_claim'
im = Marking()
im[start] = 1
im[insurance_claim] = 1

# Final marking (fm): Tokens in 'closed' only
fm = Marking()
fm[closed] = 1

# Remove unnecessary places (optional, if not needed)
# petri_utils.remove_place(net, insurance_claim)  # Uncomment if not needed
```

### Explanation:
1. **Places**:
   - `start` and `insurance_claim` represent the initial state (new claim).
   - `closed` represents the final state (closed claim).
   - Loop places (`documents_incomplete`, `re_document_submission`, `investigation_required`, `in_depth_investigation`) handle re-submission and in-depth investigation.

2. **Transitions**:
   Each transition is labeled with the activity name (e.g., `Claim Registration (CR)`).

3. **Arcs**:
   - The main flow connects places sequentially (e.g., `Claim Registration`  `Document Verification`).
   - Loops are created for re-submission (`DV`  `RDS`  `DV`) and in-depth investigation (`FC`  `II`  `FC`).

4. **Initial Marking (`im`)**:
   - Tokens are placed in `start` and `insurance_claim` to represent a new claim.

5. **Final Marking (`fm`)**:
   - A token is placed in `closed` to represent a fully processed claim.

### Notes:
- The net can be extended further (e.g., adding conditions for approval thresholds or parallel paths).
- The `remove_place` calls are commented out since `insurance_claim` is needed for the initial marking. Uncomment if you want to simplify the net.
