```sql
WITH eligible_cases AS (
    SELECT
        case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT
    e.*
FROM event_log AS e
JOIN eligible_cases AS c
    ON e.case_id = c.case_id
ORDER BY
    e.case_id,
    e."timestamp";
```

This counts events per `case_id`, keeps only cases with six or fewer events using `HAVING COUNT(*) <= 6`, and then joins back to `event_log` so the full event rows for those eligible cases are returned.

A DuckDB window-function version would also work:

```sql
SELECT * EXCLUDE (event_count)
FROM (
    SELECT
        e.*,
        COUNT(*) OVER (PARTITION BY case_id) AS event_count
    FROM event_log AS e
) AS counted_events
WHERE event_count <= 6
ORDER BY
    case_id,
    "timestamp";
```
