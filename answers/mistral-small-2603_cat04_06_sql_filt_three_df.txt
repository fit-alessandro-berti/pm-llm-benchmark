# DuckDB SQL Query to Exclude Cases with Specific Activity Sequence and Timing

Here's a DuckDB SQL query that identifies and excludes cases containing the specific activity sequence with the timing condition:

```sql
WITH
-- First, identify all cases that have the exact sequence of activities in order
sequenced_cases AS (
    SELECT DISTINCT case_id
    FROM (
        SELECT
            case_id,
            activity,
            LAG(activity) OVER (PARTITION BY case_id ORDER BY timestamp) AS prev_activity,
            LAG(timestamp) OVER (PARTITION BY case_id ORDER BY timestamp) AS prev_timestamp
        FROM event_log
    )
    WHERE
        activity = 'Confirm Order' AND
        prev_activity = 'Create Order' AND
        prev_timestamp < timestamp - INTERVAL 5 DAY
),

-- Then find cases that have the exact sequence (regardless of timing)
exact_sequence_cases AS (
    SELECT DISTINCT case_id
    FROM (
        SELECT
            case_id,
            activity,
            LAG(activity) OVER (PARTITION BY case_id ORDER BY timestamp) AS prev_activity,
            LAG(LAG(activity) OVER (PARTITION BY case_id ORDER BY timestamp))
                OVER (PARTITION BY case_id ORDER BY timestamp) AS prev_prev_activity
        FROM event_log
    )
    WHERE
        activity = 'Confirm Order' AND
        prev_activity = 'Create Order' AND
        prev_prev_activity = 'Approve Purchase Requisition'
),

-- Combine both conditions to get cases that match both the sequence and timing
cases_to_exclude AS (
    SELECT case_id FROM sequenced_cases
    INTERSECT
    SELECT case_id FROM exact_sequence_cases
)

-- Finally, return all events except those from the excluded cases
SELECT *
FROM event_log
WHERE case_id NOT IN (SELECT case_id FROM cases_to_exclude)
ORDER BY case_id, timestamp;
```

## Explanation:

1. **sequenced_cases CTE**:
   - Identifies cases where 'Confirm Order' follows 'Create Order' with more than 5 days gap
   - Uses window functions to compare each event with its immediate predecessor

2. **exact_sequence_cases CTE**:
   - Identifies cases that have the exact sequence of three activities in order
   - Uses nested window functions to look two steps back in the sequence

3. **cases_to_exclude CTE**:
   - Combines both conditions using INTERSECT to find cases that meet both criteria

4. **Final Query**:
   - Returns all events from cases that don't match the exclusion criteria
   - Results are ordered by case_id and timestamp for readability

This query ensures we only return events from cases that either:
- Don't contain the exact sequence of activities, or
- Contain the sequence but with 5 days between 'Create Order' and 'Confirm Order'
