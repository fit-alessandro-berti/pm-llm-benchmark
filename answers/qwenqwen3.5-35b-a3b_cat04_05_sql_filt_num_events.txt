# DuckDB SQL Query: Filter Cases with Six or Fewer Events

To return only the events from cases that contain six or fewer events in total, use the following DuckDB SQL query. This solution uses a window function to efficiently calculate event counts per case and filter accordingly.

```sql
SELECT *
FROM (
    SELECT *,
           COUNT(*) OVER (PARTITION BY case_id) AS event_count
    FROM event_log
) AS subquery
WHERE event_count <= 6;
```

## Explanation

1. **Window Function:** `COUNT(*) OVER (PARTITION BY case_id)` computes the total number of events for each `case_id` across the entire table.
2. **Subquery:** Wraps the result so the calculated `event_count` can be filtered in the outer query.
3. **Filter:** The `WHERE event_count <= 6` clause ensures only events from cases with six or fewer events are returned.

## Alternative: CTE-Based Approach

For improved readability, you can also use a Common Table Expression (CTE):

```sql
WITH case_counts AS (
    SELECT 
        case_id,
        COUNT(*) AS event_count
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT el.*
FROM event_log el
INNER JOIN case_counts cc ON el.case_id = cc.case_id;
```

## Performance Note

- **Window Function Approach:** Typically more performant in DuckDB as it avoids an explicit join and can be optimized more efficiently.
- **CTE Approach:** More explicit and easier to understand, useful for complex or educational contexts.

Both queries are valid; choose based on your preference for performance or clarity.
