# Process Flow Visualization Description

This visualization is a **process mining / business process flowchart** that depicts multiple interrelated procurement and order management workflows. It shows the lifecycle of different business object types and how they flow through various process steps.

## Object Types (Data Sources)

The diagram tracks **five distinct object types**, shown at the top:

| Object Type | Volume |
|---|---|
| **PurchaseRequisitionItem** | 2.0M objects |
| **SalesOrderItem** | 3.5M objects |
| **PurchaseOrderItem** | 5.0M objects |
| **VendorInvoiceItem** | 1.2M objects |
| **ReturnOrderItem** | 0.8M objects |

Each object type is color-coded with distinct colored paths (purple, orange, blue, green, red/pink) to trace its journey through the process.

## Key Process Activities

The flowchart includes these major steps with their overall volumes:

- **Create Requisition** (3.0M)  **Approve Requisition** (5.7M)
- **Create Order** (8.5M)  **Pack Goods** (2.0M)  **Ship Goods** (1.8M)  **Confirm Delivery** (1.7M)
- **Send Order** (5.2M), **Check Stock** (1.0M), **Process Return** (0.8M)
- **Approve Payment** (1.5M)  **Create Invoice** (2.0M)
- **Close Order** (1.7M)

## Flow Characteristics

- **Transition times** are annotated on edges (e.g., 300ms, 450ms, 600ms)
- Multiple **Start** and **End** nodes indicate parallel process entry/exit points
- **Feedback loops** exist (e.g., from Process Return back to Approve Requisition)
- The colored dots on each activity box indicate which object types participate in that step

## Overall Pattern

This represents an **end-to-end procure-to-pay and order-to-cash process**, showing how requisitions, purchase orders, sales orders, returns, and invoices converge and diverge through shared and distinct process steps.
