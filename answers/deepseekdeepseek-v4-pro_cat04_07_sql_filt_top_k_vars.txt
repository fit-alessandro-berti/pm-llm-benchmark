Below is a DuckDB SQL query that fulfills the requirements. It uses CTEs to build the case sequences, identify unique variants, count cases per variant, determine the top K by frequency, and finally return only the events from cases of those top variants.

**Key points**:
- The activity sequence is obtained with `list(activity ORDER BY timestamp)`, which respects the chronological order.
- Variants are compared based on the entire ordered list.
- The top K variants are selected using `DENSE_RANK()` to include all variants with the same count as the Kth rank (i.e., ties are kept). If you strictly want at most K *distinct* variants, replace `DENSE_RANK()` with `ROW_NUMBER()`.
- The parameter `K` is used in the `WHERE rnk <= K` clause. Set it to the desired integer value (here set as `3` for illustration).

```sql
-- Set the number of top variants you want to keep
-- (Replace 3 with your K value)
WITH
-- 1. Build the ordered sequence of activities for each case
case_sequences AS (
    SELECT
        case_id,
        list(activity ORDER BY timestamp) AS variant_seq
    FROM event_log
    GROUP BY case_id
),

-- 2. Count how many cases correspond to each unique variant sequence
variant_counts AS (
    SELECT
        variant_seq,
        count(*) AS case_count
    FROM case_sequences
    GROUP BY variant_seq
),

-- 3. Determine top K variants by frequency (including ties using DENSE_RANK)
top_variants AS (
    SELECT
        variant_seq,
        case_count,
        dense_rank() OVER (ORDER BY case_count DESC) AS rnk
    FROM variant_counts
    WHERE rnk <= 3   -- <--- replace 3 with your K
),

-- 4. Get the case_ids that belong to those top variants
variant_cases AS (
    SELECT cs.case_id
    FROM case_sequences cs
    INNER JOIN top_variants tv ON cs.variant_seq = tv.variant_seq
)

-- 5. Return all original events only for the selected cases
SELECT e.*
FROM event_log e
INNER JOIN variant_cases vc ON e.case_id = vc.case_id
ORDER BY e.case_id, e.timestamp;   -- optional ordering
```

**Adjusting the strictness of the top K**:
- Using `DENSE_RANK()` will keep **all variants** whose rank is  K, so more than K variants may be returned if counts tie around the Kth boundary.
- If you want **exactly K variants** and don’t care about ties, replace `dense_rank()` with `row_number()`:
  ```sql
  row_number() OVER (ORDER BY case_count DESC) AS rnk
  ```

Simply change the literal `3` to your chosen `K`.
