### Key Design Notes:
1. **Place-Transition Structure**: Each activity is a transition, and each state between activities is a place. This aligns with standard Petri net semantics where tokens flow through places.
2. **Loops via Arcs**: 
   - `p_dv`  `t_rds`  `p_dv` creates a loop allowing document re-submission indefinitely until documents pass verification.
   - `p_fc`  `t_ii`  `p_fc` creates a loop for repeated in-depth investigations.
3. **Branching**: From `p_dv`, a token can fire either `t_rds` or `t_fc`. From `p_fc`, it can fire either `t_ii` or `t_ca`. Petri nets handle this non-deterministically based on execution logs or simulation rules.
4. **Accepting Net**: The initial marking `im` places a token in `p_start`, and the final marking `fm` places a token in `p_end`, satisfying the definition of an accepting Petri net.
