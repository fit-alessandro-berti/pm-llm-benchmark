# BPMN Process Diagram Description

This is a **Business Process Model and Notation (BPMN)** diagram depicting an **order fulfillment process** organized into four horizontal swimlanes representing different departments:

## Swimlanes/Departments:

### 1. **Sales** (Top Lane)
- Process starts with a **timer event** ("Monday morning")
- Tasks include: checking order completeness, verifying customer in database
- Contains a subprocess for adding new customers to the database
- Ends with sending order details and drafting an invoice

### 2. **Production** (Second Lane)
- Triggered by receiving order details (message event)
- Uses a **parallel gateway** to split activities
- Includes: planning production and manufacturing the product
- Has a conditional check for material quality ("Material not faulty")

### 3. **Procurement** (Third Lane)
- Contains a **Material Compensation Subprocess**
- Tasks: ordering materials, checking materials, with compensation handling for returning materials
- Includes error handling for order cancellation

### 4. **Warehouse** (Bottom Lane)
- Triggered when product is complete
- Single task: "Send product"
- Ends with "Product sent" event

## Key Features:
- **Dashed lines**: Message flows between departments
- **Gateways**: Decision points (X) and parallel splits (+)
- **Events**: Timer, message, and compensation events
- Shows cross-departmental coordination in order processing
