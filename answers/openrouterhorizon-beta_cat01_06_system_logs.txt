  - CASE_QTR_REPORT: Word document “Quarterly_Report.docx” work spanning initial focus at 08:59:50 and later refocus to completion and close.

Activity naming:
- Low-level actions mapped to standardized activities:
  - FOCUS/initial open -> Open Document/Spreadsheet/PDF; repeated focus later -> Refocus/Return/Re-reference as applicable.
  - TYPING in Word -> Edit Content; in Excel -> Edit Spreadsheet; in Email -> Compose Email.
  - SAVE -> Save Document/Spreadsheet.
  - SWITCH used to indicate transitions; included as “Switch to Email” or “Re-reference Document” when relevant to the same case continuity.
  - CLICK “Open Email” -> Open Email Thread; “Reply” -> Start Reply; “Send Email” -> Send Email.
  - SCROLL in PDF/Email -> Review/Browse; HIGHLIGHT -> Annotate PDF.
  - CLOSE -> Close Document.

Event attributes:
- Case ID ties events to an artifact-based case.
- Activity Name standardized for process analysis.
- Timestamp taken from original log; App and Window preserved for traceability; Details capture typed text or action context.

This produces coherent, artifact-centric traces suitable for process mining tools (e.g., XES/CSV import), where each case forms a clear start-to-completion mini-process.
