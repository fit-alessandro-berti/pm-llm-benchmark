###  **1. POWL Model with Potential Unfairness (Bias-Prone Path)**

This model includes an **XOR branch** after the skill assessment, where applicants are either evaluated via a standard **CulturalFitCheck** or receive a subtle advantage through a **CommunityAffiliationCheck** — introducing a potential source of bias.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Loop for data completeness (if missing info)
loop_data = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck, RequestMoreInfo])

# Sequence: ReceiveApplication  loop_data  SkillAssessment
step1 = StrictPartialOrder(nodes=[ReceiveApplication, loop_data, SkillAssessment])
step1.order.add_edge(ReceiveApplication, loop_data)
step1.order.add_edge(loop_data, SkillAssessment)

# XOR choice after SkillAssessment: either standard cultural fit or community-affiliation path
xor_branch = OperatorPOWL(
    operator=Operator.XOR,
    children=[CulturalFitCheck, CommunityAffiliationCheck]
)

# Final sequence: XOR branch  ManagerialReview  FinalDecision
step2 = StrictPartialOrder(nodes=[xor_branch, ManagerialReview, FinalDecision])
step2.order.add_edge(xor_branch, ManagerialReview)
step2.order.add_edge(ManagerialReview, FinalDecision)

# Combine the entire workflow: step1 (pre-skills)  step2 (post-skills)
root = StrictPartialOrder(nodes=[step1, step2])
root.order.add_edge(step1, step2)

# Optional: Explicitly add the dependency between the end of step1 and the start of step2
# (already handled by the order above)
```

>  **Why this model is unfair**:  
> The **XOR branch** allows applicants to take **two different paths** after passing the skill assessment. One path is neutral (CulturalFitCheck), but the other gives **implicit advantage** via **CommunityAffiliationCheck**, which may lead to higher subjective scoring. Since this choice is not based on merit but on external affiliations, it introduces **unfair bias** — especially if the affiliation is not relevant to job performance.

###  **2. POWL Model Without Unfairness (Fair, Unified Evaluation)**

This model removes the **XOR branch** and ensures **all applicants go through the same cultural fit evaluation**, regardless of background. The **CommunityAffiliationCheck** is eliminated entirely, and no special path exists. This maintains **equal treatment** and eliminates the potential for bias.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities (same as before, but no CommunityAffiliationCheck)
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Loop for data completeness (same as before)
loop_data = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck, RequestMoreInfo])

# Sequence: ReceiveApplication  loop_data  SkillAssessment  CulturalFitCheck  ManagerialReview  FinalDecision
step1 = StrictPartialOrder(nodes=[ReceiveApplication, loop_data, SkillAssessment, CulturalFitCheck, ManagerialReview, FinalDecision])

# Add sequential dependencies
step1.order.add_edge(ReceiveApplication, loop_data)
step1.order.add_edge(loop_data, SkillAssessment)
step1.order.add_edge(SkillAssessment, CulturalFitCheck)
step1.order.add_edge(CulturalFitCheck, ManagerialReview)
step1.order.add_edge(ManagerialReview, FinalDecision)

# No XOR, no special paths — all applicants follow the same path
root = step1
```

>  **Why this model is fair**:  
> - All applicants go through **identical steps**.
> - **No branching** based on affiliations or local residency.
> - The **CulturalFitCheck** is applied uniformly to everyone, ensuring **merit-based evaluation**.
> - Eliminates the **potential for subjective uplift** due to community ties, thus removing a key source of bias.

###  Recommendation:
Use **Model 2** in real-world implementations to ensure **fair, transparent, and auditable hiring**.
