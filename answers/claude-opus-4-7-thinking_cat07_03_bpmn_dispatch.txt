# BPMN Diagram: Dispatch of Goods - Computer Hardware Shop

This is a **Business Process Model and Notation (BPMN)** diagram illustrating the goods dispatch process for a computer hardware shop. The diagram is organized into three horizontal swim lanes representing different departments:

## Swim Lanes (Roles)
1. **Logistics**
2. **Secretary**
3. **Warehouse**

## Process Flow

### Start
- The process begins with a **"Ship goods"** start event in the Secretary lane.

### Parallel Split
- A **parallel gateway (AND)** splits the flow into two concurrent paths:
  - One path continues in the Secretary lane
  - The other goes down to the Warehouse lane

### Secretary Path
1. **Clarify shipment method**
2. **Exclusive gateway (XOR)**: "Special handling?"
   - **Yes**  Get 3 offers from logistic companies  Select logistic company and place order
   - **No**  Check if insurance is required (uses "Order details" document and "Customer information" data store)
3. **Inclusive gateway (OR)**: 
   - "Always"  Write package label
   - "If insurance necessary"  Insure parcel (in Logistics lane), then rejoin
4. Paths merge at another **inclusive gateway**, then converge at an **exclusive gateway**

### Warehouse Path
1. **Package goods** (with a 48-hour timer boundary event)
   - If delayed  **Notify customer about delay**
2. Paths merge at an **inclusive gateway**

### Final Convergence
- A **parallel gateway** joins both the Secretary and Warehouse paths
- **Fill out Shipment paperwork**
- **Move package to pickup area**
- Ends with **"Shipment prepared"** end event

## Key BPMN Elements Used
- Start/End events (circles)
- Tasks (rounded rectangles)
- Gateways: Exclusive (X), Inclusive (O), Parallel (+)
- Timer event (clock icon)
- Data object (Order details) and Data store (Customer information)
- Sequence flows and message/data associations
