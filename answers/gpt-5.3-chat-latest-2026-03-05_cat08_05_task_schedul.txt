1. Analyzing Historical Scheduling Performance and Dynamics

Reconstructing the Actual Process

Using the MES event logs, the first step is reconstructing the real execution of jobs on the shop floor. Each Case ID represents a job, and activities correspond to task states (queue entry, setup start/end, task start/end). Using process mining:

• Process discovery algorithms (Inductive Miner or Heuristics Miner) reconstruct the actual routing patterns across machines. This reveals the true variants of job flows rather than assumed routings.

• Object-centric process mining is useful because events involve both jobs and resources (machines/operators).

• Resource-centric timelines reconstruct machine schedules by ordering events for each machine ID, revealing the exact sequence of jobs processed.

The result is two complementary views:
• Job perspective: how each order flowed through operations.
• Machine perspective: the exact sequencing of jobs and setups.

Key Metrics and Analyses

Lead Times and Flow Times

Using case-level timestamps:

• Lead time = Job completion timestamp  Job release timestamp.
• Flow time distributions derived from statistical analysis across job types and routing variants.

Metrics:
• Mean, median, and percentile lead times
• Makespan distribution across weekly production windows
• Lead time variability per routing type

Process mining tools (e.g., conformance dashboards) can visualize cumulative flow diagrams and throughput trends.

Task Waiting Times (Queue Times)

Queue time is measured between:

Queue Entry  Setup Start (or Task Start if no setup)

Analysis includes:
• Average waiting time per machine
• Queue distribution per machine and per job priority
• Time-of-day congestion patterns

Queue heatmaps reveal chronic bottlenecks.

Resource Utilization

For each machine:

Productive Time = sum(Task Start  Task End)

Setup Time = sum(Setup Start  Setup End)

Idle Time = Total Available Time  Productive  Setup  Breakdown

Utilization metrics:
• Machine utilization %
• Setup ratio (setup time / productive time)
• Operator utilization
• Starvation vs blocking periods

Sequence-Dependent Setup Analysis

Sequence-dependent setups are reconstructed by analyzing:

Previous Job  Current Job transitions on each machine.

Method:

1. Order events by machine.
2. Identify job transitions.
3. Measure setup duration.
4. Group by attributes of job pairs (material, tooling group, geometry, tolerance class).

This produces a setup-time matrix:

SetupTime(machine, job_type_A  job_type_B)

Clustering techniques can reveal families of jobs that require similar setups.

Schedule Adherence and Tardiness

Key metrics:

• Tardiness = max(0, completion_time  due_date)
• Percentage of late jobs
• Average lateness
• Maximum lateness

Additional indicators:

• Slack consumption over time
• Critical ratio trends
• Percentage of jobs entering final operation already late

Variant comparison reveals patterns associated with late jobs.

Impact of Disruptions

Breakdowns and priority changes are treated as special events.

Analysis includes:

• Delay propagation analysis (how breakdowns affect downstream queues)
• Schedule recovery time after disruptions
• Jobs most affected by breakdowns

Event correlation methods can quantify the increase in queue length or tardiness following disruptions.


2. Diagnosing Scheduling Pathologies

Bottleneck Identification

Using:

• Utilization analysis
• Queue length distributions
• Throughput constraints

Machines with:

• Utilization consistently >85%
• Long queue times
• Large downstream delays

are likely bottlenecks.

Process mining bottleneck detection highlights machines where flow time accumulates.

Variant Analysis (On-Time vs Late Jobs)

Comparing process variants of:

• On-time jobs
• Late jobs

reveals patterns such as:

• Longer queues at specific machines
• More setup-heavy sequences
• Rerouting or rework loops

This isolates problematic routing patterns.

Suboptimal Sequencing

Machine-centric sequence reconstruction reveals:

• Frequent setup-heavy transitions
• Lack of job-family batching
• Alternating job types increasing setup overhead

Metrics:

Total Setup Time / Production Time

Transition analysis highlights costly sequences.

Resource Starvation

Downstream machines may show high idle time caused by upstream bottlenecks.

Indicators:

• Idle time spikes following upstream queue congestion
• Long waiting periods for certain machines despite high overall WIP

This indicates flow imbalance.

WIP Bullwhip Effect

WIP accumulation patterns can be reconstructed using queue entry/exit events.

Symptoms include:

• Large buffers before bottleneck machines
• Oscillating queue lengths
• Sudden WIP surges after priority changes

Process mining visualizations such as WIP trend charts reveal instability caused by reactive scheduling.


3. Root Cause Analysis of Scheduling Ineffectiveness

Limitations of Static Dispatching Rules

Current rules like FCFS or EDD operate locally at each machine.

Issues:

• No awareness of downstream congestion
• No consideration of setup sequences
• No coordination across work centers

This leads to globally inefficient schedules.

Lack of Real-Time Global Visibility

Local scheduling decisions ignore:

• System-wide WIP
• Current machine loads
• Upcoming bottlenecks

This creates cascading delays.

Inaccurate Task Duration Estimates

Process mining reveals discrepancies between:

Planned Duration vs Actual Duration

Variance often depends on:

• Operator skill
• Job complexity
• Machine condition

Static planning ignores this variability.

Ignoring Sequence-Dependent Setups

Without considering setup relationships, machines may process sequences that dramatically increase setup time.

For example:

Job A  Job B may require a 45-minute setup whereas Job A  Job C requires only 10 minutes.

Poor Coordination Between Work Centers

Upstream machines release jobs faster than downstream bottlenecks can process them.

This causes WIP accumulation.

Inadequate Disruption Handling

Hot jobs and breakdowns trigger reactive decisions that destabilize schedules.

Process Mining Differentiation of Root Causes

Process mining helps separate:

Scheduling logic problems vs capacity constraints.

If:

• Utilization is low but queues are high  scheduling inefficiency.
• Utilization is extremely high  true capacity bottleneck.

Variance analysis identifies whether delays stem from unpredictable processing times or poor sequencing decisions.


4. Developing Advanced Data-Driven Scheduling Strategies

Strategy 1: Dynamic Multi-Factor Dispatching Rules

Core Logic

Instead of single rules like FCFS or EDD, each job receives a dynamic priority score:

Priority Score = w1(Due Date Slack) + w2(Remaining Processing Time) + w3(Job Priority) + w4(Setup Cost) + w5(Downstream Load)

Where:

Slack = Due Date  Estimated Remaining Processing Time

Setup cost is estimated using the setup transition matrix derived from process mining.

Use of Process Mining Insights

• Historical queue times inform downstream load estimates.
• Setup matrices estimate sequence cost.
• Duration distributions estimate remaining processing time.

Expected Impact

• Reduced tardiness by prioritizing jobs with low slack.
• Lower setup time through sequence awareness.
• Improved flow coordination across machines.

Strategy 2: Predictive Scheduling Using Machine Learning

Core Logic

Predict future system states and proactively adjust schedules.

Predictive models estimate:

• Task duration distributions
• Breakdown probability
• Future queue congestion

Models trained from historical logs include:

• Gradient boosting for task duration prediction
• Survival models for machine failure prediction
• Time-series models for queue buildup

Scheduling Algorithm

A rolling-horizon scheduler periodically recalculates priorities using predicted system states.

Benefits

• Anticipates bottlenecks
• Produces more reliable completion estimates
• Reduces unexpected delays

Expected Impact

• Improved due-date reliability
• Lower schedule volatility
• Better planning visibility.

Strategy 3: Setup Time Optimization via Job Family Sequencing

Core Logic

Use clustering on historical setup transitions to identify job families with minimal setup changes.

Construct a setup cost matrix per machine.

Scheduling approach:

• Batch jobs with similar setup requirements
• Use optimization algorithms (e.g., traveling salesman heuristics) to determine optimal job sequences minimizing setup time.

Focus primarily on bottleneck machines.

Use of Process Mining

Setup transitions mined from event logs provide empirical setup cost estimates.

Expected Impact

• Major reduction in setup overhead
• Increased effective machine capacity
• Reduced queue buildup at bottlenecks.


5. Simulation, Evaluation, and Continuous Improvement

Discrete Event Simulation

A digital twin of the shop floor is constructed using parameters derived from the logs:

• Task duration distributions
• Setup time matrices
• Machine breakdown frequencies
• Job routing probabilities
• Arrival patterns

Simulation compares:

• Current dispatching rules
• Dynamic multi-factor dispatching
• Predictive scheduling
• Setup-optimized sequencing

Test Scenarios

• Normal workload
• Peak demand
• Frequent breakdowns
• Multiple urgent job arrivals
• High setup variability

Key Evaluation KPIs

• Average tardiness
• Percentage of late jobs
• WIP levels
• Average lead time
• Machine utilization
• Throughput

Continuous Monitoring Framework

After deployment:

Process mining continuously monitors event logs to detect deviations.

Key elements:

Real-Time KPI Monitoring
• Tardiness trends
• Queue length anomalies
• Setup time drift

Drift Detection
• Statistical process control detects changes in processing times or setup durations.

Automated Feedback Loop

Scheduling parameters update periodically using newly mined data:

• Updated task duration models
• Updated setup matrices
• Updated breakdown probabilities

Adaptive Scheduling

If performance degradation is detected:

• Dispatch rule weights are recalibrated
• Predictive models retrained
• Sequencing strategies adjusted

Outcome

This creates a self-improving scheduling system combining process mining insights, predictive analytics, and simulation-driven optimization to systematically reduce tardiness, stabilize lead times, and improve shop-floor efficiency.
