###  3. Proposed Investigative SQL Queries

These queries are designed to systematically validate the hypotheses, quantify anomaly frequency, and identify patterns across orders, resources, and departments.

####  Query 1: Detect Shipping Before Mandatory Compliance Checks
*Investigates: System bypass or SLA-driven shortcuts*
```sql
SELECT 
    sc.case_id,
    sc.timestamp AS ship_time,
    cc.timestamp AS credit_check_time,
    vs.timestamp AS stock_validation_time
FROM (SELECT case_id, timestamp FROM order_event_log WHERE activity = 'Ship Goods') sc
LEFT JOIN (SELECT case_id, timestamp FROM order_event_log WHERE activity = 'Perform Credit Check') cc 
  ON sc.case_id = cc.case_id
LEFT JOIN (SELECT case_id, timestamp FROM order_event_log WHERE activity = 'Validate Stock') vs 
  ON sc.case_id = vs.case_id
WHERE (cc.timestamp IS NOT NULL AND sc.timestamp < cc.timestamp)
   OR (vs.timestamp IS NOT NULL AND sc.timestamp < vs.timestamp)
ORDER BY sc.case_id;
```

####  Query 2: Flag Orders Missing Mandatory Process Steps
*Investigates: Incomplete workflows, data entry gaps, or systemic omissions*
```sql
SELECT 
    oel.case_id,
    COUNT(*) FILTER (WHERE activity = 'Perform Credit Check') = 0 AS missing_credit_check,
    COUNT(*) FILTER (WHERE activity = 'Validate Stock') = 0 AS missing_stock_validation,
    COUNT(*) FILTER (WHERE activity = 'Confirm Shipment') = 0 AS missing_shipment_confirmation
FROM order_event_log oel
GROUP BY oel.case_id
HAVING 
    COUNT(*) FILTER (WHERE activity = 'Perform Credit Check') = 0
    OR COUNT(*) FILTER (WHERE activity = 'Validate Stock') = 0
    OR COUNT(*) FILTER (WHERE activity = 'Confirm Shipment') = 0;
```

####  Query 3: Identify Financial Sequence Violations (Payment Before Invoice)
*Investigates: Accounting control weaknesses, prepayment handling errors*
```sql
SELECT 
    pay.case_id,
    pay.timestamp AS payment_time,
    inv.timestamp AS invoice_time,
    o.order_value,
    o.order_type
FROM (SELECT case_id, timestamp FROM order_event_log WHERE activity = 'Receive Payment') pay
JOIN (SELECT case_id, timestamp FROM order_event_log WHERE activity = 'Issue Invoice') inv 
  ON pay.case_id = inv.case_id
LEFT JOIN orders o ON pay.case_id = o.case_id
WHERE pay.timestamp < inv.timestamp;
```

####  Query 4: Correlate Anomalies with Order Attributes & Resource Departments
*Investigates: Whether priority/high-value orders or specific departments drive violations*
```sql
SELECT 
    o.case_id,
    o.order_type,
    o.order_value,
    r.department,
    COUNT(*) AS total_events,
    COUNT(*) FILTER (WHERE oel.activity IN ('Perform Credit Check', 'Validate Stock')) AS compliance_steps_done
FROM order_event_log oel
JOIN orders o ON oel.case_id = o.case_id
JOIN resources r ON oel.resource = r.resource_id
GROUP BY o.case_id, o.order_type, o.order_value, r.department
HAVING COUNT(*) FILTER (WHERE oel.activity IN ('Perform Credit Check', 'Validate Stock')) < 2
ORDER BY o.order_value DESC;
```

####  Query 5: Detect Contradictory System Flags vs. Actual Execution
*Investigates: Data quality issues, UI/backend sync failures, manual overrides*
```sql
SELECT 
    conf.case_id,
    conf.additional_info AS confirmation_flags,
    ship.timestamp AS actual_ship_time,
    ship.resource AS shipping_resource
FROM order_event_log conf
JOIN order_event_log ship 
  ON conf.case_id = ship.case_id 
 AND ship.activity = 'Ship Goods'
WHERE conf.activity = 'Confirm Shipment'
  AND (conf.additional_info ILIKE '%shipment_scheduled=N%' 
       OR conf.additional_info ILIKE '%shipment_scheduled=n%');
```
