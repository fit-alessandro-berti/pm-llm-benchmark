# Insurance Claim Management Process — Accepting Petri Net

Below is the complete Python code constructing the accepting Petri net for the insurance claim process, including the re-submission loop (DV  RDS) and the investigation loop (FC  II):

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# Create the Petri net
net = PetriNet(name="insurance_claim_management")

# ----------------------------------------------------------------------
# PLACES (states between activities)
# ----------------------------------------------------------------------
p_start = PetriNet.Place(name="p_start")        # new claim arrived
p1 = PetriNet.Place(name="p1")                  # after Claim Registration
p2 = PetriNet.Place(name="p2")                  # after Document Verification
p3 = PetriNet.Place(name="p3")                  # after Fraud Check
p4 = PetriNet.Place(name="p4")                  # after Claim Assessment
p5 = PetriNet.Place(name="p5")                  # after Approval
p6 = PetriNet.Place(name="p6")                  # after Payment
p7 = PetriNet.Place(name="p7")                  # after Notification
p_end = PetriNet.Place(name="p_end")            # claim closed (final state)

for place in [p_start, p1, p2, p3, p4, p5, p6, p7, p_end]:
    net.places.add(place)

# ----------------------------------------------------------------------
# TRANSITIONS (activities)
# ----------------------------------------------------------------------
t_cr  = PetriNet.Transition(name="CR",  label="Claim Registration")
t_dv  = PetriNet.Transition(name="DV",  label="Document Verification")
t_rds = PetriNet.Transition(name="RDS", label="Re-Document Submission")
t_fc  = PetriNet.Transition(name="FC",  label="Fraud Check")
t_ii  = PetriNet.Transition(name="II",  label="In-depth Investigation")
t_ca  = PetriNet.Transition(name="CA",  label="Claim Assessment")
t_ap  = PetriNet.Transition(name="AP",  label="Approval")
t_p   = PetriNet.Transition(name="P",   label="Payment")
t_n   = PetriNet.Transition(name="N",   label="Notification")
t_cl  = PetriNet.Transition(name="CL",  label="Closure")

for transition in [t_cr, t_dv, t_rds, t_fc, t_ii, t_ca, t_ap, t_p, t_n, t_cl]:
    net.transitions.add(transition)

# ----------------------------------------------------------------------
# ARCS (control flow)
# ----------------------------------------------------------------------
# Main sequence: start -> CR -> DV
petri_utils.add_arc_from_to(p_start, t_cr, net)
petri_utils.add_arc_from_to(t_cr, p1, net)
petri_utils.add_arc_from_to(p1, t_dv, net)
petri_utils.add_arc_from_to(t_dv, p2, net)

# Loop 1: documents incomplete -> Re-Document Submission -> back to DV
petri_utils.add_arc_from_to(p2, t_rds, net)
petri_utils.add_arc_from_to(t_rds, p1, net)   # back to place before DV

# Continue: DV -> FC
petri_utils.add_arc_from_to(p2, t_fc, net)
petri_utils.add_arc_from_to(t_fc, p3, net)

# Loop 2: fraud doubts -> In-depth Investigation -> back to FC
# (can be repeated multiple times)
petri_utils.add_arc_from_to(p3, t_ii, net)
petri_utils.add_arc_from_to(t_ii, p2, net)    # back to place before FC

# Continue: FC -> CA -> AP -> P -> N -> CL -> end
petri_utils.add_arc_from_to(p3, t_ca, net)
petri_utils.add_arc_from_to(t_ca, p4, net)
petri_utils.add_arc_from_to(p4, t_ap, net)
petri_utils.add_arc_from_to(t_ap, p5, net)
petri_utils.add_arc_from_to(p5, t_p, net)
petri_utils.add_arc_from_to(t_p, p6, net)
petri_utils.add_arc_from_to(p6, t_n, net)
petri_utils.add_arc_from_to(t_n, p7, net)
petri_utils.add_arc_from_to(p7, t_cl, net)
petri_utils.add_arc_from_to(t_cl, p_end, net)

# ----------------------------------------------------------------------
# INITIAL AND FINAL MARKINGS
# ----------------------------------------------------------------------
im = Marking()
im[p_start] = 1     # a new claim has arrived

fm = Marking()
fm[p_end] = 1       # the claim is closed and archived
```

## Explanation of the model

**Main flow:**
```
p_start  CR  p1  DV  p2  FC  p3  CA  p4  AP  p5  P  p6  N  p7  CL  p_end
```

**Loop 1 — Document re-submission:**
After `DV` puts a token in `p2`, there is a choice:
- fire `FC` (documents are complete), or
- fire `RDS`, which returns the token to `p1`, so `DV` is executed again.

This loop can repeat until documents are complete.

**Loop 2 — In-depth investigation:**
After `FC` puts a token in `p3`, there is a choice:
- fire `CA` (no fraud doubts), or
- fire `II`, which returns the token to `p2`, so `FC` is executed again.

Since `II` returns to `p2`, the fraud check can be repeated multiple times, matching the requirement that "this can happen multiple times before proceeding to Claim Assessment."

**Markings:**
- The **initial marking** `im` places one token in `p_start`, representing the arrival of a new insurance claim.
- The **final marking** `fm` requires exactly one token in `p_end`, which is only reachable after `Closure (CL)` fires — i.e., the claim was paid, the customer notified, and the claim closed and archived.
