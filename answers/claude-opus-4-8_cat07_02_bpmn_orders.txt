# BPMN Process Diagram Description

This is a **Business Process Model and Notation (BPMN)** diagram illustrating an order fulfillment process across four organizational departments, represented as horizontal swimlanes.

## Swimlanes (Departments)

### 1. Sales Lane (Top)
- **Trigger:** A timer event ("Monday morning") starts the process
- **Sub-process "Check customer's order":**
  - Check order for completeness
  - Check if customer is already in the database
  - A gateway branches based on whether the customer exists:
    - **Customer not in the database**  Add customer to the database (connects to a database object)
    - **Customer in the database**  proceeds directly
  - Both paths merge at a converging gateway
- **Send order details**  triggers downstream processes
- **Draft invoice** (triggered after "Product sent" message)
- Ends with an end event

### 2. Production Lane
- **Start:** "Order details arrive" (message event)
- **Parallel gateway** (split) into:
  - Plan production  Manufacture product
  - A second branch with a message event ("Material not faulty")
- **Parallel gateway** (join) leading to "Product complete" event

### 3. Procurement Lane
- **Start:** "Order details arrive" (message event)
- **Material Compensation Subprocess:**
  - Order materials (with compensation/rewind marker)
  - Return material (compensation activity)
- **Check materials** activity with:
  - A conditional/signal event
  - **Order cancellation** path (end event with rewind marker)
  - A message event flowing upward

### 4. Warehouse Lane (Bottom)
- **Start:** "Product complete" message event
- **Send product** activity
- Ends with "Product sent" message event

## Key Flow Connections
Dashed lines represent **message flows** connecting the lanes:
- "Order details" flow from Sales to Production and Procurement
- "Product complete" and "Product sent" messages link Warehouse back to Production and Sales

This diagram models a coordinated, cross-departmental workflow with parallel processing, compensation handling, and exception/cancellation paths.
