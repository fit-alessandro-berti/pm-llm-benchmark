How cases were identified
- Object-centric grouping: Each case represents work on a single “object.”
  - Word documents: by file name (Document1.docx, Quarterly_Report.docx).
  - Spreadsheet: by file name (Budget_2024.xlsx).
  - PDF: by file name (Report_Draft.pdf).
  - Email: by email subject inferred from the click “Open Email about Annual Meeting” (case CASE_EMAIL_Annual_Meeting).
- Session stitching: Interactions on the same object within the continuous session were kept in one case. Brief context switches (minutes apart) were treated as the same case (e.g., returning to Document1.docx at 09:06 after editing Excel). The initial focus on Quarterly_Report at 08:59:50 and the later edit/close at 09:07–09:08 were merged into one case because they refer to the same document within the same morning window.
- Inbox focus at 09:01:45 was associated with the Annual Meeting email case because it immediately preceded opening that specific email; this keeps the email handling as one coherent case rather than splitting out a generic “inbox session.”

How low-level actions were mapped to standardized activities
- FOCUS or SWITCH to a specific artifact:
  - Word: Open/Focus Document
  - Excel: Open/Focus Spreadsheet
  - Acrobat: Open/Focus PDF
  - Chrome (mail): Open/Focus Email Client
- TYPING:
  - In Word: Edit Document Content
  - In Excel: Edit Spreadsheet
  - In Email: Draft Email Reply
- CLICK:
  - Open Email about X: Open Email
  - Reply to Email: Start Email Reply
  - Send Email: Send Email
- SCROLL:
  - In email: Read Email
  - In PDF: Review PDF
- HIGHLIGHT in PDF: Annotate PDF
- SAVE: Save Document / Save Spreadsheet (by app)
- CLOSE: Close Document (for the Word document window)

Import guidance
- Map columns as:
  - Case ID: case_id
  - Activity: activity_name
  - Timestamp: timestamp (ISO 8601 UTC)
- app, window, object, object_type, raw_action, details are optional event attributes useful for filtering and conformance checks.
