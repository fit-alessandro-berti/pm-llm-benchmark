Looking at the insurance claim management process, I need to construct an accepting Petri net with:
1. A main sequence flow
2. Two optional loops (document resubmission and fraud investigation)
3. Initial marking at the start
4. Final marking at closure

Let me design this step by step:

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# Create the Petri net
net = PetriNet("Insurance Claim Process")

# Create all places (states between activities)
start_place = PetriNet.Place(name="start")  # New claim arrives
after_cr = PetriNet.Place(name="after_claim_registration")
after_dv = PetriNet.Place(name="after_document_verification")
after_fc = PetriNet.Place(name="after_fraud_check")
after_ca = PetriNet.Place(name="after_claim_assessment")
after_ap = PetriNet.Place(name="after_approval")
after_p = PetriNet.Place(name="after_payment")
after_n = PetriNet.Place(name="after_notification")
end_place = PetriNet.Place(name="end")  # Claim closed

# Additional places for the loops
after_rds = PetriNet.Place(name="after_re_document_submission")
after_ii = PetriNet.Place(name="after_in_depth_investigation")

# Create all transitions (activities)
cr = PetriNet.Transition(name="claim_registration", label="Claim Registration")
dv = PetriNet.Transition(name="document_verification", label="Document Verification")
rds = PetriNet.Transition(name="re_document_submission", label="Re-Document Submission")
fc = PetriNet.Transition(name="fraud_check", label="Fraud Check")
ii = PetriNet.Transition(name="in_depth_investigation", label="In-depth Investigation")
ca = PetriNet.Transition(name="claim_assessment", label="Claim Assessment")
ap = PetriNet.Transition(name="approval", label="Approval")
p = PetriNet.Transition(name="payment", label="Payment")
n = PetriNet.Transition(name="notification", label="Notification")
cl = PetriNet.Transition(name="closure", label="Closure")

# Add all elements to the net
places = [start_place, after_cr, after_dv, after_fc, after_ca, after_ap, 
          after_p, after_n, end_place, after_rds, after_ii]
transitions = [cr, dv, rds, fc, ii, ca, ap, p, n, cl]

for place in places:
    net.places.add(place)

for transition in transitions:
    net.transitions.add(transition)

# Create arcs - main sequence
petri_utils.add_arc_from_to(start_place, cr, net)
petri_utils.add_arc_from_to(cr, after_cr, net)
petri_utils.add_arc_from_to(after_cr, dv, net)
petri_utils.add_arc_from_to(dv, after_dv, net)
petri_utils.add_arc_from_to(after_dv, fc, net)
petri_utils.add_arc_from_to(fc, after_fc, net)
petri_utils.add_arc_from_to(after_fc, ca, net)
petri_utils.add_arc_from_to(ca, after_ca, net)
petri_utils.add_arc_from_to(after_ca, ap, net)
petri_utils.add_arc_from_to(ap, after_ap, net)
petri_utils.add_arc_from_to(after_ap, p, net)
petri_utils.add_arc_from_to(p, after_p, net)
petri_utils.add_arc_from_to(after_p, n, net)
petri_utils.add_arc_from_to(n, after_n, net)
petri_utils.add_arc_from_to(after_n, cl, net)
petri_utils.add_arc_from_to(cl, end_place, net)

# Create arcs for Document Verification loop
petri_utils.add_arc_from_to(after_dv, rds, net)
petri_utils.add_arc_from_to(rds, after_rds, net)
petri_utils.add_arc_from_to(after_rds, dv, net)

# Create arcs for Fraud Check loop
petri_utils.add_arc_from_to(after_fc, ii, net)
petri_utils.add_arc_from_to(ii, after_ii, net)
petri_utils.add_arc_from_to(after_ii, fc, net)

# Create initial marking (one token in start place)
im = Marking()
im[start_place] = 1

# Create final marking (one token in end place)
fm = Marking()
fm[end_place] = 1

# Output the Petri net structure
print("Insurance Claim Process Petri Net")
print("=================================")
print(f"Initial marking: {im}")
print(f"Final marking: {fm}")
print("\nMain Sequence Flow:")
print("start -> Claim Registration -> after_CR -> Document Verification -> after_DV")
print("-> Fraud Check -> after_FC -> Claim Assessment -> after_CA -> Approval")
print("-> after_AP -> Payment -> after_P -> Notification -> after_N -> Closure -> end")
print("\nLoops:")
print("1. Document Verification loop: after_DV -> RDS -> after_RDS -> DV")
print("2. Fraud Check loop: after_FC -> II -> after_II -> FC")
```

This Petri net models the insurance claim process with:
- **Main flow**: CR  DV  FC  CA  AP  P  N  CL
- **Document loop**: DV can lead to RDS (if documents incomplete) and back to DV
- **Fraud investigation loop**: FC can lead to II (if doubts) and back to FC
- **Initial marking**: Token at start place represents a new claim
- **Final marking**: Token at end place represents a closed claim

The structure ensures that:
1. A new claim always starts at the start place
2. After document verification, the process can either continue to fraud check or loop back via document resubmission
3. After fraud check, the process can either continue to claim assessment or loop back via in-depth investigation
4. The process always ends at the end place after closure
5. Both loops can occur multiple times (as indicated by the bidirectional connections)

This design captures the complexity of real-world insurance claim processing while maintaining clear control flow through the Petri net semantics.
