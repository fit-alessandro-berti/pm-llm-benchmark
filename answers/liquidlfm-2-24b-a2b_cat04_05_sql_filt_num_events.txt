To achieve the task described, you can use a subquery to first calculate the number of events per `case_id` and then filter out the cases with more than six events. Here's a DuckDB SQL query that accomplishes this:

```sql
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
);
```

This query works as follows:

1. The subquery selects `case_id` from the `event_log` table.
2. It groups the results by `case_id`.
3. It uses the `HAVING` clause to filter out any `case_id` that has more than six events (`COUNT(*) <= 6`).
4. The outer query then selects all columns from `event_log` where the `case_id` is in the list of `case_id`s returned by the subquery, effectively filtering out events from cases with more than six events.

This approach ensures that you only get the events from cases that have six or fewer events in total.
