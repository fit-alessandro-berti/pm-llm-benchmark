# BPMN Business Process Diagram Description

This visualization is a **BPMN (Business Process Model and Notation) diagram** depicting an end-to-end order fulfillment process. It's organized into four horizontal swim lanes representing different departments:

## Swim Lanes & Their Activities

### 1. **Sales Lane** (Top)
- Starts with a timer event "Monday morning"
- **Check order for completeness**  **Check if customer is already in the database**
- A gateway decision: if customer is NOT in database  **Add customer to the database** (interacts with a database object); if customer IS in database  proceed directly
- **Send order details** (message event sent to other lanes)
- **Draft invoice** (triggered after "Product sent" message)

### 2. **Production Lane**
- Triggered by "Order details arrive" message
- **Plan production**  parallel gateway
- **Manufacture product**  ends with "Product complete"
- Includes a message event "Material not faulty" linked from Procurement

### 3. **Procurement Lane**
- Triggered by "Order details arrive" message
- Contains a **Material Compensation Subprocess** with:
  - **Order materials** task
  - **Return material** compensation task
- **Check materials** task with two outcomes:
  - Success  sends message to Production
  - Failure  **Order cancellation** (termination event)

### 4. **Warehouse Lane** (Bottom)
- Triggered by "Product complete" message
- **Send product** task
- Ends with "Product sent" message event (which feeds back to Sales for invoicing)

## Key BPMN Elements Shown
- **Circles**: Start/end events (with envelope icons for message events, clock for timer)
- **Rounded rectangles**: Tasks/activities
- **Diamonds**: Gateways (X = exclusive, + = parallel)
- **Dashed lines**: Message flows between lanes
- **Solid arrows**: Sequence flows within lanes
- **Database cylinder**: Data store

## Process Flow Summary
The diagram illustrates how a customer order flows from Sales  simultaneously to Production and Procurement  Warehouse (shipping)  back to Sales (invoicing), with compensation handling for faulty materials and order cancellation as exception paths.
