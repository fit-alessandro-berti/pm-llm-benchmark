# Comprehensive Process Mining Approach for Speedy Parcels Optimization

## 1. Process Discovery and Conformance Checking

### 1.1 Data Preprocessing and Integration

**Case Notion Design:**
The primary challenge is defining an appropriate case ID that captures the end-to-end delivery process while enabling multi-perspective analysis. I recommend a **dual-case structure**:

- **Primary Case ID: "Vehicle-Day" (e.g., V12-20241205)** - This creates a holistic process model from shift start to end, capturing the complete operational journey including travel, deliveries, and exceptions.
- **Secondary Case ID: "Package ID" (e.g., P9876)** - This enables package-level conformance checking against planned delivery sequences and time windows.

**Integration Pipeline:**
```plaintext
1. Data Harmonization:
   - Synchronize timestamps across systems using NTP logs (account for GPS device drift: ±2-5 sec)
   - Standardize location data: geofence depot (100m radius) and customer addresses
   - Enrich scanner events with GPS coordinates (match by timestamp ±30 sec window)
   - Link maintenance logs to vehicle-days via date-vehicle ID

2. Event Log Construction:
   - Extract activities from each source:
     * Driver: Start Shift, End Shift, Break Start/End
     * Dispatch: Route Assigned, Route Modified
     * Vehicle: Depart Depot, Arrive Customer (geo-fence triggered), Low Speed Detected, 
                Unscheduled Stop, Arrive Depot
     * Scanner: Delivery Success, Delivery Failed, Pickup Scanned
     * Maintenance: Maintenance Required, Maintenance Completed

3. Attribute Enrichment:
   - Calculate derived attributes: travel time between stops, dwell time, speed variance,
     distance traveled, time window compliance, stop sequence deviation
   - Flag anomalous events: speed > 110% route limit, dwell > 15 min at non-customer locations
```

**Key Challenges & Mitigations:**
- **Missing Data:** GPS dropouts in urban canyons; impute using Kalman filtering and scanner event location as anchors
- **Granularity Mismatch:** GPS produces 1,000+ events/day vs. 30-40 scanner events; aggregate GPS into "movement phases" using stay-point detection algorithms
- **Clock Skew:** Cross-system timestamp misalignment; implement a master timeline based on dispatch system with GPS/scanner event reconciliation
- **Semantic Ambiguity:** "Arrive Customer" may not match actual doorstep time; use GPS entry into 50m geofence + first scanner event logic

Employ the **Inductive Miner with Petri Net** representation for its robustness to noise and ability to handle concurrency. Supplement with **Directly-Follows Graphs (DFG)** for high-level visualization.

The actual process will reveal:
- **Happy Path:** Start Shift  Route Assigned  Depart Depot  *(Travel  Arrive Customer  [Delivery Success|Failed]  Depart Customer)*  Arrive Depot  End Shift
- **Exception Paths:**
  - **Traffic Delay Loop:** Travel  Low Speed Detected  (possible Return to Previous Location)  Travel
