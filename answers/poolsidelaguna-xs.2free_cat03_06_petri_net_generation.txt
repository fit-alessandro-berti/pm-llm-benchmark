p_after_approval = Place(name="after_approval")
p_after_payment = Place(name="after_payment")
p_after_notification = Place(name="after_notification")
end_place = Place(name="end")

# Add all places to the net
for place in [start_place, p_after_claim, p_after_registration, p_after_dv, 
              p_after_rds, p_after_fc, p_after_ii, p_after_assessment,
              p_after_approval, p_after_payment, p_after_notification, end_place]:
    net.places.add(place)

# Create transitions for each activity
t_claim = Transition(name="Insurance_Claim", label="C")
t_registration = Transition(name="Claim_Registration", label="CR")
t_dv = Transition(name="Document_Verification", label="DV")

t_payment = Transition(name="Payment", label="P")

for transition in [t_claim, t_registration, t_dv, t_rds, t_fc, t_ii, 
                   t_assessment, t_approval, t_payment, t_notification, t_closure]:

# Add arcs connecting places and transitions
# Main linear flow

**Key Features:**
1. **Linear Flow**: C  CR  DV  FC  CA  AP  P  N  CL
2. **Re-Document Submission Loop**: When documents are incomplete after DV, the process can loop back to DV via RDS
3. **In-depth Investigation Loop**: When fraud is suspected after FC, the process can loop back to FC via II (multiple times if needed)
4. **Conditional Approval**: Low-value claims can skip the approval step and proceed directly from assessment to payment
