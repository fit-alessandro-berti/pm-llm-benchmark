# Business Process Model and Notation (BPMN) Diagram

This visualization is a **BPMN process diagram** depicting an **order fulfillment workflow** across four organizational departments (swim lanes):

---

## 1. **Sales** (Top Lane)
- The process begins with a **timer event** ("Monday morning")
- **Check order for completeness**  **Check if customer is already in the database**
- An **exclusive gateway** splits the flow:
  - *Customer not in database*  **Add customer to the database** (with a database data store)
  - *Customer in the database*  proceeds directly
- Paths merge at another gateway  **Send order details**  triggers a "Product sent" message event  **Draft invoice**  process ends

## 2. **Production** (Second Lane)
- Triggered by a **message event** ("Order details arrive")
- A **parallel gateway** splits into:
  - **Plan production**
  - A path leading to a **material check** ("Material not faulty" message)
- Both converge  **Manufacture product**  **Product complete** (end signal event)

## 3. **Procurement** (Third Lane)
- Triggered by a **message event** ("Order details arrive")
- Contains a **Material Compensation Subprocess**:
  - **Order materials**  **Check materials**
  - If issues arise  **Return material** (compensation activity with compensation markers)
  - Handles **order cancellation** via a terminate/cancel event

## 4. **Warehouse** (Bottom Lane)
- Triggered by **"Product complete"** message event
- **Send product**  **Product sent** (end event)

---

## Key Features
- **Message flows** (dashed lines) connect departments, showing inter-departmental communication (e.g., "Order details" flows from Sales to Production and Procurement)
- Uses **compensation events**, **parallel/exclusive gateways**, **subprocesses**, and **message events** to model complex business logic
