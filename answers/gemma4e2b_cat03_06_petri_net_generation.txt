This scenario requires constructing a flow that includes sequential steps, conditional loops, and clear start/end markers, fitting the definition of an accepting Petri net.

We will use the following structure:
1. **Start Place:** Where the claim initiates.
2. **Sequential Steps:** CR, DV, FC, CA, AP, P, N, CL.
3. **Decision Loops:** RDS (Re-Document Submission) and II (In-depth Investigation).

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# 1. Initialize the Petri Net
net = PetriNet(name="InsuranceClaimProcess")

# --- Define Places (States) ---
# Start Place
source = PetriNet.Place(name="Claim_Start")

# Process Stages
cr = PetriNet.Place(name="Claim_Registration")
dv = PetriNet.Place(name="Document_Verification")
rds = PetriNet.Place(name="Re_Document_Submission")
fc = PetriNet.Place(name="Fraud_Check")
ii = PetriNet.Place(name="In_Depth_Investigation")
ca = PetriNet.Place(name="Claim_Assessment")
ap = PetriNet.Place(name="Approval")
p = PetriNet.Place(name="Payment")
n = PetriNet.Place(name="Notification")
cl = PetriNet.Place(name="Claim_Closure")

# --- Define Transitions (Activities) ---
execute_cr = PetriNet.Transition(name="Claim Registration", label="Register Claim")
execute_dv = PetriNet.Transition(name="Document Verification", label="Verify Documents")
execute_rds = PetriNet.Transition(name="Re-Document Submission", label="Re-Submit Documents")
execute_fc = PetriNet.Transition(name="Fraud Check", label="Check Fraud")
execute_ii = PetriNet.Transition(name="In-Depth Investigation", label="Investigate Fraud")
execute_ca = PetriNet.Transition(name="Claim Assessment", label="Assess Claim")
execute_ap = PetriNet.Transition(name="Approval", label="Approve Claim")
execute_p = PetriNet.Transition(name="Payment", label="Make Payment")
execute_n = PetriNet.Transition(name="Notification", label="Notify Customer")
execute_cl = PetriNet.Transition(name="Claim Closure", label="Close Claim")


# 2. Add Places and Transitions to the Net
net.places.add(source, cr, dv, rds, fc, ii, ca, ap, p, n, cl)
net.transitions.add(execute_cr, execute_dv, execute_rds, execute_fc, execute_ii, execute_ca, execute_ap, execute_p, execute_n, execute_cl)


# 3. Define Arcs (Flows)

# Initial Flow
petri_utils.add_arc_from_to(source, execute_cr, net)
petri_utils.add_arc_from_to(cr, dv, net)

# Document Verification Loop (DV -> RDS -> DV)
# If documents need re-submission:
petri_utils.add_arc_from_to(dv, execute_rds, net)
petri_utils.add_arc_from_to(execute_rds, dv, net)

# Fraud Check Loop (FC -> II -> FC)
# If in-depth investigation is required:
petri_utils.add_arc_from_to(fc, execute_ii, net)
petri_utils.add_arc_from_to(execute_ii, fc, net)

# Main Sequential Flow (after loops)
# Path 1: Successful verification path
petri_utils.add_arc_from_to(dv, fc, net)

# Path 2: Proceed to assessment
petri_utils.add_arc_from_to(fc, ca, net)

# Final steps
petri_utils.add_arc_from_to(ca, ap, net)
petri_utils.add_arc_from_to(ap, p, net)
petri_utils.add_arc_from_to(p, n, net)
petri_utils.add_arc_from_to(n, cl, net)


# 4. Define Initial Marking (im)
# A new claim arrives at the start.
im = Marking()
im[source] = 1

# 5. Define Final Marking (fm)
# The process is accepted when the claim is successfully closed.
fm = Marking()
fm[cl] = 1
# All other places remain empty in the final state

# 6. Verification (Optional: Print Net Structure)
print("--- Petri Net Constructed ---")
print(f"Total Places: {len(net.places)}")
print(f"Total Transitions: {len(net.transitions)}")
print("\nInitial Marking (im):")
print(im)
print("\nFinal Marking (fm):")
print(fm)
```

### Explanation of the Constructed Petri Net:

1.  **Places (States):** Represent the specific steps or conditions of the claim process (`Claim_Start`, `Claim_Registration`, `Document_Verification`, `Re_Document_Submission`, `Fraud_Check`, `In_Depth_Investigation`, `Claim_Assessment`, `Approval`, `Payment`, `Notification`, `Claim_Closure`).
2.  **Transitions (Activities):** Represent the actions performed (e.g., `Check Fraud`, `Re-Submit Documents`).
3.  **Initial Marking (`im`):** A single token starts at `Claim_Start`, representing the arrival of a new insurance claim.
4.  **Final Marking (`fm`):** The process is accepted when the token reaches the final state, `Claim_Closure`. Only this place holds the final token, indicating the claim has been successfully processed and closed.
5.  **Loops:**
    *   The arc from `Document_Verification` to `Re_Document_Submission` and back to `Document_Verification` implements the loop for re-submitting documents.
    *   The arc from `Fraud_Check` to `In_Depth_Investigation` and back to `Fraud_Check` implements the loop for in-depth investigation.
6.  **Flow:** The flow is strictly enforced: Start $\rightarrow$ Registration $\rightarrow$ Verification $\rightarrow$ (Decision Loop) $\rightarrow$ Fraud Check $\rightarrow$ (Decision Loop) $\rightarrow$ Assessment $\rightarrow$ Approval $\rightarrow$ Payment $\rightarrow$ Notification $\rightarrow$ Closure.
