# Description of the Visualization

This figure contains three related components illustrating a **proclet-based workflow model** for a laboratory visit process:

## (a) Two Proclet Classes Connected Through Two Channels

This is the main diagram showing two interconnected process flows:

### **Lab Visit (left side):**
- Begins with **Order entry** (initiated by a doctor)
- Proceeds to **Take blood sample** (by lab assistant)
- **Receive result**  **Judge results**
- A decision point "**Everything fine**" branches to either:
  - **Send report** (if yes) — the end state
  - **Do tests again**  **Receive updated result** (loop back)

### **Lab Test (right side):**
- **Perform test** (by chemical analyst)
- **Make report** (by chemical analyst)
- **Finish lab test** — the end state
- Alternative path: **Perform test again**  **Provide updated report** (by chemical analyst)

### Communication Channels:
- **Order system** (dashed box) connects the two proclet classes
- **HIS** (Hospital Information System) serves as another channel
- Channels show cardinality notations (1,1 and 1,*) indicating message multiplicities

## (b) Class Diagram
A simple UML class diagram showing that **Lab visit** (1..1) **requires** one or more (1..*) **Lab tests**.

## (c) Example of a Performative
A table showing a sample message:
- **Time:** 11:00
- **Channel:** Order system
- **Sender:** Lab visit - John
- **Receivers:** Lab test – HGB
- **Action:** Create
- **Content:** "Can you perform a HGB test for John?"
- **Scope:** Private
- **Direction:** OUT

This visualization demonstrates how **proclets** (lightweight, interacting workflow processes) communicate through channels to coordinate healthcare laboratory workflows.
