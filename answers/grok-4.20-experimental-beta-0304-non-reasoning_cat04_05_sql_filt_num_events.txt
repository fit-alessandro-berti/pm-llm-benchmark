**DuckDB SQL Query:**

```sql
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
ORDER BY case_id, timestamp;
```

**Alternative version using a CTE (often more readable):**

```sql
WITH valid_cases AS (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT e.*
FROM event_log e
JOIN valid_cases vc ON e.case_id = vc.case_id
ORDER BY e.case_id, e.timestamp;
```

Both queries achieve the same result:
- First identify all `case_id`s that have 6 or fewer events
- Then return all events belonging to those cases
- The `ORDER BY` ensures consistent output ordering (recommended for process event logs)

The first version is more concise and generally performs well in DuckDB.
