

Looking at the description, I need to model the hiring process in two variants: one with a biased XOR branch and one without.

Let me trace through the key elements:
- **Loop**: Data completeness check (check  request more info  check again)
- **Sequence**: ReceiveApplication  Loop  SkillAssessment  Cultural Fit stage  ManagerialReview  FinalDecision
- **XOR (unfair model only)**: Choice between standard CulturalFitCheck and CommunityAffiliationCheck

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# ============================================================
# POWL Model 1: With Potential Unfairness
# ============================================================
# The XOR branch after SkillAssessment allows either a standard
# CulturalFitCheck OR a CommunityAffiliationCheck that gives
# preferential treatment to locally affiliated applicants.

ReceiveApplication_1 = Transition(label="ReceiveApplication")
DataCompletenessCheck_1 = Transition(label="DataCompletenessCheck")
RequestMoreInfo_1 = Transition(label="RequestMoreInfo")
SkillAssessment_1 = Transition(label="SkillAssessment")
CulturalFitCheck_1 = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck_1 = Transition(label="CommunityAffiliationCheck")
ManagerialReview_1 = Transition(label="ManagerialReview")
FinalDecision_1 = Transition(label="FinalDecision")

# Loop for data completeness: execute DataCompletenessCheck, then optionally
# RequestMoreInfo and repeat, or exit the loop
data_loop_1 = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck_1, RequestMoreInfo_1])

# XOR branch: either standard cultural fit OR community affiliation check (bias point)
biased_xor = OperatorPOWL(operator=Operator.XOR, children=[CulturalFitCheck_1, CommunityAffiliationCheck_1])

# Full process as a partial order with strict sequential ordering
unfair_model = StrictPartialOrder(nodes=[ReceiveApplication_1, data_loop_1, SkillAssessment_1, biased_xor, ManagerialReview_1, FinalDecision_1])
unfair_model.order.add_edge(ReceiveApplication_1, data_loop_1)
unfair_model.order.add_edge(data_loop_1, SkillAssessment_1)
unfair_model.order.add_edge(SkillAssessment_1, biased_xor)
unfair_model.order.add_edge(biased_xor, ManagerialReview_1)
unfair_model.order.add_edge(ManagerialReview_1, FinalDecision_1)

print("=== POWL Model 1: With Potential Unfairness ===")
print(unfair_model)

# ============================================================
# POWL Model 2: Without Unfairness (Fair Process)
# ============================================================
# All applicants go through the same CulturalFitCheck.
# No XOR branch providing selective community-based advantages.

ReceiveApplication_2 = Transition(label="ReceiveApplication")
DataCompletenessCheck_2 = Transition(label="DataCompletenessCheck")
RequestMoreInfo_2 = Transition(label="RequestMoreInfo")
SkillAssessment_2 = Transition(label="SkillAssessment")
CulturalFitCheck_2 = Transition(label="CulturalFitCheck")
ManagerialReview_2 = Transition(label="ManagerialReview")
FinalDecision_2 = Transition(label="FinalDecision")

# Same loop for data completeness
data_loop_2 = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck_2, RequestMoreInfo_2])

# Full process: strictly sequential, everyone gets the same cultural fit evaluation
fair_model = StrictPartialOrder(nodes=[ReceiveApplication_2, data_loop_2, SkillAssessment_2, CulturalFitCheck_2, ManagerialReview_2, FinalDecision_2])
fair_model.order.add_edge(ReceiveApplication_2, data_loop_2)
fair_model.order.add_edge(data_loop_2, SkillAssessment_2)
fair_model.order.add_edge(SkillAssessment_2, CulturalFitCheck_2)
fair_model.order.add_edge(CulturalFitCheck_2, ManagerialReview_2)
fair_model.order.add_edge(ManagerialReview_2, FinalDecision_2)

print("=== POWL Model 2: Without Unfairness (Fair) ===")
print(fair_model)
```

**Key Differences:**

- **Model 1 (Unfair):** After `SkillAssessment`, an `XOR` branch lets some applicants go through `CommunityAffiliationCheck` (which implicitly boosts scores for locally affiliated candidates) instead of the standard `CulturalFitCheck`. This is the bias injection point—affiliated applicants get routed to a preferential evaluation path.

- **Model 2 (Fair):** The `XOR` branch is completely removed. Every applicant, regardless of community affiliation, proceeds through the identical `CulturalFitCheck` activity, ensuring uniform evaluation criteria and eliminating the structural opportunity for bias.
