# BPMN Visualization Description

This image shows a **Business Process Model and Notation (BPMN) diagram** depicting an end-to-end order fulfillment process. The diagram is organized into four horizontal swimlanes, each representing a different department:

## Swimlanes & Their Activities

### 1. Sales (Top Lane)
- **Start Event:** Triggered on "Monday morning" (timer event)
- **Check order for completeness**  **Check if customer is already in the database**
- A gateway (exclusive decision) splits the flow:
  - If **customer not in database**  **Add customer to the database** (writes to a database data store)
  - If **customer in database**  proceeds directly
- Flows merge, then **Send order details** (message event sent to Production and Procurement)
- Upon receiving "Product sent" message  **Draft invoice**  End event

### 2. Production (Second Lane)
- Starts when **Order details arrive** (message event)
- **Plan production**  parallel gateway
- Waits for a message: **Material not faulty**
- **Manufacture product**  ends with **Product complete** (message sent to Warehouse)

### 3. Procurement (Third Lane)
- Starts when **Order details arrive** (message event)
- Contains a **Material Compensation Subprocess**:
  - **Order materials** (with compensation handler: **Return material**)
- **Check materials** (event-based gateway) leads to either:
  - Sending a message (material OK, goes to Production)
  - **Order cancellation** (compensation throw event)  End

### 4. Warehouse (Bottom Lane)
- Triggered by **Product complete** message from Production
- **Send product**  sends "Product sent" message back to Sales

## Key BPMN Elements Used
- **Message flows** (dashed lines) connecting the lanes
- **Timer, message, and compensation events**
- **Exclusive (X) and parallel (+) gateways**
- **Data store** (database icon)
- **Subprocess** with compensation activity

Overall, the diagram illustrates how Sales, Production, Procurement, and Warehouse collaborate through message exchanges to process a customer order from receipt through invoicing.
