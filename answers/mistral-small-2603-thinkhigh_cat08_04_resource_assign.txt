### **Strategy 1: Skill-Based Routing with Proficiency Weighting**
**Problem Addressed:**
- Skill mismatches (e.g., INC-1001 assigned to Agent B12 lacking DB-SQL skill).

- Underutilization of specialists (e.g., "Security-IAM" skills idle during peak hours).

3. **Conformance Checking:** Ensures the new logic aligns with documented skill tiers (e.g., P1 tickets always go to L3).

**Implementation Details:**
- **Assignment Logic:**
  1. **Exact Match:** Assign to an agent with the *exact required skill* (e.g., "Database-SQL").
  2. **Proficiency-Weighted Match:** If no exact match, assign to the agent with the *closest skill* (e.g., "DB-SQL"  "Database-SQL") and highest historical resolution speed for that skill.
  3. **Tier Priority:** For P1/P2 tickets, prioritize higher-tier agents even if a lower-tier agent has the skill (to ensure SLA compliance).
- **Proficiency Scoring:**
  - Calculate an agent’s *skill proficiency score* = (1 / average resolution time for that skill) × (FCR rate for that skill).
  - Example: Agent X resolves "Networking-Firewall" tickets in 2 hours (score = 0.5), Agent Y in 4 hours (score = 0.25). Assign to Agent X if available.

**Data Required:**

- Historical resolution times and FCR rates per skill.
- Real-time agent availability (queue length, current workload).
