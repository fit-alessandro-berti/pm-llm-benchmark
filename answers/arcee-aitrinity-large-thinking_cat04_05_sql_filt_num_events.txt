Here is a DuckDB SQL query that filters out cases with more than six events and returns all events for the remaining cases:

```sql
WITH case_event_counts AS (
  SELECT 
    case_id, 
    COUNT(*) AS total_events
  FROM event_log
  GROUP BY case_id
)
SELECT 
  el.*
FROM event_log el
JOIN case_event_counts cec 
  ON el.case_id = cec.case_id
WHERE cec.total_events <= 6
ORDER BY el.case_id, el.timestamp;
```

**Explanation:**

1. The CTE `case_event_counts` computes the total number of events per `case_id`.
2. The main query joins the original `event_log` table with the CTE on `case_id`.
3. The `WHERE` clause filters to keep only those cases where the event count is  6.
4. The `ORDER BY` clause (optional) sorts the result by `case_id` and `timestamp` for readability.

This query correctly groups events by `case_id`, filters out cases exceeding the sixevent threshold, and returns the full set of events for the qualifying cases.
