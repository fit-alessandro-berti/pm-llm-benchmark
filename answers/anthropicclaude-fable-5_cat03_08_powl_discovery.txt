# POWL Model Construction

## Analysis of the Log

- **OR  MS  QS** always occur in sequence at the start.
- **AS** is followed by zero or more **IQC**s, possibly looping back to **AS** (case 3 even shows `IQC, IQC` consecutively, so IQC itself can repeat within one loop-back).
- **TST** may be followed by **RT**, which leads back to **TST** (cases 1, 3, 5; case 5 repeats this twice).
- **PK** always follows, then **LB** is optional (skipped in case 2).
- **DP  D** always close the case.

## POWL Structure (textual)

```
PO = ( nodes = { OR, MS, QS,
                 LOOP_AS = * ( AS, * ( IQC, tau ) ),
                 LOOP_TST = * ( TST, RT ),
                 PK,
                 XOR_LB = X ( LB, tau ),
                 DP, D },
       order = { OR --> MS,
                 MS --> QS,
                 QS --> LOOP_AS,
                 LOOP_AS --> LOOP_TST,
                 LOOP_TST --> PK,
                 PK --> XOR_LB,
                 XOR_LB --> DP,
                 DP --> D } )
```

- `* ( AS, * ( IQC, tau ) )`: execute AS; optionally execute one or more IQCs (the inner loop allows consecutive IQCs as in case 3) and return to AS; repeat until exit.
- `* ( TST, RT )`: execute TST; either exit or do RT and TST again (handles the multiple re-tests of case 5).
- `X ( LB, tau )`: labeling either happens or is skipped (case 2).

## Python (pm4py) Code

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Activities
OR  = Transition(label="OR")    # Order Received
MS  = Transition(label="MS")    # Material Sourcing
QS  = Transition(label="QS")    # Quality Screening
AS_ = Transition(label="AS")    # Assembly
IQC = Transition(label="IQC")   # In-line Quality Check
TST = Transition(label="TST")   # Testing
RT  = Transition(label="RT")    # Re-Testing
PK  = Transition(label="PK")    # Packaging
LB  = Transition(label="LB")    # Labeling
DP  = Transition(label="DP")    # Distribution Planning
D   = Transition(label="D")     # Dispatch

skip1 = SilentTransition()
skip2 = SilentTransition()

# Inner loop: allows one or more consecutive IQCs
iqc_loop = OperatorPOWL(operator=Operator.LOOP, children=[IQC, skip1])

# Assembly loop: AS, optionally (IQC+ then AS again), repeated
as_loop = OperatorPOWL(operator=Operator.LOOP, children=[AS_, iqc_loop])

# Testing loop: TST, optionally (RT then TST again), repeated
tst_loop = OperatorPOWL(operator=Operator.LOOP, children=[TST, RT])

# Optional labeling
xor_lb = OperatorPOWL(operator=Operator.XOR, children=[LB, skip2])

# Root partial order
root = StrictPartialOrder(nodes=[OR, MS, QS, as_loop, tst_loop, PK, xor_lb, DP, D])
root.order.add_edge(OR, MS)
root.order.add_edge(MS, QS)
root.order.add_edge(QS, as_loop)
root.order.add_edge(as_loop, tst_loop)
root.order.add_edge(tst_loop, PK)
root.order.add_edge(PK, xor_lb)
root.order.add_edge(xor_lb, DP)
root.order.add_edge(DP, D)
```

## Verification Against the Log

| Case | Trace | Covered by model |
|------|-------|------------------|
| 1 | OR, MS, QS, AS, IQC, AS, TST, RT, TST, PK, LB, DP, D |  (one AS-loop iteration, one re-test, LB taken) |
| 2 | OR, MS, QS, AS, IQC, AS, TST, PK, DP, D |  (no re-test, LB skipped via tau) |
| 3 | OR, MS, QS, AS, IQC, IQC, AS, IQC, AS, TST, RT, TST, PK, LB, DP, D |  (inner IQC loop allows consecutive IQCs) |
| 4 | OR, MS, QS, AS, TST, PK, LB, DP, D |  (both loops exit immediately) |
| 5 | OR, MS, QS, AS, IQC, AS, IQC, AS, TST, RT, TST, RT, TST, PK, LB, DP, D |  (two AS-loop iterations, two re-tests) |
