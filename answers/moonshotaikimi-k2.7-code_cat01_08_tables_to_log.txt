## Assumptions

- **Case identifier:** `order_id`.
- **Activity derivation:**  
  - `Orders`  **Order Created**.  
  - `Order Lines`  one **Item Picked** event per line (because each line has its own picker/timestamp).  
  - `Delivery`  one **Order Dispatched** and one **Order Arrived** event per delivery record (split shipments are kept as separate events).  
  - `Invoice`  one **Invoice Issued** event per invoice.  
  - `Payments`  one **Payment Received** event per payment, linked to the order through the invoice.
- **Resources:** Resource IDs were mapped to names/roles using the **Resources** table.  
  - Payments do not include an explicit resource, so the actor is recorded as **Payment System**.
- **Product enrichment:** Item descriptions/categories from the **Products** table are included in the `Item Picked` events.
- **Missing data:** `order_id = 1004` has no lines, deliveries, invoices, or payments, so it only contains the **Order Created** event.
