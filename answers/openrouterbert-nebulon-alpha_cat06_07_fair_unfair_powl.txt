Here are the two POWL models representing the hiring process at GlobalTeq Solutions, with and without potential unfairness in the cultural fit evaluation:

---

### **1. POWL Model with Potential Unfairness**
This model includes an XOR branch where applicants with community affiliations receive preferential treatment.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Define loop for data completeness
loop_body = StrictPartialOrder(nodes=[DataCompletenessCheck, RequestMoreInfo])
loop_body.order.add_edge(DataCompletenessCheck, RequestMoreInfo)
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[ReceiveApplication, loop_body])

# Define XOR for cultural fit vs. community affiliation
cultural_fit_xor = OperatorPOWL(
    operator=Operator.XOR,
    children=[CulturalFitCheck, CommunityAffiliationCheck]
)

# Define partial order for the main process
root = StrictPartialOrder(nodes=[
    data_loop,
    SkillAssessment,
    cultural_fit_xor,
    ManagerialReview,
    FinalDecision
])

# Set execution order
root.order.add_edge(data_loop, SkillAssessment)
root.order.add_edge(SkillAssessment, cultural_fit_xor)
root.order.add_edge(cultural_fit_xor, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)
```

**Explanation:**
- The **loop** ensures applicants provide complete data before proceeding.
- After the **SkillAssessment**, an **XOR** branch allows either a standard **CulturalFitCheck** or a **CommunityAffiliationCheck** (which may introduce bias).
- The rest of the process (**ManagerialReview**, **FinalDecision**) proceeds sequentially.

---

### **2. POWL Model Without Unfairness**
This model removes the XOR branch, ensuring all applicants undergo the same cultural fit evaluation.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")  # Only one path now
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Define loop for data completeness
loop_body = StrictPartialOrder(nodes=[DataCompletenessCheck, RequestMoreInfo])
loop_body.order.add_edge(DataCompletenessCheck, RequestMoreInfo)
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[ReceiveApplication, loop_body])

# Define partial order for the main process (no XOR branch)
root = StrictPartialOrder(nodes=[
    data_loop,
    SkillAssessment,
    CulturalFitCheck,  # All applicants go through the same check
    ManagerialReview,
    FinalDecision
])

# Set execution order
root.order.add_edge(data_loop, SkillAssessment)
root.order.add_edge(SkillAssessment, CulturalFitCheck)
root.order.add_edge(CulturalFitCheck, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)
```

**Explanation:**
- The **loop** remains for data completeness.
- After **SkillAssessment**, **all applicants** proceed to the same **CulturalFitCheck** (no XOR branch).
- The rest of the process (**ManagerialReview**, **FinalDecision**) is unchanged.

---

### **Key Differences:**
| **Feature**               | **Model with Unfairness**                     | **Model Without Unfairness**               |
|---------------------------|-----------------------------------------------|--------------------------------------------|
| **Cultural Fit Evaluation** | XOR branch: `CulturalFitCheck` or `CommunityAffiliationCheck` | Single path: `CulturalFitCheck` for all |
| **Bias Risk**             | High (preferential treatment for affiliates)  | Low (uniform evaluation)                  |
| **Process Fairness**      | Compromised by XOR branch                     | Ensured by removing XOR branch            |

These models clearly distinguish where bias may enter (XOR branch) and how to eliminate it (single evaluation path).
