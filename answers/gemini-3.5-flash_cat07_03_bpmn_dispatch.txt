Based on the provided BPMN (Business Process Model and Notation) diagram, here is a detailed description of the business process for the **"Dispatch of goods"** at a **"Computer Hardware Shop"**.

---

### **Overview**
The diagram depicts a workflow divided into a single pool (**Dispatch of goods Computer Hardware Shop**) containing three horizontal lanes representing different departments or roles:
1. **Logistics** (Top)
2. **Secretary** (Middle)
3. **Warehouse** (Bottom)

---

### **Step-by-Step Process Flow**

#### **1. Process Initiation**
* The process begins with the start event **"Ship goods"** in the **Secretary** lane.
* Immediately after starting, the process hits a **Parallel Gateway (+)**, splitting the workflow into two parallel branches that run simultaneously: one managed by the **Secretary** and the other by the **Warehouse**.

---

#### **2. Secretary Branch (Middle Lane)**
This branch handles the administrative and planning aspects of the shipment.
* **Step 1:** The Secretary performs the task **"Clarify shipment method"**, which references an input document/data object called **"Order details"**.
* **Step 2:** The flow enters an **Exclusive Gateway (X)** deciding on **"Special handling?"**:
    * **Path A: Yes (Special Handling Required)**
        * The Secretary performs the task **"Get 3 offers from logistic companies"**.
        * Subsequently, they perform the task **"Select logistic company and place order"**.
        * The flow then moves to a merging Exclusive Gateway.
    * **Path B: No (Standard Handling)**
        * The Secretary performs the task **"Check if insurance is required"**, which queries a database containing **"Customer information"**.
        * After this, an **Inclusive Gateway (O)** splits the path:
            * **Path B1 ("If insurance necessary"):** The flow goes up to the **Logistics** lane where the task **"Insure parcel"** is performed.
            * **Path B2 ("always"):** The Secretary performs the task **"Write package label"**.
            * Both paths merge back at another **Inclusive Gateway (O)** before moving to the final merging Exclusive Gateway.

Both Path A and Path B converge at an **Exclusive Gateway (X)** and proceed to the final synchronization gateway.

---

#### **3. Warehouse Branch (Bottom Lane)**
This branch handles the physical preparation of the goods.
* **Step 1:** The Warehouse staff performs the task **"Package goods"**.
* **Exception Handling (Timer):** Attached to the "Package goods" task is a **boundary non-interrupting timer event** set to **"48 hours"**. 
    * If the packaging takes longer than 48 hours, a parallel path is triggered to **"Notify customer about delay"**.
* Both the standard packaging flow and the notification flow (if triggered) merge at an **Inclusive Gateway (O)**.

---

#### **4. Synchronization and Completion**
* The outputs from the **Secretary's branch** and the **Warehouse's branch** meet at a **Parallel Gateway (+)**. This ensures both administrative preparation and physical packaging are complete before moving forward.
* Once synchronized, the workflow proceeds in the **Warehouse** lane with the final steps:
    * **"Fill out Shipment paperwork"**
    * **"Move package to pickup area"**
* The process concludes at the end event: **"Shipment prepared"**.
