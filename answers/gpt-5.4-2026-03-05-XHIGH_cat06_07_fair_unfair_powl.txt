Below are two **pm4py-style POWL models** for the hiring process.  
Both include:

- application intake,
- a **loop** for incomplete data,
- a **skill assessment**,
- managerial review,
- final decision.

They differ only in how they model the cultural-fit stage.

---

## 1) POWL model with potential unfairness

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# Activities
ra1 = Transition(label="ReceiveApplication")
rp1 = Transition(label="ResumeParsing")
dc1 = Transition(label="DataCompletenessCheck")
rmi1 = Transition(label="RequestMoreInfo")
sa1 = Transition(label="SkillAssessment")
cf1 = Transition(label="CulturalFitCheck")
ca1 = Transition(label="CommunityAffiliationCheck")
mr1 = Transition(label="ManagerialReview")
fd1 = Transition(label="FinalDecision")

# Resume parsing followed by completeness check
check_block1 = StrictPartialOrder(nodes=[rp1, dc1])
check_block1.order.add_edge(rp1, dc1)

# Loop: if data is incomplete, request more info and re-check
completeness_loop1 = OperatorPOWL(
    operator=Operator.LOOP,
    children=[check_block1, rmi1]
)

# Potentially unfair XOR:
# either standard cultural fit, or an affiliation-based path
culture_branch1 = OperatorPOWL(
    operator=Operator.XOR,
    children=[cf1, ca1]
)

# Whole process
unfair_model = StrictPartialOrder(
    nodes=[ra1, completeness_loop1, sa1, culture_branch1, mr1, fd1]
)
unfair_model.order.add_edge(ra1, completeness_loop1)
unfair_model.order.add_edge(completeness_loop1, sa1)
unfair_model.order.add_edge(sa1, culture_branch1)
unfair_model.order.add_edge(culture_branch1, mr1)
unfair_model.order.add_edge(mr1, fd1)
```

### Compact POWL view
- `CheckBlock1 = PO=(nodes={ResumeParsing, DataCompletenessCheck}, order={ResumeParsing-->DataCompletenessCheck})`
- `CompletenessLoop1 = *(CheckBlock1, RequestMoreInfo)`
- `CultureBranch1 = X(CulturalFitCheck, CommunityAffiliationCheck)`
- `UnfairModel = PO=(nodes={ReceiveApplication, CompletenessLoop1, SkillAssessment, CultureBranch1, ManagerialReview, FinalDecision},`
  `order={ReceiveApplication-->CompletenessLoop1, CompletenessLoop1-->SkillAssessment, SkillAssessment-->CultureBranch1, CultureBranch1-->ManagerialReview, ManagerialReview-->FinalDecision})`

**Where unfairness appears:**  
The bias point is the XOR branch  
`X(CulturalFitCheck, CommunityAffiliationCheck)`  
because some applicants can be routed through an affiliation-sensitive path that may favor local/community-linked candidates.

## 2) POWL model without unfairness

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# Activities
ra2 = Transition(label="ReceiveApplication")
rp2 = Transition(label="ResumeParsing")
dc2 = Transition(label="DataCompletenessCheck")
rmi2 = Transition(label="RequestMoreInfo")
sa2 = Transition(label="SkillAssessment")
cf2 = Transition(label="CulturalFitCheck")
mr2 = Transition(label="ManagerialReview")
fd2 = Transition(label="FinalDecision")

# Resume parsing followed by completeness check
check_block2 = StrictPartialOrder(nodes=[rp2, dc2])
check_block2.order.add_edge(rp2, dc2)

# Loop: if data is incomplete, request more info and re-check
completeness_loop2 = OperatorPOWL(
    operator=Operator.LOOP,
    children=[check_block2, rmi2]
)

# Whole process: no community-based branch
fair_model = StrictPartialOrder(
    nodes=[ra2, completeness_loop2, sa2, cf2, mr2, fd2]
)
fair_model.order.add_edge(ra2, completeness_loop2)
fair_model.order.add_edge(completeness_loop2, sa2)
fair_model.order.add_edge(sa2, cf2)
fair_model.order.add_edge(cf2, mr2)
fair_model.order.add_edge(mr2, fd2)
```

### Compact POWL view
- `CheckBlock2 = PO=(nodes={ResumeParsing, DataCompletenessCheck}, order={ResumeParsing-->DataCompletenessCheck})`
- `CompletenessLoop2 = *(CheckBlock2, RequestMoreInfo)`
- `FairModel = PO=(nodes={ReceiveApplication, CompletenessLoop2, SkillAssessment, CulturalFitCheck, ManagerialReview, FinalDecision},`
  `order={ReceiveApplication-->CompletenessLoop2, CompletenessLoop2-->SkillAssessment, SkillAssessment-->CulturalFitCheck, CulturalFitCheck-->ManagerialReview, ManagerialReview-->FinalDecision})`

**Why this is fairer:**  
All applicants follow the same path after `SkillAssessment`, with a single `CulturalFitCheck` and no affiliation-based XOR branch.
