**Processtree model (textual notation)**  

```
-> (
    'RA',                      Receive Application
    'VF',                      Validate Form
    'AE',                      Assess Eligibility
    X(                        exclusive choice: either skip the loop or execute it
        tau,                   “skip” –> case2 (no missinginfo loop)
        *(                     loop: zero or more repetitions of “RMI then VF”
            -> ( 'RMI', 'VF' ),    Request Missing Info    Validate Form
            tau                     redopart does nothing, just forces another iteration
        )
    ),
    'AD',                      Approval Decision
    'SC',                      Send Confirmation
    'AC'                       Archive Case
)
```

**Explanation of the structure**

* **Sequence (`->`)** at the root enforces the overall order of the main activities.  
* After the three mandatory steps `RA  VF  AE` an **exclusive choice (`X`)** decides whether the process follows the “missinginformation” loop or jumps directly to the approval decision.  
  * The left branch is a silent step (`tau`), representing the cases where no request for missing information occurs (case2).  
  * The right branch is a **loop (`*`)** whose body is the sequence `RMI  VF`.  
    * The loop’s redopart is again a silent step (`tau`); therefore each iteration produces exactly one `RMI` followed by one `VF`.  
    * Because the loop can be taken zero or more times, the XOR in front of it also allows the case where the loop is never entered (the “skip” branch).  
* After the loop (or its skip) the process continues with the fixed tail `AD  SC  AC`.

This tree captures all observed traces:

| Trace | Captured by the tree |
|-------|-----------------------|
| `RA VF AE AD SC AC` | `RA  VF  AE  (skip loop)  AD  SC  AC` |
| `RA VF AE RMI VF AD SC AC` | `RA  VF  AE  (loop: RMI VF)  AD  SC  AC` |
| `RA VF AE RMI RMI VF AD SC AC` | `RA  VF  AE  (loop: RMI VF RMI VF)  AD  SC  AC` |

The model balances **fitness** (covers all log traces), **precision** (does not allow extraneous behaviour) and **simplicity** (uses a single optional loop and a short fixed suffix).
