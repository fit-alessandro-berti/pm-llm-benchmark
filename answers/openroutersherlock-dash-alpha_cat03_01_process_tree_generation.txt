# 2. Triage sequence: TI -> TC
triage_seq = ProcessTree(operator=Operator.SEQUENCE)
TI.parent = triage_seq

triage_seq.children = [TI, TC]

initial_diag_xor = ProcessTree(operator=Operator.XOR)

treat_xor = ProcessTree(operator=Operator.XOR)
TM.parent = treat_xor
SPT.parent = treat_xor
treat_xor.children = [TM, SPT]

loop_seq = ProcessTree(operator=Operator.SEQUENCE)

overall_final_seq = ProcessTree(operator=Operator.SEQUENCE)

adv_parallel.parent = overall_final_seq
final_seq.parent = overall_final_seq

# ROOT: triage_after_initial -> overall_final_seq
root = ProcessTree(operator=Operator.SEQUENCE)

This process tree exactly models the described hospital patient management scenario:

1. **Root sequence**: Entire process flows linearly through major phases
2. **Initial registration/assessment**: A  RG  IA 
3. **Triage**: TI  TC (mandatory sequence)
4. **Initial diagnostics**: Exclusive choice CE  FBP
5. **Diagnostic-treatment loop**: After initial diagnostics, loop containing:
   - SC (mandatory)
   - Exclusive treatment choice TM  SPT  
   - RDT (exclusive choice FCE  FBA)
   - Loop can repeat until normal results (via LOOP operator semantics)
6. **Concurrent advanced care**: AI + NC (true parallel execution)
7. **Final clinical sequence**: FAS  DDI
8. **Final admin concurrency**: BL + ICP + MRF (all three in parallel)

The tree respects all operator semantics and properly sets parent-child relationships as required by pm4py's ProcessTree implementation.
