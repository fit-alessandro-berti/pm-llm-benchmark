## Explanation of Case Grouping and Activity Naming Logic

### Case Identification Strategy

I identified **5 distinct cases** based on **document-centric work sessions**. Each case represents a coherent work unit centered on a specific document or task:

- **CASE_001**: Quarterly Report editing (Quarterly_Report.docx) - initiated at 08:59:50, returned to after other tasks, and finalized at 09:08:15
- **CASE_002**: General Document editing (Document1.docx) - created, edited with budget references, and completed as an integrated work session despite interruptions
- **CASE_003**: Email handling (Annual Meeting email) - discrete email task including review, reply composition, and sending
- **CASE_004**: PDF review (Report_Draft.pdf) - brief focused review and annotation task
- **CASE_005**: Budget spreadsheet update (Budget_2024.xlsx) - isolated spreadsheet editing session

### Activity Naming Standardization

I converted low-level raw actions into **meaningful process activities**:

| Raw Actions | Standardized Activity |
|-------------|----------------------|
| FOCUS, SWITCH (to document) | Open Document |
| TYPING (content entry) | Edit Content |
| SAVE | Save Document |
| CLOSE | Close Document |
| CLICK (open email), SCROLL (email context) | Review Email / Review Content |
| TYPING (email reply) + CLICK (reply) | Compose Response |
| CLICK (send) | Send Email |
| SCROLL (PDF context) | Review Content |
| HIGHLIGHT | Annotate Document |

### Key Design Decisions

1. **Temporal Clustering**: Cases are grouped by document focus, allowing the user to return to CASE_001 and CASE_002 after context switches, reflecting realistic work patterns.

2. **Activity Abstraction**: Generic actions (TYPING, CLICK, SCROLL) are context-aware—the same action translates to different activities depending on the application and workflow state.

3. **Logical Completeness**: Each case begins with an open/focus event and ideally ends with close/save events, creating complete process narratives suitable for process mining analysis.

This structure enables meaningful process discovery, conformance checking, and performance analysis.
