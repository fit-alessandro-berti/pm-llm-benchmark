Here is a clean DuckDB-compliant SQL query that fulfills your requirements:

```sql
WITH case_counts AS (
    SELECT 
        case_id,
        COUNT(*) AS event_count
    FROM 
        event_log
    GROUP BY 
        case_id
    HAVING 
        COUNT(*) <= 6
)
SELECT 
    e.*
FROM 
    event_log e
JOIN 
    case_counts c ON e.case_id = c.case_id;
```

### Explanation:
- **Common Table Expression (CTE) `case_counts`**: Groups events by `case_id` and computes the total number of events per case using `COUNT(*)`. The `HAVING` clause filters out any cases with more than six events.
- **Main Query**: Joins the original `event_log` table with the filtered CTE (`case_counts`) to retrieve **only the full set of events belonging to cases with six or fewer events**.

This approach is efficient and idiomatic for DuckDB, leveraging its support for CTEs and standard SQL syntax.
