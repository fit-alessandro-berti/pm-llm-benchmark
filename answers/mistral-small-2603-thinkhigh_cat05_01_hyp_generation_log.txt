##  **SQL Queries to Investigate Anomalies**

### **Query 1: Out-of-Sequence Activities**
**Purpose:** Detect cases where activities occur in illogical orders.
**Logic:** Compare the actual sequence of activities against the expected process flow.

```sql
WITH expected_sequence AS (
    SELECT
        'Register Order' AS activity,
        1 AS step_order
    UNION ALL SELECT 'Perform Credit Check', 2
    UNION ALL SELECT 'Validate Stock', 3
    UNION ALL SELECT 'Confirm Shipment', 4
    UNION ALL SELECT 'Ship Goods', 5
    UNION ALL SELECT 'Issue Invoice', 6
    UNION ALL SELECT 'Receive Payment', 7
),
ordered_events AS (
    SELECT
        oel.case_id,
        oel.activity,
        oel.timestamp,
        es.step_order,
        ROW_NUMBER() OVER (PARTITION BY oel.case_id ORDER BY oel.timestamp) AS event_seq
    FROM order_event_log oel
    JOIN expected_sequence es ON oel.activity = es.activity
)
SELECT
    oe1.case_id,
    oe1.activity AS current_activity,
    oe1.timestamp AS current_timestamp,
    oe2.activity AS previous_activity,
    oe2.timestamp AS previous_timestamp,
    CASE
        WHEN oe2.step_order IS NULL THEN 'First activity (no prior step)'
        WHEN oe1.step_order < oe2.step_order THEN ' ANOMALY: Out-of-sequence activity'
        ELSE ' Correct sequence'
    END AS sequence_status
FROM ordered_events oe1
LEFT JOIN ordered_events oe2
    ON oe1.case_id = oe2.case_id
    AND oe1.event_seq = oe2.event_seq + 1
ORDER BY oe1.case_id, oe1.event_seq;
```

**Expected Output:**
Flags cases like 1002 (*Perform Credit Check* after *Ship Goods*) and 1004 (*Receive Payment* before *Issue Invoice*).

### **Query 2: Missing Mandatory Activities**
**Purpose:** Identify cases where critical steps are skipped.
**Logic:** Check for absence of required activities per case.

```sql
WITH required_activities AS (
    SELECT 'Register Order' AS activity UNION ALL
    SELECT 'Perform Credit Check' UNION ALL
    SELECT 'Validate Stock' UNION ALL
    SELECT 'Confirm Shipment' UNION ALL
    SELECT 'Ship Goods' UNION ALL
    SELECT 'Issue Invoice' UNION ALL
    SELECT 'Receive Payment'
)
SELECT
    o.case_id,
    o.order_type,
    o.order_value,
    ra.activity AS missing_activity,
    CASE
        WHEN NOT EXISTS (
            SELECT 1 FROM order_event_log oel
            WHERE oel.case_id = o.case_id AND oel.activity = ra.activity
        ) THEN ' Missing'
        ELSE ' Present'
    END AS status
FROM orders o
CROSS JOIN required_activities ra
WHERE NOT EXISTS (
    SELECT 1 FROM order_event_log oel
    WHERE oel.case_id = o.case_id AND oel.activity = ra.activity
)
ORDER BY o.case_id, ra.activity;
```

**Expected Output:**
- Case 1004: Missing *Perform Credit Check* and *Validate Stock*.
- Case 1003: Missing *Validate Stock* (if not logged).

### **Query 3: Wrong Department Performing Activities**
**Purpose:** Detect activities performed by incorrect departments.
**Logic:** Join `order_event_log` with `resources` and validate department alignment.

```sql
WITH activity_departments AS (
    SELECT
        'Register Order' AS activity,
        'Sales' AS expected_department
    UNION ALL SELECT 'Perform Credit Check', 'Finance'
    UNION ALL SELECT 'Validate Stock', 'Logistics'
    UNION ALL SELECT 'Confirm Shipment', 'Logistics'
    UNION ALL SELECT 'Ship Goods', 'Logistics'
    UNION ALL SELECT 'Issue Invoice', 'Finance'
    UNION ALL SELECT 'Receive Payment', 'Finance'
)
SELECT
    oel.case_id,
    oel.activity,
    oel.timestamp,
    oel.resource,
    r.department AS actual_department,

JOIN activity_departments ad ON oel.activity = ad.activity
```

### **Query 4: Duplicate Activities**
**Purpose:** Find cases with multiple instances of the same activity.
**Logic:** Group by `case_id` and `activity` to count occurrences.

```sql
SELECT
    case_id,
```

### **Query 5: Premature Activities (Time-Based Anomalies)**
**Purpose:** Detect illogical timing between activities (e.g., payment before invoice).
**Logic:** Compare timestamps of dependent activities.

```sql
WITH activity_pairs AS (
    SELECT
        'Receive Payment' AS later_activity,
        'Issue Invoice' AS earlier_activity
    UNION ALL SELECT 'Ship Goods', 'Confirm Shipment'
    UNION ALL SELECT 'Issue Invoice', 'Ship Goods'
)
SELECT
    oel1.case_id,
    oel1.activity AS later_activity,
    oel1.timestamp AS later_timestamp,
    oel2.activity AS earlier_activity,
    oel2.timestamp AS earlier_timestamp,
    CASE
        WHEN oel1.timestamp < oel2.timestamp THEN ' ANOMALY: Later activity occurred first'
        ELSE ' Correct order'
    END AS time_order_status
FROM order_event_log oel1
JOIN order_event_log oel2 ON oel1.case_id = oel2.case_id
JOIN activity_pairs ap ON oel1.activity = ap.later_activity AND oel2.activity = ap.earlier_activity
ORDER BY oel1.case_id;
```

**Expected Output:**
- Case 1004: *Receive Payment* (2024-02-01 09:05:00) before *Issue Invoice* (2024-02-01 09:20:00).
