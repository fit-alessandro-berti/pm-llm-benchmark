Here's a clean, efficient DuckDB SQL query that fulfills your requirements:

```sql
WITH valid_cases AS (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT e.*
FROM event_log e
JOIN valid_cases vc ON e.case_id = vc.case_id;
```

###  How it works:
1. **`valid_cases` CTE**: Groups the log by `case_id` and uses `HAVING COUNT(*) <= 6` to keep only cases with 6 or fewer events. `GROUP BY` ensures each `case_id` appears exactly once.
2. **Main `SELECT`**: Joins the original `event_log` back to `valid_cases` on `case_id`, restoring all original columns (`e.*`) for the filtered cases.
3. **Correct filtering logic**: Cases with `>6` events are excluded at the CTE level, so they never enter the final result set.

###  DuckDB-Specific Alternative (More Concise)
DuckDB supports the `QUALIFY` clause, which lets you filter on window functions directly without a `JOIN` or CTE:

```sql
SELECT case_id, activity, timestamp
FROM event_log
QUALIFY COUNT(*) OVER (PARTITION BY case_id) <= 6;
```
This is functionally identical, often faster, and highly idiomatic in DuckDB.

Both approaches correctly group, filter, and return the full event rows for cases meeting your threshold. Choose the CTE+JOIN for maximum readability/portability, or `QUALIFY` for DuckDB-native conciseness.
