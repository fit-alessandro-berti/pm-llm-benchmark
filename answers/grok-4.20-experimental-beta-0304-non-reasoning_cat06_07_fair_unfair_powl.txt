**POWL Model 1: With Potential Unfairness** (contains the biased XOR branch)

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
StandardCulturalFitCheck = Transition(label="StandardCulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Loop for data completeness: DataCompletenessCheck -> (RequestMoreInfo -> DataCompletenessCheck)*
data_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[DataCompletenessCheck, RequestMoreInfo]
)

# XOR after skill assessment: either standard cultural fit OR community-affiliation path (bias point)
cultural_xor = OperatorPOWL(
    operator=Operator.XOR,
    children=[StandardCulturalFitCheck, CommunityAffiliationCheck]
)

# Main partial order
model_unfair = StrictPartialOrder(nodes=[
    ReceiveApplication,
    data_loop,
    SkillAssessment,
    cultural_xor,
    ManagerialReview,
    FinalDecision
])

# Define ordering
model_unfair.order.add_edge(ReceiveApplication, data_loop)
model_unfair.order.add_edge(data_loop, SkillAssessment)
model_unfair.order.add_edge(SkillAssessment, cultural_xor)
model_unfair.order.add_edge(cultural_xor, ManagerialReview)
model_unfair.order.add_edge(ManagerialReview, FinalDecision)
```

**POWL Model 2: Without Unfairness** (no special community branch — all applicants go through the same cultural fit evaluation)

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# Activities (no CommunityAffiliationCheck)
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")          # same path for everyone
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Same data completeness loop
data_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[DataCompletenessCheck, RequestMoreInfo]
)

# Main partial order - no XOR for cultural evaluation
model_fair = StrictPartialOrder(nodes=[
    ReceiveApplication,
    data_loop,
    SkillAssessment,
    CulturalFitCheck,
    ManagerialReview,
    FinalDecision
])

# Define ordering - single path after skill assessment
model_fair.order.add_edge(ReceiveApplication, data_loop)
model_fair.order.add_edge(data_loop, SkillAssessment)
model_fair.order.add_edge(SkillAssessment, CulturalFitCheck)
model_fair.order.add_edge(CulturalFitCheck, ManagerialReview)
model_fair.order.add_edge(ManagerialReview, FinalDecision)
```

These two models faithfully capture the hiring process described:

- **Model 1** explicitly introduces the unfair XOR branch at the cultural-fit stage, allowing the “CommunityAffiliationCheck” path that can give preferential treatment.
- **Model 2** removes that bias point by having a single, uniform `CulturalFitCheck` for all applicants after the skill assessment.
