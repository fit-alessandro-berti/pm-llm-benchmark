Here are two efficient ways to achieve this in DuckDB. Both approaches first identify which `case_id` groups have 6 events, then return all original events that belong to those groups.

### Approach 1: Using a Subquery with `IN`
This is often the most intuitive method:  
```sql
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
);
```
**Explanation:**  
1. The subquery counts events per `case_id` and keeps only those with a count of 6 or less.  
2. The outer query returns all rows whose `case_id` appears in that filtered list.

---

### Approach 2: Using a Window Function (Potentially More Efficient on Large Data)
```sql
SELECT *
FROM (
    SELECT *,
           COUNT(*) OVER (PARTITION BY case_id) AS events_per_case
    FROM event_log
) sub
WHERE events_per_case <= 6;
```
**Explanation:**  
1. The inner query adds a column `events_per_case` by counting all rows within each `case_id` partition.  
2. The outer query filters rows where that count is 6, then returns the original columns (you can select specific columns instead of `*`).

---

### Which one to choose?
- **Approach 1** is usually clearer for most readers and works well in DuckDB.  
- **Approach 2** can be more efficient if the dataset is very large, since it avoids a subquery and scans the table only once.

Both will correctly return all events from cases with 6 or fewer events, while excluding entire cases with more than 6 events.
