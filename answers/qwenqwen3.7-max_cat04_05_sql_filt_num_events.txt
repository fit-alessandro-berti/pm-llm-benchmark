Here are two ways to write this query in DuckDB. 

### Method 1: The DuckDB-Idiomatic Way (Using `QUALIFY`)
DuckDB supports the `QUALIFY` clause, which allows you to filter directly on the results of window functions. This is the cleanest and most elegant approach:

```sql
SELECT *
FROM event_log
QUALIFY COUNT(*) OVER (PARTITION BY case_id) <= 6;
```

**How it works:**
* `COUNT(*) OVER (PARTITION BY case_id)` calculates the total number of events for each case and appends it to every row without collapsing the result set.
* `QUALIFY` acts like a `WHERE` clause, but specifically for window functions. It filters out any rows where the calculated count is greater than 6.

---

### Method 2: The Standard SQL Way (Using a CTE)
If you prefer standard SQL that is easily portable to other database systems that might not support `QUALIFY`, you can use a Common Table Expression (CTE) with a `JOIN` or `IN` clause:

```sql
WITH valid_cases AS (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT e.*
FROM event_log e
INNER JOIN valid_cases v 
    ON e.case_id = v.case_id;
```

**How it works:**
* The `valid_cases` CTE groups the log by `case_id` and uses the `HAVING` clause to isolate only the cases with 6 or fewer events.
* The main query then joins the original `event_log` table against this filtered list of `case_id`s, returning the full event details for the valid cases only. 

*Note: Both queries will return all original columns. If your table has additional columns beyond `case_id`, `activity`, and `timestamp`, they will be included automatically.*
