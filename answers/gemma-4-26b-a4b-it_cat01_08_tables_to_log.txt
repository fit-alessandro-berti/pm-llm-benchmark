### Assumptions and Data Integration Notes

To produce this event log, the following logical steps and assumptions were made:
1.  **Granularity of Picking**: Each item in the `Order Lines` table is treated as an individual "Item Picked" event. This allows for process mining of warehouse efficiency and handling of partial picks.
2.  **Resource Mapping**: Resource IDs (e.g., `SP-001`, `WH-001`) were mapped to their corresponding human-readable names and roles using the `Resources` table.
3.  **Payment Actor**: The `Payments` table does not specify a resource. In a business context, payments are initiated by the customer; therefore, the Resource is labeled as "Customer".
4.  **Incomplete Processes**: Order `1004` only contains data up to the "Item Picked" stage. This is included as-is, representing a process that has not yet reached the delivery or billing stage in the provided dataset.
5.  **Split Deliveries/Invoices**: For Order `1002`, which features multiple deliveries and multiple invoices, each delivery and each invoice is recorded as a unique event to maintain the integrity of the physical and financial flow.
6.  **Chronological Integrity**: All events have been sorted by `Case ID` and then by `Timestamp` to ensure the sequence is logically sound for process mining tools.
