**Assumption:** all events can be treated as `lifecycle:transition = complete`.

### Grouping and naming logic
- I used an **artifact-centric case notion**: each named work item became a case:
  - `DOC_Document1.docx`
  - `DOC_Quarterly_Report.docx`
  - `EMAIL_Annual_Meeting`
  - `PDF_Report_Draft.pdf`
  - `XLS_Budget_2024.xlsx`
- This is the cleanest standard event log because the log clearly names files and one email subject, while the user multitasks across them.
- Low-level actions were lifted as follows:
  - `FOCUS` / `SWITCH`  **Access ...**
  - `TYPING`  **Edit Document / Edit Spreadsheet / Compose Reply**
  - `SCROLL`  **Read Email / Review PDF**
  - `HIGHLIGHT`  **Annotate PDF**
  - `CLICK`  semantic actions like **Open Email**, **Initiate Reply**, **Send Reply**
  - `SAVE` / `CLOSE` kept as **Save** / **Close**
- The `detail` field keeps the original text/action context so activity names stay standardized.
