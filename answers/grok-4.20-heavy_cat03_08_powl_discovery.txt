- **Sequential backbone** (via top-level `StrictPartialOrder` / PO with total order): OR  MS  QS  (assembly/quality)  (testing)  PK  (optional LB)  DP  D. No true concurrency is observed in the log; all traces are linear per case.
- **Assembly/quality block** (`asm_qual = * (AS, iqcs)` with `iqcs = * (IQC, tau)`): Always starts with AS; allows zero or more groups of 1+ IQCs followed by additional AS (i.e. AS (IQC AS)*). 
  - Matches case 4 (just AS, no IQC).
  - Matches cases 1/2/5 (AS IQC AS or AS IQC AS IQC AS).
  - Matches case 3 (AS IQC IQC AS IQC AS) via consecutive IQCs inside the inner loop.
- **Testing block** (`tests = * (TST, RT)`): TST (then optionally RT + TST, repeatable). Matches all variants (no RT, one RT, two RTs).
- **Optional labeling** (`lb_choice = X (LB, tau)`): LB may be skipped (as in case 2); uses silent transition for the "do-nothing" branch.
- All leaf nodes are `Transition(label=...)` (or `SilentTransition()` for tau). Nested `OperatorPOWL` (LOOP/XOR) handle the control-flow patterns; the root PO wires the major blocks together.

This POWL reproduces all observed traces while staying compact. The nested loop for IQCs is necessary to allow consecutive IQCs (only case 3) while still guaranteeing the block always ends on AS before moving to testing.
