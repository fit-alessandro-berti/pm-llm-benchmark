# BPMN Diagram: Dispatch of Goods - Computer Hardware Shop

This is a **Business Process Model and Notation (BPMN)** diagram depicting the goods dispatch process at a computer hardware shop. The process is organized into three swim lanes representing different departments:

## Swim Lanes

### 1. **Logistics**
- Contains the "Insure parcel" task, which is triggered when insurance is necessary

### 2. **Secretary**
- **Clarify shipment method** - Initial task after the process starts
- **Special handling?** - Exclusive gateway (XOR) decision point
  - **Yes path**: Get 3 offers from logistic companies  Select logistic company and place order
  - **No path**: Check if insurance is required (uses Customer Information database and Order Details)
- **Write package label** - Always executed after insurance check

### 3. **Warehouse**
- **Package goods** - With a 48-hour timer boundary event
  - If delayed: **Notify customer about delay**
- **Fill out Shipment paperwork**
- **Move package to pickup area**
- **End event**: "Shipment prepared"

## Key Elements
- **Start Event**: Ship goods
- **Parallel Gateways (+)**: Synchronize multiple paths
- **Exclusive Gateways (X)**: Decision points
- **Intermediate Events**: Timer and conditional triggers
- **Data Objects**: Order details, Customer information
