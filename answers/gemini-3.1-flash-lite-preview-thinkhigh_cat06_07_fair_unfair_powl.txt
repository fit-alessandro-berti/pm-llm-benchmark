To model the hiring process at GlobalTeq Solutions using POWL, we define the common sequential steps and apply the specific branching logic as requested.

### 1. POWL Model with Potential Unfairness
In this model, the cultural evaluation stage is an XOR choice between a standard assessment and a biased community affiliation check.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define Activities
receive = Transition(label="ReceiveApplication")
data_check = Transition(label="DataCompletenessCheck")
request_info = Transition(label="RequestMoreInfo")
skill_test = Transition(label="SkillAssessment")
std_cultural = Transition(label="StandardCulturalFit")
comm_check = Transition(label="CommunityAffiliationCheck")
manager_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# The loop for data completeness: Execute check, potentially request more info, repeat
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# The XOR bias point: Standard path vs. Biased Community Check
cultural_choice = OperatorPOWL(operator=Operator.XOR, children=[std_cultural, comm_check])

# Sequential flow: Receive -> Data Loop -> Skill Test -> Cultural Choice -> Manager Review -> Final Decision
root_unfair = StrictPartialOrder(nodes=[receive, data_loop, skill_test, cultural_choice, manager_review, final_decision])
root_unfair.order.add_edge(receive, data_loop)
root_unfair.order.add_edge(data_loop, skill_test)
root_unfair.order.add_edge(skill_test, cultural_choice)
root_unfair.order.add_edge(cultural_choice, manager_review)
root_unfair.order.add_edge(manager_review, final_decision)
```

### 2. POWL Model Without Unfairness
In this model, we remove the XOR branching that facilitates preferential treatment, ensuring that all candidates go through a singular, standardized cultural fit assessment.

```python
# The cultural check is now a single, mandatory transition without an XOR choice
# All other steps remain identical to ensure structural consistency
cultural_eval = Transition(label="StandardCulturalFit")

# Sequential flow: Receive -> Data Loop -> Skill Test -> Standard Cultural Fit -> Manager Review -> Final Decision
root_fair = StrictPartialOrder(nodes=[receive, data_loop, skill_test, cultural_eval, manager_review, final_decision])
root_fair.order.add_edge(receive, data_loop)
root_fair.order.add_edge(data_loop, skill_test)
root_fair.order.add_edge(skill_test, cultural_eval)
root_fair.order.add_edge(cultural_eval, manager_review)
root_fair.order.add_edge(manager_review, final_decision)
```

### Explanation of the Models
*   **Loop Structure:** Both models utilize `Operator.LOOP` for the data completeness phase. This handles the requirement that missing information triggers a cycle of requests until the data is satisfactory.
*   **Unfairness Model:** The `cultural_choice` node (`Operator.XOR`) introduces the bias. It allows the system to deviate from the standard evaluation, creating a path where affiliation with local groups or specific clubs can replace a standard assessment, effectively creating a "privileged" route for certain applicants.
*   **Fairness Model:** By replacing the `Operator.XOR` branch with a single, universal `StandardCulturalFit` transition, we ensure that the evaluation criteria remain uniform for all candidates, removing the technical mechanism that allows for biased preferential treatment.
