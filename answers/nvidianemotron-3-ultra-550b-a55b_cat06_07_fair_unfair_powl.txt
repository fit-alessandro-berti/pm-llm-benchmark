### 1. POWL Model with Potential Unfairness
This model includes the `XOR` branching after the `SkillAssessment`. One branch performs a standard `CulturalFitCheck`, while the other executes a `CommunityAffiliationCheck` which introduces the subtle bias described in the text (preferential treatment for specific affiliations). The loop for data completeness is modeled using the `LOOP` operator.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# --- Define Activities (Transitions) ---
receive_app = Transition("ReceiveApplication")
data_check = Transition("DataCompletenessCheck")
request_info = Transition("RequestMoreInfo")
skill_assess = Transition("SkillAssessment")
cultural_fit = Transition("CulturalFitCheck")          # Standard evaluation path
community_check = Transition("CommunityAffiliationCheck") # Biased path (Unfairness source)
manager_review = Transition("ManagerialReview")
final_decision = Transition("FinalDecision")

# --- Define Control-Flow Operators ---

# 1. Loop for Data Completeness: * (DataCompletenessCheck, RequestMoreInfo)
# Execute DataCompletenessCheck; if incomplete, execute RequestMoreInfo and repeat Check.
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# 2. XOR Choice for Cultural Evaluation (Source of Unfairness)
# Exclusive choice between Standard Cultural Fit Check and Community Affiliation Check.
cultural_xor = OperatorPOWL(operator=Operator.XOR, children=[cultural_fit, community_check])

# --- Define Main Process Partial Order ---
# Nodes in the top-level sequence
nodes_unfair = [
    receive_app, 
    data_loop, 
    skill_assess, 
    cultural_xor, 
    manager_review, 
    final_decision
]

model_unfair = StrictPartialOrder(nodes=nodes_unfair)

# Define Execution Order (Edges)
# ReceiveApplication -> DataLoop -> SkillAssessment -> CulturalXOR -> ManagerialReview -> FinalDecision
model_unfair.order.add_edge(receive_app, data_loop)
model_unfair.order.add_edge(data_loop, skill_assess)
model_unfair.order.add_edge(skill_assess, cultural_xor)
model_unfair.order.add_edge(cultural_xor, manager_review)
model_unfair.order.add_edge(manager_review, final_decision)

# model_unfair now represents the biased process.
```

### 2. POWL Model Without Unfairness
This model removes the `XOR` branch entirely. All applicants proceed sequentially from `SkillAssessment` to a single, mandatory `CulturalFitCheck` activity. The `CommunityAffiliationCheck` activity is absent, ensuring a uniform evaluation path. The data completeness loop remains identical.

```python
# --- Define Activities (Transitions) ---
# (Re-using labels defined above; CommunityAffiliationCheck is omitted)
receive_app = Transition("ReceiveApplication")
data_check = Transition("DataCompletenessCheck")
request_info = Transition("RequestMoreInfo")
skill_assess = Transition("SkillAssessment")
cultural_fit = Transition("CulturalFitCheck") # Single, standardized evaluation for all
manager_review = Transition("ManagerialReview")
final_decision = Transition("FinalDecision")

# --- Define Control-Flow Operators ---

# 1. Loop for Data Completeness (Identical to Model 1)
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# NO XOR Operator here. CulturalFitCheck is a mandatory sequential step.

# --- Define Main Process Partial Order ---
nodes_fair = [
    receive_app, 
    data_loop, 
    skill_assess, 
    cultural_fit,   # Direct node, not wrapped in XOR
    manager_review, 
    final_decision
]

model_fair = StrictPartialOrder(nodes=nodes_fair)

# Define Execution Order (Edges)
model_fair.order.add_edge(receive_app, data_loop)
model_fair.order.add_edge(data_loop, skill_assess)
model_fair.order.add_edge(skill_assess, cultural_fit) # Direct sequence
model_fair.order.add_edge(cultural_fit, manager_review)
model_fair.order.add_edge(manager_review, final_decision)

# model_fair now represents the unbiased process.
```

### Key Differences Summary

| Feature | Model 1 (Unfair) | Model 2 (Fair) |
| :--- | :--- | :--- |
| **Cultural Evaluation Structure** | `OperatorPOWL(Operator.XOR, [CulturalFitCheck, CommunityAffiliationCheck])` | Single `Transition("CulturalFitCheck")` in sequence. |
| **CommunityAffiliationCheck** | Present as an exclusive branch option. | **Absent** (removed from model). |
| **Control Flow after SkillAssessment** | Branching (XOR): Process non-deterministically chooses a path (simulating human/subjective routing). | Linear (Sequence): Every applicant follows the exact same path. |
| **Bias Representation** | The `XOR` structure formally allows the `CommunityAffiliationCheck` path to exist, modeling the "subtle unfairness" where affiliation triggers a different evaluation logic. | By removing the `XOR` and the biased activity, the model enforces procedural equality. |
