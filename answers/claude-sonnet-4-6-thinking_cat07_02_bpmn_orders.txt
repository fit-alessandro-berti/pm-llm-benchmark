# BPMN Process Diagram Description

## Overview
This is a **Business Process Model and Notation (BPMN)** diagram depicting an order fulfillment process divided into **four swim lanes**:

---

## Swim Lanes

### 1. **Sales** (Top Lane)
- Triggered by a **timer event** ("Monday morning")
- Sequence of tasks:
  - **Check order for completeness**
  - **Check if customer is already in the database**
  - An **exclusive gateway** routes to:
    - "Customer not in database"  **Add customer to the database**
    - "Customer in database"  bypass
- Connects to a **database** artifact
- Ends with **Send order details**  **Draft Invoice** (end event)

### 2. **Production** (Second Lane)
- Starts with **Order details arrive** (message event)
- Flows through **Plan production** with parallel gateways
- **Manufacture product**  "Product complete" event
- Includes a loop for **Material not faulty** condition

### 3. **Procurement** (Third Lane)
- Contains a **Material Compensation Subprocess** including:
  - **Order materials**
  - **Return material** (compensation task)
- **Check materials** task
- Handles **Order cancellation** via termination event

### 4. **Warehouse** (Bottom Lane)
- Receives **Product complete** signal
- Executes **Send product** task
- Ends with **Product sent** event

---

## Key BPMN Elements Used
| Element | Usage |
|---|---|
| Message Events | Inter-lane communication |
| Exclusive Gateways | Decision points |
| Parallel Gateways | Concurrent flows |
| Subprocess | Material compensation |
| Dashed arrows | Message flows between lanes |
