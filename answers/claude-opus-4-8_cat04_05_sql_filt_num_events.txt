# DuckDB SQL Query: Filter Cases by Event Count

Here's a clean solution using a window function, which is the most efficient approach in DuckDB:

```sql
SELECT
    case_id,
    activity,
    timestamp
FROM (
    SELECT
        case_id,
        activity,
        timestamp,
        COUNT(*) OVER (PARTITION BY case_id) AS event_count
    FROM event_log
) sub
WHERE event_count <= 6
ORDER BY case_id, timestamp;
```

## How It Works

1. **Window function** (`COUNT(*) OVER (PARTITION BY case_id)`): Computes the total number of events per `case_id` and attaches that count to *every* row belonging to that case — without collapsing the rows.
2. **Outer filter** (`WHERE event_count <= 6`): Keeps only rows from cases whose total event count is 6 or fewer.
3. **`ORDER BY`**: Returns events in a sensible, deterministic order (per case, chronologically).

---

## Alternative: Using a Subquery with `GROUP BY`

If you prefer the more classic "compute eligible cases, then join back" pattern:

```sql
SELECT
    e.case_id,
    e.activity,
    e.timestamp
FROM event_log e
WHERE e.case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
ORDER BY e.case_id, e.timestamp;
```

Or equivalently with a CTE for readability:

```sql
WITH valid_cases AS (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT
    e.case_id,
    e.activity,
    e.timestamp
FROM event_log e
JOIN valid_cases vc
    ON e.case_id = vc.case_id
ORDER BY e.case_id, e.timestamp;
```

---

## Recommendation

| Approach | Notes |
|----------|-------|
| **Window function** | Single pass over the data; usually fastest in DuckDB; most concise. |
| **`IN` subquery / CTE join** | More explicit and readable; easy to extend (e.g., add other case-level metrics). |

For most process-mining workloads on DuckDB, I'd reach for the **window function** version first. All three produce identical result sets — every event from cases with **6 or fewer** total events.

> **Note:** The boundary is *inclusive* (`<= 6`), matching the requirement "six or fewer." If you meant strictly fewer than six, change the condition to `< 6`.
