# Process Flow Visualization Description

This is a **multi-object process flow diagram** (likely an Object-Centric Process Mining visualization) that depicts the interaction of multiple business object types through various process activities.

## Object Types (Starting Points)
The diagram tracks **5 different object types**, each with its own color:
- **PurchaseRequisitionItem** (purple) — 2.0M objects
- **SalesOrderItem** (orange) — 3.5M objects
- **PurchaseOrderItem** (blue) — 5.0M objects
- **VendorInvoiceItem** (green) — 1.2M objects
- **ReturnOrderItem** (red) — 0.8M objects

## Process Activities (Nodes)
Rectangular boxes represent activities, each showing the overall count of objects passing through:
- **Create Requisition** (3.0M)
- **Create Order** (8.5M)
- **Approve Requisition** (5.7M)
- **Pack Goods** (2.0M)
- **Send Order** (5.2M)
- **Check Stock** (1.0M)
- **Process Return** (0.8M)
- **Ship Goods** (1.8M)
- **Approve Payment** (1.5M)
- **Confirm Delivery** (1.7M)
- **Create Invoice** (2.0M)
- **Close Order** (1.7M)

## Flow Patterns
- **Purchase flow** (purple): Requisition  Approve Requisition  Send Order/Check Stock
- **Sales flow** (orange): Create Order  Pack Goods  Ship Goods  Confirm Delivery  Close Order
- **Purchase Order flow** (blue): Create Order  Send Order  Approve Payment  Create Invoice
- **Vendor Invoice flow** (green): Feeds into Approve Requisition and Create Invoice
- **Return flow** (red): Process Return  Close Order

## Edge Labels
Arrows are annotated with **transition times in milliseconds** (e.g., 300ms, 450ms, 600ms), indicating process performance/duration between activities.

## Structure
- Dashed boxes at the top/sides = object type sources
- Oval "Start" and "End" nodes mark process boundaries
- Colored dots inside activity boxes indicate which object types participate in each activity

This visualization is typical of tools like **Celonis** or similar object-centric process mining platforms, used to analyze complex end-to-end business processes (procure-to-pay, order-to-cash, returns handling) simultaneously.
