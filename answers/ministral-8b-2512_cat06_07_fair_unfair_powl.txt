Below are two POWL models representing the hiring process at **GlobalTeq Solutions**, one with potential unfairness (due to the **XOR branch for community affiliation**) and another without it.

---

### **1. POWL Model with Potential Unfairness**
This model includes an **XOR choice** where applicants are either evaluated via a **standard cultural fit check** or a **CommunityAffiliationCheck** (which subtly favors certain groups).

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
StandardCulturalFit = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Loop for data completeness
loop_data = OperatorPOWL(
    operator=Operator.LOOP,
    children=[DataCompletenessCheck, RequestMoreInfo]
)

# XOR choice: Standard Cultural Fit vs. Community Affiliation Check
xor_cultural_fit = OperatorPOWL(
    operator=Operator.XOR,
    children=[StandardCulturalFit, CommunityAffiliationCheck]
)

# Main workflow (with unfairness)
root_unfair = StrictPartialOrder(nodes=[
    ReceiveApplication,
    loop_data,
    SkillAssessment,
    xor_cultural_fit,
    ManagerialReview,
    FinalDecision
])

# Order constraints
root_unfair.order.add_edge(ReceiveApplication, loop_data)
root_unfair.order.add_edge(loop_data, SkillAssessment)
root_unfair.order.add_edge(SkillAssessment, xor_cultural_fit)
root_unfair.order.add_edge(xor_cultural_fit, ManagerialReview)
root_unfair.order.add_edge(ManagerialReview, FinalDecision)
```

**Key Issue:**
- The **XOR branch** allows some applicants to bypass a **standard cultural fit evaluation** and instead go through a **CommunityAffiliationCheck**, which may subtly favor certain groups.

---

### **2. POWL Model Without Unfairness**
This model ensures **all applicants** go through the **same cultural fit evaluation**, removing the biased XOR branch.

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Activities (same as before)
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")  # No XOR branch
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Loop for data completeness (same as before)
loop_data = OperatorPOWL(
    operator=Operator.LOOP,
    children=[DataCompletenessCheck, RequestMoreInfo]
)

# Main workflow (fair version)
root_fair = StrictPartialOrder(nodes=[
    ReceiveApplication,
    loop_data,
    SkillAssessment,
    CulturalFitCheck,  # No XOR branch
    ManagerialReview,
    FinalDecision
])

# Order constraints
root_fair.order.add_edge(ReceiveApplication, loop_data)
root_fair.order.add_edge(loop_data, SkillAssessment)
root_fair.order.add_edge(SkillAssessment, CulturalFitCheck)
root_fair.order.add_edge(CulturalFitCheck, ManagerialReview)
root_fair.order.add_edge(ManagerialReview, FinalDecision)
```

**Key Improvement:**
- **No XOR branch** exists—all applicants undergo the **same CulturalFitCheck**, ensuring fairness.

---

### **Key Differences**
| Feature | **Unfair Model** | **Fair Model** |
|---------|----------------|--------------|
| **Cultural Fit Evaluation** | XOR choice (Standard vs. CommunityAffiliationCheck) | Single path (CulturalFitCheck) |
| **Potential Bias** | Present (affiliation-based advantage) | Absent (neutral evaluation) |
| **Workflow Structure** | Branching based on affiliation | Linear, consistent process |

This demonstrates how **POWL can model both fair and biased hiring processes**, highlighting where subtle unfairness may arise.
