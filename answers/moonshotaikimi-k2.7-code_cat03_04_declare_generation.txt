declare_model = {

### Key modeling choices

- **`init`**: every case starts with **Idea Generation (IG)**.
- **`existence` / `exactly_one`**: each activity is required exactly once in every product launch trace.
- **`response` / `precedence` / `succession`**: encode the main process flow (e.g., `DD` must follow `IG`, `TFC` must follow `DD`, etc.), including the branch from `PC` to `UT`.
- **`chainresponse` / `chainprecedence` / `chainsuccession`**: model the direct, adjacent ordering of the main linear stages.
- **Negative constraints** (`noncoexistence`, `nonsuccession`, `nonchainsuccession`) are left empty because the scenario does not specify any forbidden activity relationships.

You can adjust the constraints or confidence values if some steps are optional or if the mined event log suggests weaker rules.
