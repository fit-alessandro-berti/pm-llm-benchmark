Here is the DuckDB SQL query to achieve your goal:

```sql
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
);
```

### Explanation:
1. The **subquery** (`SELECT case_id FROM event_log GROUP BY case_id HAVING COUNT(*) <= 6`) identifies all `case_id` values where the total number of events is 6 or fewer.
2. The **main query** filters the original `event_log` table to only return rows where the `case_id` is in the list generated by the subquery (i.e., cases with 6 events).
3. This ensures **all events** from qualifying cases are included in the result, while completely excluding cases with >6 events.

### Alternative (using a CTE for clarity):
```sql
WITH valid_cases AS (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
)
SELECT e.*
FROM event_log e
JOIN valid_cases v ON e.case_id = v.case_id;
```

Both versions are functionally equivalent. The `IN` subquery is concise and efficient for this use case in DuckDB.
