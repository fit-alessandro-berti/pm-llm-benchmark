#### **1. Identify Claims with Suspiciously Fast/Slow Transitions**
**Goal:** Flag claims where activity intervals deviate significantly from the temporal profile.

**Query for PN Delays (>10 days):**
```sql
SELECT
    c.claim_id,
    c.claim_type,
    c.claim_amount,
    ce_approve.timestamp AS approve_time,
    ce_notify.timestamp AS notify_time,
    EXTRACT(EPOCH FROM (ce_notify.timestamp - ce_approve.timestamp)) / 86400 AS days_delay,
    ce_approve.resource AS approver,
    ce_notify.resource AS notifier
FROM claims c
JOIN claim_events ce_approve ON c.claim_id = ce_approve.claim_id AND ce_approve.activity = 'P'
JOIN claim_events ce_notify ON c.claim_id = ce_notify.claim_id AND ce_notify.activity = 'N'
WHERE ce_approve.timestamp < ce_notify.timestamp
  AND EXTRACT(EPOCH FROM (ce_notify.timestamp - ce_approve.timestamp)) > 864000  -- >10 days (864000 seconds)
ORDER BY days_delay DESC;
```

**Query for AC Short Intervals (<4 hours):**
```sql
SELECT
    c.claim_id,
    c.claim_type,
    c.claim_amount,
    ce_assign.timestamp AS assign_time,
    ce_close.timestamp AS close_time,
    EXTRACT(EPOCH FROM (ce_close.timestamp - ce_assign.timestamp)) / 3600 AS hours_to_close,
    ce_assign.resource AS adjuster
FROM claims c
JOIN claim_events ce_assign ON c.claim_id = ce_assign.claim_id AND ce_assign.activity = 'A'
JOIN claim_events ce_close ON c.claim_id = ce_close.claim_id AND ce_close.activity = 'C'
WHERE ce_assign.timestamp < ce_close.timestamp

  -- Check if intermediate steps (E, P, N) are missing
  AND NOT EXISTS (
      SELECT 1 FROM claim_events ce

        AND ce.timestamp BETWEEN ce_assign.timestamp AND ce_close.timestamp
  )
ORDER BY hours_to_close;
```
