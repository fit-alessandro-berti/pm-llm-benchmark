# Analysis of Event Log Against Normative Process

## Conforming Cases

**Case 1:** Receive  Check  Authorize  Package  Ship  Close 
Follows the expected sufficient-inventory sequence perfectly.

**Case 4:** Receive  Check  Cancel  Close 
Follows the expected insufficient-inventory (cancellation) sequence perfectly.

## Violating Cases

### Case 2 — Violation
**Sequence:** Receive  Check  **Package items**  **Authorize payment**  Ship  Close

**Violation:** "Package items" occurs *before* "Authorize payment." The normative description explicitly states that payment authorization must occur *before* packaging ("Package items should only occur after authorization and before shipping"). Packaging items before confirming that payment is valid risks preparing goods for a customer whose payment may later fail.

### Case 3 — Violation (multiple)
**Sequence:** Receive  Check  Authorize  **Cancel order**  **Ship order**  Close

**Violations:**
1. **Shipping after cancellation:** The description is unambiguous — "Ship order should ... never [occur] after a cancellation." Once the order was canceled, shipping should not have happened.
2. **Illogical branching:** The process reached "Authorize payment" (implying inventory was sufficient) but then branched to "Cancel order," which is normally reserved for insufficient-inventory scenarios. The cancellation scenario is described as going directly from "Check inventory" to "Cancel order."
3. **Missing "Package items":** Even ignoring the cancellation, "Ship order" occurred without a preceding "Package items" activity, violating the rule that shipping must follow packaging.

### Case 5 — Violation
**Sequence:** Receive  Check  Authorize  **Ship order**  Close (no "Package items")

**Violation:** "Package items" is entirely missing. The normative flow requires: Authorize  Package  Ship. Shipping directly after authorization skips the required packaging step, violating the rule that "Ship order should only occur after packaging."
