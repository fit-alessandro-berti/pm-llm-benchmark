Here are the two POWL models representing the hiring process for GlobalTeq Solutions. The first model includes the XOR branch that introduces potential bias, while the second model removes this branch to ensure fairness.

### **POWL Model 1: With Potential Unfairness**

This model represents the workflow where an XOR choice is introduced after the skill assessment. One branch (`CulturalFitCheck`) proceeds with a standard evaluation, while the other (`CommunityAffiliationCheck`) provides a biased advantage based on community affiliation.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# 1. Define Transitions (Activities)
receive_app = Transition(label="ReceiveApplication")
data_check = Transition(label="DataCompletenessCheck")
request_more_info = Transition(label="RequestMoreInfo")
skill_assess = Transition(label="SkillAssessment")
cultural_fit = Transition(label="CulturalFitCheck")
community_affil = Transition(label="CommunityAffiliationCheck")
managerial_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# 2. Define Loop for Data Completeness (Loop Operator)
# Represents the process where missing info triggers a request loop
completeness_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_more_info])

# 3. Define XOR for Cultural Fit (Bias Point)
# One branch is standard evaluation, the other is biased community check
cultural_xor = OperatorPOWL(operator=Operator.XOR, children=[cultural_fit, community_affil])

# 4. Main Workflow Nodes
nodes = [receive_app, completeness_loop, skill_assess, cultural_xor, managerial_review, final_decision]

# 5. Create StrictPartialOrder for the main flow
model1 = StrictPartialOrder(nodes=nodes)

# 6. Add Edges (Sequential Ordering)
model1.order.add_edge(receive_app, completeness_loop)
model1.order.add_edge(completeness_loop, skill_assess)
model1.order.add_edge(skill_assess, cultural_xor)
model1.order.add_edge(cultural_xor, managerial_review)
model1.order.add_edge(managerial_review, final_decision)

# Output Model 1 Structure (Optional for verification)
print("Model 1 Nodes:", [n.label for n in nodes])
print("Model 1 Edges:", [(e.source_node.label if hasattr(e.source_node, 'label') else e.source_node, 
                       e.target_node.label if hasattr(e.target_node, 'label') else e.target_node) 
                      for e in model1.order.edges])
```

### **POWL Model 2: Without Unfairness**

This model represents a similar workflow but removes the community-based branch. All applicants undergo the standard `CulturalFitCheck` sequentially after skill assessment. No XOR operator is used for cultural evaluation.

```python

# 2. Define Loop for Data Completeness (Loop Operator)
completeness_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_more_info])

# 3. Main Workflow Nodes (No XOR or Community Affiliation Check)
nodes = [receive_app, completeness_loop, skill_assess, cultural_fit, managerial_review, final_decision]
```
