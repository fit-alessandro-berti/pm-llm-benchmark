To construct the Petri net for the insurance claim management process, we will define places as the states between activities and transitions as the activities themselves. We will implement the loops for "Re-Document Submission" and "In-depth Investigation" using back-arcs to the preceding states.

Here is the implementation using `pm4py`:

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# 1. Initialize the Petri net
net = PetriNet("Insurance_Claim_Process")

# 2. Define Places (States)
p_start = PetriNet.Place("Start")
p_reg = PetriNet.Place("After_Registration")
p_verif = PetriNet.Place("After_Verification")
p_fraud = PetriNet.Place("After_Fraud_Check")
p_assess = PetriNet.Place("After_Assessment")
p_approve = PetriNet.Place("After_Approval")
p_paid = PetriNet.Place("After_Payment")
p_notified = PetriNet.Place("After_Notification")
p_closed = PetriNet.Place("Closed")

# 3. Define Transitions (Activities)
t_reg = PetriNet.Transition("CR", "Claim Registration")
t_verif = PetriNet.Transition("DV", "Document Verification")
t_rds = PetriNet.Transition("RDS", "Re-Document Submission")
t_fc = PetriNet.Transition("FC", "Fraud Check")
t_ii = PetriNet.Transition("II", "In-depth Investigation")
t_ca = PetriNet.Transition("CA", "Claim Assessment")
t_ap = PetriNet.Transition("AP", "Approval")
t_pay = PetriNet.Transition("P", "Payment")
t_notify = PetriNet.Transition("N", "Notification")
t_close = PetriNet.Transition("CL", "Closure")

# Add elements to the net
for p in [p_start, p_reg, p_verif, p_fraud, p_assess, p_approve, p_paid, p_notified, p_closed]:
    net.places.add(p)
for t in [t_reg, t_verif, t_rds, t_fc, t_ii, t_ca, t_ap, t_pay, t_notify, t_close]:
    net.transitions.add(t)

# 4. Connect transitions with arcs
petri_utils.add_arc_from_to(p_start, t_reg, net)
petri_utils.add_arc_from_to(t_reg, p_reg, net)

# Sequence: Registration -> Verification -> Loop for RDS
petri_utils.add_arc_from_to(p_reg, t_verif, net)
petri_utils.add_arc_from_to(t_verif, p_verif, net)
petri_utils.add_arc_from_to(p_verif, t_rds, net)
petri_utils.add_arc_from_to(t_rds, p_reg, net) # Loop back for re-submission

# Sequence: Verification -> Fraud Check -> Loop for II
petri_utils.add_arc_from_to(p_verif, t_fc, net)
petri_utils.add_arc_from_to(t_fc, p_fraud, net)
petri_utils.add_arc_from_to(p_fraud, t_ii, net)
petri_utils.add_arc_from_to(t_ii, p_fraud, net) # Loop back for investigation

# Sequence: Fraud Check -> Assessment -> Approval -> Payment -> Notification -> Closure
petri_utils.add_arc_from_to(p_fraud, t_ca, net)
petri_utils.add_arc_from_to(t_ca, p_assess, net)
petri_utils.add_arc_from_to(p_assess, t_ap, net)
petri_utils.add_arc_from_to(t_ap, p_approve, net)
petri_utils.add_arc_from_to(p_approve, t_pay, net)
petri_utils.add_arc_from_to(t_pay, p_paid, net)
petri_utils.add_arc_from_to(p_paid, t_notify, net)
petri_utils.add_arc_from_to(t_notify, p_notified, net)
petri_utils.add_arc_from_to(p_notified, t_close, net)
petri_utils.add_arc_from_to(t_close, p_closed, net)

# 5. Define Markings
im = Marking()
im[p_start] = 1

fm = Marking()
fm[p_closed] = 1

print("Petri net constructed successfully.")
```

### Explanation of the Construction:
*   **Sequential Logic:** The net follows the chain from registration to closure.
*   **Looping Logic:** 
    *   **RDS:** By connecting the transition `RDS` to `p_reg`, the token is forced back to the registration state, allowing the `DV` activity to be repeated.
    *   **II:** By connecting the transition `II` back to `p_fraud`, the process allows for repeated `In-depth Investigation` before proceeding to the `Claim Assessment` stage.
*   **Markings:** The `initial marking` holds one token in the `start` place, and the `final marking` requires one token in the `closed` place to signify completion.
