Here's a DuckDB SQL query that meets your requirements:

```sql
WITH case_sequences AS (
    SELECT 
        case_id,
        activity,
        timestamp,
        -- Get the next two activities and timestamps in sequence
        LEAD(activity, 1) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_activity,
        LEAD(activity, 2) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_next_activity,
        LEAD(timestamp, 1) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_timestamp,
        LEAD(timestamp, 2) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_next_timestamp
    FROM event_log
),
filtered_cases AS (
    SELECT DISTINCT case_id
    FROM case_sequences
    WHERE 
        -- Check for the exact sequence
        activity = 'Approve Purchase Requisition'
        AND next_activity = 'Create Order'
        AND next_next_activity = 'Confirm Order'
        -- Check timing condition (more than 5 days between Create Order and Confirm Order)
        AND DATEDIFF('day', next_timestamp, next_next_timestamp) > 5
)
-- Return all events except those from filtered cases
SELECT *
FROM event_log
WHERE case_id NOT IN (SELECT case_id FROM filtered_cases)
ORDER BY case_id, timestamp;
```

**Explanation:**

1. **CTE `case_sequences`:**
   - Uses `LEAD()` window function to look ahead at the next two activities and timestamps for each event within each case
   - Orders events by timestamp to ensure proper sequence detection

2. **CTE `filtered_cases`:**
   - Identifies cases with the exact sequence: 'Approve Purchase Requisition'  'Create Order'  'Confirm Order'
   - Checks that the time between 'Create Order' and 'Confirm Order' is more than 5 days using `DATEDIFF`
   - Returns distinct `case_id`s that meet both criteria

3. **Final SELECT:**
   - Excludes all events from cases identified in `filtered_cases`
   - Returns all events from remaining cases

**Key Features:**
- Handles multiple sequences per case correctly
- Ensures "direct succession" by checking consecutive events
- Properly calculates day difference using `DATEDIFF('day', ...)`
- Excludes entire cases (not just specific sequences) as requested

**Alternative approach using EXISTS** (if you prefer):
```sql
SELECT *
FROM event_log el
WHERE NOT EXISTS (
    SELECT 1
    FROM event_log e1
    JOIN event_log e2 ON e1.case_id = e2.case_id 
        AND e2.timestamp > e1.timestamp
    JOIN event_log e3 ON e2.case_id = e3.case_id 
        AND e3.timestamp > e2.timestamp
    WHERE 
        e1.case_id = el.case_id
        AND e1.activity = 'Approve Purchase Requisition'
        AND e2.activity = 'Create Order'
        AND e3.activity = 'Confirm Order'
        AND DATEDIFF('day', e2.timestamp, e3.timestamp) > 5
        -- Ensure no other events in between
        AND NOT EXISTS (
            SELECT 1 FROM event_log e4 
            WHERE e4.case_id = e1.case_id 
            AND e4.timestamp > e1.timestamp 
            AND e4.timestamp < e2.timestamp
        )
        AND NOT EXISTS (
            SELECT 1 FROM event_log e4 
            WHERE e4.case_id = e2.case_id 
            AND e4.timestamp > e2.timestamp 
            AND e4.timestamp < e3.timestamp
        )
)
ORDER BY case_id, timestamp;
```

The first approach using window functions is generally more efficient for this type of sequence detection in DuckDB.
