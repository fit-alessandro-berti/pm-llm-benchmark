### Assumptions made
1. **Case ID** = `order_id`.
2. **Order Created** comes from `Orders.order_date`.
3. **Items Picked** is modeled as **one event per picked order line**, since each line has its own timestamp and picker.
4. **Order Dispatched** and **Order Arrived** are modeled as **one event per delivery record**, because orders can have split shipments.
5. **Invoice Issued** is modeled as **one event per invoice**.
6. **Payment Received** is modeled as **one event per payment**, linked through `invoice_id` back to `order_id`.
7. For **Resource/Actor**, I used the `Resources` table where available; otherwise the raw ID would have been used.
8. **Order 1004** has no delivery, invoice, or payment records, so its log ends after picking based on available data.
9. Product details are included for item-picking events where applicable.
