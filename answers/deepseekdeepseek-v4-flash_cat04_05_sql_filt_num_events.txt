Below is a DuckDB SQL query that accomplishes the task. It first counts events per case in a subquery, then filters the original table to keep only cases with six or fewer events.

```sql
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
);
```

**Explanation:**
- The inner subquery groups the events by `case_id`, counts how many events belong to each case, and retains only those case IDs where the count is  6.
- The outer query then selects all columns and rows from `event_log` whose `case_id` appears in that filtered list.
- This returns the complete set of events for the qualifying cases, including all original columns (e.g., `activity`, `timestamp`, etc.).

**Alternative using a window function (if you prefer a single-pass approach):**
```sql
SELECT *
FROM (
    SELECT *,
           COUNT(*) OVER (PARTITION BY case_id) AS case_event_count
    FROM event_log
) AS sub
WHERE case_event_count <= 6;
```

Either query will work in DuckDB; the first one is simpler and often more efficient for this common pattern.
