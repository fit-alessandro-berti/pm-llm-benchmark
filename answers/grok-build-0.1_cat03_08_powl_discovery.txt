**POWL Model**

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Leaf transitions
OR = Transition(label="OR")
MS = Transition(label="MS")
QS = Transition(label="QS")
AS = Transition(label="AS")
IQC = Transition(label="IQC")
TST = Transition(label="TST")
RT = Transition(label="RT")
PK = Transition(label="PK")
LB = Transition(label="LB")
DP = Transition(label="DP")
D = Transition(label="D")

# Silent transitions
skip_iqc = SilentTransition()
skip_lb = SilentTransition()

# Sub-models
iqc_loop = OperatorPOWL(operator=Operator.LOOP, children=[IQC, skip_iqc])          # * (IQC, skip)
assembly_loop = OperatorPOWL(operator=Operator.LOOP, children=[AS, iqc_loop])      # * (AS, iqc_loop)
test_loop = OperatorPOWL(operator=Operator.LOOP, children=[TST, RT])               # * (TST, RT)
lb_choice = OperatorPOWL(operator=Operator.XOR, children=[LB, skip_lb])            # X (LB, skip)

# Root partial order (strictly sequential backbone)
root = StrictPartialOrder(nodes=[OR, MS, QS, assembly_loop, test_loop, PK, lb_choice, DP, D])

root.order.add_edge(OR, MS)
root.order.add_edge(MS, QS)
root.order.add_edge(QS, assembly_loop)
root.order.add_edge(assembly_loop, test_loop)
root.order.add_edge(test_loop, PK)
root.order.add_edge(PK, lb_choice)
root.order.add_edge(lb_choice, DP)
root.order.add_edge(DP, D)
```

**Explanation of the structure**

- **Fixed prefix**: `OR  MS  QS` (always present).
- **Assembly / In-line QC phase** (`assembly_loop`):
  - Modeled as `* (AS, iqc_loop)`.
  - `iqc_loop = * (IQC, skip)` allows one or more consecutive IQC checks (the silent transition permits repeating IQC without forcing an AS in between).
  - This generates exactly the observed patterns: `AS`, `AS IQC AS`, `AS IQC AS IQC AS`, `AS IQC IQC AS IQC AS`, etc.
- **Testing / Re-testing phase** (`test_loop`):
  - `* (TST, RT)` — execute TST, then optionally RT followed by TST again.
  - Matches all observed testing loops (including multiple RT cycles in case 5).
- **Post-testing phase**:
  - `PK  lb_choice  DP  D`
  - `lb_choice = X (LB, skip)` makes labeling optional (covers both the cases that perform LB and the one that skips it).

The model is a single `StrictPartialOrder` whose nodes are the sub-POWL operators and the remaining atomic transitions. All ordering constraints are expressed via the partial-order edges; no concurrency is present in the log.
