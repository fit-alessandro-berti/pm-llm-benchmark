# POWL Model for Telecom Corporate Network Setup

Let me trace through the scenario carefully:

1. **SR  CS** (strict ordering)
2. **CS  DA, CS  ACI** (DA and ACI concurrent after CS)
3. **DA  Loop, ACI  Loop** (loop starts after both complete)
4. **Loop  XOR** (after stable config, choose deployment)
5. **XOR  QA, XOR  SA** (QA and SA concurrent)
6. **QA  GLA, SA  GLA** (GLA waits for both)

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Step 1: Define all activity nodes
SR  = Transition(label="Service Request (SR)")
CS  = Transition(label="Contract Setup (CS)")
DA  = Transition(label="Datacenter Allocation (DA)")
ACI = Transition(label="Access Control Initialization (ACI)")
BNC = Transition(label="Base Network Configuration (BNC)")
DR  = Transition(label="Debug & Reconfigure (D&R)")
CD  = Transition(label="Containerized Deployment (CD)")
MD  = Transition(label="Monolithic Deployment (MD)")
QA  = Transition(label="Quality Assurance (QA)")
SA  = Transition(label="Security Audit (SA)")
GLA = Transition(label="Go-Live Approval (GLA)")

# Step 2: Build operator nodes
# Loop: execute BNC, then either exit or execute D&R and repeat BNC
loop = OperatorPOWL(operator=Operator.LOOP, children=[BNC, DR])

# Exclusive choice: either containerized or monolithic deployment
xor = OperatorPOWL(operator=Operator.XOR, children=[CD, MD])

# Step 3: Assemble the root partial order with all nodes
root = StrictPartialOrder(nodes=[SR, CS, DA, ACI, loop, xor, QA, SA, GLA])

# Step 4: Add ordering edges (unconnected nodes remain concurrent)
root.order.add_edge(SR, CS)       # SR --> CS
root.order.add_edge(CS, DA)       # CS --> DA
root.order.add_edge(CS, ACI)      # CS --> ACI  (DA & ACI are concurrent)
root.order.add_edge(DA, loop)     # DA  --> loop
root.order.add_edge(ACI, loop)    # ACI --> loop (loop waits for both)
root.order.add_edge(loop, xor)    # loop --> xor (deployment choice after config)
root.order.add_edge(xor, QA)     # xor  --> QA
root.order.add_edge(xor, SA)     # xor  --> SA  (QA & SA are concurrent)
root.order.add_edge(QA, GLA)     # QA   --> GLA
root.order.add_edge(SA, GLA)     # SA   --> GLA (GLA waits for both)
```

## Key Design Decisions

| Scenario Aspect | POWL Construct | Reasoning |
|---|---|---|
| SR  CS | `StrictPartialOrder` edge | CS must follow SR |
| DA  ACI | No edge between DA & ACI | Concurrent (no ordering) |
| BNC + optional D&R | `OperatorPOWL(LOOP)` | Execute BNC; either exit or do D&R then BNC again |
| CD vs MD | `OperatorPOWL(XOR)` | Mutually exclusive choice |
| QA  SA | No edge between QA & SA | Concurrent; both must finish before GLA |
| GLA | Edges from QA & SA | Joins the two concurrent branches |
