# Insurance Claims Processing — Performance Analysis

**Cases with performance issues:** 2002, 2003, and especially **2005** and **2003**. Cases 2001 and 2004 are the fast "happy path" baseline (no document requests, completed in ~1.5 hrs).

---

## 2. Root-Cause Analysis by Attribute

### A. Complexity (strongest correlation)
There is a clear monotonic relationship between complexity and lead time:

- **Low complexity**  0 document requests  ~1.5 hrs
- **Medium complexity**  1 document request  ~26 hrs
- **High complexity**  2–3 document requests  48–77 hrs

**The number of "Request Additional Documents" loops is the dominant driver.** Each loop introduces a long wait (often overnight or multi-day) while the claimant supplies information. In Case 2005, three sequential requests stacked up day-over-day delays:
- 1st request: Apr 1, 11:30  2nd request: Apr 2, 17:00 (30 hrs)
- 2nd  3rd request: Apr 3, 15:00 (22 hrs)
- 3rd request  Approval: Apr 4, 10:00 (19 hrs)

### B. Region
Region B contains the two slowest medium/high cases (2002, 2005). However, the slow Region A case (2003) is also High complexity. So **region alone is not the root cause** — it co-varies with complexity. The data is too small to claim a true regional effect, but Region B + High complexity (Case 2005) is the worst observed combination and worth monitoring.

### C. Resource
- **Manager_Bill** approves *both* high-complexity claims (2003, 2005) and is associated with very long pre-approval waits. **Manager_Ann** handles only Low/Medium claims (fast). This may indicate Bill is a specialized but bottlenecked approver for complex cases.
- **Adjuster_Lisa** (Region B) handled all three of Case 2005's repeated document requests — repeated requests by the same adjuster may suggest the *first* requests were incomplete/unclear, forcing additional round-trips.
- Finance and CSR steps are consistently fast (~15–45 min) across all cases — **not bottlenecks.**

---

## 3. Explanations & Mitigation Suggestions

| Root Cause | Why It Extends Duration | Suggested Mitigation |
|------------|------------------------|----------------------|
| **Repeated "Request Additional Documents" loops** | Each loop adds overnight/multi-day waiting; effects compound for high-complexity claims | Use a **single consolidated documentation checklist** at first request so all needed docs are gathered in one round; add SLA timers and automated reminders to claimants |
| **High-complexity claims** | Require more evaluation and multiple data exchanges | Triage complexity at intake; route to a **specialist fast-track team** with extra capacity; set complexity-specific SLAs |
| **Approval bottleneck (Manager_Bill for complex claims)** | Single specialized approver creates a queue; long gaps before approval | Cross-train additional approvers for high-complexity claims; balance workload; monitor Bill's queue |
| **Adjuster requesting docs repeatedly (Lisa)** | Incomplete initial requests cause rework | Provide adjusters with standardized request templates; coach/audit on first-time-right requests |
| **Region B + High complexity** | Worst observed combination | Investigate whether Region B has staffing/process gaps for complex claims; rebalance if confirmed with larger dataset |

### Key Takeaway
The primary root cause is **complexity-driven rework** — high-complexity claims trigger multiple sequential document requests, each adding large waiting times. Resource specialization (Manager_Bill) and possible regional capacity issues (Region B) are secondary, correlated contributors. The biggest, most actionable win is to **reduce the number of documentation round-trips** by gathering complete information up front.

>  **Caveat:** With only 5 cases, these are directional findings. Region and resource effects are confounded with complexity. A larger sample would be needed to statistically isolate the independent effect of Region and individual Resources.
