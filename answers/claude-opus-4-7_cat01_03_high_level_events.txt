# Event Abstraction: From Low-Level Events to High-Level Process Steps

## 1. Proposed High-Level Activities

Based on the recurring patterns across cases A1 and B2, I identified **five distinct high-level process steps**. Each aggregates low-level events that share a common purpose, involve related resources, and occur in close temporal proximity.

---

### **Step 1: Material Preparation**

| CaseID | Start Time | End Time | Low-Level Events Included |
|--------|------------|----------|---------------------------|
| A1 | 08:00:05 | 08:00:20 | Retrieve raw metal sheet  Scan barcode  Place metal sheet on table  Align metal sheet  Preheat metal sheet |
| B2 | 08:00:05 | 08:00:25 | Retrieve raw metal sheet  Scan barcode  Place metal sheet on table  Align metal sheet  Preheat metal sheet |

**Rationale:**  
All of these events happen before any transformation occurs. They describe getting the raw material *ready* for processing — physical retrieval, identification (barcode), positioning (table placement + alignment), and thermal conditioning (preheat). Resources involved are the intake operator, a positioning robot, and a heating unit, all working on the raw input.

---

### **Step 2: Welding / Assembly**

| CaseID | Start Time | End Time | Low-Level Events Included |
|--------|------------|----------|---------------------------|
| A1 | 08:01:00 | 08:01:10 | Pick up welding tool  Weld corner A  Weld corner B |
| B2 | 08:01:03 | 08:01:12 | Pick up welding tool  Weld corner A  Weld corner B |

**Rationale:**  
These events describe the actual value-adding transformation of the part. Picking up the welding tool is a direct setup action for welding (same operator, seconds apart). Corner welds are repetitions of the same core activity. Grouping them hides the internal "per-corner" granularity while preserving the stage's meaning.

---

### **Step 3: In-Process Quality Check (Weld Inspection)**

| CaseID | Timestamp | Low-Level Event Included |
|--------|-----------|---------------------------|
| A1 | 08:01:20 | Measure weld integrity (Score: 95) |
| B2 | 08:01:22 | Measure weld integrity (Score: 93) |

**Rationale:**  
This is a *conditional/gatekeeping* event performed by a dedicated Quality Sensor, immediately after welding. It is distinct from the final visual check because (a) it uses an automated sensor, (b) it specifically evaluates the welding output, and (c) it occurs *before* finishing operations. Treating it as its own high-level step preserves the "stage-gate" nature of manufacturing QA.

---

### **Step 4: Surface Finishing (Coating & Curing)**

| CaseID | Start Time | End Time | Low-Level Events Included |
|--------|------------|----------|---------------------------|
| A1 | 08:01:30 | 08:01:45 | Apply protective coating  Dry coating |
| B2 | 08:01:35 | 08:01:50 | Apply protective coating  Dry coating |

**Rationale:**  
Coating application and drying are two halves of the same finishing operation — the coating is useless without curing, and the curing only makes sense because coating was just applied. They occur back-to-back on dedicated finishing equipment (Coating Unit + Heating Unit #2). This grouping reflects the "finishing" stage of the manufacturing flow.

---

### **Step 5: Final Inspection / Release**

| CaseID | Timestamp | Low-Level Event Included |
|--------|-----------|---------------------------|
| A1 | 08:02:00 | Visual check (Passed) |
| B2 | 08:02:05 | Visual check (Passed) |

**Rationale:**  
The last event in each case is a manual, human inspection acting as the release gate. It differs from Step 3 in both resource type (human vs. sensor) and scope (whole product vs. weld only). It is the terminal "approval" step before the product leaves the case.

---

## 2. Grouping Heuristics Applied

The abstraction logic used was:

| Heuristic | Example |
|-----------|---------|
| **Shared purpose** | All pre-processing events  *Material Preparation* |
| **Same or related resource cluster** | Operator B + welding tool  *Welding* |
| **Temporal proximity (short gaps, contiguous)** | Coating + Drying occur ~15s apart  *Surface Finishing* |
| **Functional role in the workflow** | Sensor measurement = inspection gate  *In-Process QC* |
| **Start/end markers of a phase** | "Pick up tool" is a setup sub-event of welding, not its own step |

## 4. Resulting High-Level Process Model

```
[Material Preparation] 
        
[Welding / Assembly] 
        
[In-Process Quality Check] 
        
[Surface Finishing] 
        
[Final Inspection]
```

This 5-step model compresses 12 low-level events per case into **5 meaningful stages**, making the workflow immediately readable to managers, process analysts, and for conformance checking — while the underlying granular data remains available for root-cause analysis if a stage fails (e.g., low IntegrityScore traced back to a specific weld corner).
