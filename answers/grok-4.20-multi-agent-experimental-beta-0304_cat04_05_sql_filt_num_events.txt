```sql
SELECT * FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
);
```

The query above uses a subquery with grouping/HAVING to identify valid `case_id` values (those with 6 or fewer events total), then returns all original events belonging to those cases. It follows the benchmark prompt's emphasis on identifying counts per `case_id`, excluding those >6, and returning the full events for remaining cases. (DuckDB supports this standard SQL pattern; a `QUALIFY` version could also work but the grouping approach matches the task description most directly.)
