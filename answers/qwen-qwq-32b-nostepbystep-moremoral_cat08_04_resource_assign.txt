  - Visualize how tickets flow between agents/tiers. Clustering of tickets around certain agents (e.g., Agent B12 handling many Network tickets) might reveal specialization inefficiencies.  

- **Role Discovery:**  
  - Determine if agents are operating outside their tier roles (e.g., L1 agents resolving P1 tickets) or if L2 specialists handle L1 tasks, indicating role misalignment.

### 4. Data-Driven Resource Assignment Strategies  
#### Strategy 1: **Skill-Based Routing Algorithm**  
- **Addresses:** Skill mismatches and reassignments.  
- **Process Mining Insight:** Identified frequent reassignments due to skill gaps (e.g., INC-1001’s DB-SQL issue).  
- **Implementation:**  
  - Map "Required Skill" tags to agents’ documented skills. Use a weighted algorithm prioritizing exact skill matches (e.g., prioritize Agent B15 for "App-CRM" over generalists.  
  - **Data Requirements:** Agent skill registries, ticket "Required Skill" fields, and historical resolution success rates.  
  - **Benefit:** Reduces reassignments by 30–40% and cuts resolution times.  

#### Strategy 2: **Workload-Aware Dynamic Assignment**  
- **Addresses:** Overloaded agents and uneven distribution.  
- **Process Mining Insight:** Agents like A05 are overworked due to round-robin, while others are idle.  
- **Implementation:**  
  - Integrate real-time agent availability and current workload into dispatching. Use a "least-loaded agent with required skill" rule.  
  - **Data Requirements:** Real-time agent status (busy/idle), skill matrices, and ticket priority.  
  - **Benefit:** Reduces average resolution time by 20% and evens workload.  

#### Strategy 3: **Predictive Skill Requirement Modeling**  
- **Addresses:** Incorrect initial categorization and escalations.  
- **Process Mining Insight:** Many L1 escalations stem from incorrect "Required Skill" tags.  
- **Implementation:**  
  - Train a machine learning model on historical tickets to predict "Required Skills" using ticket descriptions (e.g., "slow network"  "Networking-Firewall").  
  - **Data Requirements:** Ticket descriptions, resolution notes, and historical skill assignments.  
  - **Benefit:** Reduces escalations by 25% by improving initial routing accuracy.
