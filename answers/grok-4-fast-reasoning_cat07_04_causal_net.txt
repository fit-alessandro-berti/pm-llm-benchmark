The right panel provides visual legends for common **gateway types** in workflow diagrams. These are simplified icons showing how flows split (branch out) and join (merge back). Each type has a "split" example (outgoing arrows from a node) and a "join" example (incoming arrows to a node). They are labeled as follows:

- **XOR-Split and XOR-Join** (Exclusive OR):
  - **Split**: A single incoming arrow to a node (white square with black dots), then one outgoing arrow (only one path chosen, e.g., based on a condition like "if approved").
  - **Join**: Multiple incoming arrows (black dots) to a single node, but only one activates (waits for the active path).
  - **Icon**: Shows a diamond-like gateway with one out/one in for split; multiple in/one out for join.
  - **Usage in Main Diagram**: The branches from e to g/h and reconvergence at z exemplify this (mutually exclusive outcomes).

- **AND-Split and AND-Join** (Parallel AND):
  - **Split**: A single incoming arrow splits into multiple parallel outgoing arrows (all paths taken simultaneously, e.g., do everything at once).
  - **Join**: Multiple incoming arrows merge into one (waits for *all* paths to complete before proceeding).
  - **Icon**: Straight splits with multiple arrows fanning out evenly; joins converge similarly.

- **OR-Split and OR-Join** (Inclusive OR):
  - **Split**: A single incoming arrow branches into multiple outgoing arrows (one or more paths taken, based on conditions).
  - **Join**: Multiple incoming arrows to one node (waits for *all active* paths to complete; flexible like XOR but allows multiples).
  - **Icon**: Similar to XOR but with looping or curved arrows in joins to show partial activation; splits fan out with potential for multiples.

These gateways are standard in process modeling (e.g., BPMN) to handle concurrency and decisions. The main diagram primarily uses XOR types, while the sidebar educates on alternatives.
