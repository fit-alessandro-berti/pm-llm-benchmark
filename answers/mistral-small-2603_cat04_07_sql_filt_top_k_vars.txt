# DuckDB SQL Query for Process Variant Analysis

Here's a comprehensive DuckDB SQL query that accomplishes all the specified tasks:

```sql
WITH
-- Step 1: Get ordered activity sequences for each case
ordered_sequences AS (
    SELECT
        case_id,
        LIST(activity ORDER BY timestamp) AS activity_sequence
    FROM event_log
    GROUP BY case_id
),

-- Step 2: Count cases per variant and rank them
variant_counts AS (
    SELECT
        activity_sequence,
        COUNT(*) AS case_count,
        RANK() OVER (ORDER BY COUNT(*) DESC) AS frequency_rank
    FROM ordered_sequences
    GROUP BY activity_sequence
),

-- Step 3: Get the top K variant sequences
top_k_variants AS (
    SELECT activity_sequence
    FROM variant_counts
    WHERE frequency_rank <= K  -- Replace K with your desired number
),

-- Step 4: Get case_ids belonging to top K variants
top_k_case_ids AS (
    SELECT case_id
    FROM ordered_sequences
    WHERE activity_sequence IN (SELECT activity_sequence FROM top_k_variants)
)

-- Final result: All events from cases in top K variants
SELECT e.*
FROM event_log e
JOIN top_k_case_ids t ON e.case_id = t.case_id
ORDER BY e.case_id, e.timestamp;
```

## Explanation:

1. **ordered_sequences**: This CTE extracts the ordered sequence of activities for each case by grouping events by case_id and ordering activities by timestamp.

2. **variant_counts**: This CTE counts how many cases follow each unique activity sequence and ranks them by frequency (most common variants first).

3. **top_k_variants**: This CTE selects the top K most frequent variants (replace K with your desired number).

4. **top_k_case_ids**: This CTE identifies all case_ids that belong to the top K variants.

5. **Final query**: Joins the original event log with the top K case_ids to return all events from those cases, ordered by case_id and timestamp.

## Notes:
- Replace `K` with your desired number of top variants to include (e.g., `WHERE frequency_rank <= 5` for top 5 variants).
- The query maintains the original event order within each case.
- If you need to handle ties in frequency ranking differently, you could use `DENSE_RANK()` instead of `RANK()`.
