Below are two **pm4py POWL** models for the GlobalTeq Solutions hiring process. Both share the same application-intake, data-completeness loop, and skill-assessment stages; they differ only in how the cultural-fit stage is handled.

---

## 1. POWL Model with Potential Unfairness

In this model, after the skill assessment there is an **XOR choice**: one path runs a standard cultural-fit evaluation, while the other runs a **CommunityAffiliationCheck** that can give affiliated applicants a subtle advantage.

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Activities
ReceiveApplication            = Transition(label="ReceiveApplication")
ResumeParsingAndDataCheck     = Transition(label="ResumeParsingAndDataCheck")
RequestMoreInfo               = Transition(label="RequestMoreInfo")
SkillAssessment               = Transition(label="SkillAssessment")
CulturalFitCheck              = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck     = Transition(label="CommunityAffiliationCheck")
ManagerialReview              = Transition(label="ManagerialReview")
FinalDecision                 = Transition(label="FinalDecision")

# Loop: parse/check data; if incomplete, request more info and re-check
DataCompletenessLoop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[ResumeParsingAndDataCheck, RequestMoreInfo]
)

# XOR branch: standard cultural fit OR community-affiliation-based check
CulturalFitXor = OperatorPOWL(
    operator=Operator.XOR,
    children=[CulturalFitCheck, CommunityAffiliationCheck]
)

# Root partial order
root_unfair = StrictPartialOrder(
    nodes=[
        ReceiveApplication,
        DataCompletenessLoop,
        SkillAssessment,
        CulturalFitXor,
        ManagerialReview,
        FinalDecision
    ]
)

# Sequential edges
root_unfair.order.add_edge(ReceiveApplication,   DataCompletenessLoop)
root_unfair.order.add_edge(DataCompletenessLoop, SkillAssessment)
root_unfair.order.add_edge(SkillAssessment,      CulturalFitXor)
root_unfair.order.add_edge(CulturalFitXor,       ManagerialReview)
root_unfair.order.add_edge(ManagerialReview,     FinalDecision)
```

**Why it can be unfair:**  
The `XOR(CulturalFitCheck, CommunityAffiliationCheck)` branch creates two different evaluative paths. Applicants routed through `CommunityAffiliationCheck` may receive an implicit uplift, while non-affiliated applicants are evaluated only against the standard cultural-fit criteria.

---

## 2. POWL Model Without Unfairness

This model removes the preferential branch. Every applicant who passes the skill assessment goes through the **same** `CulturalFitCheck` activity.

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Activities
ReceiveApplication            = Transition(label="ReceiveApplication")
ResumeParsingAndDataCheck     = Transition(label="ResumeParsingAndDataCheck")
RequestMoreInfo               = Transition(label="RequestMoreInfo")
SkillAssessment               = Transition(label="SkillAssessment")
CulturalFitCheck              = Transition(label="CulturalFitCheck")
ManagerialReview              = Transition(label="ManagerialReview")
FinalDecision                 = Transition(label="FinalDecision")

# Loop: parse/check data; if incomplete, request more info and re-check
DataCompletenessLoop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[ResumeParsingAndDataCheck, RequestMoreInfo]
)

# Root partial order (no XOR; single cultural-fit path)
root_fair = StrictPartialOrder(
    nodes=[
        ReceiveApplication,
        DataCompletenessLoop,
        SkillAssessment,
        CulturalFitCheck,
        ManagerialReview,
        FinalDecision
    ]
)

# Sequential edges
root_fair.order.add_edge(ReceiveApplication,   DataCompletenessLoop)
root_fair.order.add_edge(DataCompletenessLoop, SkillAssessment)
root_fair.order.add_edge(SkillAssessment,      CulturalFitCheck)
root_fair.order.add_edge(CulturalFitCheck,     ManagerialReview)
root_fair.order.add_edge(ManagerialReview,     FinalDecision)
```

**Why it is fairer:**  
There is no `CommunityAffiliationCheck` branch and no XOR at the cultural-fit stage. All applicants proceed through the identical `CulturalFitCheck`, removing the structural source of community-based preferential treatment.
