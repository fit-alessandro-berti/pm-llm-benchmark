#### **Methodology**  
Leverage **process mining techniques** on the event log to:  
- **Discover the process model** (e.g., using tools like ProM or Celonis) to map dependencies between activities and orders.  
- **Conformance checking** to identify deviations from ideal workflows caused by constraints.

#### **Constraint Identification & Impact Metrics**  
**a. Shared Cold-Packing Stations**  
- **Identification**: Filter events for "Packing" activities with "Requires Cold Packing = TRUE" and analyze resource allocation (e.g., Station C2 in the log).  
- **Impact Metrics**:

  - **Throughput reduction**: Orders delayed due to cold-packing station unavailability.

**c. Priority Handling (Express Orders)**  
- **Identification**: Compare timestamps and activity sequences for "Express" vs. "Standard" orders.

**d. Hazardous Material Limits**  
- **Identification**: Filter events with "Hazardous Material = TRUE" in "Packing" or "Quality Check." Track concurrent hazardous orders in these activities.  
- **Impact Metrics**:  
  - **Constraint violations**: Number of times >10 hazardous orders were in packing/QC simultaneously.  
  - **Hazardous order delays**: Waiting time for hazardous orders when the limit is reached.  
  - **Throughput loss**: Orders delayed due to hazardous limit bottlenecks.  

#### **Differentiating Within- vs. Between-Instance Waits**  
- **Within-instance time**: Duration of an activity (e.g., "Packing" takes 10 minutes). Calculated as `Activity Complete Time - Activity Start Time`.  
- **Between-instance time**: Gap between completion of one activity and start of the next (e.g., waiting for a cold-packing station). Calculated as `Next Activity Start Time - Prior Activity Complete Time`.  
- **Process Mining Tools**: Use **event log timestamps** and **resource allocation logs** to distinguish these. For example, if a gap occurs while a resource is busy with another order, it is between-instance.
