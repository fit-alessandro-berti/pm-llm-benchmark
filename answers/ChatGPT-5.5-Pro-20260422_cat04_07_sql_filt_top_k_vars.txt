```sql
-- Replace ? with K, or bind it as a DuckDB parameter.
WITH case_sequences AS (
    SELECT
        case_id,
        list(activity ORDER BY "timestamp") AS activity_sequence
    FROM event_log
    GROUP BY case_id
),

variant_counts AS (
    SELECT
        activity_sequence,
        count(*) AS case_count
    FROM case_sequences
    GROUP BY activity_sequence
),

top_k_variants AS (
    SELECT
        activity_sequence,
        case_count
    FROM variant_counts
    ORDER BY case_count DESC
    LIMIT ?
),

top_k_cases AS (
    SELECT
        cs.case_id
    FROM case_sequences cs
    JOIN top_k_variants tv
        ON cs.activity_sequence = tv.activity_sequence
)

SELECT
    e.case_id,
    e.activity,
    e."timestamp"
FROM event_log e
JOIN top_k_cases tc
    ON e.case_id = tc.case_id
ORDER BY
    e.case_id,
    e."timestamp";
```

This query does the following:

1. `case_sequences` builds the ordered activity sequence for each `case_id`.
2. `variant_counts` groups identical sequences and counts how many cases follow each variant.
3. `top_k_variants` keeps the K most frequent variants.
4. `top_k_cases` identifies all cases whose sequence belongs to one of those top K variants.
5. The final `SELECT` returns only the original events from those cases.

One caveat: if multiple events in the same case can have the exact same timestamp, the ordering between those tied events is not fully deterministic unless another tie-breaker column exists, such as `event_id`. Then you would use:

```sql
list(activity ORDER BY "timestamp", event_id)
```
