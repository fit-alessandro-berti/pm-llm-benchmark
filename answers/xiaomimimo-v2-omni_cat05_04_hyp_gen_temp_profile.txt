**3. Investigate the A  C Anomaly (Premature Closure):**
```sql
-- Find claims closed very quickly after assignment and see if they lack intermediate steps.
WITH claim_flow AS (
    SELECT
        claim_id,
        MAX(CASE WHEN activity = 'A' THEN timestamp END) AS assign_time,
        MAX(CASE WHEN activity = 'E' THEN timestamp END) AS evaluate_time,
        MAX(CASE WHEN activity = 'P' THEN timestamp END) AS approve_time,
        MAX(CASE WHEN activity = 'C' THEN timestamp END) AS close_time
    FROM
        claim_events
    GROUP BY
        claim_id
)
SELECT
    claim_id,
    EXTRACT(MINUTE FROM (close_time - assign_time)) AS minutes_to_close,
    CASE WHEN evaluate_time IS NULL THEN 'Missing' ELSE 'Present' END AS evaluate_step,
    CASE WHEN approve_time IS NULL THEN 'Missing' ELSE 'Present' END AS approve_step
FROM
    claim_flow
WHERE
    close_time IS NOT NULL
    AND assign_time IS NOT NULL
    AND (close_time - assign_time) < INTERVAL '4 hours'  -- Threshold for "very fast" closure
ORDER BY
    minutes_to_close;
```
*Goal: Verify if claims closed rapidly after assignment consistently lack the 'Evaluate' and/or 'Approve' steps, indicating process shortcuts.*
