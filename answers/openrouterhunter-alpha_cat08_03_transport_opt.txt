# **Process Mining Analysis for Speedy Parcels: A Comprehensive Approach**

## **1. Process Discovery and Conformance Checking**

### **Data Preprocessing and Integration**
Integrating data from GPS, scanners, dispatch, and maintenance logs into a cohesive event log requires careful alignment:

**Key Preprocessing Steps:**
- **Unified Case ID Structure:** Use `Vehicle-Day` as the primary case ID, as suggested. Each day a vehicle operates is a distinct process instance.
- **Event Correlation:** Link scanner events to specific packages using Package ID, and correlate vehicle GPS points with driver shifts via Driver ID and timestamps.
- **Activity Standardization:** Create a unified activity taxonomy (e.g., `Depart Depot`, `Travel`, `Arrive Customer`, `Delivery Attempt`, `Maintenance Start`, `Arrive Depot`).
- **Timestamp Alignment:** Synchronize all data sources to a common time zone and ensure granular time-stamping.
- **Context Enrichment:** Merge location data with external traffic/road condition data if available; attach planned route sequences from dispatch to each case.

**Challenges:**
- **Data Silos:** Different systems may use inconsistent IDs (e.g., Vehicle ID formats).
- **Gaps & Noise:** GPS drift, missed scans, or unscheduled stops not logged in maintenance records.
- **Event Granularity:** GPS provides high-frequency data; scanner events are sparse. Downsampling GPS to meaningful state changes (e.g., moving vs. idle) is necessary.

### **Process Discovery**
Using algorithms like **Inductive Miner** or **Heuristic Miner**, I would discover the "as-is" delivery process:
- **Input:** Event log with activities: `Start Shift`, `Route Assigned`, `Depart Depot`, `Travel`, `Arrive Customer`, `Delivery Success`, `Delivery Failed`, `Depart Customer`, `Unscheduled Stop`, `Maintenance`, `Return Depot`, `End Shift`.
- **Output:** A control-flow model (e.g., Petri net, BPMN) showing the most common paths, loops (e.g., repeated delivery attempts), and rare variants (e.g., mid-route maintenance).
- **Focus Areas:** Visualize the ratio of time spent on value-added activities (delivery) vs. non-value-added (travel, waiting, idle). Highlight parallelism (e.g., multiple packages delivered in one stop).

### **Conformance Checking**
Compare the discovered model against the **dispatch-planned process** (ideal route sequence and timing):
- **Sequence Deviations:** Use token-based replay or alignments to detect unplanned activities (e.g., `Unscheduled Stop` not in the plan) or skipped planned stops.
- **Timing Deviations:** Calculate for each case the difference between planned and actual arrival/departure times at stops. Significant delays (>15 minutes) indicate bottlenecks.
- **Resource Deviations:** Check if actual driver/vehicle assignments match the plan.
- **Key Metrics:** Conformance percentage per case, and distribution of deviation magnitudes across the fleet.

## **2. Performance Analysis and Bottleneck Identification**

### **Relevant KPIs**
| KPI | Formula (from Event Log) | Goal |
|------|---------------------------|------|
| **On-Time Delivery Rate** | % of packages delivered within customer time window |  Increase |
| **Avg. Time per Delivery Stop** | Total service time at stops / number of stops |  Reduce |
| **Travel Time vs. Service Time Ratio** | Total travel time / total service time |  Reduce (optimize routing) |
| **Fuel Consumption per km** | (Estimated from GPS speed/ idling events + vehicle data) |  Reduce |
| **Vehicle Utilization Rate** | (Total moving time) / (Shift duration - maintenance) |  Increase |
| **Traffic Delay Frequency** | Number of `Low Speed Detected` events per route |  Reduce |
| **Failed Delivery Rate** | Number of `Delivery Failed` events / total delivery attempts |  Reduce |

### **Bottleneck Identification Techniques**
- **Performance Analysis in Discovered Model:** Overlay the process map with average duration per activity (e.g., heat map). Likely bottlenecks: `Travel` (especially in congested zones), `Delivery Failed` loops, `Unscheduled Stop`.
- **Dwell Time Analysis:** Calculate variance in `Arrive Customer` to `Delivery Success` time. High variance indicates inconsistent service times or parking issues.
- **Queue Analysis:** Identify backlogs by counting vehicles waiting to load/unload at depot or clustered in traffic.
- **Variant Analysis:** Compare top 5% fastest routes vs. slowest 5%. What activities differ? Do faster routes have fewer failed deliveries or less idle time?
- **Resource-Based Analysis:** Compare performance by driver, vehicle type, or time of day (using histogram of arrival times per zone).

**Quantifying Impact:** Calculate the cumulative delay contributed by each bottleneck (e.g., traffic adds 120 hours total delay per month across fleet), and the cost implication (fuel, overtime, missed SLAs).

## **3. Root Cause Analysis for Inefficiencies**

### **Potential Root Causes & Process Mining Validation**

| Inefficiency | Potential Root Cause | Process Mining Analysis for Validation |
|--------------|----------------------|----------------------------------------|
| **Late Deliveries** | Static routes can't adapt to real-time traffic. | Correlate `Low Speed Detected` events with delays. Compare routes with/without dynamic re-routing. |
| **High Failed Delivery Rate** | Inaccurate time windows; customer not home. | Cluster failed deliveries by time of day/ location. Analyze if they occur more in certain routes or after long travel times (driver behind schedule). |
| **Excessive Idle Time** | Inefficient loading/ unloading; driver breaks. | Analyze depot dwell times. Compare `Start Shift` to `Depart Depot` duration across drivers/ days. |
| **Frequent Unscheduled Stops** | Poor vehicle maintenance; aging fleet. | Correlate `Unscheduled Stop` with vehicle age, mileage (from GPS), and maintenance logs. |
| **High Fuel Costs** | Aggressive driving; prolonged idling. | Analyze GPS speed patterns (harsh acceleration/ braking) and idling durations per driver. |

**Variant Analysis Example:** Compare the process variants of the top 10% vs. bottom 10% performing drivers (based on on-time rate). Do high performers have fewer `Delivery Failed` events? Do they spend less time in `Travel`? This can reveal best practices.

## **4. Data-Driven Optimization Strategies**

### **Strategy 1: Dynamic Time-Window Adjustment & Customer Communication**
- **Target Inefficiency:** High failed delivery rate.
- **Root Cause:** Static time windows don't account for real-time delays; customers unavailable.
- **Process Mining Insight:** Analysis shows 40% of failed deliveries occur after the vehicle is already >30 minutes behind schedule. Variant analysis indicates failed attempts cluster in afternoon routes.
- **Proposal:** Implement a system that updates customer ETA in real-time based on current vehicle progress (using GPS and discovered process average times). Send SMS/app notifications with dynamic 2-hour windows.
- **Expected Impact:** Reduce failed deliveries by 25-30%, decreasing re-delivery costs and improving on-time rate.

### **Strategy 2: Route Optimization Based on Historical Performance Patterns**
- **Target Inefficiency:** High travel time vs. service time ratio; traffic delays.
- **Root Cause:** Routes planned using static maps, not accounting for historical traffic patterns or stop-time variability.
- **Process Mining Insight:** Conformance checking reveals planned routes frequently deviate due to traffic bottlenecks at specific intersections at specific times. Dwell time analysis shows certain addresses consistently require >10 minutes.
- **Proposal:** Use process-discovered travel times between stops (segmented by time of day and day of week) to re-optimize route sequences. Cluster stops with high dwell times together to minimize time variability.
- **Expected Impact:** Reduce travel time by 10-15%, increase stops per hour, and lower fuel consumption.

### **Strategy 3: Predictive Maintenance Scheduling**
- **Target Inefficiency:** Costly unscheduled stops and vehicle downtime.
- **Root Cause:** Maintenance based on fixed schedules, not actual usage and wear patterns.
- **Process Mining Insight:** Correlate `Unscheduled Stop` events with vehicle mileage (from GPS), harsh braking events (from speed data), and past maintenance activities. Predict failure likelihood.
- **Proposal:** Develop a risk score for each vehicle based on operational patterns. Schedule proactive maintenance during off-peak hours or before predicted high-impact failures.
- **Expected Impact:** Reduce unscheduled stops by 40-50%, increase vehicle availability, and lower emergency repair costs.

## **5. Operational Constraints and Monitoring**

### **Accounting for Constraints**
- **Driver Hours:** Optimization algorithms must incorporate max shift duration and mandatory break times, as seen in `End Shift` and break events in the log.
- **Vehicle Capacity:** Use package dimensions/weight from dispatch to ensure optimized routes don㦙 exceed capacity.
- **Customer Time Windows:** Process-mined successful delivery patterns will inform feasible time windows; optimization must respect these.

### **Continuous Monitoring Plan**
**Dashboard Key Views:**
1. **Process Health Dashboard:** Real-time process map highlighting deviations from optimized process. Metrics: Conformance rate, avg. process duration.
2. **Performance Trend Dashboard:** KPI trends over time (on-time rate, stops/hour). Compare before/after optimization implementation.
3. **Bottleneck Alert Dashboard:** Automated alerts when process metrics degrade (e.g., travel time exceeds threshold in a zone).
4. **Driver/Vehicle Scorecards:** Individual performance metrics to identify outliers for coaching or reward.

**Process Views to Track:**
- **Drift Detection:** Monitor for new process variants emerging post-optimization that may indicate new inefficiencies.
- **Seasonality Analysis:** Compare process performance across months to anticipate seasonal patterns (e.g., holiday traffic).
- **Impact Analysis:** A/B test optimized routes against legacy routes in controlled areas to quantify improvement.

By implementing this process mining approach, Speedy Parcels can transition from reactive problem-solving to proactive, data-driven operational excellence, directly addressing punctuality and cost concerns.
