## POWL Model 1: With Potential Unfairness

In this model, after the skill assessment there is an **XOR branch**: applicants either go through a standard `CulturalFitCheck` or take the `CommunityAffiliationCheck` path that introduces a subtle advantage.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# --- Activities ---
receive        = Transition(label="ReceiveApplication")
data_check     = Transition(label="DataCompletenessCheck")
request_info   = Transition(label="RequestMoreInfo")
skill          = Transition(label="SkillAssessment")
cultural_fit   = Transition(label="CulturalFitCheck")
community       = Transition(label="CommunityAffiliationCheck")  # bias source
manager_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# --- Loop for data completeness ---
# Execute DataCompletenessCheck; if incomplete -> RequestMoreInfo, then re-check.
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# --- XOR branch: standard cultural fit OR community-affiliation path (BIAS) ---
fit_choice = OperatorPOWL(operator=Operator.XOR, children=[cultural_fit, community])

# --- Overall partial order (sequential flow) ---
root_unfair = StrictPartialOrder(
    nodes=[receive, data_loop, skill, fit_choice, manager_review, final_decision]
)
root_unfair.order.add_edge(receive, data_loop)
root_unfair.order.add_edge(data_loop, skill)
root_unfair.order.add_edge(skill, fit_choice)
root_unfair.order.add_edge(fit_choice, manager_review)
root_unfair.order.add_edge(manager_review, final_decision)
```

**Where the unfairness appears:** The `OperatorPOWL(XOR, [CulturalFitCheck, CommunityAffiliationCheck])` node lets some applicants follow the `CommunityAffiliationCheck` path, which grants a subjective uplift. This selective routing tilts the evaluation in favor of affiliated/local candidates.

---

## POWL Model 2: Without Unfairness

Here the XOR branch is removed. **Every** applicant undergoes the *same* `CulturalFitCheck`, eliminating the affiliation-based shortcut. The data-completeness loop and the sequential skill check are retained.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# --- Activities ---
receive        = Transition(label="ReceiveApplication")
data_check     = Transition(label="DataCompletenessCheck")
request_info   = Transition(label="RequestMoreInfo")
skill          = Transition(label="SkillAssessment")
cultural_fit   = Transition(label="CulturalFitCheck")   # single uniform path
manager_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# --- Loop for data completeness (unchanged) ---
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# --- Overall partial order (sequential, no XOR / no bias branch) ---
root_fair = StrictPartialOrder(
    nodes=[receive, data_loop, skill, cultural_fit, manager_review, final_decision]
)
root_fair.order.add_edge(receive, data_loop)
root_fair.order.add_edge(data_loop, skill)
root_fair.order.add_edge(skill, cultural_fit)
root_fair.order.add_edge(cultural_fit, manager_review)
root_fair.order.add_edge(manager_review, final_decision)
```

---

## Summary of the Difference

| Aspect | Model 1 (Unfair) | Model 2 (Fair) |
|---|---|---|
| Data completeness | `LOOP(DataCompletenessCheck, RequestMoreInfo)` | Same |
| Skill assessment | Sequential `SkillAssessment` | Same |
| Cultural evaluation | **`XOR(CulturalFitCheck, CommunityAffiliationCheck)`** — selective advantage path | **Single `CulturalFitCheck`** — uniform for all |
| Managerial review & decision | Sequential | Same |
| Bias source | The XOR routing to `CommunityAffiliationCheck` | Removed |

Both models share the loop (data completeness) and the strictly ordered sequence (skill  cultural  review  decision). The **only structural difference** is the XOR choice node in Model 1, which is exactly where the subtle bias is injected; removing it (Model 2) guarantees every applicant follows an identical merit-based evaluation path.
