# BPMN Diagram Description: Dispatch of Goods Process

This is a **Business Process Model and Notation (BPMN)** diagram depicting the "Dispatch of goods" process for a **Computer Hardware Shop**. The diagram is organized into three horizontal swimlanes (roles).

## Swimlanes (Pools/Lanes)
1. **Logistics**
2. **Secretary**
3. **Warehouse**

## Process Flow

### Start
- The process begins with a **start event** labeled **"Ship goods"** (in the Secretary lane).
- This flows into a **parallel gateway (AND-split)**, which splits the process into two concurrent paths.

### Path 1 – Secretary Lane (Shipment Setup)
- **Clarify shipment method** task
- Leads to an **exclusive gateway (XOR)** labeled **"Special handling?"**
  - **No**  goes up to **"Check if insurance is required"** task (which references the *Customer information* data store), then through an event-based/intermediate gateway labeled **"always"** to **"Write package label."**
    - From here, an optional branch labeled **"If insurance necessary"** goes up to the **Logistics** lane to **"Insure parcel."**
    - Both paths converge at a **merging gateway**.
  - **Yes**  goes to **"Get 3 offers from logistic companies"**  **"Select logistic company and place order"**
- These branches rejoin at an **exclusive gateway (XOR-merge)**.

*Note:* An **"Order details"** document artifact is shown near the Secretary tasks.

### Path 2 – Warehouse Lane (Packaging)
- **Package goods** task, which has an attached **timer boundary event ("48 hours")**.
  - If the timer triggers (delay)  **"Notify customer about delay"**
- Paths merge at an **intermediate gateway**.

### Convergence
- Both major paths (shipment setup and packaging) come together at a **parallel gateway (AND-join)** in the Warehouse lane.
- This leads to:
  - **Fill out Shipment paperwork**
  - **Move package to pickup area**

### End
- The process concludes with an **end event** labeled **"Shipment prepared."**

## Key BPMN Elements Used
- **Events:** Start, intermediate, timer boundary, end
- **Gateways:** Parallel (AND), Exclusive (XOR), event-based
- **Tasks/Activities:** Rounded rectangles
- **Artifacts:** Data store (Customer information), Document (Order details)
- **Sequence flows:** Solid arrows
- **Associations:** Dashed lines (to data objects)

This diagram effectively illustrates the coordination between departments to prepare a shipment, including conditional handling, insurance decisions, and delay notifications.
