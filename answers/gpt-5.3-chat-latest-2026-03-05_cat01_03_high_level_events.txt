High-level process derived from the event sequences appears consistent across cases. The lowlevel events naturally cluster into four coherent phases based on temporal proximity, resource types, and functional purpose.

High-Level Step 1: Material Preparation and Setup  
Low-level events grouped:
- Retrieve raw metal sheet  
- Scan barcode  
- Place metal sheet on table  
- Align metal sheet  
- Preheat metal sheet  

Rationale:
These events all prepare the raw material before fabrication begins. They occur sequentially at the beginning of each case, involve preparation resources (Operator A, Robot Arm #2, Heating Unit #1), and transform the material from storage state to a ready-to-process state. The barcode scan also logically belongs here because it registers the material before processing.

Proposed name:
Material Preparation & Setup

General Rule for Aggregating Low-Level Events

Events can be grouped into higher-level stages using three main criteria:

- Functional similarity: activities contributing to the same manufacturing objective (e.g., preparation, welding, finishing).  
- Resource context: activities performed by the same operator type or machine group (e.g., welding operations by Operator B).  
- Temporal proximity and dependency: events that occur consecutively with no unrelated tasks between them and represent sequential sub-steps of a single stage.

Using these rules allows the full event log to be consistently abstracted into a clearer, interpretable manufacturing workflow.
