### 1. POWL Model with Potential Unfairness 
```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="Receive_Applications")
DataCompletenessCheck = Transition(label="Data_Completeness_Check")
RequestMoreInfo = Transition(label="Request_More_Info")  # Loop-back activity
SendQuestionnare = Transition(label="Send_Questionnaire")
EvaluateSkills = Transition(label="Skill_Assessment")
ApplyForAffiliation = Transition(label="Apply_For_Affiliation")  # CommunityCheck branch
StandardFitCheck = Transition(label="Standard_Cultural_Fit_Check")
CommunityAffiliationCheck = Transition(label="Community_Affiliation_Check")  # Bias source
ManagerReview = Transition(label="Managerial_Review")
FinalDecision = Transition(label="Final_Decision")

# Unfair process structure
unfair_model = StrictPartialOrder(nodes=[
    # Skill + Cultural check XOR
    OperatorPOWL(operator=Operator.XOR, children=[
        EvaluateSkills,
        OperatorPOWL(operator=Operator.LOOP, children=[StandardFitCheck, CommunityAffiliationCheck])
    ]),
    ManagerReview,
    FinalDecision
])

# Add dependencies
unfair_model.order.add_edge(ReceiveApplication, DataCompletenessCheck)
unfair_model.order.add_edge(DataCompletenessCheck, RequestMoreInfo)  # Loop if incomplete
unfair_model.order.add_edge(ReceiveApplication, SendQuestionnare)
unfair_model.order.add_edge(SendQuestionnare, EvaluateSkills)
unfair_model.order.add_edge(EvaluateSkills, unfair_model.children[0])
```

**Bias Origin:**  
The XOR operator (`EvaluateSkills  [StandardFitCheck OR CommunityAffiliationCheck]`) creates selective advantage. Candidates affiliated with local groups take the biased branch, receiving implicit score adjustments absent in the standard branch.

### 2. POWL Model Without Unfairness 
```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition

# Define activities (aligned with first model)
ReceiveApplication = Transition(label="Receive_Applications")
DataCompletenessCheck = Transition(label="Data_Completeness_Check")
RequestMoreInfo = Transition(label="Request_More_Info")
SendQuestionnare = Transition(label="Send_Questionnaire")
EvaluateSkills = Transition(label="Skill_Assessment")
StandardFitCheck = Transition(label="Standard_Cultural_Fit_Check")
ManagerReview = Transition(label="Managerial_Review")
FinalDecision = Transition(label="Final_Decision")

# Recursive loop for fairness
fair_model = StrictPartialOrder(nodes=[
    # Looped process without bias
    OperatorPOWL(operator=Operator.LOOP, children=[
        DataCompletenessCheck,
        OperatorPOWL(operator=Operator.LOOP, children=[RequestMoreInfo, SendQuestionnare])
    ]),
    EvaluateSkills,
    StandardFitCheck,
    ManagerReview,
    FinalDecision
])

# Dependencies (same as unfair model logic)
fair_model.order.add_edge(ReceiveApplication, DataCompletenessCheck)
fair_model.order.add_edge(DataCompletenessCheck, RequestMoreInfo)
fair_model.order.add_edge(RequestMoreInfo, SendQuestionnare)
fair_model.order.add_edge(SendQuestionnare, EvaluateSkills)
fair_model.order.add_edge(EvaluateSkills, StandardFitCheck)  # Only standard branch
fair_model.order.add_edge(StandardFitCheck, ManagerReview)
```

**Bias Removed:**  
The strict loop structure (`[DataCompletenessCheck  [RequestMoreInfo  SendQuestionnare]  EvaluateSkills  StandardFitCheck  ManagerReview]`) eliminates any XOR choice. All candidates share identical pathways, with no community-affiliation branch influencing evaluations.
