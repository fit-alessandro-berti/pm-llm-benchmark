SELECT o.case_id, o.order_type, o.order_value,

       COUNT(CASE WHEN oel.activity = 'Confirm Shipment' THEN 1 END) AS shipment_confirmation_count
FROM orders o
LEFT JOIN order_event_log oel ON o.case_id = oel.case_id
GROUP BY o.case_id, o.order_type, o.order_value
HAVING COUNT(CASE WHEN oel.activity IN ('Perform Credit Check', 'Validate Stock', 'Confirm Shipment') THEN 1 END) < 3;

### **2. Out-of-Order Activities**
**Anomaly:**
- **Case 1003**: "Ship Goods" happens before "Confirm Shipment".
- **Case 1004**: "Receive Payment" happens before "Issue Invoice".

**Hypotheses:**
- Data entry errors (timestamps incorrectly logged).
- Process bypasses (e.g., goods shipped without confirmation).
- Fraudulent activities (e.g., receiving payment before invoice).

**SQL Queries to Investigate:**
```sql
-- Find cases where activities occur out of the expected order
WITH activity_order AS (
  SELECT
    'Register Order' AS activity,
    1 AS expected_order
  UNION ALL SELECT 'Perform Credit Check', 2
  UNION ALL SELECT 'Validate Stock', 3
  UNION ALL SELECT 'Confirm Shipment', 4
  UNION ALL SELECT 'Ship Goods', 5
  UNION ALL SELECT 'Issue Invoice', 6
  UNION ALL SELECT 'Receive Payment', 7
)
SELECT oel.case_id, oel.activity, oel.timestamp,
       ao.expected_order,
       LAG(ao.expected_order) OVER (PARTITION BY oel.case_id ORDER BY oel.timestamp) AS prev_expected_order
FROM order_event_log oel
JOIN activity_order ao ON oel.activity = ao.activity
WHERE oel.activity IN ('Ship Goods', 'Receive Payment', 'Confirm Shipment')
  AND LAG(ao.expected_order) OVER (PARTITION BY oel.case_id ORDER BY oel.timestamp) > ao.expected_order;
```

### **8. Duplicate or Missing Event IDs**
**Anomaly:**
- Event IDs are not sequential (e.g., gaps in `event_id` for a `case_id`).

**Hypotheses:**
- Logging failures (e.g., events not recorded).
- Data corruption.

**SQL Queries to Investigate:**
```sql
-- Check for gaps in event_id sequence per case_id
WITH numbered_events AS (
  SELECT
    case_id,
    event_id,
    activity,
    ROW_NUMBER() OVER (PARTITION BY case_id ORDER BY event_id) AS expected_event_id
  FROM order_event_log
)
SELECT case_id, event_id, activity,
       expected_event_id,
       event_id - expected_event_id AS gap
FROM numbered_events
WHERE event_id != expected_event_id
ORDER BY case_id, event_id;
```
