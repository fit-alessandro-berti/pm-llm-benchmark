```sql
-- Replace ? with the desired K value, e.g. 10
WITH case_sequences AS (
    SELECT
        case_id,
        list(activity ORDER BY timestamp) AS activity_sequence
    FROM event_log
    GROUP BY case_id
),

variant_counts AS (
    SELECT
        activity_sequence,
        COUNT(*) AS case_count
    FROM case_sequences
    GROUP BY activity_sequence
),

top_k_variants AS (
    SELECT
        activity_sequence
    FROM variant_counts
    ORDER BY case_count DESC
    LIMIT ?
),

top_k_cases AS (
    SELECT
        cs.case_id
    FROM case_sequences cs
    INNER JOIN top_k_variants tkv
        ON cs.activity_sequence = tkv.activity_sequence
)

SELECT
    el.*
FROM event_log el
INNER JOIN top_k_cases tkc
    ON el.case_id = tkc.case_id
ORDER BY
    el.case_id,
    el.timestamp;
```

This query works as follows:

1. `case_sequences` builds the ordered activity sequence for each `case_id`.
2. `variant_counts` groups identical sequences and counts how many cases follow each variant.
3. `top_k_variants` selects the K most frequent variants.
4. `top_k_cases` finds all cases whose sequence belongs to one of those top K variants.
5. The final query returns all original events belonging only to those cases.
