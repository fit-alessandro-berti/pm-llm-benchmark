### POWL Model with Potential Unfairness
This model introduces an XOR branch after the skill assessment. One branch leads to a standard `CulturalFitCheck`, while the other executes a `CommunityAffiliationCheck` – a path that implicitly favors applicants with specific local connections or club memberships. This branching creates the opportunity for subtle bias.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# ----- Activities -----
rec_app      = Transition(label="ReceiveApplication")
data_check   = Transition(label="DataCompletenessCheck")
req_info     = Transition(label="RequestMoreInfo")
skill_assess = Transition(label="SkillAssessment")
cultural_fit = Transition(label="CulturalFitCheck")
community_aff= Transition(label="CommunityAffiliationCheck")
mgmt_review  = Transition(label="ManagerialReview")
final_dec    = Transition(label="FinalDecision")

# ----- Control-flow operators -----
# Loop: incomplete data triggers re-request
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, req_info])
# XOR: biased selection between standard cultural fit and community affiliation
fit_xor   = OperatorPOWL(operator=Operator.XOR, children=[cultural_fit, community_aff])

# ----- Partial order with unfairness -----
root_unfair = StrictPartialOrder(
    nodes=[rec_app, data_loop, skill_assess, fit_xor, mgmt_review, final_dec]
)
# Sequential ordering
root_unfair.order.add_edge(rec_app, data_loop)
root_unfair.order.add_edge(data_loop, skill_assess)
root_unfair.order.add_edge(skill_assess, fit_xor)
root_unfair.order.add_edge(fit_xor, mgmt_review)
root_unfair.order.add_edge(mgmt_review, final_dec)
```
