### 2. Root Cause Analysis (Attribute Correlation)

By analyzing the event logs, we can correlate the delays with specific attributes: **Complexity**, **Resource**, and **Region**.

#### A. Complexity Level (Primary Driver of Process Flow)
*   **Observation:** Low Complexity cases (2001, 2004) bypass the "Request Additional Documents" activity entirely. Medium and High Complexity cases trigger this activity.
*   **Correlation:**
    *   **Low:** 0 Requests.
    *   **Medium:** 1 Request (Case 2002).
    *   **High:** 2 Requests (Case 2003) to 3 Requests (Case 2005).
*   **Root Cause:** Higher complexity inherently requires more verification, leading to iterative loops of "Request Additional Documents." Each loop adds significant wait time (often overnight).

#### B. Resource Efficiency (Adjuster Performance)
*   **Observation:** We compare how Adjuster_Mike (Region A) and Adjuster_Lisa (Region B) handled High Complexity claims (Case 2003 vs. Case 2005).
*   **Comparison:**
    *   **Adjuster_Mike (Case 2003):** Sent the 1st Request at 11:00 and the 2nd Request at 17:00 on the **same day**.
    *   **Adjuster_Lisa (Case 2005):** Sent the 1st Request at 11:30 (Day 1), 2nd Request at 17:00 (Day 2), and 3rd Request at 15:00 (Day 3).
*   **Root Cause:** Adjuster_Lisa (Region B) is significantly slower in processing document requests. The gap between requests in Case 2005 is consistently **24+ hours**, whereas Mike managed a same-day turnaround in Case 2003. This suggests a resource-specific efficiency gap or workload issue in Region B.

#### C. Region (Process Consistency)

*   **Root Cause:** While Region B has the worst overall duration (Case 2005), the issue is not purely geographic. It is linked to the resources assigned there (Adjuster_Lisa). However, Region B also handled the Medium complexity case (2002) which had a 20-hour wait between "Request" and "Approve," compared to Region A's High complexity case where the wait was shorter relative to the complexity. This implies Region B's approval workflow (Manager/Finance handoff) may be slower after documentation is gathered.

### 3. Explanations and Mitigation Strategies

#### Explanation of Performance Issues
The extended durations are caused by a combination of **process loops** and **resource latency**.
1.  **The Loop Effect:** High-complexity claims trigger the "Request Additional Documents" activity. If the initial assessment is incomplete, the case loops back. Case 2005 looped 3 times.
2.  **The Latency Effect:** Even when documents are requested, the time taken to process them varies by resource. Adjuster_Lisa waits until the next day to re-evaluate or send new requests, whereas Adjuster_Mike does this within hours.
3.  **The Approval Bottleneck:** In Case 2005, there is a significant gap between the final document request (04-03 15:00) and Approval (04-04 10:00), suggesting the Manager (Bill) or Finance team in Region B is slower to finalize High Complexity claims.

#### Suggestions to Mitigate Root Causes

**1. Implement a "Single-Point" Document Request Policy**
*   **Issue:** Adjusters are requesting documents piecemeal (Case 2005 had 3 separate requests).
*   **Solution:** Mandate that Adjusters must request **all** missing documentation in a single communication. If a claim is missing 3 items, send one request for all 3, rather than three separate requests.
*   **Expected Impact:** Reduces Case 2005's duration from 77 hours to ~48 hours by eliminating redundant loops.

**2. Resource Training and Workload Balancing**
*   **Issue:** Adjuster_Lisa (Region B) shows slower turnaround times on document reviews compared to Adjuster_Mike.
*   **Solution:** Conduct training for Region B staff on initial claim assessment accuracy. Additionally, review workload distribution; ensure High Complexity claims are not disproportionately assigned to one resource.
*   **Expected Impact:** Reduces the wait time between "Request" and "Approve" activities.

**3. SLA Escalation for Document Loops**
*   **Issue:** There is no cap on how many times a claim can loop for documents.
*   **Solution:** Set a Service Level Agreement (SLA) where a claim cannot be requested for documents more than **2 times**. If documentation is still missing after the 2nd request, the claim must be escalated to a Senior Adjuster or Manager for a final decision (approve with partial docs or reject).
*   **Expected Impact:** Prevents "zombie cases" that drag on for days due to unresponsive claimants or indecisive adjusters.

**4. Automated Document Validation**
*   **Issue:** Human review is required to validate documents, causing delays.
*   **Solution:** Implement Optical Character Recognition (OCR) and automated validation for standard documents (e.g., IDs, invoices). If the system validates the document instantly, the "Request" activity is skipped.
*   **Expected Impact:** Drastically reduces the duration of Medium and High complexity cases by automating the initial "Evaluate" and "Request" steps.
