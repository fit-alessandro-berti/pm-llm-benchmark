Below is a complete Python construction of an **accepting Petri net** for the insurance claim management process. It uses:

* `pm4py.objects.petri_net.obj.PetriNet`
* `pm4py.objects.petri_net.obj.Marking`
* `pm4py.objects.petri_net.utils.petri_utils`

The net models the normal sequence of activities and includes two optional **repeatable loops**:

1. **DV  RDS  DV** for incomplete documents.
2. **FC  II  FC** for repeated in-depth investigations.

```python
from pm4py.objects.petri_net.obj import PetriNet, Marking
from pm4py.objects.petri_net.utils import petri_utils

# 1. Create the Petri net
net = PetriNet(name="insurance_claim_process")

# 2. Create places
source       = PetriNet.Place(name="source")           # initial state
after_C      = PetriNet.Place(name="after_C")
after_CR     = PetriNet.Place(name="after_CR")
after_DV     = PetriNet.Place(name="after_DV")
after_FC     = PetriNet.Place(name="after_FC")
after_CA     = PetriNet.Place(name="after_CA")
after_AP     = PetriNet.Place(name="after_AP")
after_P      = PetriNet.Place(name="after_P")
after_N      = PetriNet.Place(name="after_N")
sink         = PetriNet.Place(name="sink")             # final state

for p in [source, after_C, after_CR, after_DV, after_FC,
          after_CA, after_AP, after_P, after_N, sink]:
    net.places.add(p)

# 3. Create transitions (labels are the readable activity names)
C   = PetriNet.Transition(name="C",   label="Insurance Claim (C)")
CR  = PetriNet.Transition(name="CR",  label="Claim Registration (CR)")
DV  = PetriNet.Transition(name="DV",  label="Document Verification (DV)")
RDS = PetriNet.Transition(name="RDS", label="Re-Document Submission (RDS)")
FC  = PetriNet.Transition(name="FC",  label="Fraud Check (FC)")
II  = PetriNet.Transition(name="II",  label="In-depth Investigation (II)")
CA  = PetriNet.Transition(name="CA",  label="Claim Assessment (CA)")
AP  = PetriNet.Transition(name="AP",  label="Approval (AP)")
P   = PetriNet.Transition(name="P",   label="Payment (P)")
N   = PetriNet.Transition(name="N",   label="Notification (N)")
CL  = PetriNet.Transition(name="CL",  label="Closure (CL)")

for t in [C, CR, DV, RDS, FC, II, CA, AP, P, N, CL]:
    net.transitions.add(t)

# 4. Add arcs

# Main flow
petri_utils.add_arc_from_to(source,   C,   net)
petri_utils.add_arc_from_to(C,        after_C,  net)
petri_utils.add_arc_from_to(after_C,  CR,  net)
petri_utils.add_arc_from_to(CR,       after_CR, net)
petri_utils.add_arc_from_to(after_CR, DV,  net)
petri_utils.add_arc_from_to(DV,       after_DV, net)

# DV loop: documents incomplete -> re-submit -> verify again
petri_utils.add_arc_from_to(after_DV, RDS, net)
petri_utils.add_arc_from_to(RDS,      after_CR, net)

# Continue to fraud check
petri_utils.add_arc_from_to(after_DV, FC,  net)
petri_utils.add_arc_from_to(FC,       after_FC, net)

# FC loop: doubts -> in-depth investigation -> fraud check again
petri_utils.add_arc_from_to(after_FC, II,  net)
petri_utils.add_arc_from_to(II,       after_DV, net)

# Continue main flow
petri_utils.add_arc_from_to(after_FC, CA,  net)
petri_utils.add_arc_from_to(CA,       after_CA, net)
petri_utils.add_arc_from_to(after_CA, AP,  net)
petri_utils.add_arc_from_to(AP,       after_AP, net)
petri_utils.add_arc_from_to(after_AP, P,   net)
petri_utils.add_arc_from_to(P,        after_P,  net)
petri_utils.add_arc_from_to(after_P,  N,   net)
petri_utils.add_arc_from_to(N,        after_N,  net)
petri_utils.add_arc_from_to(after_N,  CL,  net)
petri_utils.add_arc_from_to(CL,       sink,     net)

# 5. Define initial and final markings
im = Marking()
im[source] = 1

fm = Marking()
fm[sink] = 1

print("Accepting Petri net constructed.")
print("Initial marking:", im)
print("Final marking:  ", fm)
```

### How the net behaves

| Path | Description |
|------|-------------|
| `source  C  CR  DV  FC  CA  AP  P  N  CL  sink` | Happy path: no re-submission and no investigation. |
| `...  DV  RDS  DV  FC  ...` | Documents are incomplete, so **Re-Document Submission** loops back to **Document Verification**. This can happen any number of times. |
| `...  FC  II  FC  CA  ...` | Fraud check raises doubts, triggering **In-Depth Investigation**, which loops back to **Fraud Check**. This can also happen any number of times. |

The **initial marking** places one token in `source`, representing a newly arrived claim. The **final marking** requires one token in `sink`, representing a successfully closed claim.
