**DuckDB SQL Query**

```sql
WITH 
-- Step 1: Extract the ordered sequence of activities per case
case_sequences AS (
    SELECT 
        case_id,
        list_agg(activity ORDER BY "timestamp") AS sequence
    FROM event_log
    GROUP BY case_id
),

-- Step 2: Count cases per variant (process variant)
variant_stats AS (
    SELECT 
        sequence,
        COUNT(*) AS frequency
    FROM case_sequences
    GROUP BY sequence
),

-- Step 3: Identify the top K variants
top_k_variants AS (
    SELECT sequence
    FROM variant_stats
    ORDER BY frequency DESC
    LIMIT K  -- <-- Replace K with desired value (e.g. 5, 10)
)

-- Step 4: Return all events belonging to cases that follow a top-K variant
SELECT 
    el.*
FROM event_log el
JOIN case_sequences cs 
    ON el.case_id = cs.case_id
WHERE cs.sequence IN (SELECT sequence FROM top_k_variants)
ORDER BY 
    el.case_id,
    el."timestamp";
```

### Key Design Choices

| Aspect                        | Implementation                                      | Rationale |
|------------------------------|-----------------------------------------------------|---------|
| **Sequence extraction**      | `list_agg(activity ORDER BY "timestamp")`           | Preserves exact order using DuckDB's ordered aggregate |
| **Variant representation**   | `LIST` type                                         | Exact matching (no risk of collision vs string concatenation) |
| **Top-K selection**          | `ORDER BY frequency DESC LIMIT K`                   | Standard and efficient |
| **Filtering**                | `WHERE ... IN (SELECT ...)`                         | Clean separation between variant ranking and event filtering |
| **Final output**             | All original columns (`el.*`)                       | Returns the filtered event log as requested |
