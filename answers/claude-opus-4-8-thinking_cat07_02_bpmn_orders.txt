# BPMN Process Diagram Description

This is a **Business Process Model and Notation (BPMN)** diagram illustrating an order fulfillment process across four organizational departments (swimlanes).

## Swimlanes (Departments)

### 1. Sales
- **Trigger:** Timer event "Monday morning"
- **Subprocess "Check customer's order":**
  - Check order for completeness
  - Check if customer is already in the database
  - Gateway (XOR) splits: if "Customer not in the database"  Add customer to the database (connects to a **database** store); if "Customer in the database"  proceeds
  - Gateways merge before exiting
- Send order details
- Receives "Product sent" message
- Draft invoice  End event

### 2. Production
- **Start:** "Order details arrive" (message event)
- **Parallel gateway (AND)** splits work:
  - Plan production
  - Manufacture product
- Includes a "Material not faulty" message event (boundary/intermediate)
- Ends with "Product complete"

### 3. Procurement
- **Start:** "Order details arrive" (message event)
- **Material Compensation Subprocess:**
  - Order materials (with compensation marker)
  - Return material (compensation activity)
- Check materials
- Branches to either a message event or "Order cancelation"

### 4. Warehouse
- **Start:** "Product complete" (message event)
- Send product
- "Product sent" message event (end)

## Key Flow Connections
- **Dashed lines** represent **message flows** between lanes (e.g., "Order details" passing from Sales to Production/Procurement; "Product sent" notifications)
- **Solid lines** represent **sequence flows** within each lane

## Notable Elements
- **Gateways:** Exclusive (X) for decisions, Parallel (+) for concurrent tasks
- **Events:** Timer (clock), Message (envelope), Compensation, Cancel
- **Data store:** Customer database

This diagram models a complete order-to-delivery workflow with coordination between sales, production, procurement, and warehousing functions.
