**POWL Model with Potential Unfairness:**

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="ReceiveApplication")
ResumeParsingCheck = Transition(label="ResumeParsingCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Define loop for data completeness
data_completeness_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[ResumeParsingCheck, RequestMoreInfo]
)

# Define XOR branch for biased cultural evaluation
cultural_evaluation_xor = OperatorPOWL(
    operator=Operator.XOR,
    children=[CulturalFitCheck, CommunityAffiliationCheck]
)

# Build partial order with potential unfairness
root_unfair = StrictPartialOrder(nodes=[
    ReceiveApplication,
    data_completeness_loop,
    SkillAssessment,
    cultural_evaluation_xor,
    ManagerialReview,
    FinalDecision
])

# Define execution order
root_unfair.order.add_edge(ReceiveApplication, data_completeness_loop)
root_unfair.order.add_edge(data_completeness_loop, SkillAssessment)
root_unfair.order.add_edge(SkillAssessment, cultural_evaluation_xor)
root_unfair.order.add_edge(cultural_evaluation_xor, ManagerialReview)
root_unfair.order.add_edge(ManagerialReview, FinalDecision)
```

**POWL Model Without Unfairness:**

```python
# Reuse basic activities from previous model
# Define modified cultural evaluation path
CulturalFitCheckUnbiased = Transition(label="CulturalFitCheck")

# Build partial order with fair evaluation
root_fair = StrictPartialOrder(nodes=[
    ReceiveApplication,
    data_completeness_loop,
    SkillAssessment,
    CulturalFitCheckUnbiased,
    ManagerialReview,
    FinalDecision
])

# Define execution order
root_fair.order.add_edge(ReceiveApplication, data_completeness_loop)
root_fair.order.add_edge(data_completeness_loop, SkillAssessment)
root_fair.order.add_edge(SkillAssessment, CulturalFitCheckUnbiased)
root_fair.order.add_edge(CulturalFitCheckUnbiased, ManagerialReview)
root_fair.order.add_edge(ManagerialReview, FinalDecision)
```

**Key Differences:**

1. **XOR Branch Removal:**  
   The fair model eliminates the `OperatorPOWL` with XOR choice between cultural evaluation paths, replacing it with a single standardized `CulturalFitCheck`.

2. **Explicit Sequence Enforcement:**  
   Both models maintain the same core structure for initial steps (loop for data completeness) and final steps (managerial review), but the fair model removes the conditional path that could introduce community-based bias.

3. **Transparency:**  
   The fair model ensures all candidates follow identical evaluation steps after the skill assessment, removing subjective branching that could advantage specific groups.
