```markdown
# Invoice Approval Process

## Start Event
- An invoice is received (typically via email).

## Activities
1. **Check PO Number**
   - Performed by: Accounts Payable Clerk
   - Description: Verify if the invoice has a valid purchase order (PO) number.

2. **Request PO from Supplier** (if no PO)
   - Performed by: Accounts Payable Clerk
   - Description: Send the invoice back to the supplier requesting a valid PO number.

3. **Log Invoice**
   - Performed by: Accounts Payable Clerk
   - Description: Enter the invoice details into the accounting system.

4. **Forward to Purchasing**
   - Performed by: Accounts Payable Clerk
   - Description: Send the invoice to the Purchasing department for confirmation of goods/services received.

5. **Confirm Receipt and Match PO**
   - Performed by: Purchasing Officer
   - Description: Compare invoice details (quantities, prices, delivery charges) against the PO.

6. **Resolve Discrepancies** (if any)
   - Performed by: Accounts Payable Clerk, Purchasing Officer, or Accounts Payable Supervisor
   - Description: Address any discrepancies by contacting the supplier or internal clarification. Escalate to supervisor if unresolved.

7. **Manager Approval**
   - Performed by: Department Manager
   - Description: Review the invoice, PO, and confirmation details. Approve or reject based on review.

8. **Schedule Payment**
   - Performed by: Accounts Payable Clerk
   - Description: Prepare the invoice for the next payment cycle.

## Sequence Flow
1. Start with receiving an invoice.
2. Check if the invoice has a valid PO number.
   - If no PO, request PO from supplier and wait for response (loop back to start if PO is provided).
   - If PO exists, log the invoice into the accounting system.
3. Forward the logged invoice to the Purchasing department.
4. Purchasing checks the invoice against the PO.
   - If discrepancy found, resolve discrepancies (may involve Supplier Relationship Manager).
     - If resolved, proceed to next step.
     - If not resolved, escalate to Accounts Payable Supervisor.
   - If no discrepancy, send confirmation back to Accounts Payable.
5. Accounts Payable matches invoice line items against the PO in the system.
   - If discrepancy found, email Purchasing to clarify.
     - If resolved, proceed to next step.
     - If not resolved, escalate to supervisor.
   - If no discrepancy, send to Department Manager for approval.
6. Department Manager reviews the invoice.
   - If approved, proceed to schedule payment.
   - If rejected, investigate further (may loop back to previous steps for resolution).
7. Schedule the invoice for payment.

## Gateways (Decision Points)
1. **PO Check Gateway**
   - Type: Exclusive (XOR)
   - Conditions:
     - PO exists?  Log Invoice
     - No PO?  Request PO from Supplier

2. **Purchasing Confirmation Gateway**
   - Type: Exclusive (XOR)
   - Conditions:
     - Discrepancy found?  Resolve Discrepancies
     - No discrepancy?  Send confirmation to AP

3. **Invoice Matching Gateway**
   - Type: Exclusive (XOR)
   - Conditions:
     - Discrepancy found?  Email Purchasing to clarify
     - No discrepancy?  Send to Manager for approval
     - Can't resolve?  Escalate to Supervisor

4. **Manager Approval Gateway**
   - Type: Exclusive (XOR)
   - Conditions:
     - Approved?  Schedule Payment
     - Rejected?  Investigate Further

## Roles
- **Accounts Payable Clerk**: Checks PO, logs invoice, matches invoice to PO, schedules payment.
- **Purchasing Officer**: Confirms receipt, matches invoice to PO.
- **Department Manager**: Approves invoice.
- **Accounts Payable Supervisor**: Handles exceptions and escalations.
- **Supplier Relationship Manager**: Helps resolve supplier-related issues.

## Artifacts
- **Invoice**: The document that moves through the process.
- **Purchase Order (PO)**: Referenced for matching and validation.
- **Accounting System**: Used for logging and matching invoices.
- **Confirmation Email**: Sent from Purchasing to Accounts Payable.
- **Notification**: Sent to Manager for approval.

## Exceptions/Variations
1. **Invoice without a PO**:
   - Sent back to supplier to request PO.
   - If no PO can be obtained, escalated to supervisor to decide on retrospective PO or rejection.

2. **Discrepancies in invoice details**:
   - Resolved by contacting the supplier or internal clarification.
   - If unresolved, escalated to supervisor.

3. **Manager rejects invoice**:
   - Requires further investigation, possibly looping back to earlier steps for resolution.
```
