**DuckDB SQL Query:**

```sql
WITH ordered_events AS (
    SELECT 
        case_id,
        activity,
        timestamp,
        list(activity ORDER BY timestamp) AS variant
    FROM event_log
    GROUP BY case_id
),

variant_counts AS (
    SELECT 
        variant,
        COUNT(*) AS frequency,
        ROW_NUMBER() OVER (ORDER BY COUNT(*) DESC) AS rank
    FROM ordered_events
    GROUP BY variant
),

top_k_variants AS (
    SELECT variant
    FROM variant_counts
    WHERE rank <= :K
),

top_k_cases AS (
    SELECT o.case_id
    FROM ordered_events o
    JOIN top_k_variants t ON o.variant = t.variant
)

SELECT 
    e.case_id,
    e.activity,
    e.timestamp
FROM event_log e
JOIN top_k_cases t ON e.case_id = t.case_id
ORDER BY e.case_id, e.timestamp;
```

**Explanation:**

- `ordered_events`: Builds the exact activity sequence (`list`) for each `case_id` by ordering activities by `timestamp`.
- `variant_counts`: Counts how many cases share each sequence and ranks them by frequency using `ROW_NUMBER()`.
- `top_k_variants`: Selects only the top K most frequent variants (using parameter `:K`).
- `top_k_cases`: Identifies all `case_id`s that follow one of the top K variants.
- Final `SELECT`: Returns all original events belonging to those cases, ordered chronologically within each case.

This query correctly:
- Preserves the timestamp-based order when building variants
- Groups by complete activity sequence
- Identifies top K variants by frequency
- Returns only events from cases belonging to those top K variants

**Usage:** Replace `:K` with your desired number (e.g. `5` or use a parameter binding).
