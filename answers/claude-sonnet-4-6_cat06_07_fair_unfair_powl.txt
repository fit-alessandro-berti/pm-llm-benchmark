## Analysis of the Process Structure

**Common elements in both models:**
1. `ReceiveApplication`  start
2. **Loop** for data completeness: `DataCompletenessCheck` *(do)* + `RequestMoreInfo` *(redo)*
3. `SkillAssessment`  scored automatically
4. Cultural fit stage  **differs between models**
5. `ManagerialReview`
6. `FinalDecision`

## Model 1: POWL with Potential Unfairness

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

#  Activities 
ReceiveApplication      = Transition(label="ReceiveApplication")
DataCompletenessCheck   = Transition(label="DataCompletenessCheck")
RequestMoreInfo         = Transition(label="RequestMoreInfo")
SkillAssessment         = Transition(label="SkillAssessment")
CulturalFitCheck        = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview        = Transition(label="ManagerialReview")
FinalDecision           = Transition(label="FinalDecision")

#  Loop: keep asking for more info until application is complete 
#    * ( DataCompletenessCheck, RequestMoreInfo )
#     execute DataCompletenessCheck; if incomplete execute RequestMoreInfo then repeat
data_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[DataCompletenessCheck, RequestMoreInfo]
)

#  XOR: standard cultural fit  vs.  community-affiliation fast-track 
#    X ( CulturalFitCheck, CommunityAffiliationCheck )
#     THIS is the bias injection point
cultural_xor = OperatorPOWL(
    operator=Operator.XOR,
    children=[CulturalFitCheck, CommunityAffiliationCheck]
)

#  Root partial order (strict sequential pipeline) 
root = StrictPartialOrder(
    nodes=[
        ReceiveApplication,
        data_loop,
        SkillAssessment,
        cultural_xor,
        ManagerialReview,
        FinalDecision
    ]
)

# Enforce left-to-right sequential ordering
root.order.add_edge(ReceiveApplication,    data_loop)
root.order.add_edge(data_loop,             SkillAssessment)
root.order.add_edge(SkillAssessment,       cultural_xor)
root.order.add_edge(cultural_xor,          ManagerialReview)
root.order.add_edge(ManagerialReview,      FinalDecision)
```

### Why this model contains unfairness

```
ReceiveApplication
        
        
* ( DataCompletenessCheck, RequestMoreInfo )    loop until complete
        
        
  SkillAssessment
        
        
X ( CulturalFitCheck  |  CommunityAffiliationCheck )    BIAS POINT
                                      
        standard path           affiliation path
        (pure merit)          (implicit score uplift)
              
                         
                  ManagerialReview
                         
                         
                   FinalDecision
```

The `XOR` node routes some applicants through `CommunityAffiliationCheck` instead of the standard `CulturalFitCheck`. Applicants recognized as local residents or members of the "Global Business Leaders Club" can be steered into this branch, receiving an implicit advantage that is unrelated to job-relevant merit.

## Model 2: POWL Without Unfairness

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

#  Activities 
ReceiveApplication    = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo       = Transition(label="RequestMoreInfo")
SkillAssessment       = Transition(label="SkillAssessment")
CulturalFitCheck      = Transition(label="CulturalFitCheck")   # single unified path
ManagerialReview      = Transition(label="ManagerialReview")
FinalDecision         = Transition(label="FinalDecision")

#  Loop: keep asking for more info until application is complete 
#    * ( DataCompletenessCheck, RequestMoreInfo )
data_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[DataCompletenessCheck, RequestMoreInfo]
)

#  Root partial order 
#    No XOR branch at the cultural-fit stage  every applicant takes the
#    identical CulturalFitCheck; community affiliation plays no role.
root = StrictPartialOrder(
    nodes=[
        ReceiveApplication,
        data_loop,
        SkillAssessment,
        CulturalFitCheck,      #  replaces the XOR; one fair path for all
        ManagerialReview,
        FinalDecision
    ]
)

# Enforce left-to-right sequential ordering
root.order.add_edge(ReceiveApplication, data_loop)
root.order.add_edge(data_loop,          SkillAssessment)
root.order.add_edge(SkillAssessment,    CulturalFitCheck)
root.order.add_edge(CulturalFitCheck,   ManagerialReview)
root.order.add_edge(ManagerialReview,   FinalDecision)
```

### Why this model is fair

```
ReceiveApplication
        
        
* ( DataCompletenessCheck, RequestMoreInfo )    loop until complete
        
        
  SkillAssessment
        
        
  CulturalFitCheck         single, uniform evaluation for ALL applicants
                            (no affiliation-based branching)
        
  ManagerialReview
        
        
  FinalDecision
```

By **removing the XOR branch** and replacing it with a single mandatory `CulturalFitCheck`, every applicant—regardless of community membership, local residency, or educational background—is evaluated using exactly the same criteria. The loop for data completeness and the skill-assessment gate are retained, as these represent legitimate, merit-based process controls.

## Side-by-side Summary

| Aspect | Model 1 (Unfair) | Model 2 (Fair) |
|---|---|---|
| Data completeness | `* (DataCompletenessCheck, RequestMoreInfo)` | `* (DataCompletenessCheck, RequestMoreInfo)` |
| Skill gate | `SkillAssessment` | `SkillAssessment` |
| Cultural fit stage | `X (CulturalFitCheck, CommunityAffiliationCheck)` | `CulturalFitCheck` only |
| Bias mechanism | XOR routes affiliated applicants to a lenient branch | No branching; uniform evaluation |
| Post-fit review | `ManagerialReview  FinalDecision` | `ManagerialReview  FinalDecision` |
