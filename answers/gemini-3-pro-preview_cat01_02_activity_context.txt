### 3. Justification for Naming Choices

**1. `Initial Doc Verification` vs. `Re-verify Documents`**
*   **Original Activity:** `Verify Documents`
*   **Justification:** The first instance is a mandatory step in the "happy path." The second instance only appears if there was a failure or adjustment in the process (Looping). Naming the second instance `Re-verify` allows process analysts to easily count how many cases required rework, distinguishing "First Time Right" cases from those needing correction.

**2. `Call: Request Docs`**
*   **Original Activity:** `Call Customer` (Case C001)
*   **Justification:** The context explicitly states "request missing docs." This is a blockage activity where the process cannot proceed without input.

**3. `Call: Clarify Discrepancy`**
*   **Original Activity:** `Call Customer` (Case C002, 1st instance)
*   **Justification:** The context indicates an "account discrepancy." This is an investigative step, different from a simple document request.

**4. `Call: Confirmation`**
*   **Original Activity:** `Call Customer` (Case C002, 2nd instance)
*   **Justification:** Occurring *after* the `Adjust Account` activity, this call is purely informational to satisfy customer service protocols, rather than to gather data.

**5. `Call: Branch Logistics`**
*   **Original Activity:** `Call Customer` (Case C003)
*   **Justification:** The resource is a `BranchOfficer` and the context is "Customer stepped out." This is physically distinct from a remote support call; it is a logistical coordination unique to the in-person channel.
