## Explanation of the Logic

### 1. Case Identification Strategy
I used the **document/artifact** (the `Window` title combined with the application) as the natural **case identifier**. Each distinct artifact represents a logical unit of work that a knowledge worker is performing:

- **Quarterly_Report.docx**  CASE-001
- **Document1.docx**  CASE-002
- **Email (Annual Meeting)**  CASE-003
- **Report_Draft.pdf**  CASE-004
- **Budget_2024.xlsx**  CASE-005

This artifact-centric approach was chosen because the user **multitasks and interleaves** across applications. A simple time-based or session-based segmentation would mix unrelated activities. The artifact-based view cleanly captures *concurrent* and *resumed* work — for example, the user returns to **Document1.docx** at 09:06 after working in Excel, and to **Quarterly_Report.docx** at 09:07 after originally opening it at 08:59. This interleaving is preserved correctly because events are grouped by artifact, not by contiguous time blocks.

### 2. Activity Name Standardization
Raw low-level actions were mapped to higher-level, process-relevant activities:

| Raw Action | Standardized Activity | Rationale |
|-----------|----------------------|-----------|
| FOCUS / SWITCH (into artifact) | Open Document / Resume Document | Marks the start or resumption of work on a case |
| TYPING (in docs) | Draft Content | Authoring text |
| TYPING (in Excel) | Edit Spreadsheet | Data entry |
| TYPING (in email reply) | Compose Reply | Email authoring |
| SAVE | Save Document / Save Spreadsheet | Persistence milestone |
| SCROLL / HIGHLIGHT | Review / Annotate Document | Reading & marking up |
| CLICK (open email) | Open Email | |
| CLICK (send) | Send Email | Completion of email handling |
| CLOSE | Close Document | Case completion |

### 3. Coherent Narrative
The log tells the story of a single worker assembling a **Quarterly Report**: they open the report, draft a working document, confirm meeting logistics via email, review a draft PDF, update supporting budget figures, link the budget back into the working document, then finalize the executive summary in the report itself. The artifact-based casing reveals this realistic, interleaved knowledge-work pattern — ideal for discovering rework loops and cross-application dependencies in a process mining tool.

> **Tip for analysis:** Sorting the full table by `Timestamp` reproduces the real user session, while filtering by `Case ID` isolates each artifact's lifecycle for conformance or performance analysis.
