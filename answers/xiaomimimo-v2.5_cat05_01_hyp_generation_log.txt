## Proposed SQL Queries for Investigation

### Query 1: Detect cases where Ship Goods occurs before Perform Credit Check

```sql
-- Cases where goods were shipped before credit was checked
SELECT
    ship.case_id,
    cc.timestamp AS credit_check_time,
    ship.timestamp AS ship_time,
    ship.timestamp - cc.timestamp AS time_gap,
    o.order_value,
    o.order_type
FROM order_event_log ship
JOIN order_event_log cc
    ON ship.case_id = cc.case_id
    AND ship.activity = 'Ship Goods'
    AND cc.activity = 'Perform Credit Check'
    AND ship.timestamp < cc.timestamp
JOIN orders o ON ship.case_id = o.case_id
ORDER BY ship.timestamp;
```

### Query 2: Detect cases where Ship Goods occurs before Validate Stock

```sql
-- Cases where goods were shipped before stock was validated
SELECT
    ship.case_id,
    sv.timestamp AS stock_validation_time,
    ship.timestamp AS ship_time,
    o.order_value,
    o.order_type
FROM order_event_log ship
LEFT JOIN order_event_log sv
    ON ship.case_id = sv.case_id
    AND sv.activity = 'Validate Stock'
    AND sv.timestamp < ship.timestamp
JOIN orders o ON ship.case_id = o.case_id
WHERE ship.activity = 'Ship Goods'
  AND (sv.event_id IS NULL OR ship.timestamp < sv.timestamp)
ORDER BY ship.timestamp;
```

### Query 3: Detect cases missing key activities entirely

```sql
-- Cases missing any of the required activities
SELECT
    o.case_id,
    o.order_type,
    o.order_value,
    CASE WHEN cc.event_id IS NULL THEN 'MISSING' ELSE 'OK' END AS credit_check_status,
    CASE WHEN sv.event_id IS NULL THEN 'MISSING' ELSE 'OK' END AS stock_validation_status,
    CASE WHEN cs.event_id IS NULL THEN 'MISSING' ELSE 'OK' END AS confirm_shipment_status
FROM orders o
LEFT JOIN order_event_log cc
    ON o.case_id = cc.case_id AND cc.activity = 'Perform Credit Check'
LEFT JOIN order_event_log sv
    ON o.case_id = sv.case_id AND sv.activity = 'Validate Stock'
LEFT JOIN order_event_log cs
    ON o.case_id = cs.case_id AND cs.activity = 'Confirm Shipment'
WHERE cc.event_id IS NULL
   OR sv.event_id IS NULL
   OR cs.event_id IS NULL
ORDER BY o.order_value DESC;
```

### Query 4: Detect cases where Receive Payment happens before Ship Goods

```sql
-- Payment received before goods shipped — potential prepaid anomaly
SELECT
    pay.case_id,
    pay.timestamp AS payment_time,
    ship.timestamp AS ship_time,
    o.order_value,
    o.order_type
FROM order_event_log pay
JOIN order_event_log ship
    ON pay.case_id = ship.case_id
    AND pay.activity = 'Receive Payment'
    AND ship.activity = 'Ship Goods'
    AND pay.timestamp < ship.timestamp
JOIN orders o ON pay.case_id = o.case_id
ORDER BY o.order_value DESC;
```

### Query 5: Activity ordering violations — Confirm Shipment after Ship Goods

```sql
-- Confirm Shipment happening after Ship Goods (reverse order)
SELECT
    ship.case_id,
    cs.timestamp AS confirm_time,
    ship.timestamp AS ship_time,
    cs.additional_info AS confirm_info,
    r.role AS confirm_resource_role,
    r.department AS confirm_department
FROM order_event_log ship
JOIN order_event_log cs
    ON ship.case_id = cs.case_id
    AND ship.activity = 'Ship Goods'
    AND cs.activity = 'Confirm Shipment'
    AND cs.timestamp > ship.timestamp
LEFT JOIN resources r ON cs.resource = r.resource_id
ORDER BY ship.timestamp;
```

### Query 6: Identify resources involved in anomalous cases

```sql
-- Resources that participated in cases with out-of-order activities
SELECT
    el.resource,
    r.role,
    r.department,
    COUNT(DISTINCT el.case_id) AS anomalous_case_count,
    STRING_AGG(DISTINCT el.activity, ', ' ORDER BY el.activity) AS activities_performed
FROM order_event_log el
JOIN resources r ON el.resource = r.resource_id
WHERE el.case_id IN (
    -- Subquery: cases where Ship Goods precedes Credit Check
    SELECT s.case_id
    FROM order_event_log s
    JOIN order_event_log c
        ON s.case_id = c.case_id
        AND s.activity = 'Ship Goods'
        AND c.activity = 'Perform Credit Check'
        AND s.timestamp < c.timestamp
    UNION
    -- Cases missing Credit Check
    SELECT o.case_id
    FROM orders o
    WHERE NOT EXISTS (
        SELECT 1 FROM order_event_log el
        WHERE el.case_id = o.case_id
        AND el.activity = 'Perform Credit Check'
    )
    UNION
    -- Cases missing Validate Stock
    SELECT o.case_id
    FROM orders o
    WHERE NOT EXISTS (
        SELECT 1 FROM order_event_log el
        WHERE el.case_id = o.case_id
        AND el.activity = 'Validate Stock'
    )
)
GROUP BY el.resource, r.role, r.department
ORDER BY anomalous_case_count DESC;
```

### Query 7: Average time between activities per case — detect unusual timing patterns

```sql
-- Time gap between Register Order and the next activity
-- Anomalous cases show very short or very long gaps
SELECT
    reg.case_id,
    reg.timestamp AS register_time,
    next_act.activity AS next_activity,
    next_act.timestamp AS next_time,
    next_act.timestamp - reg.timestamp AS gap,
    o.order_value
FROM order_event_log reg
JOIN LATERAL (
    SELECT *
    FROM order_event_log el
    WHERE el.case_id = reg.case_id
      AND el.activity != 'Register Order'
    ORDER BY el.timestamp ASC
    LIMIT 1
) next_act ON TRUE
JOIN orders o ON reg.case_id = o.case_id
WHERE reg.activity = 'Register Order'
ORDER BY gap ASC;
```

### Query 8: Full process variant analysis — group cases by their activity sequence

```sql
-- Identify all unique process variants and their frequency
SELECT
    STRING_AGG(el.activity, '  ' ORDER BY el.timestamp) AS process_variant,
    COUNT(DISTINCT el.case_id) AS case_count,
    STRING_AGG(DISTINCT CAST(el.case_id AS VARCHAR), ', ') AS case_ids
FROM order_event_log el
GROUP BY el.case_id
ORDER BY case_count DESC;
```

## Summary of Risk Assessment

| Case | Order Value | Anomalies | Risk Level |
|------|------------|-----------|------------|
| 1001 | $1,250 | None — follows standard process |  Low |
| 1002 | $2,000 | Credit Check & Validate Stock done AFTER shipment; `attempted_early=Y` |  High |
| 1003 | $1,500 | Missing Validate Stock; Confirm Shipment after Ship Goods; `late_confirmation=Y` |  High |
| 1004 | $3,000 | Payment before any processing; Missing Credit Check & Validate Stock; Invoice after Payment; `shipment_scheduled=N` |  Critical |

**Key Recommendations:**
1. **Enforce sequential controls** at the system level — Ship Goods should not be executable without prior Perform Credit Check and Validate Stock.
2. **Investigate LogisticsMgr_2 and WarehouseClerk_3** who are involved in multiple anomalous cases.
3. **Audit Case 1004** as the highest-value order with the most severe control gaps — potential fraud or serious system malfunction.
4. **Implement automated alerts** for activities logged out of the expected sequence.
5. **Review the `priority` order type** — Case 1002 suggests priority orders may be systematically bypassing controls.
