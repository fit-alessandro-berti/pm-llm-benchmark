Below is a consolidated **event log** suitable for process mining.  
I used **`order_id` as the case ID** and created one row per meaningful event.  
For line items, deliveries, invoices, and payments, I modeled each record as a separate event when it represents a distinct lifecycle step.

### Assumptions Made
- **Order Created** comes from the `Orders` table.
- **Item Picked** is modeled as one event per row in `Order Lines`.
- **Order Dispatched** and **Order Arrived** are modeled as one event per delivery record in `Delivery`.
- **Invoice Issued** is modeled as one event per invoice record in `Invoice`.
- **Payment Received** is modeled as one event per payment record in `Payments`.
- When multiple events of the same type exist for one order, they are included separately in chronological order.
- Timestamps are taken directly from the source tables and assumed to be accurate event times.
