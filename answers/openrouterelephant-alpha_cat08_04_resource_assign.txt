**Process Mining Approach to Optimizing Resource Assignment in TechSolve’s IT Service Desk**

---

### 1. Analyzing Resource Behavior and Assignment Patterns

To analyze agent and tier performance, we apply **process mining** techniques to the event log, transforming event data into actionable insights:

**Key Metrics to Analyze:**
- **Workload Distribution:** Total ticket volume handled per agent, per tier, and per skill set.
- **Processing Times:** Average handling time per agent/tier, segmented by ticket priority, category, and required skill.
- **First-Call Resolution (FCR) Rate for L1:** Percentage of tickets resolved at L1 without escalation.
- **Escalation Frequency:** Rate of escalations from L1  L2 and L2  L3, and reassignments within tiers.
- **Skill Utilization:** Ratio of tickets requiring a specific skill to agent proficiency in that skill.
- **SLA Compliance:** Percentage of tickets resolved within SLA timelines, segmented by priority and assignment path.

**Process Mining Techniques:**
- **Resource Interaction Analysis:** Maps how agents interact with tickets over time, revealing handover patterns and idle times.
- **Social Network Analysis (Handover Graphs):** Visualizes frequency and direction of ticket reassignments (e.g., how often L1 hands off to Agent B12).
- **Role Discovery:** Clusters agents based on behavior (e.g., "quick-resolvers," "escalation-prone," "specialist").
- **Conformance Checking:** Compares *actual* assignment paths against *intended* logic (e.g., "L1 should handle all P3 unless skill match fails").

**Skill Utilization Analysis:**
- Cross-reference **Required Skill** (e.g., "Database-SQL") with **Agent Skills** to calculate:
  - **Skill Mismatch Rate:** % of tickets where assigned agent lacks required skill.
  - **Underutilization:** % of specialized agents (e.g., "DB-SQL") handling tasks below their skill level (e.g., basic OS issues).
  - **Overutilization:** % of specialized agents overloaded with tickets matching their skills, leading to delays.

---

### 2. Identifying Resource-Related Bottlenecks and Issues

**Bottleneck Identification via Process Mining:**
- **Skill Bottlenecks:** Identify queues where tickets with specific required skills (e.g., "Networking-Firewall") wait longer due to insufficient qualified agents.
- **Reassignment Loops:** Detect cases with multiple reassignments (e.g., INC-1001 reassigned twice) and calculate added delay per reassignment.
- **Incorrect Initial Assignments:** Measure % of L1 assignments where escalation was immediate (within <30 mins), indicating poor initial routing.
- **Overloaded/Underutilized Agents:** Use workload heatmaps to flag agents with >90% utilization or <40% utilization.

**Quantifying Impact:**
- **Delay per Reassignment:** Average time added per reassignment (e.g., 15–45 mins based on log timestamps).
- **SLA Breach Correlation:** Regression analysis showing % of P2/P3 breaches linked to:
  - Skill mismatch (e.g., 60% of P2 breaches involved tickets routed to agents lacking required skill).
  - Tier escalation delays (e.g., L2 queue wait time >2 hours correlates with 70% of SLA misses).
- **Example Calculation:** If 30% of tickets require "App-CRM" but only 15% of L1 agents have this skill, and 40% of these tickets are reassigned, the reassignment rate directly contributes to SLA breaches.

---

### 3. Root Cause Analysis for Assignment Inefficiencies

**Potential Root Causes:**
- **Deficient Assignment Rules:** Round-robin ignores skills/workload, leading to mismatches.
- **Incomplete Skill Profiles:** Agents may lack updated skill tags (e.g., "Agent A02" listed as "Basic-Troubleshoot" only).
- **Poor Ticket Categorization:** Misclassified categories (e.g., "Network" vs. "Security") lead to wrong skill routing.
- **Lack of Real-Time Visibility:** Dispatcher assigns without live workload data.
- **L1 Under-empowerment:** Insufficient training/tools cause unnecessary escalations.

**Variant & Decision Mining:**
- **Variant Analysis:** Compare "smooth" cases (no reassignment) vs. "chaotic" cases (multiple reassignments). Key differences may include:
  - Ticket description keywords (e.g., "database" triggers SQL skill need).
  - Initial assigned agent’s historical FCR rate.
  - Time-of-day patterns (e.g., night shifts have fewer specialized agents).
- **Decision Mining:** Analyze dispatcher decision points to identify rules used (or ignored) during assignment.

---

### 4. Developing Data-Driven Resource Assignment Strategies

**Strategy 1: Skill-Based Routing with Proficiency Weighting**
- **Issue Addressed:** Skill mismatch and underutilization.
- **How It Works:** Assign tickets to the *most skilled available agent* (not just L1), using a weighted score combining:
  - Skill match (binary: yes/no).
  - Proficiency level (e.g., "Expert" vs. "Basic").
  - Current workload (queue length).
- **Data Required:** Updated agent skill matrix, ticket skill requirements.
- **Expected Benefits:** 20–30% reduction in reassignments; improved FCR for L1.

**Strategy 2: Workload-Aware Dynamic Assignment**
- **Issue Addressed:** Overloaded/underutilized agents.
- **How It Works:** Real-time algorithm distributing tickets based on:
  - Current queue length per agent.
  - Historical handling time per ticket type.
  - Skill availability (penalize assigning to overloaded specialists).
- **Data Required:** Live agent status (available/busy), historical processing times.
- **Expected Benefits:** Balanced workload; 15–25% reduction in queue wait times.

**Strategy 3: Predictive Escalation Criteria**
- **Issue Addressed:** Unnecessary escalations from L1.
- **How It Works:** Use historical data to build a model predicting escalation likelihood based on:
  - Ticket category + priority.
  - L1 agent’s past FCR rate for similar tickets.
  - Time spent in L1 before escalation.
- **Data Required:** Labeled dataset of "resolved at L1" vs. "escalated" cases.
- **Expected Benefits:** Reduce L2/L3 load by 10–20%; faster first-contact resolution.

---

### 5. Simulation, Implementation, and Monitoring

**Simulation with Process Mining Models:**
- Build a **discrete-event simulation** using mined process models:
  - Input: Agent pool, skill profiles, ticket arrival rates.
  - Test strategies (e.g., "What if we route 50% of P3 tickets directly to App-CRM specialists?").
  - Measure outcomes: SLA compliance, average delay, reassignment rate.
- Use tools like **SimPy** or **ProM** to simulate before rollout.

**Monitoring Post-Implementation:**
- **KPIs to Track:**
  - SLA breach rate (by priority).
  - Ticket reassignment frequency.
  - Agent utilization rate (target: 70–85%).
  - First-call resolution rate (L1).
  - Average time in queue per skill type.
- **Process Mining Dashboards:**
  - Real-time "Assignment Flow" network graph.
  - Skill utilization heatmap.
  - SLA compliance trend by tier.
- **Feedback Loop:** Monthly re-run process mining to detect drift and refine rules.

---

**Conclusion:** By leveraging process mining to analyze actual resource behavior, TechSolve can replace heuristic assignments with a dynamic, skill-aware system. This will reduce delays from reassignments, improve SLA compliance, and ensure specialized agents are deployed where their expertise adds maximum value.
