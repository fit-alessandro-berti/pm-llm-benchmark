# POWL Models for GlobalTeq Solutions Hiring Process

Below are two POWL models constructed using `pm4py`, reflecting the described hiring workflow. The first model includes an XOR branch that introduces potential bias through community affiliation checks, while the second model ensures a fair, uniform process.

## Model 1: POWL Model WITH Potential Unfairness

This model introduces an XOR branch after the skill assessment, where one path applies a community-based advantage.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Loop for data completeness: check -> if missing, request more info -> check again
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck, RequestMoreInfo])

# XOR branch: standard cultural fit OR biased community affiliation check
cultural_xor = OperatorPOWL(
    operator=Operator.XOR,
    children=[CulturalFitCheck, CommunityAffiliationCheck]
)

# Build the partial order
root_unfair = StrictPartialOrder(
    nodes=[
        ReceiveApplication,
        data_loop,
        SkillAssessment,
        cultural_xor,
        ManagerialReview,
        FinalDecision
    ]
)

# Add edges (sequential ordering)
root_unfair.order.add_edge(ReceiveApplication, data_loop)
root_unfair.order.add_edge(data_loop, SkillAssessment)
root_unfair.order.add_edge(SkillAssessment, cultural_xor)
root_unfair.order.add_edge(cultural_xor, ManagerialReview)
root_unfair.order.add_edge(ManagerialReview, FinalDecision)
```

### Explanation of Model 1
- `ReceiveApplication`  `data_loop` (ensures the resume has all required information; if not, `RequestMoreInfo` triggers a re-check).
- `data_loop`  `SkillAssessment` (scored automatically; qualifying candidates move on).
- `SkillAssessment`  `cultural_xor` **(BIAS POINT)**: An exclusive choice between a neutral `CulturalFitCheck` and a `CommunityAffiliationCheck` that provides implicit score adjustments for members of a local group.
- `cultural_xor`  `ManagerialReview`  `FinalDecision`.

## Model 2: POWL Model WITHOUT Unfairness

This model removes the XOR branch so that **all** applicants undergo the same cultural fit evaluation. The loop for data completeness and the sequential structure remain.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="ReceiveApplication")
DataCompletenessCheck = Transition(label="DataCompletenessCheck")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Loop for data completeness (same as before)
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[DataCompletenessCheck, RequestMoreInfo])

# Build the partial order — NO XOR branch, uniform cultural check
root_fair = StrictPartialOrder(
    nodes=[
        ReceiveApplication,
        data_loop,
        SkillAssessment,
        CulturalFitCheck,
        ManagerialReview,
        FinalDecision
    ]
)

# Add edges (sequential ordering)
root_fair.order.add_edge(ReceiveApplication, data_loop)
root_fair.order.add_edge(data_loop, SkillAssessment)
root_fair.order.add_edge(SkillAssessment, CulturalFitCheck)
root_fair.order.add_edge(CulturalFitCheck, ManagerialReview)
root_fair.order.add_edge(ManagerialReview, FinalDecision)
```

### Explanation of Model 2
- `ReceiveApplication`  `data_loop`  `SkillAssessment` (identical to Model 1).
- `SkillAssessment`  `CulturalFitCheck` directly — **no XOR choice**, so every applicant is evaluated by the same standard, merit-based assessment.
- `CulturalFitCheck`  `ManagerialReview`  `FinalDecision`.

## Key Differences

| Aspect | Model 1 (Unfair) | Model 2 (Fair) |
|--------|------------------|----------------|
| Cultural fit step | XOR with `CommunityAffiliationCheck` | Single `CulturalFitCheck` |
| Potential for bias | **Yes** (selective uplift for affiliated applicants) | **None** (uniform path) |
| Loop for data completeness | Present | Present |
| Sequential flow | Yes | Yes |

By replacing the `OperatorPOWL` XOR node with a single `Transition` in the fair model, we eliminate the structural mechanism that allows preferential treatment, ensuring procedural equity.
