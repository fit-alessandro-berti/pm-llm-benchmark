**Explanation of the constraints**

| Constraint type | Typical meaning in the process |
|-----------------|---------------------------------|
| **existence** | Each listed activity must appear at least once in every trace. |
| **absence** | The activity *Rejection* (or any undesired step) must never occur. |
| **exactly_one** | *Approval Gate* and *Final Launch* are required to occur exactly once. |
| **init** | Every trace must begin with *Idea Generation*. |
| **responded_existence** | If a source activity occurs, a target activity must also occur somewhere later (e.g., if a design draft is created, eventually a feasibility check must be performed). |
| **coexistence** | Both activities must appear in the trace (order free). |
| **response** | When the source occurs, the target must appear later (e.g., after prototyping a lab test must follow). |
| **precedence** | The target may only occur after the source has occurred. |
| **succession** | When the source occurs, the target must occur immediately after. |
| **altresponse** | At least one of the alternative targets must appear after the source. |
| **altprecedence** | At least one of the alternative sources must precede the target. |
| **altsuccession** | At least one of the alternative targets must immediately follow the source. |
| **chainresponse** | Immediate response – source followed instantly by target. |
| **chainprecedence** | Immediate precedence – target starts right after source finishes. |
| **chainsuccession** | Immediate succession – source followed instantly by target. |
| **noncoexistence** | The two activities cannot both be present in the same trace. |
| **nonsuccession** | The two activities cannot appear in that particular order. |
| **nonchainsuccession** | The two activities cannot be adjacent in the trace. |
