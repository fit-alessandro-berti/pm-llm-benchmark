```sql
  SELECT 
    activity_sequence,
    COUNT(*) AS case_count
  FROM case_sequences

),

  SELECT activity_sequence
  FROM variant_counts
  ORDER BY case_count DESC

FROM event_log e
INNER JOIN case_sequences cs ON e.case_id = cs.case_id
INNER JOIN top_variants tv ON cs.activity_sequence = tv.activity_sequence

```

**Explanation:**

1. **`case_sequences` CTE**: Uses `LIST_AGG` with `ORDER BY timestamp` to create the ordered sequence of activities for each `case_id`

2. **`variant_counts` CTE**: Groups by the activity sequence and counts how many cases follow each variant

3. **`top_variants` CTE**: Orders variants by case count (descending) and limits to top K

4. **Final SELECT**: Joins back to the original `event_log` to return all events from cases whose activity sequence matches one of the top K variants

**To use this query:**
- Replace `?` with your desired K value (e.g., `3` for top 3 variants)
- The result includes all original columns from `event_log`, ordered by `case_id` and `timestamp`
