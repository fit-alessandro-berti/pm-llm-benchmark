# Process Diagram Description: Dispatch of Goods

This is a **BPMN (Business Process Model and Notation)** diagram depicting the "Dispatch of goods" workflow for a Computer Hardware Shop. The process involves three distinct roles organized in swimlanes.

---

## Roles (Swimlanes)

| Lane | Responsibility |
|------|---------------|
| **Logistics** | Insurance coordination |
| **Secretary** | Administrative & documentation tasks |
| **Warehouse** | Physical packaging & handling |

---

## Process Flow

### 1. Start & Parallel Split
The process begins with **"Ship goods"**, which immediately splits into two parallel tracks via a **parallel gateway** — one for administrative work (Secretary) and one for physical preparation (Warehouse).

---

### 2. Secretary Lane (Administrative Track)

After clarifying the shipment method, an **exclusive gateway** asks: **"Special handling?"**

- **If YES:**
  - Get 3 offers from logistic companies  Select logistic company and place order

- **If NO:**
  - Check if insurance is required (using Order details and Customer information as data inputs)
  - Another gateway checks if insurance is necessary:
    - **Insurance needed**  Escalates to the **Logistics** lane to "Insure parcel"
    - **Always continues**  Write package label

Both branches converge via exclusive gateways.

---

### 3. Warehouse Lane (Physical Track)

- **Package goods**
- A **48-hour timer event** begins:
  - If there's a delay  **Notify customer about delay**
- Converges at an exclusive gateway

---

### 4. Final Convergence & Completion

A **parallel gateway** ensures both the Secretary and Warehouse tracks are complete before the final steps:

1. **Fill out shipment paperwork**
2. **Move package to pickup area**
3. **End Event: Shipment prepared**

---

## Key Process Features

| BPMN Element | Location | Purpose |
|---|---|---|
| Parallel Gateway (+) | Start & near-end | Allows admin and warehouse work simultaneously |
| Exclusive Gateway (X) | "Special handling?" | Routes based on handling requirements |
| Timer Event | 48-hour wait | Manages delay notification |
| Data Object | Order details, Customer info | Inputs for insurance check |
| Sub-process | Logistics lane | External insurance coordination |

This design efficiently parallelizes administrative and physical tasks, only synchronizing at the end to ensure everything is ready before the package moves to the pickup area.
