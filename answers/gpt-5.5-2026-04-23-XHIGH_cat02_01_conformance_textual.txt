Under the intended interpretation, the process allows two valid paths:

1. **Fulfillment path:**  
   **Receive order  Check inventory  Authorize payment  Package items  Ship order  Close order**

2. **Cancellation path:**  
   **Receive order  Check inventory  Cancel order  Close order**

Based on that, the violating cases are **Case 2, Case 3, and Case 5**.

| Case ID | Status | Explanation |
|---|---|---|
| **1** | **Conforms** | Follows the normal fulfillment path exactly: receive, check inventory, authorize payment, package, ship, close. |
| **2** | **Violates** | The case performs **“Package items” before “Authorize payment.”** The description states that payment should be authorized before packaging, and that packaging should only occur after authorization. The observed sequence is: Receive  Check  Package  Authorize  Ship  Close. |
| **3** | **Violates** | Several issues occur. After “Authorize payment,” the case performs **“Cancel order,”** which does not match either expected branch. If inventory was insufficient, cancellation should have occurred directly after the inventory check, without payment authorization. If inventory was sufficient, the case should have continued to packaging and shipping. More seriously, the case then performs **“Ship order” after “Cancel order,”** which is explicitly forbidden by the description. It also ships without a preceding “Package items” step. |
| **4** | **Conforms** | Follows the cancellation path correctly: receive, check inventory, cancel order, close order. |
| **5** | **Violates** | The case skips **“Package items”** and goes directly from **“Authorize payment”** to **“Ship order.”** The normative process requires items to be packaged before shipping. |

Therefore, the nonconforming cases are:

**Case 2, Case 3, and Case 5.**
