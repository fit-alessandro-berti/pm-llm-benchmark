### Analysis of POWL Models

#### Standard Hire-to-Retire Process
The normative sequence of activities is:
1. **Post_Job_Ad**  2. **Screen_Candidates**  3. **Conduct_Interviews**  4. **Make_Hiring_Decision**  5. **Onboard_Employee**  6. **Add_to_Payroll**  7. **Close_Case**

All steps are mandatory and must occur in this order.

---

### Model 1 Anomalies
- **Missing precedence**: There is no edge from `Conduct_Interviews` to `Make_Hiring_Decision`. Instead, both activities are direct successors of `Screen_Candidates`. This allows two possible execution orders:
  - Correct: `Screen  Interview  Decide`
  - Incorrect: `Screen  Decide  Interview` (decision before interviews)
- **Severity**: Moderate. The process still requires all activities, and the incorrect order can be avoided if the execution respects the intended logic, but the model does not enforce the necessary constraint.

### Model 2 Anomalies
- **Decoupled screening**: `Screen_Candidates` has only an incoming edge from `Post_Job_Ad` and no outgoing edges. It can be executed at any time after posting, even after `Close_Case`, making it independent of the main process flow.
- **Interview before screening**: `Conduct_Interviews` is directly after `Post_Job_Ad` (no dependency on `Screen_Candidates`), allowing interviews to occur before screening.
- **Optional payroll**: The XOR operator makes `Add_to_Payroll` optional (can be skipped via a silent transition). This is a critical omission.
- **Repetitive onboarding**: The LOOP operator allows `Onboard_Employee` to be repeated multiple times, which is not typical for a single hiring case.
- **Severity**: High. The model fundamentally alters the process by removing mandatory steps, decoupling key activities, and introducing unnecessary repetition.

---

### Which Model Is More Aligned with the Normative Process?
**Model 1** is closer to the normative process.  
- It preserves all mandatory activities in a linear (partial) order, with only one missing constraint (interview  decision).  
- The anomaly is a single precedence error that can still produce correct behavior if the execution chooses the right order.  
- Model 2, by contrast, introduces multiple structural changes that violate essential process logic: skipping payroll, isolating screening, and allowing unbounded repetition. These deviations are more severe and make the process functionally incorrect.

**Conclusion**: Model 1 aligns more closely with the standard Hire-to-Retire workflow.
