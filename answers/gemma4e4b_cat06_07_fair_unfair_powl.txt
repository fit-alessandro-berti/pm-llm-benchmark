## POWL Models for GlobalTeq Solutions Hiring Process

We will use the provided POWL framework structure to model the two scenarios.

### 1. POWL Model with Potential Unfairness (Biased Path)

In this model, the XOR choice at the Cultural Fit stage creates an explicit alternative path (`CommunityAffiliationCheck`) which, according to the description, provides a preferential score uplift, thereby introducing potential systemic bias.

**Model Logic:**
1. Data validation loop ensures completeness.
2. Skills are tested.
3. *Choice:* The applicant either takes the standard `CulturalFitCheck` **OR** the biased `CommunityAffiliationCheck`.
4. Managerial review follows the selection path.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# --- 1. Define Core Activities ---
# Initialization
receive = Transition(label="ReceiveApplication")
resume_parse = Transition(label="ResumeParsing")
check_data = Transition(label="CheckDataCompleteness")
request_info = Transition(label="RequestMoreInfo")

# Skill Assessment
skill_assess = Transition(label="SkillAssessment")

# --- 2. Define Biased Cultural Fit Choices (The XOR Branch) ---
# Path A: Standard, Merit-Based Check
standard_fit = Transition(label="StandardCulturalFitCheck")

# Path B: Affiliation Check (The source of bias)
community_affil = Transition(label="CommunityAffiliationCheck") 

# The XOR operator represents the choice between two paths after skill assessment
cultural_fit_choice = OperatorPOWL(operator=Operator.XOR, children=[standard_fit, community_affil])

# --- 3. Final Stages ---
manager_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# --- 4. Build the Loop (Data Completeness Check) ---
# The loop: If parsing fails, check data, request info, and then loop back to resume parsing.
loop_resume = OperatorPOWL(operator=Operator.LOOP, children=[resume_parse, check_data, request_info])

# --- 5. Build the Main Workflow (StrictPartialOrder) ---
biased_hiring_process = StrictPartialOrder(nodes=[
    receive, 
    loop_resume, 
    skill_assess, 
    cultural_fit_choice, 
    manager_review, 
    final_decision
])

# --- 6. Define the Partial Order (Edges) ---
# 1. Start to Data Validation
biased_hiring_process.order.add_edge(receive, loop_resume)

# 2. Data Validation Loop (Handles self-contained loop structure)
# The loop feeds back into itself
# This is tricky in a straight SPO definition, but conceptually, the loop handles the internal cycle.
# We link the output of the loop (resume_parse) to the next main stage.
biased_hiring_process.order.add_edge(loop_resume, skill_assess) 

# 3. Skill Check to Cultural Fit Choice
biased_hiring_process.order.add_edge(skill_assess, cultural_fit_choice)

# 4. Cultural Fit Choice must precede Manager Review
# Since the XOR node represents a required choice before moving on, we link it.
biased_hiring_process.order.add_edge(cultural_fit_choice, manager_review)

# 5. Manager Review must precede Final Decision
biased_hiring_process.order.add_edge(manager_review, final_decision)

print("--- POWL Model 1: Biased Cultural Fit Process ---")
# In a real environment, this object would be saved/run. We print the structure overview.
print(f"Nodes: {biased_hiring_process.nodes}")
print(f"Edges: {biased_hiring_process.order.edges}")
```

### 2. POWL Model Without Unfairness (Standardized Path)

In this model, the XOR choice is eliminated. All applicants, regardless of their community affiliations, must proceed through the single, objective `StandardCulturalFitCheck`. This removes the optional, biased branching point.

**Model Logic:**
1. Data validation loop ensures completeness (same as M1).
2. Skills are tested (same as M1).
3. *Fixed Path:* All applicants must undergo the standard `CulturalFitCheck`.
4. Managerial review follows the single standardized path.

```python
# Re-using previously defined components for efficiency
# (In a runnable script, we would define these again or pass them)
# receive, loop_resume, skill_assess, manager_review, final_decision

# --- 1. Define Fixed Cultural Fit Check ---
standard_fit = Transition(label="StandardCulturalFitCheck") 
# Note: We explicitly exclude the 'CommunityAffiliationCheck'

# --- 2. Build the Main Workflow (StrictPartialOrder) ---
fair_hiring_process = StrictPartialOrder(nodes=[
    receive, 
    loop_resume, 
    skill_assess, 
    standard_fit,  # Mandatory, standardized single step
    manager_review, 
    final_decision
])

# --- 3. Define the Partial Order (Edges) ---
# 1. Start to Data Validation
fair_hiring_process.order.add_edge(receive, loop_resume)

# 2. Data Validation Loop (Handles self-contained loop structure)
fair_hiring_process.order.add_edge(loop_resume, skill_assess) 

# 3. Skill Check to Standard Cultural Fit (Direct sequential link)
fair_hiring_process.order.add_edge(skill_assess, standard_fit)

# 4. Standard Cultural Fit must precede Manager Review
fair_hiring_process.order.add_edge(standard_fit, manager_review)

# 5. Manager Review must precede Final Decision
fair_hiring_process.order.add_edge(manager_review, final_decision)

print("\n\n--- POWL Model 2: Standardized, Unbiased Cultural Fit Process ---")
print(f"Nodes: {fair_hiring_process.nodes}")
print(f"Edges: {fair_hiring_process.order.edges}")
```

### Summary of Bias Mitigation

| Feature | Model 1 (Biased) | Model 2 (Fair) | Bias Mitigation |
| :--- | :--- | :--- | :--- |
| **Cultural Fit Structure** | `XOR` choice between `StandardCulturalFitCheck` and `CommunityAffiliationCheck`. | Direct sequential transition (StandardCulturalFitCheck only). | **Eliminating Choice:** By replacing the optional, affinity-based `XOR` branching with a single, mandatory sequential step, the system forces all candidates to be judged by the same objective measure. |
| **Process Flow** | Allows applicants to skip objective review if an affiliation exists. | Forces all applicants through the same assessment regardless of background. | **Transparency:** The process becomes deterministic at the critical decision point, making the criteria for passing uniform and measurable. |
