# BPMNStyle Textual Process Model: Invoice Approval Workflow

## 1. Participants (Pools & Lanes)

| Pool | Lane / Role | Notes |
|------|-------------|-------|
| **Organization** | **Accounts Payable (AP) Clerk** | Receives invoice, checks PO, logs invoice, matches line items, schedules payment. |
| **Organization** | **Purchasing Officer** | Confirms goods/services received by comparing invoice to PO. |
| **Organization** | **Department Manager** | Final approver after PO match and purchasing confirmation. |
| **Organization** | **AP Supervisor** | Handles exceptions, escalations, and invoices without a PO. |
| **Organization** | **Supplier Relationship Manager (SRM)** | Assists AP/Purchasing in supplierside issue resolution. |
| **Supplier** *(external)* | **Supplier** | Sends invoice and, when asked, provides corrected invoice or PO reference. |

---

## 2. HighLevel Flow

```
[Start] Invoice received by AP Clerk
    
    
Check for valid PO number
    
     No / missing PO  Request PO details from Supplier
                             (loop back to Check PO; if still unresolved,
                              escalate to AP Supervisor for retrospective PO or reject)
    
     Yes  Log invoice into accounting system
                  
                  
          Forward to Purchasing for confirmation
                  
                  
          Compare invoice details to PO
                  
    
                                 
 Discrepancy found             Everything correct
                                 
                                 
Resolve discrepancy        Send confirmation to AP
(loop back)                      
                                 
    
                  
                  
          Match invoice line items to PO
                  
    
                                 
 Mismatch                     Match OK
                                 
                                 
Clarify with Purchasing    Send approval request to Manager
(escalate to Supervisor if unresolved)
                  
                  
          Manager reviews & decides
                  
    
                              
 Approve      Reject       Hold / Investigate
                              
                              
Schedule     End:         AP Clerk investigates
payment      Rejected      (loop back to matching)
    
    
[End] Payment scheduled
```

## 3. Detailed BPMN Element List

| ID | Element | Type | Lane | Description / Inputs / Outputs |
|----|---------|------|------|--------------------------------|
| **E1** | **Invoice received** | Start Event (Message) | AP Clerk | Triggered by supplier email containing invoice. |
| **T1** | **Check invoice for valid PO** | Task | AP Clerk | Inputs: invoice. Output: PO validation result. |
| **G1** | **PO present and valid?** | XOR Gateway | AP Clerk | Decision: does the invoice reference an existing PO? |
| **T2** | **Request missing PO details from supplier** | Task | AP Clerk | Send email asking supplier for PO reference or revised invoice. |
| **T3** | **Log invoice in accounting system** | Task | AP Clerk | Inputs: invoice + valid PO. Output: accounting system entry. |
| **T4** | **Forward invoice to Purchasing** | Task | AP Clerk | Send invoice/PO package to Purchasing Officer for receipt confirmation. |
| **T5** | **Compare invoice to PO** | Task | Purchasing Officer | Check quantities, prices, delivery charges against PO. |
| **G2** | **Invoice details correct?** | XOR Gateway | Purchasing Officer | Decision: any discrepancy between invoice and PO? |
| **T6** | **Resolve invoice discrepancy** | Task (subprocess) | Purchasing Officer / AP Clerk / SRM | Contact supplier directly, ask AP to hold invoice, or involve SRM to verify contract terms. |
| **T7** | **Send confirmation to AP** | Task | Purchasing Officer | Output: confirmation email that goods/services were received and invoice is OK. |
| **T8** | **Match invoice line items to PO** | Task | AP Clerk | Systematic matching of invoice lines to PO lines. |
| **G3** | **Line items match?** | XOR Gateway | AP Clerk | Decision: successful match? |
| **T9** | **Clarify discrepancy with Purchasing** | Task | AP Clerk | Email Purchasing for clarification; may escalate if unresolved. |
| **T10** | **Escalate to AP Supervisor** | Task | AP Clerk / AP Supervisor | Supervisor handles complex mismatches or missing PO exceptions. |
| **T11** | **Send approval request to Manager** | Task (system autoemail) | AP Clerk / System | Triggered only after confirmation and successful PO match. |
| **T12** | **Review invoice, PO and confirmation** | Task | Department Manager | Manager examines invoice details and confirmation. |
| **G4** | **Manager decision** | XOR Gateway | Department Manager | Approve / Reject / Hold. |
| **T13** | **Approve invoice** | Task | Department Manager | System approval recorded. |
| **T14** | **Reject invoice** | Task | Department Manager | Invoice rejected; no payment. |
| **T15** | **Hold invoice and request investigation** | Task | Department Manager | Sends invoice back to AP Clerk for further review. |
| **T16** | **Schedule payment** | Task | AP Clerk | Add invoice to next payment cycle. |
| **E2** | **Payment scheduled** | End Event | AP Clerk | Process completes for approved invoices. |
| **E3** | **Invoice rejected** | End Event | Department Manager | Process ends without payment. |

## 4. Sequence Flows

1. **E1**  **T1**
2. **T1**  **G1**
3. **G1** (No / missing PO)  **T2**  *(message to Supplier)*  **T1** *(recheck after supplier response)*
4. **G1** (Yes)  **T3**
5. **T3**  **T4**
6. **T4**  **T5**
7. **T5**  **G2**
8. **G2** (Discrepancy)  **T6**  *(revised invoice received or issue resolved)*  **T5**
9. **G2** (Correct)  **T7**
10. **T7**  **T8**
11. **T8**  **G3**
12. **G3** (Mismatch)  **T9**
    - If resolved  **T8**
    - If unresolved  **T10**  *(supervisor resolves)*  **T8**
13. **G3** (Match OK)  **T11**
14. **T11**  **T12**
15. **T12**  **G4**
16. **G4** (Approve)  **T13**  **T16**  **E2**
17. **G4** (Reject)  **T14**  **E3**
18. **G4** (Hold / Investigate)  **T15**  **T8** *(or, for complex cases, via Supervisor/SRM)*  loop back to Manager approval.

| Supplier | AP Clerk | Revised invoice or PO reference |

## 6. Key Decision Rules

| Gateway | Condition | Next Step |
|---------|-----------|-----------|
| **G1** | Invoice has a valid PO number | Log invoice in system |
| **G1** | PO missing or invalid | Request details from supplier; if unresolved, escalate to Supervisor |
| **G2** | Invoice quantities/prices/charges match PO | Send confirmation to AP |
| **G2** | Discrepancy found | Resolve with supplier/AP/SRM; loop back to comparison |
| **G3** | Lineitem match successful | Request Manager approval |
| **G3** | Lineitem mismatch | Clarify with Purchasing; escalate to Supervisor if unresolved |
| **G4** | Manager approves | Schedule payment |
| **G4** | Manager rejects | End — invoice not paid |
| **G4** | Manager holds / requests investigation | Return to AP for investigation and rematching |

## 7. Exceptions, Variations & Ambiguities

- **Missing PO handling:**  
  Mary describes sending the invoice back to the supplier first. Rita notes that some invoices (e.g., informally contracted services) never have a PO and may require a **retrospective PO** or rejection. The model therefore shows the primary path as *request supplier for PO*, with an escalation path to the **AP Supervisor** when a PO cannot be obtained.

- **Discrepancy resolution ownership:**  
  Dan states Purchasing may contact the supplier directly or ask AP to hold the invoice. Mary states AP emails Purchasing for clarification and escalates to the Supervisor if unresolved. Karen may be pulled in for contract/relationship issues. The model consolidates this as a shared **“Resolve discrepancy”** task involving Purchasing, AP, SRM, and sometimes the Supervisor.

- **Manager approval trigger:**  
  Triggered automatically by the system once Purchasing has confirmed receipt and AP has matched the invoice to the PO.

- **Manager rejection vs. hold:**  
  Rejection is a terminal state; a hold returns the invoice to AP for investigation and rematching.

- **Payment scheduling:**  
  Occurs only after Manager approval. The exact timing of the payment run is not specified in the interviews.
