### 1. Analyzing Resource Behavior and Assignment Patterns

**Key Metrics & Process Mining Techniques**

Using the event log data, we will examine resource performance and assignment patterns through several lenses:

* **Workload Distribution:**  
  Count of tickets handled per agent/tier, active working time (sum of durations between START and COMPLETE events), idle/waiting time, and queue lengths before assignment events. This reveals imbalances and underutilization.

* **Processing Times:**  
  Mean and median handling time per activity (e.g., “Work L1”, “Work L2”) broken down by agent, tier, ticket category, and required skill. This highlights efficiency variations and potential training gaps.

* **First-Call Resolution (FCR) Rate for L1:**  
  Percentage of tickets resolved entirely at L1 without escalation. Calculated by checking whether a case has an “Escalate” event after L1 work.

* **Escalation & Reassignment Frequency:**  
  Number of escalations per ticket, reassignments within the same tier, and handovers between agents. Frequent handovers indicate assignment inefficiencies.

* **Skill Utilization:**  
  For each agent, compute the proportion of tickets they worked on where the required skill matches one of their documented skills. Also measure “overqualification” – instances where an agent possesses advanced skills but works on lowcomplexity tickets.

**Process Mining Techniques Applied**

* **Resource Profiles & Activity Matrix:**  
  Show which agents perform which activities, how often, and with what performance. Compare with intended tier responsibilities (e.g., L1 should handle basic troubleshooting, not specialized tasks).

* **Social Network Analysis (Handover Networks):**  
  Build a graph where nodes are agents and edges represent ticket handovers (e.g., from Agent A05 to Agent B12). Edge weight indicates frequency. This uncovers the actual escalation and reassignment flows, highlighting bottlenecks, unexpected handovers, or agents who act as hubs for reassignments.

* **Role Discovery:**  
  Cluster agents based on their executed activities, handled ticket types, and performance patterns. Compare discovered roles with the official tier classification to detect mismatches (e.g., some L2 agents routinely doing L1 work).

* **Performance Analysis with Time Perspectives:**  
  Use Gantt charts, process maps with timing information, and the “performance spectrum” to visualize where delays occur (e.g., long waiting times before “Work L2 Start”). Correlate these delays with resource availability and skill shortages.
