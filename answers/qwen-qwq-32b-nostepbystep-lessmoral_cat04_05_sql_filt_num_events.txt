To address the task, the solution involves filtering cases based on their event count and returning all events from eligible cases. Here's the SQL query:

```sql
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
);
```

### Explanation:
1. **Subquery (Inner SELECT):**
   - **`SELECT case_id FROM event_log GROUP BY case_id`:** Groups all events by `case_id` to calculate the count of events per case.
   - **`HAVING COUNT(*) <= 6`:** Filters groups to retain only `case_id` values where the total number of events (via `COUNT(*)`) is 6. This excludes cases with >6 events.

2. **Main Query:**  
   - **`SELECT * FROM event_log WHERE case_id IN (...)`:** Retrieves all events from the original table where the `case_id` is in the list of valid cases identified by the subquery.

### Key Considerations:
- **Efficiency:** The subquery first identifies valid `case_id`s, minimizing the dataset processed in the main query. This approach avoids processing large groups of invalid cases.
- **Edge Cases:** 
  - Cases with 0 events are impossible (since `case_id` must exist in the table to be grouped).
  - Cases with exactly 6 events are included, while those exceeding 6 are excluded.

This solution ensures only events from eligible cases are returned, adhering to the requirement.
