# Dispatch of Goods – BPMN Process Diagram

This is a **Business Process Model and Notation (BPMN)** diagram depicting the **"Dispatch of Goods"** process for a **Computer Hardware Shop**. The process is organized into three horizontal **swim lanes** representing different departments/roles:

---

## Swim Lanes

### 1. **Logistics** (Top Lane)
- Contains a single task: **"Insure parcel"** – triggered conditionally when insurance is determined to be necessary.

### 2. **Secretary** (Middle Lane)
The Secretary handles two parallel tracks after the process starts:

**Track A – Insurance & Labeling:**
- Receives **Order details** (document) and accesses **Customer information** (database)
- **"Check if insurance is required"** – evaluates whether insurance is needed
- If insurance is necessary, it triggers "Insure parcel" in the Logistics lane
- **"Write package label"** – always performed regardless of insurance decision
- Both paths merge before continuing

**Track B – Shipment Method:**
- **"Clarify shipment method"** – determines how goods will be shipped
- **"Special handling?"** (exclusive gateway):
  - **Yes**  **"Get 3 offers from logistic companies"**  **"Select logistic company and place order"**
  - **No**  proceeds directly forward
- Paths converge at a merging exclusive gateway

### 3. **Warehouse** (Bottom Lane)
- **"Package goods"** – runs in parallel with Secretary activities
- A **timer event (48 hours)** is attached: if packaging exceeds 48 hours, **"Notify customer about delay"** is triggered
- After packaging completes, the flow waits at an **event-based gateway** for synchronization
- A **parallel join gateway** merges the Warehouse and Secretary flows
- **"Fill out shipment paperwork"**  **"Move package to pickup area"**
- Process ends with **"Shipment prepared"** (end event)

---

## Key BPMN Elements Used
| Symbol | Meaning |
|--------|---------|
| **Circle (thin)** | Start event ("Ship goods") |
| **Circle (bold)** | End event ("Shipment prepared") |
| **+ Diamond** | Parallel gateway (fork/join) |
| **X Diamond** | Exclusive gateway (decision) |
| ** Diamond** | Event-based/Inclusive gateway |
| **Clock circle** | Timer boundary event (48 hours) |
| **Database icon** | Data store (Customer information) |
| **Document icon** | Data object (Order details) |

---

## Process Flow Summary
The process begins with **"Ship goods"** and immediately splits into **three parallel branches**: clarifying shipment method, checking insurance/writing labels, and packaging goods. All branches must complete before shipment paperwork is filled out and the package is moved to the pickup area, concluding with **"Shipment prepared."**
