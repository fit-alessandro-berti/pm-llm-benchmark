Premature Closure (A  C)** | A direct partial order edge from *Assign Adjuster (A)* to *Close Claim (C)* **bypasses E, P, and N**. | Allows claims to be closed immediately after assignment, without any evaluation, approval, or notification. | `root.order.add_edge(A, C)` is explicitly added, overriding the intended sequence. |

```sql
-- Example: Do skipped notifications (Query 3) correlate with low-value claims?
WITH skipped_notifications AS (
    SELECT DISTINCT c.claim_id, c.claim_amount, c.claim_type
    FROM claims c
    JOIN (
        SELECT claim_id, MIN(timestamp) AS first_p_time
        FROM claim_events
        WHERE activity = 'P'
        GROUP BY claim_id
    ) ce_p ON c.claim_id = ce_p.claim_id
    JOIN (
        SELECT claim_id, MIN(timestamp) AS first_c_time
        FROM claim_events
        WHERE activity = 'C'
        GROUP BY claim_id
    ) ce_c ON c.claim_id = ce_c.claim_id
    WHERE ce_p.first_p_time < ce_c.first_c_time
      AND NOT EXISTS (
          SELECT 1
          FROM claim_events ce_n
          WHERE ce_n.claim_id = c.claim_id
            AND ce_n.activity = 'N'
            AND ce_n.timestamp > ce_p.first_p_time
            AND ce_n.timestamp < ce_c.first_c_time
      )
)
SELECT
    claim_type,
    AVG(claim_amount) AS avg_claim_amount,
    COUNT(*) AS skipped_count
FROM skipped_notifications
GROUP BY claim_type
ORDER BY avg_claim_amount;
```

**Expected Insight:**
- If **`avg_claim_amount` is low** for skipped notifications, strongly supports **H2.1 (policy exception for small claims)**.
