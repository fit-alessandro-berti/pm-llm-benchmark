The following DuckDB SQL query identifies cases that contain the exact three-event sequence `'Approve Purchase Requisition'  'Create Order'  'Confirm Order'` in direct succession, and where the time between the `'Create Order'` and `'Confirm Order'` in that sequence is more than 5 days. All events from those cases are then excluded, returning only events from the remaining cases.

```sql
WITH flagged_cases AS (
    SELECT DISTINCT case_id
    FROM (
        SELECT
            case_id,
            activity,
            timestamp,
            LAG(activity) OVER (PARTITION BY case_id ORDER BY timestamp) AS prev_activity,
            LEAD(activity) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_activity,
            LEAD(timestamp) OVER (PARTITION BY case_id ORDER BY timestamp) AS next_timestamp
        FROM event_log
    ) t
    WHERE activity = 'Create Order'
      AND prev_activity = 'Approve Purchase Requisition'
      AND next_activity = 'Confirm Order'
      AND (next_timestamp - timestamp) > INTERVAL '5 days'
)
SELECT *
FROM event_log
WHERE case_id NOT IN (SELECT case_id FROM flagged_cases)
ORDER BY case_id, timestamp;  -- optional ordering for readability
```

### Explanation

1. **Window functions** `LAG` and `LEAD` look at the previous and next event for each row (within the same `case_id`, ordered by `timestamp`).
2. The filter picks only `'Create Order'` events whose immediate predecessor is `'Approve Purchase Requisition'` and immediate successor is `'Confirm Order'`, forming the required direct succession.
3. For these events, the difference between the `next_timestamp` (the `'Confirm Order'` timestamp) and the current `timestamp` (`'Create Order'`) is compared to `INTERVAL '5 days'`. Only sequences where this interval exceeds 5 days are kept.
4. `DISTINCT case_id` collects all cases that contain at least one such sequence.
5. Finally, all events from the original `event_log` whose `case_id` is *not* in the flagged list are returned.

This ensures that cases matching both the exact sequence and the timing condition are fully excluded, as required.
