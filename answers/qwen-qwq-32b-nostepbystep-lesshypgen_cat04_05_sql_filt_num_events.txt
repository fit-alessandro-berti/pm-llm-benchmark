Here is the DuckDB SQL query that fulfills the requirements:

```sql
SELECT *
FROM event_log
WHERE case_id IN (
    SELECT case_id
    FROM event_log
    GROUP BY case_id
    HAVING COUNT(*) <= 6
);
```

**Explanation:**
1. The subquery `(SELECT case_id FROM event_log GROUP BY case_id HAVING COUNT(*) <=6)`:
   - Groups the table by `case_id` to calculate the number of events per case.
   - Uses `HAVING COUNT(*) <=6` to retain only those `case_id` groups that have 6 or fewer events.

2. The outer query uses `WHERE case_id IN (...)` to return all original event rows where the `case_id` appears in the filtered list of acceptable case IDs.

This approach ensures we only return events from cases that meet the "6 events" criterion while preserving all columns (including `activity` and `timestamp`) for those qualifying cases.
