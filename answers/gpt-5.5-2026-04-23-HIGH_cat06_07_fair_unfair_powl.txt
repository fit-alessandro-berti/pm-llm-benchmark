Below are two POWL models in the style of the provided `pm4py` example.

## 1. POWL Model with Potential Unfairness

This model includes an XOR branch after the skill assessment. One path uses the standard `CulturalFitCheck`, while the other uses `CommunityAffiliationCheck`, which represents the potentially unfair path where applicants with certain local/community affiliations may receive a subtle advantage.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# Activities
receive_application = Transition(label="ReceiveApplication")
data_completeness_check = Transition(label="DataCompletenessCheck")
request_more_info = Transition(label="RequestMoreInfo")
skill_assessment = Transition(label="SkillAssessment")

cultural_fit_check = Transition(label="CulturalFitCheck")
community_affiliation_check = Transition(label="CommunityAffiliationCheck")

managerial_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# Loop for missing or incomplete data:
# Execute DataCompletenessCheck, then either exit the loop,
# or RequestMoreInfo and repeat DataCompletenessCheck.
data_completion_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[
        data_completeness_check,
        request_more_info
    ]
)

# Potentially unfair XOR:
# After SkillAssessment, applicants may go through either the standard
# CulturalFitCheck or the CommunityAffiliationCheck path.
# The CommunityAffiliationCheck path represents the possible unfair uplift.
biased_cultural_branch = OperatorPOWL(
    operator=Operator.XOR,
    children=[
        cultural_fit_check,
        community_affiliation_check
    ]
)

# Main process as a partial order
powl_with_unfairness = StrictPartialOrder(
    nodes=[
        receive_application,
        data_completion_loop,
        skill_assessment,
        biased_cultural_branch,
        managerial_review,
        final_decision
    ]
)

# Sequential ordering of the main process
powl_with_unfairness.order.add_edge(receive_application, data_completion_loop)
powl_with_unfairness.order.add_edge(data_completion_loop, skill_assessment)
powl_with_unfairness.order.add_edge(skill_assessment, biased_cultural_branch)
powl_with_unfairness.order.add_edge(biased_cultural_branch, managerial_review)
powl_with_unfairness.order.add_edge(managerial_review, final_decision)
```

Conceptually:

```text
PO=(
  nodes={
    ReceiveApplication,
    *(DataCompletenessCheck, RequestMoreInfo),
    SkillAssessment,
    X(CulturalFitCheck, CommunityAffiliationCheck),
    ManagerialReview,
    FinalDecision
  },
  order={
    ReceiveApplication --> *(DataCompletenessCheck, RequestMoreInfo),
    *(DataCompletenessCheck, RequestMoreInfo) --> SkillAssessment,
    SkillAssessment --> X(CulturalFitCheck, CommunityAffiliationCheck),
    X(CulturalFitCheck, CommunityAffiliationCheck) --> ManagerialReview,
    ManagerialReview --> FinalDecision
  }
)
```

The potential unfairness appears here:

```text
X(CulturalFitCheck, CommunityAffiliationCheck)
```

because the `CommunityAffiliationCheck` branch may introduce preferential treatment for applicants with local/community affiliations.

## 2. POWL Model Without Unfairness

This model removes the community-affiliation-specific branch. Every applicant follows the same cultural fit evaluation after the skill assessment.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# Activities
receive_application = Transition(label="ReceiveApplication")
data_completeness_check = Transition(label="DataCompletenessCheck")
request_more_info = Transition(label="RequestMoreInfo")
skill_assessment = Transition(label="SkillAssessment")

cultural_fit_check = Transition(label="CulturalFitCheck")

managerial_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# Loop for missing or incomplete data:
# Execute DataCompletenessCheck, then either exit the loop,
# or RequestMoreInfo and repeat DataCompletenessCheck.
data_completion_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[
        data_completeness_check,
        request_more_info
    ]
)

# Main process as a partial order
powl_without_unfairness = StrictPartialOrder(
    nodes=[
        receive_application,
        data_completion_loop,
        skill_assessment,
        cultural_fit_check,
        managerial_review,
        final_decision
    ]
)

# Sequential ordering of the main process
powl_without_unfairness.order.add_edge(receive_application, data_completion_loop)
powl_without_unfairness.order.add_edge(data_completion_loop, skill_assessment)
powl_without_unfairness.order.add_edge(skill_assessment, cultural_fit_check)
powl_without_unfairness.order.add_edge(cultural_fit_check, managerial_review)
powl_without_unfairness.order.add_edge(managerial_review, final_decision)
```

Conceptually:

```text
PO=(
  nodes={
    ReceiveApplication,
    *(DataCompletenessCheck, RequestMoreInfo),
    SkillAssessment,
    CulturalFitCheck,
    ManagerialReview,
    FinalDecision
  },
  order={
    ReceiveApplication --> *(DataCompletenessCheck, RequestMoreInfo),
    *(DataCompletenessCheck, RequestMoreInfo) --> SkillAssessment,
    SkillAssessment --> CulturalFitCheck,
    CulturalFitCheck --> ManagerialReview,
    ManagerialReview --> FinalDecision
  }
)
```

In this fairer version, all candidates follow:

```text
SkillAssessment --> CulturalFitCheck --> ManagerialReview --> FinalDecision
```

There is no separate `CommunityAffiliationCheck` branch, so the model removes the potential source of preferential treatment based on local or community affiliation.
