### 1. POWL Model with Potential Unfairness
This model includes an exclusive choice (`XOR`) after the skill assessment. Applicants are routed either to a standard cultural fit evaluation or to a `CommunityAffiliationCheck` that implicitly grants a scoring advantage, representing the bias point.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition
from pm4py.objects.process_tree.obj import Operator

# --- Activity Definitions ---
receive_app = Transition(label="ReceiveApplication")
data_check = Transition(label="DataCompletenessCheck")
request_info = Transition(label="RequestMoreInfo")
skill_assess = Transition(label="SkillAssessment")
cultural_fit = Transition(label="CulturalFitCheck")
community_check = Transition(label="CommunityAffiliationCheck")
manager_review = Transition(label="ManagerialReview")
final_decision = Transition(label="FinalDecision")

# --- Control Flow Structures ---
# Loop: Check data -> if incomplete, request info -> loop back to check
data_loop = OperatorPOWL(operator=Operator.LOOP, children=[data_check, request_info])

# XOR Branch (Bias Point): Standard cultural fit OR community affiliation path
# The community path provides implicit subjective uplifts as described
cultural_xor = OperatorPOWL(operator=Operator.XOR, children=[cultural_fit, community_check])

# --- Partial Order (Sequential Execution) ---
unfair_model = StrictPartialOrder(nodes=[
    receive_app, data_loop, skill_assess, cultural_xor, manager_review, final_decision
])

unfair_model.order.add_edge(receive_app, data_loop)
unfair_model.order.add_edge(data_loop, skill_assess)
unfair_model.order.add_edge(skill_assess, cultural_xor)
unfair_model.order.add_edge(cultural_xor, manager_review)
unfair_model.order.add_edge(manager_review, final_decision)
```
