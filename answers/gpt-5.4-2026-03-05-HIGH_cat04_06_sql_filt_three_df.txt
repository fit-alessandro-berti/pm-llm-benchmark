```sql
WITH sequenced AS (
    SELECT
        case_id,
        activity,
        "timestamp",
        LEAD(activity, 1)  OVER (PARTITION BY case_id ORDER BY "timestamp") AS act_2,
        LEAD(activity, 2)  OVER (PARTITION BY case_id ORDER BY "timestamp") AS act_3,
        LEAD("timestamp", 1) OVER (PARTITION BY case_id ORDER BY "timestamp") AS ts_2,
        LEAD("timestamp", 2) OVER (PARTITION BY case_id ORDER BY "timestamp") AS ts_3
    FROM event_log
),
cases_to_exclude AS (
    SELECT DISTINCT case_id
    FROM sequenced
    WHERE activity = 'Approve Purchase Requisition'
      AND act_2   = 'Create Order'
      AND act_3   = 'Confirm Order'
      AND (ts_3 - ts_2) > INTERVAL '5 days'
)
SELECT e.*
FROM event_log e
WHERE NOT EXISTS (
    SELECT 1
    FROM cases_to_exclude x
    WHERE x.case_id = e.case_id
)
ORDER BY e.case_id, e."timestamp";
```

How it works:
- `LEAD(...)` looks at the next 2 events within each `case_id`, ordered by timestamp.
- The `cases_to_exclude` CTE finds cases where three consecutive events are exactly:
  1. `Approve Purchase Requisition`
  2. `Create Order`
  3. `Confirm Order`
- It then keeps only those cases where `Confirm Order - Create Order > 5 days`.
- The final query returns all events except those belonging to those matching cases.

Note:
- If multiple events in the same case can share the exact same timestamp, add a tie-breaker column to the `ORDER BY` inside the window functions, e.g. `ORDER BY "timestamp", event_id`, so the event order is deterministic.
