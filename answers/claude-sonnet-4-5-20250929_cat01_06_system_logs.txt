## Explanation of Case Identification and Activity Naming Logic

### Case Identification Logic:

I applied a **document-centric case identification strategy**, where each unique document or resource the user interacts with represents a separate case. This approach is based on the following reasoning:

1. **Natural Work Units**: Each document, email, PDF, or spreadsheet represents a distinct work artifact with its own lifecycle (open  work  save  close).

2. **Resource-Based Grouping**: Events are grouped by the combination of application and specific window/document name:
   - **CASE-001**: All activities related to `Document1.docx`
   - **CASE-002**: All activities related to the Annual Meeting email in Gmail
   - **CASE-003**: All activities related to `Report_Draft.pdf`
   - **CASE-004**: All activities related to `Budget_2024.xlsx`
   - **CASE-005**: All activities related to `Quarterly_Report.docx`

3. **Temporal Continuity**: Even when the user switches between applications, returning to the same document continues the same case (e.g., Document1.docx was revisited after working on Excel).

### Activity Naming Logic:

I transformed low-level system actions into **higher-level, semantically meaningful activities** that reflect user intent:

- **FOCUS**  **"Open Document/PDF/Spreadsheet/Email"**: Initial engagement with a resource
- **TYPING**  **"Edit Document/Spreadsheet"** or **"Compose Email"**: Content creation activities
- **SAVE**  **"Save Document/Spreadsheet"**: Persisting changes
- **CLICK (with context)**  **"Open Email," "Reply to Email," "Send Email"**: Context-specific actions
- **SCROLL**  **"Read Email"** or **"Review PDF"**: Content consumption activities
- **HIGHLIGHT**  **"Annotate PDF"**: Document annotation activity
- **CLOSE**  **"Close Document"**: Completing work on a resource

This transformation creates a process mining event log that tells coherent stories about how the user works with different documents and resources, making it suitable for analysis of work patterns, task durations, and process flows.
