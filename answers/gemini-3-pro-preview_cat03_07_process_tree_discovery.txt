1.  **Main Flow:** All cases start with *Receive Application (RA)* $\rightarrow$ *Validate Form (VF)* $\rightarrow$ *Assess Eligibility (AE)*.
2.  **The Decision/Rework Point:** After *Assess Eligibility*, the process either moves directly to *Approval Decision (AD)* (Case 2) or enters a rework phase.
3.  **Rework Logic:**
    *   In Case 1, we see *Request Missing Info (RMI)* $\rightarrow$ *Validate Form (VF)*.
    *   In Case 3, we see *Request Missing Info (RMI)* happening twice before *Validate Form (VF)*. This implies *RMI* can repeat (a loop) before re-validation occurs.

4.  **Conclusion:** All cases end with *Approval Decision (AD)* $\rightarrow$ *Send Confirmation (SC)* $\rightarrow$ *Archive Case (AC)*.

**Model Structure:**
To capture the optional rework (which can be skipped) and the repeating *RMI* activity, we use a nested loop structure within the main sequence.

*   **Main Sequence:** `RA` $\rightarrow$ `VF` $\rightarrow$ `AE` $\rightarrow$ `[Rework Loop]` $\rightarrow$ `AD` $\rightarrow$ `SC` $\rightarrow$ `AC`.
*   **Rework Loop:** A loop where the "start" is a silent step ($\tau$). This allows the process to execute the silent step and exit immediately (skipping rework), or execute the silent step, perform the "redo" body, and loop back.
*   **Rework Body:** A sequence containing `[RMI Loop]` $\rightarrow$ `VF`.
*   **RMI Loop:** A loop `* ( RMI, tau )`. This executes `RMI` once, then optionally loops back (executing $\tau$ then `RMI` again) to capture the `RMI -> RMI` behavior in Case 3.

from pm4py.objects.process_tree.obj import ProcessTree, Operator

# 1. Create the Root Node (Main Sequence)
root = ProcessTree(operator=Operator.SEQUENCE)

# 2. Define Main Path Activities (Start)
ra = ProcessTree(label="Receive Application (RA)", parent=root)
vf_start = ProcessTree(label="Validate Form (VF)", parent=root)
ae = ProcessTree(label="Assess Eligibility (AE)", parent=root)

# 3. Define the Rework Loop Logic
# We need a loop that allows skipping (Case 2) or executing RMI+ -> VF (Cases 1 & 3).
# Structure: * ( tau, Sequence( *(RMI, tau), VF ) )
rework_loop = ProcessTree(operator=Operator.LOOP, parent=root)

# 3a. Left child of Rework Loop (The 'Do' part): Silent Step (tau)
# Executing this and exiting corresponds to "No Rework".
loop_start_tau = ProcessTree(label=None, parent=rework_loop)

# 3b. Right child of Rework Loop (The 'Redo' part): Sequence of RMI -> VF
rework_body_seq = ProcessTree(operator=Operator.SEQUENCE, parent=rework_loop)

rmi_loop.children = [rmi_leaf, rmi_redo_tau]

# 3d. Inside Rework Body: The Re-Validation Activity
vf_rework = ProcessTree(label="Validate Form (VF)", parent=rework_body_seq)

# Connect Rework Body Children
rework_body_seq.children = [rmi_loop, vf_rework]

# Connect Rework Loop Children
rework_loop.children = [loop_start_tau, rework_body_seq]

# 4. Define Main Path Activities (End)
ad = ProcessTree(label="Approval Decision (AD)", parent=root)
sc = ProcessTree(label="Send Confirmation (SC)", parent=root)

# 5. Assemble the Root
root.children = [ra, vf_start, ae, rework_loop, ad, sc, ac]
