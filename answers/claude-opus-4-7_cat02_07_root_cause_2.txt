## 2. Root Cause Analysis

### Root Cause #1: Complexity Drives Multiple Document Requests
The strongest correlation is between **Complexity** and duration:

| Complexity | # Doc Requests | Avg Duration |
|------------|----------------|--------------|
| Low | 0 | ~1.5h |
| Medium | 1 | ~26h |
| High | 2ñ3 | ~48ñ77h |

- Case 2003 (High): **2** "Request Additional Documents" events
- Case 2005 (High): **3** "Request Additional Documents" events
- Each request introduces long waiting periods (often overnight gaps of 6ñ22 hours).

### Root Cause #2: Bottleneck Between Document Request and Approval
Largest time gaps occur between the last document request and claim approval:
- Case 2003: Last doc request 04-01 17:00  Approval 04-02 16:00 = **23 hours**
- Case 2005: Last doc request 04-03 15:00  Approval 04-04 10:00 = **19 hours**
- Case 2002: Doc request 04-01 14:00  Approval 04-02 10:00 = **20 hours**

**Manager_Bill** specifically handles High-complexity approvals and appears slower than **Manager_Ann** (who handles Low/Medium cases more swiftly).

### Root Cause #3: Adjuster_Lisa Over-Requesting Documents
- **Adjuster_Lisa** (Region B) issued **3 doc requests** for Case 2005 and **1** for Case 2002.
- **Adjuster_Mike** (Region A) issued **2 doc requests** for Case 2003.
- Adjuster_Lisa appears to request documents iteratively rather than consolidating needs upfront, extending lead times.

### Root Cause #4: Region B + High Complexity = Worst Performance
- Case 2005 (Region B, High) is nearly **60% longer** than Case 2003 (Region A, High).
- Region B's pipeline (CSR_Paul  Adjuster_Lisa  Manager_Bill  Finance_Carl) appears less coordinated for complex cases.

---

## 3. Explanations & Recommendations

### Why these issues occur:
1. **Complex claims inherently require more information**, but iterative rather than consolidated requests multiply the waiting time.
2. **Managers handling high-complexity approvals (Manager_Bill)** may face workload or expertise bottlenecks.
3. **Adjuster behavior differences**: Lisa's fragmented approach to requesting documents may reflect inadequate training or unclear documentation checklists.
4. **Customer response times** between document requests naturally add overnight/multi-day waiting periods.

### Recommendations:
1. **Introduce a "complete document checklist"** for high-complexity claims at the Evaluate Claim stage - require adjusters to request everything at once.
2. **Set SLAs on approval steps**, especially for Manager_Bill - investigate whether workload rebalancing or additional approvers are needed.
3. **Coach Adjuster_Lisa** on front-loading document requests; benchmark against best performers.
4. **Create a "High-Complexity Fast Track"** with dedicated resources and parallel processing where possible.
5. **Implement customer self-service portals** for document submission to shrink the wait time between request and receipt.
6. **Region B process review**: Investigate whether handoffs between Paul, Lisa, Bill, and Carl have coordination gaps compared to Region A.

### Summary
The dominant root cause is **High Complexity  multiple iterative document requests  long waits for customer response and manager approval**, compounded by **Region B's specific resource chain (Lisa + Bill)** performing worse than Region A on complex cases.
