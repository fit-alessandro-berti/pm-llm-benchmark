## Rationale for Groupings

### 1. Material Preparation
- **Why grouped:** These events all occur at the beginning of each case, are temporally contiguous, and serve the shared purpose of getting the raw sheet ready for welding.
- **Logical flow:**  
  `Retrieve`  `Scan` (identification)  `Place`  `Align` (positioning)  `Preheat` (thermal conditioning).
- **Resource pattern:** Operator A handles handling/identification, the robot arm performs precise alignment, and the heating unit preconditions the material.

### 2. Welding / Assembly
- **Why grouped:** The welding tool is picked up immediately before the two welding actions, and all three events are performed by the same operator using the same tool.
- **Logical flow:**  
  `Pick up welding tool`  `Weld corner A`  `Weld corner B`.
- **Outcome:** The raw material is transformed into an assembled part.

### 3. In-Process Weld Inspection
- **Why grouped separately:** This is a dedicated quality-check event performed by a specialized `Quality Sensor`, not by the welding operator. It validates the previous stage before the process continues.
- **Key data:** `IntegrityScore` can be preserved as a stage-level attribute.

### 4. Surface Finishing
- **Why grouped:** Coating application and drying are sequential, use related equipment, and together form a single surface-treatment phase.
- **Logical flow:**  
  `Apply protective coating`  `Dry coating`.

### 5. Final Visual Inspection
- **Why grouped:** This is the last event in every case and represents a final human confirmation step (`Check: Passed`). It confirms that the entire product meets visual quality standards.

## General Rules for Grouping the Full Log

To apply this abstraction consistently across the full event log, use the following rules:

1. **Start-of-case material handling:** Any sequence beginning with `Retrieve`, `Scan`, `Place`, `Align`, and `Preheat` maps to **Material Preparation**.
2. **Tool pickup followed by joining actions:** Events involving tool retrieval and any `Weld` activities map to **Welding / Assembly**.
3. **Quality sensor checks immediately after welding:** `Measure weld integrity` maps to **In-Process Weld Inspection**.
4. **Coating and drying pair:** `Apply protective coating` followed by `Dry coating` maps to **Surface Finishing**.
5. **Final human verification:** The last `Visual check` event in a case maps to **Final Visual Inspection**.

Using these rules, every low-level trace can be collapsed into the five high-level stages shown above, preserving the key attributes (e.g., integrity score, coating type, pass/fail status) at the aggregated stage level.
