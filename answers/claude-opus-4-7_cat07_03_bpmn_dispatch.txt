# BPMN Diagram: Dispatch of Goods - Computer Hardware Shop

This is a **Business Process Model and Notation (BPMN)** diagram illustrating the goods dispatch process for a computer hardware shop. The diagram is organized into three horizontal swim lanes representing different departments:

## Swim Lanes (Departments)
1. **Logistics** (top)
2. **Secretary** (middle)
3. **Warehouse** (bottom)

## Process Flow

**Start:** The process begins with a "Ship goods" event in the Secretary lane.

### Parallel Split
After starting, a parallel gateway (+) splits the flow into two concurrent paths:

**Path 1 - Secretary Lane:**
- **Clarify shipment method** task
- Decision gateway: **Special handling?**
  - **Yes branch:** Get 3 offers from logistic companies  Select logistic company and place order
  - **No branch:** Check if insurance is required (uses *Customer information* data store and *Order details* document)
- Inclusive gateway (O): If insurance is necessary, route to Logistics lane for **Insure parcel**, then (always) **Write package label**
- Paths merge at another inclusive gateway, then converge at an exclusive gateway (X)

**Path 2 - Warehouse Lane:**
- **Package goods** task (with a 48-hour timer attached)
  - If delay occurs: **Notify customer about delay**
- Paths merge at an inclusive gateway

### Parallel Join and Completion
Both main paths synchronize at a parallel gateway (+) in the Warehouse lane, then proceed to:
- **Fill out Shipment paperwork**
- **Move package to pickup area**

**End:** The process concludes with the "Shipment prepared" end event.

## Key BPMN Elements Used
- **Pools/Lanes** for organizational responsibility
- **Tasks** (rounded rectangles) for activities
- **Gateways**: Parallel (+), Exclusive (X), and Inclusive (O)
- **Timer event** (48 hours) for boundary timing
- **Data objects** (Order details document, Customer information database)
- **Start and End events** (circles)
