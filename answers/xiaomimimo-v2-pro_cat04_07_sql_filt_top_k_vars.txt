Here's a comprehensive DuckDB SQL query that accomplishes all your requirements:

```sql
WITH case_sequences AS (
    -- Step 1: Get ordered sequence of activities for each case
    SELECT 
        case_id,
        LIST(activity ORDER BY timestamp) AS activity_sequence,
        STRING_AGG(activity, '  ' ORDER BY timestamp) AS variant_description
    FROM event_log
    GROUP BY case_id
),

variant_counts AS (
    -- Step 2 & 3: Count cases per variant and rank by frequency
    SELECT 
        activity_sequence,
        variant_description,
        COUNT(*) AS case_count,
        DENSE_RANK() OVER (ORDER BY COUNT(*) DESC) AS variant_rank
    FROM case_sequences
    GROUP BY activity_sequence, variant_description
),

top_variants AS (
    -- Filter to only top K variants
    SELECT 
        activity_sequence,
        variant_description,
        case_count,
        variant_rank
    FROM variant_counts
    WHERE variant_rank <= 5  -- Change 5 to your desired K value
),

cases_in_top_variants AS (
    -- Get case IDs that belong to top variants
    SELECT cs.case_id
    FROM case_sequences cs
    INNER JOIN top_variants tv 
        ON cs.activity_sequence = tv.activity_sequence
)

-- Step 4: Return all events for cases in top variants
SELECT 
    el.case_id,
    el.activity,
    el.timestamp
FROM event_log el
INNER JOIN cases_in_top_variants ctv 
    ON el.case_id = ctv.case_id
ORDER BY 
    ctv.case_id,
    el.timestamp;
```

## Key Features Explained:

1. **Case Sequences**: Uses `LIST()` or `ARRAY_AGG()` with `ORDER BY timestamp` to create ordered activity sequences
2. **Variant Identification**: Groups identical sequences to identify unique process variants
3. **Ranking**: Uses `DENSE_RANK()` to handle ties properly when multiple variants have the same frequency
4. **Top K Filtering**: Replace `5` with your desired K value
5. **Complete Event Return**: Returns all original events for cases in the top variants
