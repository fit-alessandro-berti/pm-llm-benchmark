Here’s a compact DuckDB query that does exactly what you described:

```sql
-- Return only events belonging to cases that have  6 events total
SELECT e.*
FROM   event_log AS e
JOIN (
        SELECT case_id
        FROM   event_log
        GROUP  BY case_id
        HAVING COUNT(*) <= 6          -- keep only cases with 6 or fewer rows
      ) AS small_cases
  ON e.case_id = small_cases.case_id
ORDER BY e.case_id, e.timestamp;      -- optional, for a tidy output
```

**How it works**

1. **Subquery (`small_cases`)**  
   - Groups the table by `case_id`.  
   - `HAVING COUNT(*) <= 6` keeps only those `case_id`s whose total number of rows (events) is six or fewer.

2. **Join back to the original table**  
   - The `JOIN` matches every row in `event_log` to the filtered list of `case_id`s, thereby returning the full event details (`e.*`) for the allowed cases.

3. **Optional ordering**  
   - `ORDER BY e.case_id, e.timestamp` makes the result easier to read, showing events in chronological order per case.

If you prefer a **windowfunction** version (no explicit join), you can also write:

```sql
SELECT *
FROM (
        SELECT e.*,
               COUNT(*) OVER (PARTITION BY case_id) AS case_evt_cnt
        FROM   event_log e
     ) sub
WHERE case_evt_cnt <= 6
ORDER BY case_id, timestamp;
```

Both approaches yield the same set of events: every event belonging to a case that contains **six or fewer** total events, and none from cases with more than six events.
