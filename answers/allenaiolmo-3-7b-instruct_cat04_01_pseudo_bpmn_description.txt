The process begins when a customer submits a request. This could be for a standard service (e.g., routine delivery) or a custom service (e.g., tailored solution).

- **Purpose:** To acknowledge receipt of the customer's request.

### **3. First Gateway: "Check Request Type" (XOR Split)**

This is a **decision point**: the request must be classified as either **Standard** or **Custom**.

#### **A. If Request Type = Standard**
- **Path: Task B1 – "Perform Standard Validation"**
    - **Purpose:** Basic validation to ensure the request meets standard criteria (e.g., valid data, sufficient resources).
    - **Next: "Run Parallel Checks" (AND Gateway)**
        - **Tasks C1 (Credit Check) and C2 (Inventory Check)**
            - **Purpose:** These parallel checks verify the customer’s creditworthiness (C1) and whether required resources (e.g., inventory items) are available (C2).
        - **After both checks pass:**

            - The process moves toward finalizing the standard request.

#### **B. If Request Type = Custom**
- **Path: Task B2 – "Perform Custom Feasibility Analysis"**
    - **Purpose:** A more in-depth analysis to determine if the custom request can be fulfilled given internal constraints.
    - **Gateway: "Is Customization Feasible?" (XOR)**
        - **If Yes:** Go to Task E1 – "Prepare Custom Quotation"
        - **If No:** Go to Task E2 – "Send Rejection Notice" and end the process (unless there's a loop, see below).

### **4. Post-Validation / Feasibility Gateway: "Is Approval Needed?" (XOR)**

After the core validation/feasibility work (standard or custom path), the process asks: **Does this request require managerial approval?**

#### **A. If Approval is Needed**
- **Task F – "Obtain Manager Approval"**
    - The request is sent to management for review.
    - **Gateway: "Is Approval Granted?" (XOR)**
        - **If Yes:** Proceed to Task G – "Generate Final Invoice"
        - **If No:** Go to Task H – "Re-evaluate Conditions"
            - This triggers a **loop back**. The process likely returns to the previous point where decisions could be reconsidered (see below for details).

#### **B. If No Approval Needed**
- Directly proceed to Task G – "Generate Final Invoice"
    - The request is considered ready for fulfillment without managerial intervention.

### **5. Task I – "Send Confirmation to Customer"**

Once the request is finalized (after all validations, approvals, and invoice generation), a confirmation is sent to the customer, marking the end of the process.
