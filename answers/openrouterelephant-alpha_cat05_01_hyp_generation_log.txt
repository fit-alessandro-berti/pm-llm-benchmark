## **3. SQL Queries to Investigate Further**

### **Query 1: Identify Cases with Out-of-Order Activities**
Find cases where a later activity in the process occurs before an earlier one.

```sql
SELECT 
    oel1.case_id,
    oel1.activity AS early_activity,
    oel1.timestamp AS early_timestamp,
    oel2.activity AS late_activity,
    oel2.timestamp AS late_timestamp
FROM order_event_log oel1
JOIN order_event_log oel2
    ON oel1.case_id = oel2.case_id
    AND oel1.event_id < oel2.event_id
    -- Define disorder: later activity should come before earlier in sequence
    AND (
        (oel1.activity = 'Ship Goods' AND oel2.activity = 'Confirm Shipment')
        OR (oel1.activity = 'Confirm Shipment' AND oel2.activity = 'Ship Goods')
        OR (oel1.activity = 'Ship Goods' AND oel2.activity = 'Validate Stock')
        OR (oel1.activity = 'Confirm Shipment' AND oel2.activity = 'Perform Credit Check')
        OR (oel1.activity = 'Ship Goods' AND oel2.activity = 'Perform Credit Check')
    )
    AND oel1.timestamp > oel2.timestamp
ORDER BY oel1.case_id, oel1.timestamp;
```

### **Query 2: Find Cases Missing Critical Steps (e.g., Credit Check)**
Identify orders that skipped "Perform Credit Check".

```sql
SELECT 
    oel.case_id,
    o.order_value,
    r.role AS last_role,
    MAX(oel.timestamp) AS last_event_time
FROM order_event_log oel
JOIN orders o ON oel.case_id = o.case_id
LEFT JOIN resources r ON oel.resource = r.resource_id
WHERE oel.activity != 'Perform Credit Check'
    AND oel.case_id NOT IN (
        SELECT DISTINCT case_id 
        FROM order_event_log 
        WHERE activity = 'Perform Credit Check'
    )
GROUP BY oel.case_id, o.order_value, r.role;
```

### **Query 3: Cases with Missing Activity Sequences (e.g., no Validate Stock before Ship Goods)**

```sql
SELECT 
    oel.case_id,
    o.order_value,
    o.order_type
FROM orders o
WHERE o.case_id IN (
    SELECT DISTINCT case_id 
    FROM order_event_log 
    WHERE activity = 'Ship Goods'
)
AND o.case_id NOT IN (
    SELECT DISTINCT case_id 
    FROM order_event_log 
    WHERE activity = 'Validate Stock'
);
```

### **Query 4: Detect Payments Before Invoices (Potential Data Issue)**

```sql
SELECT 
    oel1.case_id,
    oel1.timestamp AS payment_time,
    oel2.timestamp AS invoice_time,
    oel1.additional_info->'amount' AS payment_amount,
    oel2.additional_info->'invoice_id' AS invoice_id
FROM order_event_log oel1
JOIN order_event_log oel2
    ON oel1.case_id = oel2.case_id
    AND oel1.timestamp < oel2.timestamp
WHERE oel1.activity = 'Receive Payment'
    AND oel2.activity = 'Issue Invoice';
```

---

### **Query 5: Resource Role vs. Activity Mismatch (e.g., non-Finance performing Finance activities)**

```sql
SELECT 
    oel.case_id,
    oel.activity,
    r.role,
    r.department,
    oel.timestamp
FROM order_event_log oel
JOIN resources r ON oel.resource = r.resource_id
WHERE (oel.activity = 'Perform Credit Check' AND r.role NOT LIKE '%Credit%')
   OR (oel.activity = 'Issue Invoice' AND r.department != 'Finance')
   OR (oel.activity = 'Receive Payment' AND r.department != 'Finance');
```

### **Query 6: Identify Early/Suspicious Shipment Attempts (e.g., `attempted_early=Y`)**

```sql
SELECT 
    case_id,
    resource,
    timestamp,
    additional_info
FROM order_event_log
WHERE activity = 'Confirm Shipment'
    AND additional_info LIKE '%attempted_early%';
```
