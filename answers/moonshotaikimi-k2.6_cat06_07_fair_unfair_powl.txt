Below are two POWL models written in Python (using **pm4py**).  
- **Model1** includes the XOR branch that can introduce bias: after the skill assessment some applicants are routed through a *standard* culturalfit check while others are routed through a **CommunityAffiliationCheck** that implicitly gives them an advantage.  
- **Model2** removes that source of unfairness: every applicant follows the same linear sequence, undergoing the identical culturalfit evaluation.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# =============================================================================
# MODEL 1: POWL WITH POTENTIAL UNFAIRNESS
# =============================================================================
# Activities
receive1          = Transition(label="ReceiveApplication")
data_check1       = Transition(label="DataCompletenessCheck")
request_info1     = Transition(label="RequestMoreInfo")
skill_assess1     = Transition(label="SkillAssessment")
cultural_fit1     = Transition(label="CulturalFitCheck")
community_aff1    = Transition(label="CommunityAffiliationCheck")
managerial_rev1   = Transition(label="ManagerialReview")
final_decision1   = Transition(label="FinalDecision")

# Loop: check data completeness; if incomplete, request more info and check again
loop1 = OperatorPOWL(operator=Operator.LOOP, children=[data_check1, request_info1])

# XOR: either standard cultural fit OR community-affiliation-based evaluation
xor_bias1 = OperatorPOWL(operator=Operator.XOR, children=[cultural_fit1, community_aff1])

# Root partial order: sequential flow with the biased XOR branch
root1 = StrictPartialOrder(nodes=[
    receive1, loop1, skill_assess1, xor_bias1, managerial_rev1, final_decision1
])
root1.order.add_edge(receive1,        loop1)
root1.order.add_edge(loop1,           skill_assess1)
root1.order.add_edge(skill_assess1,   xor_bias1)
root1.order.add_edge(xor_bias1,       managerial_rev1)
root1.order.add_edge(managerial_rev1, final_decision1)


# =============================================================================
# MODEL 2: POWL WITHOUT UNFAIRNESS
# =============================================================================
# Activities (same labels, fresh objects)
receive2          = Transition(label="ReceiveApplication")
data_check2       = Transition(label="DataCompletenessCheck")
request_info2     = Transition(label="RequestMoreInfo")
skill_assess2     = Transition(label="SkillAssessment")
cultural_fit2     = Transition(label="CulturalFitCheck")
managerial_rev2   = Transition(label="ManagerialReview")
final_decision2   = Transition(label="FinalDecision")

# Loop for data completeness (identical to Model 1)
loop2 = OperatorPOWL(operator=Operator.LOOP, children=[data_check2, request_info2])

# Root partial order: strictly linear flow — NO XOR, NO CommunityAffiliationCheck
root2 = StrictPartialOrder(nodes=[
    receive2, loop2, skill_assess2, cultural_fit2, managerial_rev2, final_decision2
])
root2.order.add_edge(receive2,        loop2)
root2.order.add_edge(loop2,           skill_assess2)
root2.order.add_edge(skill_assess2,   cultural_fit2)
root2.order.add_edge(cultural_fit2,   managerial_rev2)
root2.order.add_edge(managerial_rev2, final_decision2)
```
