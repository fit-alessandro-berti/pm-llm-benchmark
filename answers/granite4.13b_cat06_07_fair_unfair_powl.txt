Below are two complete POWL (Partially Ordered Workflow Language) models that faithfully encode the hiring process described for GlobalTeq Solutions. Each model reflects a different handling of the culturalfit / communityaffiliation step, thereby illustrating where potential unfairness can arise and how it can be mitigated.

---

### 1 POWL Model with Potential Unfairness  

**Purpose:** To capture the current workflow that includes an XOR branching after the skill assessment. One branch goes through a standard culturalfit evaluation (ActivityCulturalFitCheck), while the other branch – **CommunityAffiliationCheck** – gives a subtle advantage to applicants affiliated with the “Global Business Leaders Club” or local residents.

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition

# ------------------------------------------------------------------
# 1. ReceiveApplication (start)
ReceiveApp = Transition(label="ReceiveApplication")

# 2. DataCompletenessCheck (loop for missing info)
DataCompCheck = Transition(label="DataCompletenessCheck")
MissingInfo = SilentTransition()          # loop back to request more details
ReceiveApp.order.add_edge(DataCompCheck, MissingInfo)
MissingInfo.order.add_edge(MissingInfo, ReceiveApp)

# 3. SkillAssessment (automated test)
SkillTest = Transition(label="SkillAssessment")

# Decision point – XOR between standard cultural fit and community affiliation
# ------------------------------------------------------------------
# A) Standard Cultural Fit Evaluation
CulturalFitStandard = Transition(label="CulturalFitCheck")   # uniform evaluation for all

# B) Community Affiliation Branch (subject to bias)
CommunityAffilCheck = Transition(label="CommunityAffiliationCheck")
# Subbranch: grant a slight uplift if applicant is in the Global Business Leaders Club or local resident
BiasUplift = SilentTransition()        # represents implicit score adjustment
CommunityAffilCheck.order.add_edge(BiasUplift, CulturalFitStandard)
BiasUplift.order.add_edge(BiasUplift, CommunityAffilCheck)

# Combine into XOR (either standard cultural fit OR community affiliation path)
XOR_Cultural_Fit = OperatorPOWL(operator=Operator.XOR,
                                 children=[CulturalFitStandard, CommunityAffiliationCheck])
SkillTest.order.add_edge(SkillTest, XOR_Cultural_Fit)

# 4. Managerial Review & Final Decision
ManagerReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")

# Execution order:
ReceiveApp --> DataCompletenessCheck (loop) --> SkillAssessment --> 
XOR_Cultural_Fit (standard vs community path) --> ManagerialReview --> FinalDecision

# ------------------------------------------------------------------
# POWL Model definition
root = StrictPartialOrder(nodes=[ReceiveApp, DataCompCheck,
                                SkillTest, XOR_Cultural_Fit,
                                CulturalFitStandard, CommunityAffiliationCheck])

root.order.add_edge(ReceiveApp, DataCompCheck)
root.order.add_edge(DataCompCheck, SkillTest)
root.order.add_edge(SkillTest, XOR_Cultural_Fit)

# Standard cultural fit path
root.order.add_edge(XOR_Cultural_Fit, CulturalFitStandard)

# Community-affiliation path (includes bias uplift)
root.order.add_edge(XOR_Cultural_Fit, CommunityAffiliationCheck)
root.order.add_edge(CommunityAffiliationCheck, BiasUplift)
root.order.add_edge(BiasUplift, CulturalFitStandard)   # returns to uniform evaluation

# Postassessment steps
root.order.add_edge(CulturalFitStandard, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)

root.order.add_edge(CommunityAffiliationCheck, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)
```

**Key Features Demonstrating Potential Unfairness:**
- The **XOR_Cultural_Fit** node splits the path based on community affiliation.
- Applicants who are members of the “Global Business Leaders Club” or local residents receive an *implicit* score uplift via `BiasUplift`.
- This creates a subtle bias because not all candidates start from equal footing.

### 2 POWL Model Without Unfairness  

**Purpose:** To represent the same hiring process but with **no special communitybased branch**. All applicants undergo the identical culturalfit evaluation (ActivityCulturalFitCheck). This removes any opportunity for bias based on affiliation.

```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition

# ------------------------------------------------------------------
# 1. ReceiveApplication (start)
ReceiveApp = Transition(label="ReceiveApplication")

# 2. DataCompletenessCheck (loop for missing info)
DataCompCheck = Transition(label="DataCompletenessCheck")
MissingInfo = SilentTransition()          # loop back to request more details
ReceiveApp.order.add_edge(DataCompCheck, MissingInfo)
MissingInfo.order.add_edge(MissingInfo, ReceiveApp)

# 3. SkillAssessment (automated test)
SkillTest = Transition(label="SkillAssessment")

# ------------------------------------------------------------------
# 4. Uniform Cultural Fit Evaluation (no community bias branch)
CulturalFitUniform = Transition(label="CulturalFitCheck")   # uniform for everyone

# Execution order remains the same, but after skill assessment we go directly to
# Managerial Review and Final Decision without branching.

# Steps:
ReceiveApp --> DataCompletenessCheck (loop) --> SkillAssessment --> 
CulturalFitUniform --> ManagerialReview --> FinalDecision

# ------------------------------------------------------------------
# POWL Model definition
root = StrictPartialOrder(nodes=[ReceiveApp, DataCompCheck,
                                SkillTest, CulturalFitUniform])

root.order.add_edge(ReceiveApp, DataCompCheck)
root.order.add_edge(DataCompCheck, SkillTest)
root.order.add_edge(SkillTest, CulturalFitUniform)

root.order.add_edge(CulturalFitUniform, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)
```

**Key Features Removing Potential Unfairness:**
- **No XOR branch**: Every applicant proceeds through the same `CulturalFitCheck`.
- No special treatment for community affiliations or local residency.
- The workflow remains linear after skill assessment, eliminating any chance of implicit score adjustments based on affiliation.

These two models illustrate how the hiring process can be structured to either embed subtle biases (Model1) or eliminate them entirely (Model2). You can import and execute these definitions in a PM4Py environment to analyze the effect of such design choices on fairness metrics.
