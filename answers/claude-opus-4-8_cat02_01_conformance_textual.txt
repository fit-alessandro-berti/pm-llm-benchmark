# Analysis of Event Log Against Normative Process

## Normative Reference Flows
- **Sufficient inventory:** Receive  Check  Authorize Payment  Package  Ship  Close
- **Insufficient inventory:** Receive  Check  Cancel  Close

---

## Case-by-Case Evaluation

###  Case 1 — COMPLIANT
Sequence: Receive  Check  Authorize  Package  Ship  Close
This exactly matches the normative "sufficient inventory" flow. No violations.

---

###  Case 2 — VIOLATION
Sequence: Receive  Check  **Package items  Authorize payment**  Ship  Close

**Violation:** Packaging occurs *before* payment authorization.
The description explicitly states: *"Package items should only occur after authorization and before shipping."* Here, items are packaged before the payment is authorized, reversing the required order. This violates the rule that authorization must precede packaging.

---

###  Case 3 — VIOLATION (multiple)
Sequence: Receive  Check  Authorize  **Cancel order  Ship order**  Close

**Violations:**
1. **Shipping after cancellation:** The description states *"Ship order should only occur after packaging and never after a cancellation."* This case ships an order that was already canceled — a direct and explicit violation.
2. **Inconsistent branching / payment before cancellation:** Payment was authorized, then the order was canceled. The two branches (fulfillment vs. cancellation) are mutually exclusive paths; mixing "Authorize payment" with "Cancel order" is logically inconsistent.
3. **Missing Package step:** Even on the fulfillment path, "Ship order" must follow "Package items," which never occurs here.

This is the most severely non-compliant case.

---

###  Case 4 — COMPLIANT
Sequence: Receive  Check  Cancel  Close
This exactly matches the normative "insufficient inventory" flow. No violations.

---

###  Case 5 — VIOLATION
Sequence: Receive  Check  Authorize  **Ship order**  Close (no Package)

**Violation:** Shipping occurs *without* packaging.
The description requires: *"Ship order should only occur after packaging... items must be prepared before they can be shipped."* Here, "Package items" is skipped entirely, so the order is shipped without being prepared. This violates the mandatory Authorize  Package  Ship sequence.

## Notes on Ambiguity
- **Case 3** is unambiguous regardless of interpretation — shipping after cancellation is explicitly forbidden.
- **Case 5** could theoretically be argued as a "minor" deviation if packaging were considered optional, but the text clearly states items *must be prepared before they can be shipped*, so it is a genuine violation.
- All compliant cases (1 and 4) correctly begin with "Receive order" and end with "Close order," satisfying the global constraints that Close must always be the final activity and Check must follow Receive before any payment/fulfillment decisions.
