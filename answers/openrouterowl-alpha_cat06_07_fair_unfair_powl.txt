# POWL Models for GlobalTeq Solutions Hiring Process

## Model 1: POWL Model with Potential Unfairness

This model includes an XOR branch that gives preferential treatment to applicants with certain community affiliations, introducing subtle bias into the process.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="ReceiveApplication")
ResumeParsing = Transition(label="ResumeParsing")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
StandardCulturalFit = Transition(label="StandardCulturalFit")
CommunityAffiliationCheck = Transition(label="CommunityAffiliationCheck")
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")
skip = SilentTransition()

# Loop for data completeness: either exit (skip) or request more info and re-parse
data_completeness_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[skip, RequestMoreInfo]
)
# After requesting more info, we resume parsing
# Note: The loop means: either exit the loop (skip) or execute RequestMoreInfo then re-enter ResumeParsing
# Actually, let's restructure: the loop is around (ResumeParsing, RequestMoreInfo)
resumeparsing_and_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[ResumeParsing, RequestMoreInfo]
)

# XOR branching for cultural fit: standard path vs. community affiliation check
cultural_fit_xor = OperatorPOWL(
    operator=Operator.XOR,
    children=[StandardCulturalFit, CommunityAffiliationCheck]
)

# Build the main partial order
# Sequence: ReceiveApplication -> Loop(ResumeParsing, RequestMoreInfo) -> SkillAssessment -> XOR CulturalFit -> ManagerialReview -> FinalDecision

root = StrictPartialOrder(
    nodes=[
        ReceiveApplication,
        resumeparsing_and_loop,
        SkillAssessment,
        cultural_fit_xor,
        ManagerialReview,
        FinalDecision
    ]
)

root.order.add_edge(ReceiveApplication, resumeparsing_and_loop)
root.order.add_edge(resumeparsing_and_loop, SkillAssessment)
root.order.add_edge(SkillAssessment, cultural_fit_xor)
root.order.add_edge(cultural_fit_xor, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)

# Visualize and verify
from pm4py.algo.transformation.powl.converter import apply as powl_to_petri
net, im, fm = powl_to_petri(root)
from pm4py.visualization.powl.visualizer import apply as visualize_powl
gviz_unfair = visualize_powl(root)
```

## Model 2: POWL Model Without Unfairness

This model removes the XOR branching for cultural fit. All applicants undergo the same standardized cultural fit evaluation, eliminating the source of bias.

```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
ReceiveApplication = Transition(label="ReceiveApplication")
ResumeParsing = Transition(label="ResumeParsing")
RequestMoreInfo = Transition(label="RequestMoreInfo")
SkillAssessment = Transition(label="SkillAssessment")
CulturalFitCheck = Transition(label="CulturalFitCheck")  # Single unified cultural fit check
ManagerialReview = Transition(label="ManagerialReview")
FinalDecision = Transition(label="FinalDecision")
skip = SilentTransition()

# Loop for data completeness: either exit (skip) or request more info and re-parse
resumeparsing_and_loop = OperatorPOWL(
    operator=Operator.LOOP,
    children=[ResumeParsing, RequestMoreInfo]
)

# No XOR branching - single standardized cultural fit check for all applicants

# Build the main partial order
# Sequence: ReceiveApplication -> Loop(ResumeParsing, RequestMoreInfo) -> SkillAssessment -> CulturalFitCheck -> ManagerialReview -> FinalDecision

root = StrictPartialOrder(
    nodes=[
        ReceiveApplication,
        resumeparsing_and_loop,
        SkillAssessment,
        CulturalFitCheck,
        ManagerialReview,
        FinalDecision
    ]
)

root.order.add_edge(ReceiveApplication, resumeparsing_and_loop)
root.order.add_edge(resumeparsing_and_loop, SkillAssessment)
root.order.add_edge(SkillAssessment, CulturalFitCheck)
root.order.add_edge(CulturalFitCheck, ManagerialReview)
root.order.add_edge(ManagerialReview, FinalDecision)

# Visualize and verify
from pm4py.algo.transformation.powl.converter import apply as powl_to_petri
net, im, fm = powl_to_petri(root)
from pm4py.visualization.powl.visualizer import apply as visualize_powl
gviz_fair = visualize_powl(root)
```

## Summary of Key Differences

| Aspect | Model 1 (Potentially Unfair) | Model 2 (Fair) |
|--------|------------------------------|----------------|
| **Cultural Fit Stage** | XOR branch: `StandardCulturalFit` OR `CommunityAffiliationCheck` | Single path: unified `CulturalFitCheck` |
| **Bias Mechanism** | Applicants with community affiliations receive uplift via `CommunityAffiliationCheck` path | All applicants evaluated through identical process |
| **Loop for Data Completeness** | Present in both models | Present in both models |
| **Overall Structure** | Receive  Loop(Parse,Request)  Skill  XOR(CulturalFit,CommunityCheck)  Review  Decision | Receive  Loop(Parse,Request)  Skill  CulturalFitCheck  Review  Decision |

The critical difference is that **Model 1** contains an **XOR operator** (`cultural_fit_xor`) that allows the process to diverge into two distinct cultural-fit evaluation paths depending on whether a community affiliation is detected. This introduces the potential for uneven treatment. **Model 2** replaces this with a single, standardized `CulturalFitCheck` activity, ensuring all applicants are assessed through the same criterion and removing the avenue for subconscious or systemic bias.
