# High-Level Process Step Aggregation

## Proposed High-Level Activities

### 1. **Material Preparation**
Groups the initial handling and setup of the raw material before any transformation begins.

| Low-Level Events Included | Resource(s) |
|---|---|
| Retrieve raw metal sheet | Operator A |
| Scan barcode | Operator A |
| Place metal sheet on table | Operator A |
| Align metal sheet | Robot Arm #2 |
| Preheat metal sheet | Heating Unit #1 |

**Rationale:** All these events occur at the start of the case and serve a single purpose — getting the raw material identified, positioned, and thermally ready for processing. They are temporally contiguous (within ~15–20 seconds) and each is a prerequisite for welding. Although two resources are involved (operator + robot + heater), they collectively perform "setup."

---

### 2. **Welding / Assembly**
Groups all actions related to joining/forming the product structure.

| Low-Level Events Included | Resource(s) |
|---|---|
| Pick up welding tool | Operator B |
| Weld corner A | Operator B |
| Weld corner B | Operator B |

**Rationale:** These events are tightly coupled in time (~20 seconds), executed by the same operator (Operator B), and all involve the welding tool. The tool pickup is an enabling sub-step for the welding actions that follow, so it belongs in the same functional phase.

---

### 3. **In-Process Quality Check (Weld Inspection)**
A dedicated verification of the assembly output before further processing.

| Low-Level Events Included | Resource(s) |
|---|---|
| Measure weld integrity | Quality Sensor #1 |

**Rationale:** Although only one event, it represents a distinct quality-gate phase. It is executed by a different resource type (Quality Sensor) and yields a measurement (IntegrityScore). Keeping it separate from welding allows downstream analysis of defect/rework patterns.

---

### 4. **Surface Treatment**
Groups coating application and drying.

| Low-Level Events Included | Resource(s) |
|---|---|
| Apply protective coating | Coating Unit #3 |
| Dry coating | Heating Unit #2 |

**Rationale:** Both events relate to finishing the surface of the welded part. Coating application and subsequent drying are logically a single process ("coat  cure") and are performed by finishing/heating equipment.

---

### 5. **Final Inspection**
Groups the end-of-line verification.

| Low-Level Events Included | Resource(s) |
|---|---|
| Visual check | Operator C |

**Rationale:** This is the terminal event of each case, performed by a dedicated QA operator (Operator C). It verifies the completed product before release and is clearly separated from the in-process weld check.

---

## Grouping Rules (Generalizable Heuristics)

The mapping above was derived using the following heuristics:

1. **Temporal proximity** — events within a short time window ( ~30s here) tend to belong together.
2. **Resource continuity** — consecutive events performed by the same operator/machine suggest one activity (e.g., Operator B's tool pickup + welds).
3. **Functional purpose** — events that serve the same goal (preparing material, finishing surface) are grouped, even when different resources are involved.
4. **Quality gates as anchors** — inspection/measurement events break the flow between logical stages and are treated as their own step.
5. **Tool/Material state changes** — transitions in material state (raw  aligned  welded  coated  inspected) mark boundaries between phases.
