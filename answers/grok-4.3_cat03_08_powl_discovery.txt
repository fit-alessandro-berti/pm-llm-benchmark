```python
import pm4py
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Leaf transitions
or_act = Transition(label="OR")
ms = Transition(label="MS")
qs = Transition(label="QS")
as_act = Transition(label="AS")
iqc = Transition(label="IQC")
tst = Transition(label="TST")
rt = Transition(label="RT")
pk = Transition(label="PK")
lb = Transition(label="LB")
dp = Transition(label="DP")
d = Transition(label="D")

# Silent transitions for loops and choice
skip_iqc = SilentTransition()
skip_lb = SilentTransition()

# IQC loop: allows repeating IQC (multiple consecutive IQC)
# * (IQC, tau)  IQC (tau IQC)* exit
iqc_loop = OperatorPOWL(operator=Operator.LOOP, children=[iqc, skip_iqc])

# Assembly loop: * (AS, iqc_loop)  AS (iqc_loop AS)* exit
# Captures variable AS/IQC interleaving with possible consecutive IQC
assembly_loop = OperatorPOWL(operator=Operator.LOOP, children=[as_act, iqc_loop])

# Testing loop: * (TST, RT)  TST (RT TST)* exit
# Captures possible re-testing
tst_loop = OperatorPOWL(operator=Operator.LOOP, children=[tst, rt])

# Labeling choice: X (LB, tau)  either LB or skip
lb_choice = OperatorPOWL(operator=Operator.XOR, children=[lb, skip_lb])

# Overall process as strict partial order (sequential flow)
root = StrictPartialOrder(nodes=[
    or_act, ms, qs, assembly_loop, tst_loop, pk, lb_choice, dp, d
])

# Add ordering edges (total order in this process)
root.order.add_edge(or_act, ms)
root.order.add_edge(ms, qs)
root.order.add_edge(qs, assembly_loop)
root.order.add_edge(assembly_loop, tst_loop)
root.order.add_edge(tst_loop, pk)
root.order.add_edge(pk, lb_choice)
root.order.add_edge(lb_choice, dp)
root.order.add_edge(dp, d)
```

**Explanation of the POWL model:**

- **Start sequence** (`OR  MS  QS`): Always executed first in strict order.
- **Assembly & In-line QC loop** (`assembly_loop`): 
  - Starts with `AS`.
  - The inner `iqc_loop` (* (IQC, tau)) allows any number (1) of consecutive `IQC` steps when repeating.
  - The outer loop allows arbitrary interleaving of `AS` and groups of `IQC` (including `IQC` `IQC`), always ending with `AS` before exiting to testing. This matches all observed patterns (e.g., direct `AS`, `AS IQC AS`, `AS IQC IQC AS IQC AS`, etc.).
- **Testing loop** (`tst_loop` = * (TST, RT)): Allows zero or more re-tests (`RT`  `TST`) before proceeding to packaging.
- **Labeling choice** (`lb_choice` = X (LB, tau)): Optional `LB`; can be skipped (as in Case 2).
- **End sequence** (`PK  lb_choice  DP  D`): Always executed after the loops.

This structure compactly captures all variations in the provided event log using the allowed POWL constructs (loops, exclusive choice, silent transitions, and partial ordering).
