# BPMN-Like Process Model: Invoice Approval Process

## 1. PROCESS OVERVIEW

**Process Name:** Invoice Approval and Payment Workflow
**Process Objective:** To systematically review, validate, and approve supplier invoices for payment in alignment with purchase orders and organizational policies.

---

## 2. ROLES/PARTICIPANTS

| Role | Description |
|------|-------------|
| **Accounts Payable (AP) Clerk** | Initial recipient and processor of invoices; logs invoices, performs matching, and coordinates approvals |
| **Purchasing Officer** | Validates invoice details against purchase orders and confirms receipt of goods/services |
| **Department Manager** | Final internal approver before payment scheduling |
| **AP Supervisor** | Handles exceptions, escalations, and complex discrepancy resolution |
| **Supplier Relationship Manager** | Facilitates supplier communication for issue resolution |
| **Supplier** | External party issuing invoices; responds to queries about discrepancies |
| **Accounting System** | Automated system for logging invoices, generating notifications, and tracking approval status |

---

## 3. ARTIFACTS/INPUTS/OUTPUTS

| Artifact | Type | Description |
|----------|------|-------------|
| **Invoice** | Input | Document received from supplier (typically via email) containing billing details |
| **Purchase Order (PO)** | Input | Authorized document confirming ordered goods/services |
| **Confirmation Email** | Communication | Email from Purchasing confirming goods/services receipt |
| **Retrospective PO** | Exception Artifact | PO created after the fact for informally contracted services |
| **Approved Invoice Record** | Output | Invoice record with manager approval in the system |
| **Payment Schedule** | Output | List of approved invoices queued for payment run |

## 6. EXCEPTIONS AND VARIATIONS

### 6.1 Exception Paths

| Exception | Description | Handling |
|-----------|-------------|----------|
| **Missing PO Number** | Invoice arrives without PO reference | Loop back to supplier for PO or use retrospective PO process |
| **Pricing Discrepancy** | Billed amount differs from PO | Contact supplier for correction or validate against contract terms |
| **Quantity Discrepancy** | Units billed differ from ordered | Verify delivery and adjust invoice accordingly |
| **Missing Items** | Line items missing from invoice | Request revised invoice or documentation |
| **Budget Variance** | Significant difference from budgeted amount | Manager flags for additional scrutiny |
| **Informal Contracts** | Services rendered without formal PO | Supervisor creates retrospective PO with department head approval |

### 6.2 Variation Notes

- **Sequencing Variation:** According to Dan's account, Purchasing sometimes contacts suppliers directly to hold invoices while Dan coordinates with AP. This parallel communication is absorbed into the escalation and resolution activities above.
- **Manager Involvement Timing:** Carlos indicates he receives the notification after Purchasing confirms and the system shows invoice-to-PO matching. This is captured in Activity 12.
- **Payment Scheduling:** Mary states she "schedules it for payment" after manager approval. Carlos and Dan suggest the invoice returns to AP for this step. This is modeled as Activity 15.
