## Explanation of Logic

### Case Identification Strategy
I grouped events into **cases based on the artifact (document/email/file) the user was working on**, since process mining typically benefits from tracing the lifecycle of a logical work object. This yielded **5 cases**:

- **C1 – Quarterly Report Preparation** (`Quarterly_Report.docx`): Opened initially, left open in the background, and revisited at the end.
- **C2 – Draft New Document** (`Document1.docx`): Active drafting work, interrupted by other tasks, then resumed and closed.
- **C3 – Handle Annual Meeting Email**: A full email-reply workflow in Google Chrome.
- **C4 – Review Report Draft PDF** (`Report_Draft.pdf`): Review and annotation in Adobe Acrobat.
- **C5 – Update 2024 Budget** (`Budget_2024.xlsx`): Spreadsheet update cycle.

This approach highlights **task switching and multitasking patterns**—for example, Case C2 and C1 both have interruptions (visible as gaps in their event timelines), which is a valuable signal for productivity/process analysis.

### Activity Name Standardization
Raw low-level actions were abstracted into standard process verbs:

| Raw Action | Standardized Activity |
|------------|-----------------------|
| FOCUS (first time on doc) | Open Document / Open Spreadsheet / Open PDF |
| FOCUS (returning) | Resume Document |
| SWITCH (to new context) | Open … / Resume … |
| TYPING | Edit Document / Edit Spreadsheet / Compose Reply |
| SAVE | Save Document / Save Spreadsheet |
| CLOSE | Close Document |
| CLICK (Open Email) | Open Email |
| CLICK (Reply) | Initiate Reply |
| CLICK (Send) | Send Email |
| SCROLL (in email/PDF) | Review Email / Review PDF |
| HIGHLIGHT | Annotate/Highlight PDF |

### Narrative
The log tells the story of a knowledge worker starting their morning by opening a quarterly report, getting sidetracked drafting a new document, pausing to handle an email about an annual meeting, reviewing a PDF draft, updating the budget spreadsheet, returning to finalize and close the draft document, and finally completing work on the quarterly report. The event log preserves this **multitasking behavior** while making each work stream analyzable as an independent process instance.
