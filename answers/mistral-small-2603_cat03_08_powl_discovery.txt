```python
from pm4py.objects.powl.obj import StrictPartialOrder, OperatorPOWL, Transition, SilentTransition
from pm4py.objects.process_tree.obj import Operator

# Define activities
OR = Transition(label="OR")  # Order Received
MS = Transition(label="MS")  # Material Sourcing
QS = Transition(label="QS")  # Quality Screening
AS = Transition(label="AS")  # Assembly
IQC = Transition(label="IQC")  # In-line Quality Check
TST = Transition(label="TST")  # Testing
RT = Transition(label="RT")  # Re-Testing
PK = Transition(label="PK")  # Packaging
LB = Transition(label="LB")  # Labeling
DP = Transition(label="DP")  # Distribution Planning
D = Transition(label="D")    # Dispatch

# Define silent transition for skipped labeling
skip_LB = SilentTransition()

# Define the testing loop (TST -> RT -> TST)
testing_loop = OperatorPOWL(operator=Operator.LOOP, children=[TST, RT])

# Define the labeling choice (LB or skip)
labeling_choice = OperatorPOWL(operator=Operator.XOR, children=[LB, skip_LB])

# Define the main process structure
root = StrictPartialOrder(nodes=[
    OR, MS, QS, AS, IQC, testing_loop, PK, labeling_choice, DP, D
])

# Define the partial order constraints
root.order.add_edge(OR, MS)
root.order.add_edge(MS, QS)
root.order.add_edge(QS, AS)
root.order.add_edge(AS, IQC)
root.order.add_edge(IQC, testing_loop)
root.order.add_edge(testing_loop, PK)
root.order.add_edge(PK, labeling_choice)
root.order.add_edge(labeling_choice, DP)
root.order.add_edge(DP, D)
```

### Explanation:

1. **Activities**: Each activity in the process is represented as a `Transition` with its corresponding label.

2. **Silent Transition**: Used to represent the optional labeling step (`skip_LB`).

3. **Testing Loop**: The testing process is modeled as a loop (`Operator.LOOP`) between `TST` and `RT` because:
   - Testing (`TST`) is performed first
   - If the product fails (`RT`), it goes back to testing (`TST`)
   - This repeats until the product passes testing

4. **Labeling Choice**: The labeling step is optional, so it's modeled as an exclusive choice (`Operator.XOR`) between `LB` and the silent transition `skip_LB`.

5. **Partial Order**: The strict partial order defines the execution sequence:
   - Linear sequence from `OR` to `MS` to `QS` to `AS`
   - `AS` must complete before `IQC` can start
   - `IQC` must complete before the testing loop can start
   - The testing loop must complete before packaging (`PK`)
   - Packaging must complete before the labeling choice
   - The labeling choice must complete before distribution planning (`DP`)
   - Distribution planning must complete before dispatch (`D`)

This POWL model captures all the observed behavior in the event log, including the optional labeling step and the repeated testing process when products fail initial testing.
